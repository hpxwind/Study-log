文件上传漏洞

问多web站点都有文件上传的接口，比如：

1、注册时上传头像（jpg，png，gif）

2、上传文件附件（doc，xls等）

而后台开发时候并没有对上传的文件功能进行安全考虑或者采用了有缺陷的措施，导致攻击者可以通过一些手段绕过安全措施从而上传一些恶意文件（一句话木马等。）

发现漏洞思路：

1、对文件上传地方按照要求上传文件,查看返回路径（路径，提示等）；

2、尝试上传不同类型的“恶意”文件，比如xx.php，分析结果，

3、查看html源码，看是否通过js在前端做了上传限制，可以绕过，

4、尝试使用不同的方式进行绕过，黑白名单绕过/MIME绕过/目录0x00截断绕过等，

5、猜测或者结合其他漏洞（敏感信息泄露等）得到木马路径，链接测试。

  • 发现过滤在前端的情况

    直接bp抓包，或者删除前端的onchange属性

  • MIME

    MIME：多用途互联网邮件扩展类型，是设定某种扩展名的文件用一种应用程序来打开的方式类型，当该扩展文件名被访问的时候，浏览器会自动使用指定应用程序来打开，多用于指定一些客户端自定义问文件名，以及一些媒体打开方式。

    每个MIME类型由两部分组成，前面是数据的大类别，例如生音audio，图像image等，后面定义具体类型。

    常见MIME类型：

    超文本标记语言文本 .html/ .html text/html

    普通文本 .txt text/plain

    RTF文本 .rtf application/rtf

    GIF图形 .gif image/gif

    JPEG .ipeg .jpg image/jpeg

    $_FILES() 可以从客户端计算机向远程服务器上传文件

    第一个参数是表单的 input name

    第二个可以是 name, type, size, tmp_name 或 error

    $_FILES['file']['name'] # 被上传文件的名称
$_FILES['file']['type'] # 被上传文件的类型
$_FILES['file']['size'] # 被上传文件的大小（字节）
$_FILES['file']['tmp_name'] # 存储在服务器的文件的临时副本名称
$_FILES['file']['error'] # 由文件上传导致的错误代码
  • getimagesize

    Getimagesize() 返回结果中有文件大小和文件类型，如果用这个函数来获取类型从而判断是否是图片的话，会存在问题。

    可以绕过，因为图片头可以伪造。

    xxd aaa.png | head -n 1 xxd查看文件16进制，head -n 1指定看第一行。

    图片木马制作：

    方法1：直接伪造头部GIF89A

    方法2：CMD：copy /b test.png + muma.php ccc.png

    方法3：使用GIMP（开源图片修改软件），通过增加备注，写入命令行

    上传后，重要的是执行包含在图片中的恶意代码，这时候，结合文件包含漏洞，让包含漏洞中的include()进行包含，

    include()函数如果代码出错，会跳过出错的部分，往下执行，直到执行完毕，

    让其执行含有恶意代码的图片，会执行到恶意代码部分。

    $html='';
if(isset($_POST['submit'])){
    $type=array('jpg','jpeg','png');//指定类型
    $mime=array('image/jpg','image/jpeg','image/png');
    $save_path='uploads'.date('/Y/m/d/');//根据当天日期生成一个文件夹
    $upload=upload('uploadfile','512000',$type,$mime,$save_path);//调用函数
    if($upload['return']){
        $html.="<p class='notice'>文件上传成功</p><p class='notice'>文件保存的路径为：{$upload['save_path']}</p>";
    }else{
        $html.="<p class=notice>{$upload['error']}</p>";

    }
}
