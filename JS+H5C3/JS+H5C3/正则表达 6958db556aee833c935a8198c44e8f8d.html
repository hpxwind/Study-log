正则表达

正则表达式（RegExp）是用于匹配字符串中字符组合的模式。JavaScript 中的正则表达式通过 RegExp 对象实现

  • 1、创建正则表达式
    // 字面量方式
const regex1 = /pattern/flags;

// 构造函数方式
const regex2 = new RegExp('pattern', 'flags');
  • 2、常用元字符
    .      - 匹配除换行符外的任意字符
\d     - 匹配数字（等价于 [0-9]）
\w     - 匹配字母、数字或下划线（等价于 [A-Za-z0-9_]）
\s     - 匹配空白字符（空格、制表符等）
^      - 匹配字符串开头
$      - 匹配字符串结尾
[abc]  - 匹配方括号内的任意字符
[^abc] - 匹配不在方括号内的任意字符
a|b    - 匹配 a 或 b
\D \W \S - 非\d \w \s
    // () 整体
console.log(/(abc)+/.test("abcabc"))

// a|b 或 |
// (asd|bcd)  | 匹配左右整体 有asd或bcd
console.log(/a|b/.test("ac"))

// [] 代表1个 [abc] 匹配a，b，c中的一个
console.log(/[abc]/.test("a"))

// /[adcdef]{3,5}/ 匹配3~5个adcdef中的字符
// [a-zA-Z0-9_] === \w
// [0-9] === \d
console.log(/[adcdef]{3,5}/.test("abc"))
    // 一位数子 \d ,连续两位 \d\d, 连续三位 \d\d\d
var str = "absc1"
var reg = /\d\d/
// test为弱匹配，返回布尔值
console.log(reg.test(str))

// 一位非数字 \D
var str2 = "123213"
var reg2 = /\D/
console.log(reg2.test(str2))

// 一位空白内容 \s (空格 缩进 换行符号)
var str3 = "dnwamsd\ndwad"
var reg3 = /\s/
console.log(reg3.test(str3)) 

// 一位空白内容 \S
var str4 = "   \n1"
var reg4 = /\S/
console.log(reg4.test(str4))

// 一位字母或数字或下划线 \w
var str5 = "1a"
var reg5 = /\w/
console.log(reg5.test(str5))

// 一位非字母或数字或下划线 \W
var str6 = "{231{"
var reg6 = /\W\W/
console.log(reg6.test(str6))

// 任意内容(除了换行符) .
var str7 = "\n"
var reg7 = /./
console.log(reg7.test(str7))

// 开头必须匹配 ^\d 开头必须为数字， ^123 开头必须为123
var str8 = "123"
var reg8 = /^123/
console.log(reg8.test(str8))

// 结尾必须匹配 \d$ 结尾必须为数字， 123$ 结尾必须为123
var str9 = "123"
var reg9 = /123$/
console.log(reg9.test(str9))
  • 3、量词
    *       - 匹配 0 次或多次
+       - 匹配 1 次或多次
?       - 匹配 0 次或 1 次
{n}     - 匹配 n 次
{n,}    - 匹配至少 n 次
{n,m}   - 匹配 n 到 m 次
asd|da  - 匹配 asd 或 da整体
()      - 匹配整体
    // * 匹配多次 0~多次
console.log(/\d*/.test("123"))

// + 至少出现 1 次
console.log(/\d+/.test("a123"))
console.log(/\d+/.test("a"))

// ? 匹配0次或1次
console.log(/\d?/.test("a112312"))

// {} 匹配指定次数 只能指定前面一个字符
console.log(/\d{2}/.test("a1"))
console.log(/abc{2}/.test("abcc")) // true

// {n, } >= n次
console.log(/\d{2,}/.test("a11"))

// {n, m} n~m次
console.log(/\d{2,3}/.test("a111"))

  • 5、匹配方法
    test()    - 测试是否匹配，返回布尔值 // test为弱匹配，返回布尔值
exec()    - 执行匹配，返回匹配结果数组
match()   - 字符串方法，返回匹配结果

  • 正则表达式的两大特性

    1、懒惰：无法记住上一次位置。解决：可以使用全局标识符标识g

    2、贪婪：越多的比配越好。解决：非贪婪模式在结尾加 ? 如：

    *? +? ?? {n,}? {n,m}?

    var str = `<p class="active"><span></span></p>`
var myreg = /<p.*>/ // 如果是贪婪，匹配到的是<p class="active"><span></span></p>，因为中间.*会尽可能多的匹配到最远的>的位置
var myreg2 = /<p.*?>/ // 非贪婪，匹配到的是<p class="active">

  • 案例
    • 密码强度匹配：/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[^\w]).{8,}$/
      ^ 和 $ 是锚点：^ 匹配字符串开头，$ 匹配字符串结尾，确保整个字符串都被规则约束（避免部分匹配）。
---------
(?=.*[a-z]) -> 确保字符串中至少包含 1 个小写字母（无论位置在哪里，前面可以有任意字符）
(?=...)：正向预查（零宽断言），表示 “从当前位置开始，后面的内容必须满足... 条件”，但不消耗字符（仅做判断）。
.*：. 匹配任意单个字符（除换行符）
[a-z]：字符类，匹配任意小写字母（a 到 z）
-------------
(?=.*[A-Z]) -> 确保字符串中至少包含 1 个大写字母。
-------------
(?=.*\d) -> 确保字符串中至少包含 1 个数字。
--------
(?=.*[^\w]) -> 确保字符串中至少包含 1 个特殊字符。
[\w_]：字符类，\w 等价于 [A-Za-z0-9_]
[^\w]：特殊字符，非[A-Za-z0-9_]
-------------
.{8,} -> 确保密码长度至少 8 位。
. 匹配任意字符（除换行符）。
{8,}：量词，表示匹配前面的字符至少 8 次（即总长度≥8）。
