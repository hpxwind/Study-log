Jsonp

jsonp(JSON with padding)是json的一种‘使用模式’，可以让网页从别的域名（网站）获取资料，即跨域读取数据。

为什么从不同域中需要jsonp？

因为存在同源策略（CORS）。同源策略：同域名，同端口，同协议。

解决：

1、Access-Control-Allow-Origin:属性

2、jsonp：前后端必须协作

jsonp原理：动态创建script标签，src属性指向没有跨域限制,指向一个接口，接口返回的格式一定是****()函数表达式.

需要注意，

1、后端接口形式必须为****()形式，需要与后端沟通

2、需要使用onload remove()删除创建的多余的script标签

3、jsonp只支持get

<button id="btn">jsonp</button>
<script>
    function test (obj) {
        console.log(obj);
    }

    btn.onclick = function () {
        var oscript = document.createElement('script');
        oscript.src = './01.txt'; // 未来的接口地址
        document.body.appendChild(oscript);
    }
    
    oscript.onload = function (){
		    oscript.remove() // 删除当前新建的节点
    }
</script>

<script>
    // https://www.baidu.com/sugrec?pre=1&p=3&ie=utf-8&json=1&prod=pc&from=pc_web&sugsid=60277,62327,63145,63327,63947,64247,64363,64439,64450,64462,64475,64488,64505,64515,64524,64419,64557,64566,64587,64593,64592,64600,64603,64634,64657,64671,64713,64740,64742,64738,64760&wd=a&req=2&csor=1&cb=ker&_=1755958739139
    
    mysearch.oninput = function (e) {
        var keyword = e.target.value
        if (!keyword) {
            list.innerHTML = ''
            return
        }
        // console.log(e.target.value)
        var oscript = document.createElement('script')
        oscript.src = `https://sp0.baidu.com/5a1Fazu8AA54nxGko9WTAnF6hhy/su?wd=${encodeURIComponent(keyword)}&cb=test`
        document.body.appendChild(oscript)
        
        oscript.onload = function () {
            oscript.remove()
        }
    }

    function test(obj) {
        console.log(obj)

        if (obj && Array.isArray(obj.s)) {
            list.innerHTML = obj.s.map(item => {
                return `<li>${item}</li>`
            }).join('')
        } else {
            list.innerHTML = ''
        }
    }
    
</script>
