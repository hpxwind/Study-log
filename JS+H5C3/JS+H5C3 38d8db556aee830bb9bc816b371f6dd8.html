  • 多库并存解决

    当页面引入多个 JavaScript 库（如同时使用 jQuery、Prototype、Zepto 等）时，可能会出现 全局变量冲突（最常见的是 $ 符号被多个库占用）。解决多库并存的核心是 隔离各自的全局变量，确保每个库能独立运行。

    多数库会暴露全局变量（如 jQuery 暴露 $ 和 jQuery，Prototype 也可能使用 $），当多个库使用相同的全局变量名时，后引入的库会覆盖先引入的库的变量，导致前者功能失效。如：

    <!-- 先引入 jQuery -->
<script src="jquery.js"></script>
<script>
  console.log($); // 输出 jQuery 函数（正常）
</script>

<!-- 再引入 Prototype（也使用 $ 符号） -->
<script src="prototype.js"></script>
<script>
  console.log($); // 输出 Prototype 的 $ 函数（jQuery 的 $ 被覆盖，冲突！）
</script>

    解决方案：

    jQuery 提供了专门的 API 解决冲突，核心是 释放 $ 符号的控制权，让其他库可以使用 $，同时保留 jQuery 变量供自身使用。

    • 1、jQuery.noConflict()释放$使用权

      jQuery.noConflict() 会将 $ 还原为引入 jQuery 之前的状态（可能是其他库的 $ 或 undefined），同时返回 jQuery 自身，方便用自定义变量引用。

      <script src="prototype.js"></script> <!-- 先引入其他库 -->
<script src="jquery.js"></script>     <!-- 后引入 jQuery -->

<script>
  // 释放 $ 控制权：$ 还原为 Prototype 的 $
  const $j = jQuery.noConflict(); 

  // 现在：
  // - $ 指向 Prototype 的功能
  // - $j 指向 jQuery 的功能（可用 $j 替代 $ 调用 jQuery 方法）
  console.log($);      // Prototype 的 $ 函数
  console.log($j);     // jQuery 函数
  console.log(jQuery); // 仍指向 jQuery（未被覆盖）

  // 使用 jQuery 时用 $j
  $j('div').css('color', 'red');

  // 使用 Prototype 时用 $
  $('div').addClassName('active');
</script>
    • 2、彻底释放$和jQuery（自定义变量名）

      若其他库也使用 jQuery 变量（极少见），可通过 jQuery.noConflict(true) 彻底释放 $ 和 jQuery，并用自定义变量保存 jQuery。

      <script src="other-lib.js"></script> <!-- 假设该库也使用 jQuery 变量 -->
<script src="jquery.js"></script>

<script>
  // 彻底释放 $ 和 jQuery，返回 jQuery 自身
  const myJQ = jQuery.noConflict(true);

  console.log($);      // 其他库的 $
  console.log(jQuery); // 其他库的 jQuery
  console.log(myJQ);   // 我们的 jQuery

  // 使用 jQuery 时用 myJQ
  myJQ('div').hide();
</script>
    • 3、立即执行函数（IIFE）内使用$

      若仅在局部代码中使用 jQuery 的 $，可通过立即执行函数（IIFE）将 jQuery 作为参数传入，在函数内部用 $ 指代，不影响全局 $。

      <script src="prototype.js"></script>
<script src="jquery.js"></script>

<script>
  // 释放全局 $
  jQuery.noConflict();

  // 在 IIFE 内部，$ 仅指代 jQuery（不影响外部）
  (function($) {
    // 这里的 $ 是 jQuery
    $(document).ready(function() {
      $('div').text('用 jQuery 操作');
    });
  })(jQuery); // 将 jQuery 作为参数传入，函数内用 $ 接收

  // 外部的 $ 仍指向 Prototype
  console.log($); // Prototype 的 $ 函数
</script>
    • 4、结合DOM加载完成事件（ready方法）

      jQuery 的 ready 方法支持传入一个函数，该函数的第一个参数会被自动设置为 jQuery，可在内部用 $ 指代。

      <script src="prototype.js"></script>
<script src="jquery.js"></script>

<script>
  // 释放全局 $
  jQuery.noConflict();

  // ready 方法的回调中，$ 自动指向 jQuery
  jQuery(document).ready(function($) {
    // 这里的 $ 是 jQuery
    $('div').css('background', 'gray');
  });

  // 外部 $ 仍为 Prototype 的 $
  $('div').addClassName('test');
</script>
  • JSON格式字符串

    json只接收6种数据类型

    类型说明示例
    字符串必须用双引号包裹（不能用单引号），支持转义字符（\n、\" 等）。"name": "Alice"
    数字整数、浮点数、负数，不支持 NaN、Infinity。"age": 20、"score": 95.5
    布尔值仅 true 或 false（小写）。"isStudent": true
    null表示空值（小写），不能用 undefined。"address": null
    对象（Object）键值对集合，键必须用双引号包裹，值可以是任意 JSON 类型。{"id": 1, "name": "Bob"}
    • JSON.parse() & JSON.stringify()
      JSON.parse()	用于将一个 JSON 字符串转换为 JavaScript 对象。
const jsonStr = '{"name":"Bob","birth":"2000-01-01","scores":[90,85,95]}';

// 基本用法
const obj = JSON.parse(jsonStr);
console.log(obj.name); // "Bob"
console.log(obj.scores[0]); // 90

// 转换日期字符串为Date对象
const objWithDate = JSON.parse(jsonStr, (key, value) => {
  if (key === "birth") {
    return new Date(value); // 将"2000-01-01"转为Date对象
  }
  return value;
});
console.log(objWithDate.birth instanceof Date); // true

----------------------------------------------------------------------
JSON.stringify()	用于将 JavaScript 值转换为 JSON 字符串。
const obj = { name: "Alice", age: 20, isStudent: true };

// 基本用法
const jsonStr = JSON.stringify(obj);
console.log(jsonStr); 
// 输出：{"name":"Alice","age":20,"isStudent":true}（严格JSON格式）

// 美化输出（缩进2个空格）
const prettyJson = JSON.stringify(obj, null, 2);
console.log(prettyJson);
/* 输出：
{
  "name": "Alice",
  "age": 20,
  "isStudent": true
}
*/

// 过滤属性（只保留name）
const filteredJson = JSON.stringify(obj, ["name"]);
console.log(filteredJson); // {"name":"Alice"}
  • href="#"与href="javascript:void(0)"的区别
    	# 包含了一个位置信息，默认的锚是#top 也就是网页的上端。而javascript:void(0), 仅仅表示一个死链接。
	在页面很长的时候会使用 # 来定位页面的具体位置，格式为：# + id。如果你要定义一个死链接请使用 javascript:void(0) 。

实例
<a href="javascript:void(0);">点我没有反应的!</a>
<a href="#pos">点我定位到指定位置!</a>

<br>
<p id="pos">尾部定位点</p>

// 阻止链接跳转，URL不会有任何变化
<a href="javascript:void(0)" rel="nofollow ugc">点击此处</a>

// 虽然阻止了链接跳转，但URL尾部会多个#，改变了当前URL。（# 主要用于配合 location.hash）
<a href="#" rel="nofollow ugc">点击此处</a>

// 同理，# 可以的话，? 也能达到阻止页面跳转的效果，但也相同的改变了URL。（? 主要用于配合 location.search）
<a href="?" rel="nofollow ugc">点击此处</a>

// Chrome 中即使 javascript:0; 也没变化，firefox中会变成一个字符串0
<a href="javascript:0" rel="nofollow ugc">点击此处</a>
  • const & let & var区别

    1，作用域

    var声明的变量的作用域只能是全局或者整个函数块

    let可以是全局或函数块，也可以是if、while、switch、等{}限定代码块中。

    function varTest() {
  var a = 1;
  {
    var a = 2; // 函数块中，同一个变量
    console.log(a); // 2
  }
  console.log(a); // 2
}

function letTest() {
  let a = 1;
	{
    let a = 2; // 代码块中，新的变量

	}

}
