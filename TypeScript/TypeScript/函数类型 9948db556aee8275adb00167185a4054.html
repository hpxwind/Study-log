函数类型

TypeScript（TS）的函数类型是对函数输入（参数）和输出（返回值）的类型约束，核心是明确函数的参数类型、返回值类型，以及函数本身的类型签名

  • 直接在函数定义中标注类型

    (参数名: 类型, ...) => 返回值类型

    // 普通函数
function add(a: number, b: number): number {
  return a + b;
}

// 箭头函数
const multiply = (x: number, y: number): number => {
  return x * y;
};

// 无返回值（void 类型）
function logMsg(msg: string): void {
  console.log(msg);
  // 无 return 或 return undefined 均符合 void
}

// 返回值为 never（永远不会执行完，如抛出错误/无限循环）
function throwError(msg: string): never {
  throw new Error(msg);
}
  • 先定义函数类型别名，再赋值

    通过 type 定义函数类型签名，再将符合该签名的函数赋值给变量，适合多个函数共享同一类型约束

    // 定义函数类型别名：接收两个 number，返回 number
type CalculateFunc = (a: number, b: number) => number;

// 符合类型的函数
const add: CalculateFunc = (x, y) => x + y; // 无需重复标注参数类型（TS 自动推导）
const subtract: CalculateFunc = (a, b) => a - b;

// 错误示例：参数类型不匹配（TS 报错）
// const invalidFunc: CalculateFunc = (a: string, b: string) => a + b;
  • 可选参数

    用 ? 标记参数为可选，可选参数必须放在必选参数之后

    // age 是可选参数（number | undefined）
function getUserInfo(name: string, age?: number): string {
  return age ? `${name}, ${age}岁` : name;
}

getUserInfo("张三"); // 合法
getUserInfo("李四", 20); // 合法
// getUserInfo("王五", "20"); // 报错：age 需为 number | undefined
  • 默认参数

    参数指定默认值后，TS 会自动推导类型（也可显式标注），默认参数可放在必选参数前（但需传 undefined 触发默认值）。

    // 默认参数：gender 默认为 "male"，类型推导为 string
function getPerson(name: string, gender = "male"): string {
  return `${name}, ${gender}`;
}

getPerson("张三"); // 张三, male
getPerson("李四", "female"); // 李四, female

// 默认参数在前（需传 undefined 触发默认值）
function getInfo(age = 18, name: string): string {
  return `${name}, ${age}岁`;
}
getInfo(undefined, "王五"); // 王五, 18岁
  • 剩余参数

    用 ... 接收剩余参数，类型通常为数组（类型[]）

    // 剩余参数 nums 为 number 数组
function sum(...nums: number[]): number {
  return nums.reduce((total, num) => total + num, 0);
}

sum(1, 2, 3); // 6
sum(1, 2, 3, 4); // 10
// sum(1, "2"); // 报错：剩余参数需为 number
  • 无参数 / 无返回值的函数类型
    // 无参数，返回 string
type NoParamFunc = () => string;
const getHello: NoParamFunc = () => "hello";

// 无参数，无返回值
type VoidFunc = () => void;
const doNothing: VoidFunc = () => {};
  • 函数重载（处理多参数 / 返回值组合）

    当一个函数需要接收不同类型 / 数量的参数，并返回不同类型时，可通过函数重载定义多个类型签名，TS 会根据入参匹配对应的签名。

    // 步骤1：定义重载签名（仅声明，无实现）
function formatData(data: string): string; // 入参 string → 返回 string
function formatData(data: number): number[]; // 入参 number → 返回 number[]

// 步骤2：实现函数（需兼容所有重载签名）
function formatData(data: string | number): string | number[] {
  if (typeof data === "string") {
    return data.toUpperCase();
  } else {
    return [data, data * 2, data * 3];
  }
}

// 调用：TS 自动匹配签名
const strRes = formatData("hello"); // 类型为 string
const numRes = formatData(10); // 类型为 number[]
  • 将函数作为参数
    // 定义接收回调函数的函数：回调接收 string，无返回值
function handleData(data: string, callback: (res: string) => void): void {
  const processed = data + "_processed";
  callback(processed);
}

// 调用：回调函数符合类型约束
handleData("test", (res) => {
  console.log(res); // res 类型为 string
});
  • 函数类型中的this类型（显式约束）

    TS 允许显式指定函数内部 this 的类型（需放在第一个参数位置，仅做类型约束，不实际传参）

    // 定义 this 的类型
type User = { name: string; age: number };

// 显式约束 this 为 User 类型
function printUser(this: User): void {
  console.log(`${this.name}, ${this.age}岁`);
}

// 绑定 this 并调用
const user: User = { name: "张三", age: 20 };
const boundPrint = printUser.bind(user);
boundPrint(); // 张三, 20岁

// 错误调用：this 类型不匹配
// printUser(); // 报错：this 应为 User，但实际是 undefined

误区：

  1. void 不是 undefined：void 表示 “无返回值”，函数可返回 undefined，但 undefined 类型表示 “明确返回 undefined”；
function fn1(): void { return; } // 合法
function fn2(): undefined { return undefined; } // 必须显式返回 undefined
  1. never 类型不可赋值给其他类型：never 表示函数永远不会完成（如抛错 / 死循环），是所有类型的子类型，但无类型是 never 的子类型；
  1. 函数类型匹配是 “协变” 的：只要参数类型兼容、返回值类型兼容，即可赋值（TS 严格模式下会检查参数类型的逆变）。
