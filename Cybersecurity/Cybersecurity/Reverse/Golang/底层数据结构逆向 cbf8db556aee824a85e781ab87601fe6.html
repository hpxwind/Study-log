    type StringHeader struct {
	Data uintptr // 字符串首地址
	Len int // 字符串长度
}

    逆向时候可以发现如下代码

  • slice
    type SliceHeader struct {
	Data uintptr // 数据指针
	Len int // 当前长度
	Cap int // 可容纳的长度
}

    数组转换切片

    一个数组转换成相等大小的切片

    slice切片类型由元素的类型组成

    创建切片

    go:
slice1 := make0([]int, len)

asm:
call   runtime_makeslice([]int, len, cap)
mov    [rsp+28h], rdx // rdx = makeslice 创建的数组基地址
mov    [rsp+30h], rax // rax = len
mov    [rsp+38h], rcx // rcx = cap

    访问（读写）slice元素

    访问slice元素时候会检查是否越界（len），若越界则调用runtime_paniclndex

    最后用slice的数据地址计算元素的实际地址

    访问过程中还有一些runtime_writeBarrier相关的调用代码，忽略即可

    Slice append

    slice.append(slice1, 123)

    append的代码一般以inline形式呈现，流程如下

    1. 先检测目标slice1.len + 1 与slice1.cap的大小关系
    1. 若slice1.len + 1 > slice1.cap 则调用runtime_growslice扩容
    1. 写入append数据
    1. 更新原始切片信息

    示例：

    m := make)(map[string]int)
m["k1"] = 7

    map的访问

    runtime_mapaccess1_xxx 函数实现访问map中的元素

    func runtime_mapaccess1_xxx(t *maptype, h *hmap, key unsafe.Pointer) unsafe.Pointer

    Map的迭代访问，迭代map需要mapiterinit 和 mapiternext函数支持

    mapiterinit 赋值初始化迭代器

    mapiternext 迭代器前进，迭代器是一个键值对数据结构

    func mapiterinit(mapType *byte, hmap map[any]any, hiter *any)
func mapiternext(hiter *any)

    hiter的数据结构类似于

    type hiter struct {
	KeyData uintptr // key 数据指针
	ValueData uintptr // value 数据指针
}

    第一项指向key内存， 第二项指向value内存，如果第一项为0，表示迭代完成
