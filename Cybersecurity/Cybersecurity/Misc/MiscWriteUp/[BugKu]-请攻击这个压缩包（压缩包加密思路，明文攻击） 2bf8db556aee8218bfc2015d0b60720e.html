[BugKu]-请攻击这个压缩包（压缩包加密思路，明文攻击）

对于加密有三种思路

1、暴力破解

2、考虑zip伪加密

3、明文攻击

1、拖进010Editor查看，发现是真加密，而且密钥任何提示，暴力很难有收获。

2、尝试明文攻击：

明确算法，和加密zip版本：右键一个文件—>属性

看到算法伪ZipCrypto Store, 这种传统的压缩算法满足明文攻击的前提条件

但如果是：

AES-256 Deflate

AES-256 Store

明文攻击也无法进行了

3、明文攻击需要拥有一个与压缩包内一样的文件才能完成，

但利用ZipCrypto Store的特性，只需要知道加密压缩包内容的12个字节就可以对压缩包进行明文攻击

而压缩包内的文件为：flag.png，其12字节的文件头是固定的，与任何其他png文件一样为：89504E470D0A1A0A0000000D49484452

因此只需要自己创建一个只有这个头文件的png图片就可以攻击了。

echo 89504E470D0A1A0A0000000D49484452 | xxd -r -ps > png_header
// xxd -r -ps：将16进制数转换成ASCII文本
//JPG：FFD8FFE115004578696600004D4D02A0
time bkcrack -C file.zip -c flag.png -p png_header -o 0 > 1.log&
// time：加上time参数查看计算爆破时间
// -C：选择加密压缩包
// -c：选择压缩包的密文部分
// -p：选择的明文文件
// -o：指定的明文在压缩包内目标文件的偏移量
// > 1.log&：后台运行，并将结果存入1.log
tail -f 1.log // 查看破解进度和 结果
bkcrack -C file.zip -c flag.png -k 92802c24 9955f8d6 65c652b8 -d 1.png
// -k：输入三段密钥
// -d：输出

最后查看1.png得到flag。
