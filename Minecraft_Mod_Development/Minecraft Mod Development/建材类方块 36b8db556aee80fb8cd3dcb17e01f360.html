    public static final RegistryObject<WallBlock> ICE_ETHER_WALL =
            registerBlock("ice_ether_wall",
                    () -> new WallBlock(BlockBehaviour.Properties.of()
                            .strength(3.0f, 5.0f)));

    public static final RegistryObject<DoorBlock> ICE_ETHER_DOOR =
            registerBlock("ice_ether_door",
                    () -> new DoorBlock(BlockBehaviour.Properties.of()
                            .strength(3.0f, 5.0f)
                            .noOcclusion(), BlockSetType.IRON)); // IRON只能红石打开

    public static final RegistryObject<TrapDoorBlock> ICE_ETHER_TRAPDOOR =
            registerBlock("ice_ether_trapdoor",
                    () -> new TrapDoorBlock(BlockBehaviour.Properties.of()
                            .strength(3.0f, 2.0f), BlockSetType.OAK));
    1. StairBlock:楼梯:.strength(3.0F, 6.0F)
    1. SlabBlock:台阶:.strength(2.0F, 5.0F)
    1. ButtonBlock:按钮:
      () -> new ButtonBlock(BlockBehaviour.Properties.of().strength(3.0f, 2.0f),
            BlockSetType.OAK,40,false)
      1. Properties：属性
      1. BlockSetType.OAK：木质按钮（决定声音、材质行为）
      1. 40：按下后恢复延迟（tick）
      1. false：不是木质压力板行为
    1. PressurePlateBlock:压力板:
      () -> new PressurePlateBlock(PressurePlateBlock.Sensitivity.EVERYTHING,
            BlockBehaviour.Properties.of().strength(1.0f, 2.0f),
            BlockSetType.OAK)
      1. EVERYTHING：玩家、怪物、物品、实体都能触发
      1. BlockSetType.OAK：声音、行为类型
    1. FenceGateBlock:栅栏门:.strength(1.0f, 2.0f)/WoodType.OAK
    1. FenceBlock:栅栏:.strength(1.0f, 2.0f)
    1. WallBlock:墙:.strength(3.0f, 5.0f)
    1. DoorBlock:门:
      () -> new DoorBlock(BlockBehaviour.Properties.of().strength(3.0f, 5.0f).noOcclusion(),
            BlockSetType.IRON)
      1. noOcclusion()：不遮挡视线与光线
      1. BlockSetType.IRON：铁门，只能用红石打开
      1. 想做木门就用：BlockSetType.OAK
    1. TrapDoorBlock:陷阱门:.strength(1.0f, 2.0f)/BlockSetType.IRON

    请注意门，陷阱门这些多状态的物品（方块模型 ≠ 物品栏模型）

    1. 如下这些方法是 Forge 内置封装好的
      stairsBlock(...)
slabBlock(...)
buttonBlock(...)
pressurePlateBlock(...)
fenceBlock(...)
fenceGateBlock(...)
wallBlock(...)
    1. 而门有两个部分，无法直接生成
      门（Door）门有两个独立部分：
1.下半部门 door_bottom
2.上半部门 door_top
它们是两个完全不同的模型 + 两张不同纹理, 因此必须写成
doorBlockWithRenderType(
    门,
    底部纹理,
    顶部纹理,
    渲染类型 // "cutout"  // 镂空渲染
);
    1. 而陷阱门有开启/关闭，上下表面，朝向，必须指定渲染层等特殊需求，无法直接生成
      trapdoorBlockWithRenderType(
    陷阱门,
    纹理,
    true,        // 是否木质
    "cutout"     // 镂空渲染
);
blockItem(陷阱门, "_bottom"); // 物品显示底部模型
    1. 最下面的blockItem

      有些方块的物品模型，不能直接用方块模型,比如：楼梯（inventory 模型是特殊的）台阶 栅栏 栅栏门 陷阱门 门, 它们在物品栏里显示的模型，和放在世界里的方块模型不一样。

    按钮、栅栏、墙在物品栏里的模型是特殊的、独立的、固定的

    1. button_inventory:按钮

      在世界里的按钮是：贴在墙上的小方块

      但物品栏里的按钮模型是:button_inventory → 这是一个固定的特殊模型，不是方块模型

    1. fence_inventory:栅栏

      世界里栅栏是 3D 立体栅栏

      但物品栏里显示的是：fence_inventory → 一个扁平的图标模型

    1. wall_inventory:墙

      世界里墙是高低不平的连接墙

      但物品栏里是：wall_inventory → 也是固定特殊模型

    1. 这三个特殊的方法：
      private <T extends Block> void fenceItem(RegistryObject<T> block, RegistryObject<Block> base) {
    this.withExistingParent(block.getId().getPath(), mcLoc("block/fence_inventory"))  // 继承原版栅栏物品模型
            .texture("texture", 材质位置);  // 使用你的自定义方块材质
}
      • 继承原版物品模型button_inventory / fence_inventory / wall_inventory
      • 替换成你的材质让模型显示你的冰以太方块纹理
