数据类型转换

1、基本数据类型转换

尽量不适用强制转换，因为存在精度损失的情况

byte -> short -> int -> long -> float -> double

double num = 10.5;
int i = (int)num;

2、引用数据类型的转换

前提是需要转换的两种数据类型必须有继承的关系

自动转换

强制转换

前提：Student继承Person 
public class Student extends Person {
}
可以转换：
public class Main {
    public static void main(String[] args) {
        Student student = new Student();
        Person person = new Person();
        // 自动类型转换
        person = student;
        // 强制类型转换
        student = (Student)person;
    }
}

实现类和接口也可以进行转换，因为接口和实现类本质上也是父类和子类的继承关系：接口是父类，实现类就是子类

接口是由抽象类继承而来

抽象父类：

public abstract class Test1 {
	public abstract void test();
}

非抽象子类：

public class Test2 extends Test1 {
	@Override
	public void test() {
	
	}
}

抽象父类可以优化成接口：

public interface Test1 {
	public abstract void test()
}

非抽象子类变成实现类

public class Test2 implements Test1 {
	@Override
	public void test() {
	
	}
}

接口不能进行强制类型转换，因为接口无法实例化（new)

接口和抽象类不同

·关键字不同

·抽象类中可以包括非抽象方法，接口中全部是抽象方法

·具体的方式不同，接口通过实现类，抽象类通过继承

·抽象类单继承，接口多实现

父类中的构造器，在创建子类对象的时候会自动调用，非抽象类，调用构造器来创建对象，创建父类对象抽象类，调用父类构造器，但不会创建对象。
