    • __iter__ &迭代器，生成器，可迭代对象

      迭代器

      # 迭代器类型的定义：
	1. 当类中定义了__iter__和__next__两个方法。
	2. __iter__方法需要返回对象本身，即self。
	3. __next__方法，返回下一个数据，如果没有数据了，需要抛出一个StopIteration的异常。
# 创建迭代器对象：
	class IT(object):
		def __init__(self):
			self.counter=0
		def __iter__(self):
			return self
		def __next__(self):
			self.counter+=1
			if self.counter==3:
				raise StopIteration()
			return self.counter
# 根据类实例化创建一个迭代器对象
	obj1=IT()
	# v1=obj1.__next__()
	# v2=obj1.__next__()
	# v3=obj1.__next__() # 异常
	v1=next(obj1) #  obj.__next__()
	print(v1)
	obj2=IT()
	for item in obj2: # 首先会执行迭代器对象的__iter__方法并获取返回值，一直去反复指向next(对象)
		print(item)
	#结果：
	# 1
	# 2

      生成器

      # 本质和迭代器相同，只是编写和表现形式不同而已

# 创建生成对象（内部是根据生成器类generator创建的对象）

      可迭代对象

      # 如果一个类中有__iter__方法且返回一个迭代器对象
# 则这个类创建的对象为可迭代对象
class F(object):
	def __iter__(self):
		return 迭代器对象
obj=F() # obj是可迭代对象
for item in obj:
	pass

      综上：

      一个类中存在__iter__方法，返回一个迭代器对象，

      实例化接收到的就是可迭代对象（可用于for循环遍历）

      当使用for循环遍历可迭代对象时，内部先指向obj.__iter__并获取迭代器对象

      在for循环内部，会不断指向迭代器对象的next（）方法。

      如：

      v1=range(10) # v1是可迭代对象

      v2=v1.__iter__() # v2就是迭代器对象

      v2.__next__()

      特性可迭代对象（如 range）迭代器（如 v2）
      定义实现 __iter__() 方法的对象实现 __iter__() 和 __next__() 的对象
      能否生成元素不能，需通过迭代器生成能，逐个生成元素
      状态无状态，可多次获取迭代器有状态，记录当前位置
      典型场景容器（列表、字典）、生成器表达式流式数据、惰性计算

      例如：range()的实现

      # 基于可迭代对象
class IterRange(object):
	def __init__(self, num):
		self.num=num
		self.conuter=-1
	def __iter__(self):
		return self
	def __next__(self):
		self.counter+=1
			if self.counter==self.num
				raise StopIteration()
			return self.counter

class Xrange(object):
	def __init__(self, max_num):
		self.max_num=max_num
	def __iter__(self):
		return IterRange(self.max_num):

obj=Xrange(100)
for item in obj:
	print(item)

    • 生成器表达式

      from typing import Generator

      # 生成器表达式，不会立即计算所有结果
gen: Generator = (x ** 2 for x in range(10) if x % 2 == 0)
# 遍历生成器，按需生成元素
for num in gen:
    print(num)
# 输出：0 4 16 36 64

      特点

      • 节省内存：处理百万级数据时，生成器表达式优势明显，列表推导式可能因内存不足报错。
      • 惰性计算：只有在迭代（如 for 循环、next() 调用）时才会计算下一个元素。

      元组的“伪推导式”（生成器+元组构造）

      # 用生成器表达式生成元素，再转成元组
tup = tuple(x ** 2 for x in range(5))
print(tup)  
# 输出：(0, 1, 4, 9, 16)

      自定义类的“推导式风格”构建（借助生成器+类逻辑）

    def __init__(self, name, age):

        self.age = age
