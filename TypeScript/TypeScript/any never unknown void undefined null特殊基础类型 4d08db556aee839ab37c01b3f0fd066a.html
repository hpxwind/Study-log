any/never/unknown/void/undefined/null特殊基础类型

  • any

    any 是 TS 为兼容原生 JS 设计的 “逃生舱”，相当于放弃类型约束，回到纯 JS 的灵活性（但也失去了 TS 的类型安全）。

    // 声明 any 类型变量
let value: any = 100;
value = "hello"; // 合法：any 可赋值任意类型
value = true;    // 合法

// 访问任意属性/方法（无类型检查）
value.foo; // 不报错（即使 value 没有 foo 属性）
value.bar(); // 不报错（即使 value 不是函数）
value[0]; // 不报错（即使 value 不是数组）
    • 禁止滥用：any 会让 TS 失去类型校验的核心价值，大量使用等同于放弃 TS；
    • 隐式 any：未标注类型且 TS 无法推导时，会默认推断为 any（严格模式 strict: true 下会报错）：
    // strict: true 时，以下代码报错（隐式 any）
function fn(x) { // x 被推断为 any
  return x + 1;
}
  • never

    never 表示永不可能出现的值的类型，是所有类型的子类型，但没有任何类型是 never 的子类型（除了自身）.

    1.函数返回never（永不执行完）

    // 抛出错误的函数：返回 never
function throwError(msg: string): never {
  throw new Error(msg); // 无返回，永远抛出错误
}

// 无限循环的函数：返回 never
function infiniteLoop(): never {
  while (true) {} // 永远不会退出循环
}

    2.穷尽检查（确保联合类型无遗漏）

    type Direction = "left" | "right" | "up" | "down";

function handleDir(dir: Direction): string {
  switch (dir) {
    case "left": return "左";
    case "right": return "右";
    case "up": return "上";
    case "down": return "下";
    // 若遗漏分支，TS 会报错：switch 未覆盖所有联合类型
    // 若添加 default，用 never 确保无遗漏
    default:
      const _exhaustiveCheck: never = dir;
      return _exhaustiveCheck;
  }
}

    若 Direction 新增值（如 "forward"），但 switch 未处理，TS 会在 default 处报错（因为 dir 无法赋值给 never），强制开发者处理所有分支。

    never 不能被显式赋值（除了 never 本身）：

    let n: never;
// n = 100; // 报错：number 不能赋值给 never
// n = undefined; // 报错
n = throwError("error"); // 合法：throwError 返回 never

    never 与 void 区别：

    • void：函数无返回值（可返回 undefined）；
    • never：函数永远不会执行到返回步骤（如抛错 / 死循环）。

  • unknown

    unknown 表示未知类型，与 any 类似（可赋值任意值），但不可随意访问属性 / 调用方法，必须先通过类型守卫缩小类型范围 —— 是更安全的 “任意类型”。

    let value: unknown = 100;
value = "hello"; // 合法：unknown 可赋值任意类型
value = true;    // 合法

// 禁止直接访问属性/方法（区别于 any）
// value.foo; // 报错：unknown 类型不能访问 foo 属性
// value.bar(); // 报错：unknown 类型不能调用 bar 方法

// 必须先做类型守卫
if (typeof value === "string") {
  value.toUpperCase(); // 合法：TS 推导为 string 类型
}

if (typeof value === "number") {
  value.toFixed(1); // 合法：TS 推导为 number 类型
}

    对比any

    特性unknownany
    赋值规则可接收任意值，不可随意赋值给其他类型可接收 / 赋值任意类型
    属性 / 方法访问需类型守卫，否则报错无限制，不报错
    类型安全✅ 安全（强制类型校验）❌ 不安全（跳过校验）
    let u: unknown = 100;
let a: any = 200;

let num: number = a; // 合法：any 可赋值给任意类型
// let num2: number = u; // 报错：unknown 不可直接赋值给 number

// unknown 需类型守卫后赋值
if (typeof u === "number") {
  let num2: number = u; // 合法
}
  • void

    void 表示函数没有返回值，或变量仅能赋值 undefined（严格模式下）。

    1.函数返回void

    // 无 return 语句：返回 void
function logMsg(msg: string): void {
  console.log(msg);
}

// return undefined：也符合 void
function fn(): void {
  return undefined;
}

const res = logMsg("hello"); // res 类型为 void

    2.void 类型变量

    let v: void;
v = undefined; // 合法（严格模式下，void 仅能赋值 undefined）
// v = null; // 严格模式下报错（非严格模式允许）

    与undefined对比

    • void：侧重 “函数无返回值” 的语义；
    • undefined：侧重 “值不存在” 的语义；
    • 函数返回 void 时，实际返回值是 undefined，但 TS 用 void 强调 “无返回” 的语义。

  • undefined/null
    • undefined：表示变量已声明但未赋值；
    • null：表示变量值为 “空”（有意清空的值）。
    let u: undefined = undefined;
let n: null = null;

// 非严格模式下，undefined/null 可赋值给任意类型
// 严格模式（strictNullChecks: true）下，必须显式标注
let str: string | undefined = "hello";
str = undefined; // 合法

let num: number | null = 100;
num = null; // 合法

关键配置：strictNullChecks

  • strictNullChecks: true（严格模式默认开启）：undefined/null 只能赋值给自身或显式包含它们的联合类型；
  • strictNullChecks: false：undefined/null 可赋值给任意类型（兼容旧 JS 行为，不推荐）。
