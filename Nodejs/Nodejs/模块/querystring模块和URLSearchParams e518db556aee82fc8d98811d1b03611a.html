querystring模块和URLSearchParams

querystring专门处理url的查询部分（?后面的部分）

它提供了一套简洁的 API，实现查询字符串与 JavaScript 对象之间的相互转换，同时支持对特殊字符的编码（escape）与解码（unescape），解决 URL 中非法字符（如空格、中文、特殊符号）的传输问题。

  • 作用
    • 特殊字符（如 &、=、空格、中文）会破坏 URL 结构，导致解析错误；
    • 手动遍历对象拼接字符串效率低且易出错；
    • 不同后端框架对查询字符串的格式要求不同（如是否支持数组、嵌套对象）。

    querystring 模块的核心作用就是标准化这些操作，解决上述痛点。

  • 转换类：字符串→对象
    • querystring.parse(str[,esp[,eq[,options]]])

      将查询字符串转换为Js对象

      如："?name=foo&age=20" → { name: "foo", age: "20" }。

      参数类型默认值说明
      strString无（必传）要解析的 URL 查询字符串（可带 ?，也可不带，模块会自动忽略 ?）。
      sepString"&"键值对之间的分隔符（如 & 表示 key1=val1&key2=val2）。
      eqString"="键与值之间的分隔符（如 = 表示 key=val）。
      optionsObject{ maxKeys: 1000 }配置项：- maxKeys：最大解析键数（默认 1000，设为 0 表示无限制）。

      如：

      const querystring = require('querystring');

// 解析带 ? 的查询字符串
const str1 = '?name=张三&age=20&isStudent=true';
const obj1 = querystring.parse(str1);
console.log(obj1); 
// 输出：{ name: '张三', age: '20', isStudent: 'true' } 
// 注意：解析后的值均为字符串类型（如 age 是 "20"，不是数字 20）

// 解析不带 ? 的查询字符串
const str2 = 'page=1&size=10&keyword=node';
const obj2 = querystring.parse(str2);
console.log(obj2); 
// 输出：{ page: '1', size: '10', keyword: 'node' }
    • 自定义分隔符querystring.parse(str, ';', ':')

      如果查询字符串用非默认分隔符（如 ; 分隔键值对，: 分隔键与值），可手动指定 sep 和 eq：

      const str = 'name:李四;age:22;hobby:code;hobby:game';
const obj = querystring.parse(str, ';', ':');
console.log(obj); 
// 输出：{ name: '李四', age: '22', hobby: ['code', 'game'] } 
// 注意：重复的键会自动转为数组

    • querystring.stringify(obj[[,esp[,eq[,options]]])

      将Js对象转换成url查询对象

      如：{ name: "foo", age: "20" } → "name=foo&age=20"。

      参数类型默认值说明
      objObject无（必传）要转换的 JavaScript 对象（值可为字符串、数字、数组、布尔值）。
      sepString"&"键值对之间的分隔符（同 parse）。
      eqString"="键与值之间的分隔符（同 parse）。
      optionsObject{ encodeURIComponent: querystring.escape }配置项：- encodeURIComponent：自定义编码函数（默认用 querystring.escape）。
      const querystring = require('querystring');

const obj1 = { name: '王五', age: 25, isWorker: false };
const str1 = querystring.stringify(obj1);
console.log(str1); 
// 输出："name=%E7%8E%8B%E4%BA%94&age=25&isWorker=false" 
// 注意：中文自动被编码（"王五" → "%E7%8E%8B%E4%BA%94"）

// 手动拼接 URL
const baseUrl = 'https://example.com/api/user';
const fullUrl = `${baseUrl}?${str1}`;
console.log(fullUrl); 
// 输出："https://example.com/api/user?name=%E7%8E%8B%E4%BA%94&age=25&isWorker=false"
    • 处理数组与自定义分隔符
      数组会被转为 key=val1&key=val2 的格式；
可自定义 sep 和 eq 改变分隔符：
      const obj = { name: '赵六', hobbies: ['reading', 'running'], gender: 'male' };

// 自定义分隔符为 ; 和 =
const str = querystring.stringify(obj, ';', '=');
console.log(str); 
// 输出："name=%E8%B5%B5%E5%85%AD;hobbies=reading;hobbies=running;gender=male"
  • 编解码类：特殊字符处理

    URL 中不允许直接包含空格、中文、&、= 等特殊字符，必须先编码（转为 %XX 格式），传输到服务端后再解码（还原为原字符）。querystring 提供两个方法专门处理此过程。

    • querystring.escape(str)

      将字符串编码为 URL 安全格式（即把特殊字符转为 %XX 形式）。

      底层逻辑与 encodeURIComponent 类似，但对部分字符（如 '）的编码规则略有差异（querystring.escape 会编码 '，而 encodeURIComponent 不会）。

      const querystring = require('querystring');

const str = '张三&age=20';
const escapedStr = querystring.escape(str);
console.log(escapedStr); 
// 输出："%E5%BC%A0%E4%B8%89%26age%3D20" 
// 解析："张"→%E5%BC%A0，"三"→%E4%B8%89，"&"→%26，"="→%3D
    • querystring.unescape(str)

      将 querystring.escape 编码后的字符串解码为原字符串（escape 的逆操作）。

      const querystring = require('querystring');

const escapedStr = "%E5%BC%A0%E4%B8%89%26age%3D20";
const unescapedStr = querystring.unescape(escapedStr);
console.log(unescapedStr); 
// 输出："张三&age=20"（还原为原字符串）

  • 注意

    1、值的类型问题

    querystring.parse解析后，所有值都是字符串类型，即如果有类似age=20，解析后20会编程字符串’20’

    const obj = querystring.parse('age=20&isStudent=true');
obj.age = Number(obj.age); // 转为数字
obj.isStudent = obj.isStudent === 'true'; // 转为布尔值
console.log(obj); // { age: 20, isStudent: true }

    2、嵌套对象不支持

    querystring仅支持扁平对象，不支持嵌套对象（如 { user: { name: "foo", age: 20 } }）若强行转换，嵌套部分会被转为字符串：

    const obj = { user: { name: "foo", age: 20 } };
const str = querystring.stringify(obj);
console.log(str); // "user=[object Object]"（嵌套对象被转为字符串，无法正常解析）

    3、与URLSearchParams区别

    特性querystring 模块URLSearchParams 类
    所属模块独立内置模块url 模块的子功能
    浏览器支持不支持（仅 Node.js）支持（浏览器 + Node.js）
    嵌套对象支持不支持不支持
    API 风格函数式（如 parse/stringify）面向对象（如 get()/set()）
    扩展功能无（仅基础编解码 / 转换）支持 forEach、has() 等
  • 若在 Node.js 环境中处理简单查询字符串，querystring 足够轻量；
  • 若需 浏览器 / Node.js 跨环境兼容，或需要更灵活的操作（如动态添加 / 删除键值对），推荐使用 URLSearchParams。

相比传统的字符串拼接或 querystring 模块，URLSearchParams 更符合现代 Web 标准，操作更简洁。

  • 查询字符串创建
    // 带 ? 或不带 ? 均可
const params1 = new URLSearchParams("name=Alice&age=20");
const params2 = new URLSearchParams("?name=Bob&gender=male");

    从对象创建

    const params = new URLSearchParams({
  name: "Charlie",
  hobbies: ["reading", "sports"] // 数组会被转为多个键值对
});
// 等价于 "name=Charlie&hobbies=reading&hobbies=sports"

    二维数组创建

    const params = new URLSearchParams([
  ["name", "David"],
  ["age", "25"],
  ["hobbies", "music"]
]);
// 等价于 "name=David&age=25&hobbies=music"
  • 获取参数方法
    方法说明示例
    get(key)获取指定键的第一个值（字符串）params.get("name") → "Alice"
    getAll(key)获取指定键的所有值（数组，适合多值）params.getAll("hobbies") → ["reading", "sports"]
    has(key)判断是否存在指定键（返回布尔值）params.has("age") → true

  • 添加/修改参数方法
    方法说明示例
    append(key, value)为指定键添加一个值（保留现有值）params.append("hobbies", "coding")
    set(key, value)设置指定键的值（覆盖现有值）params.set("age", "21")

  • 删除参数方法
    方法说明示例
    delete(key)删除指定键的所有值params.delete("gender")
  • 遍历方法
    const params = new URLSearchParams("name=Alice&age=20&hobbies=reading");

// 1. for...of 遍历 [key, value]
for (const [key, value] of params) {
  console.log(`${key}: ${value}`); // name: Alice, age: 20, hobbies: reading
}

// 2. 遍历所有键
for (const key of params.keys()) {
  console.log(key); // name, age, hobbies
}

// 3. 遍历所有值
for (const value of params.values()) {
  console.log(value); // Alice, 20, reading
}

// 4. forEach 遍历
params.forEach((value, key) => {
  console.log(`${key}=${value}`);
});
  • 转换字符串方法
    方法说明示例
    toString()转换为查询字符串（不带 ?）params.toString() → "name=Alice&age=20"
  • 编码与解码

    URLSearchParams 会自动处理特殊字符的编码（如中文、空格、& 等），无需手动调用 encodeURIComponent：（但不会处理单引号）

    const params = new URLSearchParams();
params.append("name", "张三");
params.append("msg", "hello world!");

console.log(params.toString()); 
// 输出："name=%E5%BC%A0%E4%B8%89&msg=hello+world!"
// 解析："张三" 被编码为 %E5%BC%A0%E4%B8%89，空格被编码为 +

    解码也会自动获得，通过get()获取的值就是解码后的原始值：

    const params = new URLSearchParams("name=%E5%BC%A0%E4%B8%89");
console.log(params.get("name")); // 输出："张三"（自动解码）
  • 与URL对象配合使用

    URL 对象的 searchParams 属性直接返回 URLSearchParams 实例，可无缝配合操作 URL：

    // 创建 URL 对象
const url = new URL("https://example.com/api/users");

// 通过 searchParams 修改查询参数
url.searchParams.set("page", "1");
url.searchParams.append("hobby", "reading");
url.searchParams.append("hobby", "sports");

console.log(url.href); 
// 输出："https://example.com/api/users?page=1&hobby=reading&hobby=sports"

// 修改参数
url.searchParams.set("page", "2");
console.log(url.href); 
// 输出："https://example.com/api/users?page=2&hobby=reading&hobby=sports"

// 删除参数
url.searchParams.delete("hobby");
console.log(url.href); 
// 输出："https://example.com/api/users?page=2"
  • 注意
    1. 值的类型：所有参数值都会被转为字符串（例如数字 20 会变为 "20"，布尔值 true 会变为 "true"）。
    1. 不支持嵌套对象：URLSearchParams 仅支持扁平结构，无法直接处理嵌套对象（如 { user: { name: "Alice" } }），需手动转为字符串（如 JSON 序列化）。
    1. 浏览器兼容性：现代浏览器（Chrome、Firefox、Edge 等）均支持，但 IE 完全不支持，需通过 polyfill 兼容（如 url-search-params-polyfill）。
    1. 与 querystring 模块的区别：
      • URLSearchParams 是面向对象的 API，支持动态修改；
      • querystring 是 Node.js 模块，主要提供 parse 和 stringify 函数；
      • URLSearchParams 跨浏览器和 Node.js 环境，而 querystring 仅在 Node.js 中可用。
