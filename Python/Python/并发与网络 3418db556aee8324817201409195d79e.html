  • 进程与线程

    线程：计算机中可以被cpu调度的最小单元（真正在工作）

    进程：是计算机资源分配的最小单元

    一个进程中可以有多个线程，同一个进程中的线程可以共享此进程中的资源

    import multiprocessing
import requests

def download_file(url, filename):
    """下载文件并保存到本地"""
    try:
        response = requests.get(url, stream=True)
        response.raise_for_status()  # 检查请求是否成功
        
        with open(filename, 'wb') as f:
            for chunk in response.iter_content(chunk_size=8192):
                f.write(chunk)
        
        print(f"下载完成: {filename}")
    except Exception as e:
        print(f"下载失败 {filename}: {e}")

if __name__ == "__main__":
    # 文件列表：(文件名, 下载URL)
    file_list = [
        ("file1.txt", "https://example.com/file1.txt"),
        ("file2.txt", "https://example.com/file2.txt"),
        ("file3.txt", "https://example.com/file3.txt"),
    ]
    
    # 创建并启动进程
    processes = []
    for filename, url in file_list:
        p = multiprocessing.Process(target=download_file, args=(url, filename))
        processes.append(p)
        p.start()
    
    # 等待所有进程完成
    for p in processes:
        p.join()
    
    print("所有下载任务已完成")
    维度多进程（Multiprocessing）多线程（Multithreading）
    基本单位操作系统分配资源的最小单位（如内存、CPU 时间片）进程内的执行单元，共享进程的资源（内存、文件句柄等）
    资源隔离每个进程有独立的地址空间，数据不共享同一进程内的线程共享内存，数据可直接访问
    切换开销进程切换需保存整个上下文（内存状态、寄存器等），开销大线程切换只需保存少量寄存器状态，开销较小
    并发实现操作系统层面调度，可利用多核 CPU 真正并行执行操作系统调度线程，但受 GIL 限制（Python 中）无法真正并行
    创建代价启动进程需复制进程空间，耗时较长（毫秒级）启动线程只需分配栈空间，耗时极短（微秒级）
  • GIL锁

    GIL，全局解释器锁，是CPython解释器特有

    让一个进程中同一个时刻只有一个线程可以被cpu调用（同一时刻，仅允许一个线程执行 Python 字节码）

    1、利用计算机多核优势，如果想让CPU同时处理一些任务，适合用多进程开发（资源开销大）

    如果程序时计算密集型，用多进程

    >>> 多线程在 CPU 密集型任务中比单线程更慢（因线程切换增加额外开销）。

    2、不利用多核优势，适合用多线程开发

    如果时IO密集类型，用多线程（网络请求，文件读写）

    >>> GIL不影响IO密集型多线程效率，甚至因线程切换开销小，而性能优异

    >>> 每个进程有独立的 Python 解释器和 GIL，互不干扰。

    进程是计算机中资源分配的最小单位；一个进程中存在多个线程，同一个进程中的线程共享资源。

    • fork

      unix专属，高效高风险

      >>> 直接赋值父进程内存（写的时候复制，实际修改才拷贝），子进程继承所有变量，模块，文件句柄等。

      >>> 若父进程有多线程和锁，子进程可能复制锁但为负值解锁线程，导致死锁

      """ Unix系统运行 """
import os
import time

global_var = "父进程的变量" # 子进程直接继承

def worker():
	print("aa")
	print("bb")

if __name__ == "__main__":
	pid=os.fork() # 创建子进程
	if pid==0:
		worker() # 子进程执行
	else:
		time.sleep(1)
		print("cc")

      子进程直接访问 global_var，无需传参，启动极快。

    • spawn

      跨平台，安全但缓慢

      >>>子进程重启Python解释器，仅继承显示传递的参数，父进程的全局变量，模块需要重新加载。

      >>>Windows仅支持spawn，且必须再__main__中启动，否则创建递归进程

      from multiprocessing import Process, set_start_method
import os

def worker(name):
    print(f"子进程: {name}, PID: {os.getpid()}")
    # 无法直接访问父进程的 global_var（若定义在 __main__ 外会报错）
    print(f"参数: {name}")

if __name__ == "__main__":
    set_start_method('spawn')  # 显式启用 spawn 模式（Windows 默认无需显式）
    p = Process(target=worker, args=("任务1",))
    p.start()
    p.join()
    print("父进程结束")

      子进程需通过 args 传参，启动时重新加载模块，速度慢但内存省。

    • forkserver

      Unix专属，折中方案

      >>>

      1. 主进程先启动一个 “服务器进程”（仅加载 Python 解释器和基础模块）。
      1. 当需要创建子进程时，主进程通过 Unix socket 通知服务器，由服务器调用 fork 创建子进程。
      1. 子进程继承服务器进程的干净状态（无主进程的多线程锁、大量变量等），避免死锁。

      >>> 多线程环境下创建子进程，或频繁创建 / 销毁进程的场景。

      from multiprocessing import Process, set_start_method
import threading
import os

lock = threading.Lock()  # 父进程的锁
shared_data = 0

def worker():
    global shared_data
    with lock:  # forkserver 模式下，子进程不会继承主进程的锁！
        shared_data += 1
    print(f"子进程修改后: {shared_data}")

if __name__ == "__main__":
    set_start_method('forkserver')  # 启动 forkserver 模式
    p = Process(target=worker)
    p.start()
    p.join()
    print(f"父进程最终: {shared_data}")  # 输出 1（无锁竞争，安全）
