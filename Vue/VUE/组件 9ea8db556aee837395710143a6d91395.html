  • 生命周期

    Vue 的生命周期指的是组件从创建到销毁的完整过程，每个阶段会触发特定的 “钩子函数”，开发者可通过这些钩子在不同阶段执行逻辑（如初始化数据、操作 DOM、清理资源等）。

    • vue2（选项式 API）
      钩子函数触发时机核心作用可操作资源
      beforeCreate实例初始化后，数据观测（data）前组件刚创建，尚未初始化数据和方法无法访问 data、methods、DOM（$el 不存在）
      created实例创建完成，数据观测和方法初始化后初始化数据、发送异步请求（如接口调用）、定义初始状态可访问 data、methods，但无法操作 DOM（$el 仍未生成）
      beforeMount模板编译完成，挂载到 DOM 前可修改数据（不会触发重新渲染）模板已编译，但未挂载到页面，$el 为虚拟 DOM（非真实 DOM）
      mounted模板挂载到真实 DOM 后操作 DOM（如初始化第三方库：ECharts、地图）、监听 DOM 事件可访问真实 DOM（$el 存在），子组件也已挂载完成
      beforeUpdate数据更新后，DOM 重新渲染前获取更新前的 DOM 状态（如旧数据对应的 DOM）可访问更新前的 DOM 和数据，修改数据不会触发额外更新
      updated数据更新且 DOM 重新渲染后处理更新后的 DOM 逻辑（如基于新 DOM 计算尺寸）可访问更新后的 DOM 和数据，禁止在此修改数据（会导致无限循环）
      beforeDestroy实例销毁前清理资源（清除定时器、解绑事件监听、取消未完成的请求）实例仍完整可用，可访问 data、DOM 等
      destroyed实例销毁后最终清理（极少用，一般在 beforeDestroy 中完成）所有指令、事件监听、子组件已销毁，data 和 DOM 不可靠再用

      父子组件生命周期执行顺序

      父子组件的生命周期会按 “先父后子，再父后子” 的顺序执行：

      • 挂载阶段：父 beforeCreate → 父 created → 父 beforeMount → 子 beforeCreate → 子 created → 子 beforeMount → 子 mounted → 父 mounted
      • 更新阶段：父 beforeUpdate → 子 beforeUpdate → 子 updated → 父 updated
      • 销毁阶段：父 beforeDestroy → 子 beforeDestroy → 子 destroyed → 父 destroyed
    • vue3（组合式 API为主）

      Vue 3 兼容 Vue 2 的大部分生命周期钩子，但推荐使用组合式 API 的生命周期函数（更贴合逻辑聚合）。核心变化：

      • 移除 beforeCreate 和 created（由 setup 替代，setup 在这两个钩子前执行）；
      • 销毁相关钩子重命名（beforeDestroy → onBeforeUnmount，destroyed → onUnmounted）；
      组合式 API 钩子对应 Vue 2 钩子触发时机核心作用
      setupbeforeCreate + created实例初始化后立即执行初始化数据、定义响应式变量、注册方法（替代 Vue 2 的 data 和 methods）
      onBeforeMountbeforeMount模板挂载前同 Vue 2 beforeMount
      onMountedmounted模板挂载后同 Vue 2 mounted
      onBeforeUpdatebeforeUpdate数据更新、DOM 渲染前同 Vue 2 beforeUpdate
      onUpdatedupdated数据更新、DOM 渲染后同 Vue 2 updated
      onBeforeUnmountbeforeDestroy实例销毁前同 Vue 2 beforeDestroy
      onUnmounteddestroyed实例销毁后同 Vue 2 destroyed

      Vue 3 新增钩子（调试 / 特殊场景）

      • onRenderTracked：组件渲染时追踪依赖（调试用，查看哪些数据被追踪）；
      • onRenderTriggered：组件更新时触发（调试用，查看是什么数据导致更新）；
      • onErrorCaptured：捕获子组件抛出的错误（返回 false 可阻止错误向上传播）。
    • 区别
      1. Vue 2 vs Vue 3：
        • Vue 2 用选项式 API（methods、data 中定义钩子）；
        • Vue 3 推荐组合式 API（setup 中通过 import 引入钩子，如 import { onMounted } from 'vue'）。
      1. 常见场景：
        • 初始化数据 / 发请求：Vue 2 用 created，Vue 3 用 setup；
        • 操作 DOM：两者均用 mounted/onMounted；
        • 清理资源：两者均用 beforeDestroy/onBeforeUnmount。
      1. 注意点：
        • updated/onUpdated 中禁止修改数据（避免无限更新）；
        • 父子组件通信相关逻辑，需遵循生命周期执行顺序（如父组件在 mounted 中给子组件传值，子组件可在 mounted 中接收）。
  • 组件的基本定义

    组件本质是 “可复用的 Vue 实例”，拥有与根实例类似的选项（data、methods、computed 等），但 **data 必须是函数 **（避免组件复用时光芒数据污染）。

    1、vue2中定义组件（选项式API）

    // 定义一个简单的组件
const MyComponent = {
  // data 必须是函数，返回一个对象
  data() {
    return {
      message: "我是组件"
    }
  },
  template: `
    <div class="my-component">
      <p>{{ message }}</p>
    </div>
  `,
  methods: {
    handleClick() {
      console.log("组件被点击");
    }
  }
};

    2、vue3中定义组件（组合式API，<script setup>）

    <template>
  <div class="my-component">
    <p>{{ message }}</p>
    <button @click="handleClick">点击</button>
  </div>
</template>

<script setup>
import { ref } from 'vue';
// 响应式数据
const message = ref("我是Vue3组件");
// 方法
const handleClick = () => {
  console.log("组件被点击");
};
</script>
  • 全局组件与局部组件

    组件需 “注册” 后才能使用，根据注册范围分为全局组件和局部组件。

    1、全局组件

    • 特点：注册后可在任何 Vue 实例或组件的模板中使用（包括子组件内部）。
    • 注册方式：通过 Vue.component(组件名, 组件配置) 注册（Vue2）；Vue3 中通过 app.component(组件名, 组件) 注册。

    1、vue2中全局组件示例

    <div id="app">
  <!-- 全局组件可直接使用 -->
  <global-component></global-component>
</div>

<script>
// 1. 定义组件配置
const GlobalComponent = {
  template: `<div>我是全局组件</div>`
};

// 2. 全局注册（需在根实例创建前注册）
Vue.component('global-component', GlobalComponent);

// 3. 创建根实例
new Vue({ el: '#app' });
</script>

    2、vue3中全局组件示例

    <div id="app">
  <global-component></global-component>
</div>

<script>
import { createApp } from 'vue';
// 定义组件
const GlobalComponent = {
  template: `<div>我是Vue3全局组件</div>`
};

// 创建app实例并全局注册组件
const app = createApp({});
app.component('global-component', GlobalComponent); // 全局注册
app.mount('#app');
</script>

    2、局部组件

    • 特点：仅能在注册它的父组件模板中使用，避免全局污染。
    • 注册方式：在父组件的 components 选项中声明（Vue2）；Vue3 中在 <script setup> 中直接引入即可（自动局部注册）。

    1、vue2局部组件示例

    <div id="app">
  <!-- 局部组件只能在当前根实例中使用 -->
  <local-component></local-component>
</div>

<script>
// 1. 定义局部组件
const LocalComponent = {
  template: `<div>我是局部组件</div>`
};

// 2. 在父组件（此处为根实例）的 components 中注册
new Vue({
  el: '#app',
  components: {
    'local-component': LocalComponent // 键为组件名，值为组件配置
  }
});
</script>

    2、vue3局部组件示例（<script setup>）

    <!-- 父组件 App.vue -->
<template>
  <div>
    <!-- 直接使用引入的局部组件 -->
    <local-component></local-component>
  </div>
</template>

<script setup>
// 引入组件（自动局部注册，无需显式声明）
import LocalComponent from './LocalComponent.vue';
</script>

    全局 vs 局部组件：如何选择？

    • 全局组件：适合通用组件（如按钮、输入框），需频繁在各处使用。
    • 局部组件：适合业务相关组件（如订单列表、用户信息），仅在特定场景使用，避免全局命名冲突。
  • 父传子（通过 props 传递数据）

    父组件向子组件传递数据的核心是 props：父组件通过 “自定义属性” 传递数据，子组件通过 props 选项声明接收的数据。

    1、基本用法（vue2）

    <!-- 父组件 -->
<template>
  <div>
    <!-- 父组件通过自定义属性传递数据 -->
    <child-component 
      :title="parentTitle" 
      :user-age="userAge"
    ></child-component>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';
export default {
  components: { ChildComponent },
  data() {
    return {
      parentTitle: "来自父组件的标题",
      userAge: 20
    }
  }
};
</script>


<!-- 子组件 ChildComponent.vue -->
<template>
  <div>
    <p>标题：{{ title }}</p>
    <p>年龄：{{ userAge }}</p>
  </div>
</template>

<script>
export default {
  // 子组件通过 props 声明接收的属性（数组形式，仅指定名称）
  props: ['title', 'userAge'], 
  // 注意：HTML中属性名不区分大小写，父组件传递时建议用 kebab-case（如 user-age），子组件 props 用 camelCase（userAge）自动映射
}
</script>

    2、props的单项数据流

    • 规则：父组件数据更新时，子组件会自动同步，但子组件不能直接修改 props（避免数据流混乱）。
    • 解决：若子组件需修改 props，可将 props 赋值给子组件的 data，或通过计算属性处理。
    // 子组件中安全使用 props
export default {
  props: ['initialCount'],
  data() {
    return {
      // 将 props 赋值给 data，子组件修改 count 不会影响父组件
      count: this.initialCount 
    }
  },
  methods: {
    increment() {
      this.count++; // 安全修改
    }
  }
}
  • 子转父（通过 $emit 触发事件）

    子组件向父组件传递数据的核心是 $emit：子组件通过 $emit('事件名', 数据) 触发自定义事件，父组件通过 @事件名 监听并接收数据。

    1、基本用法（vue2）

    <!-- 子组件 ChildComponent.vue -->
<template>
  <button @click="sendDataToParent">向父组件传值</button>
</template>

<script>
export default {
  data() {
    return {
      childMessage: "我是子组件的数据"
    }
  },
  methods: {
    sendDataToParent() {
      // 触发自定义事件（事件名建议用 kebab-case），并传递数据
      this.$emit('send-message', this.childMessage, 123); 
    }
  }
}
</script>


<!-- 父组件 -->
<template>
  <div>
    <!-- 父组件监听子组件的自定义事件 -->
    <child-component @send-message="handleMessage"></child-component>
    <p>从子组件接收的数据：{{ receivedData }}</p>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';
export default {
  components: { ChildComponent },
  data() {
    return {
      receivedData: null
    }
  },
  methods: {
    // 接收子组件传递的参数（按触发时的顺序接收）
    handleMessage(msg, num) {
      this.receivedData = `消息：${msg}，数字：${num}`;
    }
  }
}
</script>

    2、vue3中的子传父（emit函数）

    Vue3 中通过 defineEmits 声明自定义事件（更规范），使用 emit 函数触发：

    <!-- 子组件 Child.vue -->
<template>
  <button @click="sendData">传递数据</button>
</template>

<script setup>
// 声明自定义事件（返回 emit 函数）
const emit = defineEmits(['send-message']); 
const sendData = () => {
  emit('send-message', 'Vue3子组件数据'); // 触发事件
};
</script>


<!-- 父组件 Parent.vue -->
<template>
  <child-component @send-message="handleMsg"></child-component>
</template>

<script setup>
import ChildComponent from './Child.vue';
const handleMsg = (data) => {
  console.log('接收数据：', data);
};
</script>
  • 属性验证与默认属性

    props 支持属性验证（确保传递的数据类型 / 格式正确）和默认属性（当父组件未传递时使用默认值），通过 “对象形式的 props” 配置。

    1、基本验证规则（Vue2）

    export default {
  // props 为对象形式，键是属性名，值是验证规则
  props: {
    // 1. 类型验证（Number/String/Boolean/Array/Object/Function）
    age: {
      type: Number, // 必须是数字类型
      required: true // 父组件必须传递该属性
    },

    // 2. 默认值（当父组件未传递时使用）
    name: {
      type: String,
      default: '默认名称' // 字符串默认值
    },

    // 3. 数组/对象的默认值（必须是函数返回）
    tags: {
      type: Array,
      default: () => ['前端', 'Vue'] // 数组默认值需用函数
    },

    // 4. 自定义验证函数（返回 true 则验证通过）
    score: {
      type: Number,
      validator: (value) => {
        // 验证规则：分数必须在 0-100 之间
        return value >= 0 && value <= 100;
      }
    }
  }
}

    2、验证失败后果

    • 开发环境下，Vue 会在控制台抛出警告（如 “属性 age 期望是 Number 但得到 String”）。
    • 生产环境下，验证会被跳过（不影响性能）。

    3、Vue3中属性验证（defineProps）

    Vue3 中通过 defineProps 的泛型或对象形式声明验证规则：

    <script setup>
// 对象形式声明 props 及验证规则
const props = defineProps({
  age: {
    type: Number,
    required: true
  },
  name: {
    type: String,
    default: '默认名称'
  }
});
</script>
  • 中间人模式（状态提升）

    定义：当多个子组件需要共享状态或相互通信时，将共享状态 “提升” 到它们的共同父组件（中间人），通过父组件中转数据的模式。

    核心逻辑：子组件 → 父组件（通过 $emit）传递数据，父组件 → 其他子组件（通过 props）传递数据，实现子组件间间接通信。

    原理：

    • 子组件通过 $emit 将数据变化通知父组件；
    • 父组件更新自身状态后，通过 props 将新状态传递给所有依赖该状态的子组件；
    • 适用于层级较浅的组件间通信（避免深层嵌套导致的代码冗余）。

    场景：两个子组件共享“计数器”状态

    <!-- 父组件（中间人） -->
<template>
  <div>
    <ChildA :count="count" @update-count="count = $event"></ChildA>
    <ChildB :count="count" @update-count="count = $event"></ChildB>
  </div>
</template>

<script>
import ChildA from './ChildA.vue';
import ChildB from './ChildB.vue';
export default {
  components: { ChildA, ChildB },
  data() { return { count: 0 }; } // 共享状态提升到父组件
};
</script>


<!-- 子组件 ChildA.vue -->
<template>
  <div>
    <p>ChildA: {{ count }}</p>
    <button @click="$emit('update-count', count + 1)">+1</button>
  </div>
</template>
<script> export default { props: ['count'] }; </script>


<!-- 子组件 ChildB.vue -->
<template>
  <div>
    <p>ChildB: {{ count }}</p>
    <button @click="$emit('update-count', count - 1)">-1</button>
  </div>
</template>
<script> export default { props: ['count'] }; </script>
  • 中央事件总线（EventBus）

    定义：通过一个全局的 “事件中心” 实现任意组件间通信（包括非父子、跨层级组件），组件可向中心发送 / 监听事件。

    核心逻辑：创建一个空的 Vue 实例作为事件总线，通过 $on 监听事件、$emit 发送事件、$off 移除事件。

    1、vue2实现EventBus

    // 1. 创建事件总线（bus.js）
import Vue from 'vue';
export default new Vue(); // 空Vue实例作为总线


// 2. 组件A（发送事件）
import bus from './bus.js';
export default {
  methods: {
    sendData() {
      // 发送事件：第一个参数是事件名，后续是传递的数据
      bus.$emit('send-message', '来自组件A的数据', 123);
    }
  }
};


// 3. 组件B（监听事件）
import bus from './bus.js';
export default {
  mounted() {
    // 监听事件：第一个参数是事件名，第二个是回调函数
    this.eventHandler = (msg, num) => {
      console.log('接收数据：', msg, num); // 输出：来自组件A的数据 123
    };
    bus.$on('send-message', this.eventHandler);
  },
  beforeDestroy() {
    // 移除事件监听（必须！避免内存泄漏）
    bus.$off('send-message', this.eventHandler);
  }
};

    2、vue3实现（使用mitt库）

    Vue3 移除了 $on/$off，推荐用轻量库 mitt 替代：

    npm install mitt
    // 1. 创建总线（bus.js）
import mitt from 'mitt';
export default mitt(); // 创建mitt实例


// 2. 组件A（发送事件）
import bus from './bus.js';
const sendData = () => {
  bus.emit('send-message', 'Vue3数据'); // 发送事件
};


// 3. 组件B（监听事件）
import bus from './bus.js';
import { onMounted, onUnmounted } from 'vue';

onMounted(() => {
  // 监听事件
  const handler = (data) => {
    console.log('接收：', data);
  };
  bus.on('send-message', handler);

  // 组件卸载时移除监听
  onUnmounted(() => {
    bus.off('send-message', handler);
  });
});

    注意：

    • 必须在组件销毁时移除事件监听，否则可能导致多次触发；
    • 适用于中小型项目，大型项目建议用 Vuex/Pinia 管理状态。
  • ref 组件通信

    定义：父组件通过 ref 属性获取子组件实例或 DOM 元素，直接访问子组件的数据 / 方法，实现父子组件通信。

    1、访问DOM元素

    <template>
  <div>
    <input ref="inputRef" type="text">
    <button @click="focusInput">聚焦输入框</button>
  </div>
</template>

<script>
// Vue2
export default {
  methods: {
    focusInput() {
      this.$refs.inputRef.focus(); // 通过$refs获取DOM并调用方法
    }
  }
};

// Vue3（<script setup>）
import { ref } from 'vue';
const inputRef = ref(null); // 与模板中ref="inputRef"对应
const focusInput = () => {
  inputRef.value.focus(); // .value访问DOM
};
</script>

    2、访问子组件实例

    <!-- 子组件 Child.vue -->
<template>
  <div>{{ count }}</div>
</template>
<script>
// Vue2
export default {
  data() { return { count: 0 }; },
  methods: {
    increment() { this.count++; }
  }
};

// Vue3（<script setup>需暴露方法）
import { ref } from 'vue';
const count = ref(0);
const increment = () => { count.value++; };
// 暴露给父组件的属性/方法（setup语法糖中默认私有）
defineExpose({ count, increment });
</script>


<!-- 父组件 Parent.vue -->
<template>
  <div>
    <Child ref="childRef"></Child>
    <button @click="callChildMethod">调用子组件方法</button>
  </div>
</template>

<script>
// Vue2
import Child from './Child.vue';
export default {
  components: { Child },
  methods: {
    callChildMethod() {
      this.$refs.childRef.increment(); // 调用子组件方法
      console.log('子组件count：', this.$refs.childRef.count); // 访问子组件数据
    }
  }
};

// Vue3（<script setup>）
import { ref } from 'vue';
import Child from './Child.vue';
const childRef = ref(null);
const callChildMethod = () => {
  childRef.value.increment(); // 调用子组件方法
  console.log('子组件count：', childRef.value.count);
};
</script>

    注意：

    • 避免过度依赖 ref 直接操作子组件（破坏封装性），优先用 props/$emit；
    • ref 访问在组件挂载后才有效（mounted 钩子中）。
  • 动态组件

    定义：通过 <component :is="组件名"></component> 动态渲染不同组件，实现组件的切换展示。

    1、基本用法：

    <template>
  <div>
    <!-- 按钮切换组件 -->
    <button @click="currentComponent = 'ComponentA'">显示A</button>
    <button @click="currentComponent = 'ComponentB'">显示B</button>
    
    <!-- 动态组件：:is绑定组件名 -->
    <component :is="currentComponent"></component>
  </div>
</template>

<script>
import ComponentA from './ComponentA.vue';
import ComponentB from './ComponentB.vue';

export default {
  components: { ComponentA, ComponentB }, // 局部注册组件
  data() {
    return {
      currentComponent: 'ComponentA' // 默认显示A
    }
  }
};
</script>

    2、结合keep-alive缓存组件状态

    默认切换动态组件时，旧组件会被销毁，新组件重新创建。keep-alive 可缓存组件实例，保留状态（如表单输入值）：

    <!-- 缓存所有动态组件 -->
<keep-alive>
  <component :is="currentComponent"></component>
</keep-alive>

<!-- 只缓存指定组件（include/exclude） -->
<keep-alive include="ComponentA,ComponentB" exclude="ComponentC">
  <component :is="currentComponent"></component>
</keep-alive>

    keep-alive 相关生命周期

    • 组件被缓存时触发 deactivated；
    • 组件被激活时触发 activated。
  • slot 插槽

    定义：插槽是组件的 “占位符”，允许父组件向子组件插入自定义内容，实现组件内容的灵活定制。

    核心：作用域插槽实现了 “子组件提供数据，父组件决定渲染方式” 的灵活协作。

    1、默认插槽（匿名插槽）

    子组件定义一个无名称的插槽，父组件插入的内容会填充到该插槽。

    <!-- 子组件 Child.vue -->
<template>
  <div class="card">
    <div class="header">卡片标题</div>
    <!-- 默认插槽：父组件内容会插入这里 -->
    <slot></slot> 
    <div class="footer">卡片底部</div>
  </div>
</template>


<!-- 父组件使用 -->
<template>
  <Child>
    <!-- 插入到子组件的默认插槽 -->
    <p>这是父组件插入的内容</p>
    <button>父组件的按钮</button>
  </Child>
</template>

    2、具名插槽（Named Slot）

    当子组件有多个插槽时，用 name 属性区分，父组件通过 v-slot:名称（简写 #名称）指定插入位置。

    <!-- 子组件 Child.vue -->
<template>
  <div class="layout">
    <!-- 头部插槽 -->
    <slot name="header"></slot>
    <!-- 内容插槽 -->
    <slot name="content"></slot>
    <!-- 底部插槽 -->
    <slot name="footer"></slot>
  </div>
</template>


<!-- 父组件使用（Vue2/Vue3通用） -->
<template>
  <Child>
    <!-- 插入到header插槽 -->
    <template v-slot:header>
      <h1>页面标题</h1>
    </template>
    
    <!-- 插入到content插槽（简写#content） -->
    <template #content>
      <p>页面内容...</p>
    </template>
    
    <!-- 插入到footer插槽 -->
    <template #footer>
      <p>版权信息</p>
    </template>
  </Child>
</template>

    3、作用域插槽（Scoped Slot）

    子组件向插槽传递数据（子组件 → 父组件），父组件可使用这些数据定制插槽内容。

    <!-- 子组件 Child.vue -->
<template>
  <div>
    <!-- 子组件向插槽传递数据（:user="user"） -->
    <slot name="user-info" :user="user" :age="20"></slot>
  </div>
</template>
<script>
export default {
  data() { return { user: { name: '张三' } }; }
};
</script>


<!-- 父组件使用 -->
<template>
  <Child>
    <!-- 通过v-slot接收子组件传递的数据（user和age） -->
    <template #user-info="slotProps">
      <p>姓名：{{ slotProps.user.name }}</p>
      <p>年龄：{{ slotProps.age }}</p>
    </template>
    
    <!-- 解构赋值简化（推荐） -->
    <template #user-info="{ user, age }">
      <p>姓名：{{ user.name }}，年龄：{{ age }}</p>
    </template>
  </Child>
</template>
  • 过度（transition）与动画

    Vue 提供 <transition> 组件封装过渡效果，通过 CSS 类或 JavaScript 钩子实现元素进入 / 离开的动画。

    1、单个元素/组件的过度（<transition>）

    <template>
  <div>
    <button @click="show = !show">切换</button>
    <!-- 包裹需要过渡的元素，name指定过渡类的前缀 -->
    <transition name="fade">
      <p v-if="show">这是过渡元素</p>
    </transition>
  </div>
</template>

<script>
export default { data() { return { show: true }; } };
</script>

<style>
/* 进入/离开的初始状态 */
.fade-enter-from, .fade-leave-to {
  opacity: 0;
  transform: translateX(20px);
}

/* 进入/离开的过渡状态 */
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.3s, transform 0.3s;
}

/* 进入结束的状态（可选，通常与默认一致） */
.fade-enter-to, .fade-leave-from {
  opacity: 1;
  transform: translateX(0);
}
</style>

    过渡类名说明（以 name="fade" 为例）：

    • fade-enter-from：进入开始（瞬间）；
    • fade-enter-active：进入过程中（持续到结束）；
    • fade-enter-to：进入结束（瞬间）；
    • fade-leave-from：离开开始（瞬间）；
    • fade-leave-active：离开过程中（持续到结束）；
    • fade-leave-to：离开结束（瞬间）。

    2、列表过度（<transition-group>）

    用于 v-for 渲染的列表，需为每个子元素设置唯一 key：

    list-move 类：当列表项位置变化（如删除中间项）时，触发移动动画，实现平滑过渡。

    <template>
  <div>
    <button @click="addItem">添加</button>
    <!-- 列表过渡：必须指定tag（渲染为指定标签） -->
    <transition-group name="list" tag="ul">
      <li v-for="item in list" :key="item.id">
        {{ item.text }}
        <button @click="removeItem(item.id)">删除</button>
      </li>
    </transition-group>
  </div>
</template>

<script>
export default {
  data() {
    return { list: [], nextId: 1 };
  },
  methods: {
    addItem() {
      this.list.push({ id: this.nextId++, text: `项目${this.nextId}` });
    },
    removeItem(id) {
      this.list = this.list.filter(item => item.id !== id);
    }
  }
};
</script>

<style>
/* 进入/离开动画 */
.list-enter-from, .list-leave-to {
  opacity: 0;
  transform: translateY(30px);
}
.list-enter-active, .list-leave-active {
  transition: all 0.3s;
}

/* 列表项移动动画（关键！） */
.list-move {
  transition: transform 0.3s;
}

/* 避免删除时元素突然消失 */
.list-leave-active {
  position: absolute;
}
</style>

    3、可复用过度（封装为组件）

    将 <transition> 封装为可复用组件，在多个地方使用：

    <!-- FadeTransition.vue（可复用过渡组件） -->
<template>
  <transition name="fade">
    <slot></slot> <!-- 插槽接收需要过渡的内容 -->
  </transition>
</template>

<style>
.fade-enter-from, .fade-leave-to { opacity: 0; }
.fade-enter-active, .fade-leave-active { transition: opacity 0.3s; }
</style>


<!-- 使用可复用过渡 -->
<template>
  <FadeTransition>
    <p v-if="show">使用复用的过渡效果</p>
  </FadeTransition>
</template>

<script>
import FadeTransition from './FadeTransition.vue';
export default { components: { FadeTransition }, data() { return { show: true }; } };
</script>
  • Diff算法

    定义：Vue 的虚拟 DOM（Virtual DOM）对比算法，用于计算新旧虚拟 DOM 树的差异，只更新变化的部分（最小化 DOM 操作），提升性能。

    核心原理

    1. 同级对比：只对比同一层级的节点（不跨层级对比），降低复杂度（时间复杂度 O (n)）。
    1. 节点类型判断：
      • 若标签名不同（如 div → p），直接销毁旧节点，创建新节点；
      • 若标签名相同，对比属性（class、style 等），更新变化的属性。
    1. 列表节点对比：依赖 key 标识节点唯一性：
      • 有 key 时：通过 key 匹配新旧节点，只移动 / 更新变化的节点；
      • 无 key 时：默认按索引匹配，可能导致错误复用（如列表排序 / 删除时状态错乱）。

    为什么key很重要?

    <!-- 反例：用索引作为key，删除第一项时的问题 -->
<ul>
  <li v-for="(item, index) in list" :key="index">
    {{ item }} <input type="text">
  </li>
</ul>
    • 原始列表：[A, B, C] → 输入框分别输入 a, b, c；
    • 删除第一项后，新列表为 [B, C]，索引变为 0, 1；
    • Vue 会认为索引 0 的节点还是原来的节点，复用输入框，导致 B 对应的输入框显示 a（错误复用）。

    正例：用唯一 id 作为 key，确保节点正确匹配，避免状态错乱。

    vue3 对 Diff 算法的优化

    Vue3 对 Diff 算法的优化

    • 静态节点提升：编译时标记不会变化的节点（如纯文本），diff 时直接跳过；
    • 事件缓存：相同事件处理函数（如 @click="handleClick"）缓存，避免重复创建；
    • 片段（Fragment）：支持多根节点，减少不必要的父容器 DOM 节点。

    总结：

    特性核心作用适用场景
    中间人模式父组件中转子组件数据浅层子组件间共享状态
    中央事件总线全局事件中心实现任意组件通信中小型项目跨组件通信
    ref 通信父组件直接访问子组件 / DOM父子组件紧急通信（谨慎使用）
    动态组件动态切换组件展示标签页、表单步骤等切换场景
    插槽（slot）父组件定制子组件内容组件内容灵活化（如卡片、列表项）
    Transition元素进入 / 离开动画单个元素切换（如弹窗、提示）
    TransitionGroup列表项添加 / 删除 / 移动动画动态列表（如待办事项、商品列表）
    Diff 算法高效对比虚拟 DOM，最小化 DOM 操作Vue 内部渲染优化（开发者需正确用 key）
