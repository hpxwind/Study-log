信息收集

主动搜集与被动搜集

收集内容的核心：

1、域名或子域名信息

子域名枚举，DNS记录，域名注册信息，域名历史解析记录

2、IP与端口信息

开放的端口，服务器真实IP，端口对应服务，服务版本

3、技术栈与架构信息

服务器软件，web架构，编程语言，CMS系统，数据库类型

4、敏感信息挖掘

目录遍历与文件泄露，robot.txt中的隐藏路径，公开github仓库

5、第三方证书等信息

工具：

1. 域名与子域名枚举工具

  • Amass（主动 + 被动）功能：多源子域名枚举（结合 DNS 服务器、搜索引擎、证书透明度日志等），支持被动模式（减少目标检测风险）。特点：数据来源广（超过 50 个数据源），可导出详细报告。
    • 安装与使用（kali）

      安装（Linux）

      # 方法一：直接下载预编译二进制文件（推荐）
wget https://github.com/OWASP/Amass/releases/download/v3.21.1/amass_linux_amd64.zip
unzip amass_linux_amd64.zip
cd amass_linux_amd64
chmod +x amass
./amass -h  # 验证安装

# 方法二：通过 go 命令安装（需先安装 Go 环境）
go install -v github.com/OWASP/Amass/v3/...@master

      配置API密钥

      Amass 通过第三方 API 获取更全面的数据，需在配置文件（默认 ~/.config/amass/config.ini）中添加 API 密钥：

      [virustotal]
apikey = your_virustotal_api_key

[shodan]
apikey = your_shodan_api_key

[censysio]
id = your_censys_id
secret = your_censys_secret

      基础用法：

      1、被动模式

      amass enum -d baidu.com -passive

      参数解析：

      • d example.com：指定目标域名。
      • passive：仅使用被动数据源（不发送 DNS 请求），隐蔽性高。

      2、主动模式（验证子域名有效性）

      amass enum -d example.com -active -brute -min-for-recursive 2

      参数解析：

      • active：主动验证子域名（发送 DNS 请求）。
      • brute：启用字典爆破（结合内置字典生成子域名）。
      • min-for-recursive 2：发现至少 2 个子域名时，递归爆破其下的子域名（如 dev.example.com → api.dev.example.com）。

      -o aubdomains.txt 输出到指定文件中

      其他用法

      1、指定数据源

      amass enum -d example.com -src -only censys,virustotal
      • src：显示每个子域名的来源。
      • only censys,virustotal：仅使用指定的数据源。

      2、结合自定义字典

      amass enum -d example.com -wordlist /path/to/wordlist.txt

      • -wordlist：指定自定义字典文件，可结合公开字典（如 SecLists/Discovery/DNS/）。

      3、网络拓扑分析

      amass intel -d example.com -asn -cidr
      amass enum -d example.com -o out.json
amass viz -d3 out.json  # 生成 D3.js 可视化页面
# 打开index.html，可查看域名，ip，ASN之间的关系
  • Sublist3r（被动为主）功能：基于搜索引擎（Google、Bing、Yahoo）和字典枚举子域名，轻量易用。特点：适合快速初步枚举，支持多线程。
  • OneForAll（国产工具）功能：整合多种子域名枚举方法（DNS 爆破、证书解析、API 查询等），支持批量处理。特点：中文友好，结果去重，适合国内目标。
    • 安装与使用（win）
      # 1. 克隆仓库
git clone https://github.com/shmilylty/OneForAll.git
cd OneForAll

# 2. 安装依赖
pip3 install -r requirements.txt

# 3. 配置 API（可选但推荐）
# 编辑 config/api.py，填入各类 API 密钥（如证书透明度、ZoomEye 等）

      基本使用

      进入文件目录

      1、单个扫描

      python oneforall.py --target baidu.com run
      • 关键参数：
        • -target：指定目标域名。
        • -output：指定结果输出路径（默认在 results 目录）。
        • -format：指定输出格式（csv、json、txt 等）。

      2、多个扫描

      python oneforall.py --targets domains.txt run

      • domains.txt 需包含多个域名，每行一个。

      3、指定扫描模块

      python oneforall.py --target baidu.com --module brute run
      • -module 可选值：
        • brute：字典爆破。
        • search：搜索引擎查询。
        • api：调用第三方 API。
        • all（默认）：启用所有模块。

      4、自定义字典

      python oneforall.py --target baidu.com --wordlist mydict.txt run

      • mydict.txt 需包含自定义子域名前缀（如 admin, api）。

      5、深度扫描

      python oneforall.py --target baidu.com --deep 2 run

      • mydict.txt 需包含自定义子域名前缀（如 admin, api）。

      1、调用api（需要预先配置）

      python3 oneforall.py --target example.com --api True run

      • 支持的 API 包括：证书透明度日志、DNSDB、Shodan 等。

      2、端口扫描

      python3 oneforall.py --target example.com --port medium run

      • --port 可选值：small（常用端口）、medium（500 个端口）、large（全端口）。

3. 端口扫描与服务识别工具

  • Nmap（主动扫描）功能：端口扫描（全端口 / 常用端口）、服务版本识别（sV参数）、操作系统探测（O）。示例：nmap -sV -p 1-1000 目标IP（扫描 1-1000 端口并识别服务版本）。特点：功能全面，支持脚本扩展（如漏洞检测脚本）。
  • Masscan（高速扫描）功能：超高速端口扫描（比 Nmap 快 10 倍以上），适合大规模 IP 段扫描。特点：速度快，但精度略低，需配合 Nmap 验证。
  • Zmap（批量扫描）功能：针对整个互联网的端口扫描（如扫描全网 80 端口开放情况），适合宏观统计。

4. 技术栈识别工具

  • Wappalyzer（浏览器插件 / CLI）功能：识别网站的服务器、框架、CMS、编程语言等技术栈（通过 HTTP 头、页面特征）。特点：图形化插件易用，CLI 版本支持批量扫描。
  • WhatWeb（命令行）功能：通过指纹库（如 HTML 标签、Cookie、响应头）识别网站技术栈，支持自定义规则。示例：whatweb 目标URL（输出目标使用的技术，如 “Nginx 1.21.0, WordPress 6.0”）。
  • BuiltWith（在线工具）功能：可视化展示网站技术栈（包括服务器、CDN、分析工具等），适合非技术人员。

5. 敏感信息挖掘工具

  • Dirsearch（目录爆破）功能：通过字典枚举网站隐藏目录和文件（如/admin、/backup.zip），支持自定义字典。示例：dirsearch -u 目标URL -w /usr/share/wordlists/dirb/common.txt。
  • Gobuster（目录 / 子域名爆破）功能：多模式（目录爆破、子域名爆破），支持 HTTP/HTTPS，速度快。
  • GitHack（代码泄露）功能：检测并下载目标网站的.git目录泄露（可能包含完整源代码）。
  • Burp Suite（集成工具）功能：爬虫（爬取网站所有链接）、参数分析、敏感信息搜索（如页面中的邮箱、密钥），是渗透测试核心工具。

6. 被动信息搜集平台

  • Shodan（设备搜索引擎）功能：搜索联网设备（服务器、摄像头等）的开放端口、服务信息（如 “apache country:CN” 搜索中国的 Apache 服务器）。特点：适合发现暴露在公网的敏感服务（如未授权的 MySQL）。
  • Censys（证书与设备搜索）功能：通过 SSL 证书、IP 端口信息搜索目标，支持复杂过滤（如 “certificate.issuer:Let's Encrypt”）。
  • ZoomEye（钟馗之眼，国产）功能：类似 Shodan，侧重国内目标，支持按地理位置、端口、服务搜索。
  • Google Hacking（搜索引擎语法）功能：通过特殊语法精准搜索目标信息，如：
    • site:目标域名 filetype:sql（搜索目标域名下的.sql 备份文件）；
    • site:目标域名 intitle:后台管理（搜索后台登录页）。

7. 自动化信息收集框架

  • Recon-ng功能：集成多种信息收集模块（子域名枚举、WHOIS 查询、端口扫描等），支持模块化扩展，适合自动化工作流。
  • SpiderFoot功能：开源情报收集框架，自动关联多源信息（域名、IP、邮箱等），生成可视化报告，适合新手。
