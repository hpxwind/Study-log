泛型

泛型（Generic）是 TypeScript 中类型复用与类型安全兼顾的核心特性，本质是 “类型参数化”—— 允许在定义函数、接口、类时，不预先指定具体类型，而是在使用时动态传入类型（类似函数的 “参数”，只不过参数是 “类型” 而非 “值”）。

  1. 避免重复编写多个相似类型的代码（如 string[]、number[] 可统一为 T[]）；
  1. 保持类型安全（替代 any，编译时就能检测类型错误，而非运行时）；
  1. 实现 “类型联动”（如函数输入类型与输出类型一致）。
  • 应用
    1. 通用工具函数：如数组处理（getFirst、flatten）、对象合并（merge）、类型转换（transform），避免重复编写 string/number/User 等专属函数；
    1. 数据结构封装：如栈、队列、链表、字典等，支持任意类型数据存储；
    1. API 响应封装：用泛型接口 ApiResponse<T> 统一响应格式，不同接口传入不同 T（如 User、User[]）；
    1. 前端组件封装：如 React/Vue 的通用组件（表格、弹窗、下拉框），通过泛型指定 props 类型（如 TableProps<T> 接收表格数据类型 T）；
    1. 类型工具库：如 lodash 的 TypeScript 版本，通过泛型保证函数输入输出的类型一致性。
  • 格式

    泛型的核心是类型参数（通常用 <T> 表示，T 是 Type 的缩写，也可用 U、V 等大写字母，语义化命名更佳，如 <Item>、<Data>）。

    // 泛型函数：<T> 是类型参数，使用时传入具体类型
function 函数名<T>(参数: T): T { ... }

// 泛型接口：类型参数可作用于属性/方法
interface 接口名<T> { ... }

// 泛型类：类型参数可作用于属性/方法/构造函数
class 类名<T> { ... }

    关键概念

    • 类型参数（Type Parameter）：<T> 中的 T，类似 “类型占位符”，使用时指定具体类型（如 string、User）；
    • 类型实参（Type Argument）：使用泛型时传入的具体类型（如 func<string>() 中的 string）；
    • 类型推断：TypeScript 可根据传入的 “值参数” 自动推导类型实参，无需显式声明（如 func(123) 自动推断 T=number）。

  • 泛型函数

    解决 “同一函数处理不同类型数据，但需保持类型一致” 的问题（替代 any，兼顾灵活与安全）。

    例如：处理数组的通用函数

    // 泛型函数：返回数组第一个元素，输入输出类型一致
function getFirstElement<T>(arr: T[]): T | undefined {
  return arr[0];
}

// 用法1：显式指定类型实参 <string>
const str = getFirstElement<string>(["a", "b", "c"]);
str.toUpperCase(); // 合法：TypeScript 知道 str 是 string

// 用法2：类型推断（推荐）：根据传入的数组类型自动推导 T=number
const num = getFirstElement([1, 2, 3]);
num.toFixed(2); // 合法：num 是 number

// 用法3：处理自定义类型数组
interface User { id: number; name: string }
const users: User[] = [{ id: 1, name: "张三" }, { id: 2, name: "李四" }];
const user = getFirstElement(users);
console.log(user?.name); // 合法：user 是 User 类型

    例如：多类型参数函数

    当函数需要处理多个不同类型的参数时，可定义多个类型参数（T、U、V 等）

    // 泛型函数：合并两个不同类型的对象，返回交叉类型 T & U
function merge<T, U>(obj1: T, obj2: U): T & U {
  return { ...obj1, ...obj2 };
}

// 类型推断：T=User，U=Contact（复用之前定义的 User/Contact 接口）
const merged = merge({ id: 1, name: "张三" }, { phone: "13800138000" });
console.log(merged.id); // 合法（来自 User）
console.log(merged.phone); // 合法（来自 Contact）

  • 泛型接口

    用于定义 “结构固定，但属性 / 方法类型可变” 的接口，常见于配置对象、API 响应、函数类型定义。

    示例：泛型接口定义对象解构

    // 泛型接口：Data 类型可变，结构固定（有 data 和 message 字段）
interface ApiResponse<T> {
  code: number;
  message: string;
  data: T; // 泛型字段：根据实际响应动态指定类型
}

// 用法1：响应数据是 User 类型
type UserResponse = ApiResponse<User>;
const userRes: UserResponse = {
  code: 200,
  message: "success",
  data: { id: 1, name: "张三" }
};

// 用法2：响应数据是 User[] 类型
type UserListResponse = ApiResponse<User[]>;
const userListRes: UserListResponse = {
  code: 200,
  message: "success",
  data: [{ id: 1, name: "张三" }, { id: 2, name: "李四" }]
};

    例如：泛型接口定义函数类型

    // 泛型接口：定义“输入 T 类型，返回 U 类型”的函数结构
interface TransformFn<T, U> {
  (input: T): U;
}

// 实现接口：将 number 转为 string
const numberToString: TransformFn<number, string> = (num) => {
  return num.toString();
};

// 实现接口：将 User 转为 { userId: number; userName: string }
const userTransform: TransformFn<User, { userId: number; userName: string }> = (user) => {
  return { userId: user.id, userName: user.name };
};
  • 泛型类

    用于创建 “属性 / 方法类型可变” 的类，常见于数据结构（如栈、队列、链表）。

    例如：泛型栈实现

    class Stack<T> {
  private items: T[] = [];

  // 入栈：接收 T 类型值
  push(item: T): void {
    this.items.push(item);
  }

  // 出栈：返回 T 类型值（或 undefined）
  pop(): T | undefined {
    return this.items.pop();
  }

  // 查看栈顶：返回 T 类型值（或 undefined）
  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }
}

// 用法1：创建 number 类型的栈
const numberStack = new Stack<number>();
numberStack.push(1);
numberStack.push(2);
const num = numberStack.pop(); // num 是 number 类型

// 用法2：创建 string 类型的栈
const stringStack = new Stack<string>();
stringStack.push("a");
stringStack.push("b");
const str = stringStack.peek(); // str 是 string 类型

// 用法3：创建 User 类型的栈
const userStack = new Stack<User>();
userStack.push({ id: 1, name: "张三" });
const user = userStack.peek(); // user 是 User 类型
  • 泛型约束（Generic Constraints）

    默认情况下，泛型 T 可以是任意类型，但有时需要限制泛型的范围（如要求 T 必须包含某个属性、继承某个接口），避免访问不存在的属性 / 方法。

    语法：用 extends 指定约束条件（T extends 约束类型）。

    例如：约束泛型必须包含某个属性

    // 定义约束接口：必须有 id 属性
interface HasId {
  id: number;
}

// 泛型约束：T 必须继承 HasId（即 T 必须有 id 属性）
function getId<T extends HasId>(obj: T): number {
  return obj.id; // 此时可安全访问 id，不会报错
}

// 合法：User 接口有 id 属性，满足约束
getId({ id: 1, name: "张三" });

// 合法：直接传入有 id 的对象
getId({ id: 2 });

// 非法：对象没有 id 属性，不满足约束 → 报错
// getId({ name: "李四" }); // 报错：Property 'id' is missing

    例如：约束泛型为对象类型（排除基本类型）

    // 约束 T 为对象类型（object 包含数组、对象，但排除 string/number/boolean 等）
function getProp<T extends object, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]; // 安全访问 obj 的 key 属性
}

// 合法：obj 是对象，key 是 obj 的属性
const user = { id: 1, name: "张三" };
getProp(user, "name"); // 返回 string 类型

// 合法：数组也是 object，key 是数组的索引（number 类型）
const arr = [1, 2, 3];
getProp(arr, 0); // 返回 number 类型

// 非法：obj 是基本类型（string），不满足 T extends object → 报错
// getProp("hello", "length"); // 报错：Type 'string' does not satisfy the constraint 'object'
    关键：K extends keyof T 是 “索引类型查询”，
keyof T 会提取 T 的所有属性名组成联合类型（如 keyof User = "id" | "name"），
确保 key 是 T 中存在的属性，避免访问不存在的键。
  • 默认类型参数（Default Type Parameters）

    可给泛型参数指定默认值，当用户未传入类型实参且 TypeScript 无法推断时，使用默认类型。

    语法：<T = 默认类型>

    例如：带默认类型的泛型函数

    // 泛型默认值：T 默认为 string
function createArray<T = string>(length: number, value: T): T[] {
  return Array(length).fill(value);
}

// 用法1：未传入类型，使用默认值 string
const strArr = createArray(3, "a"); // strArr: string[]

// 用法2：传入类型 number，覆盖默认值
const numArr = createArray<number>(3, 1); // numArr: number[]

// 用法3：类型推断（无需传入类型）
const boolArr = createArray(3, true); // boolArr: boolean[]

    例如：带默认类型的泛型接口

    // 泛型默认值：Data 默认为 void（无数据）
interface ApiResponse<T = void> {
  code: number;
  message: string;
  data?: T; // 可选字段，默认无数据
}

// 用法1：无数据响应（使用默认类型 void）
const successRes: ApiResponse = {
  code: 200,
  message: "操作成功"
};

// 用法2：有数据响应（指定 T 为 User）
const userRes: ApiResponse<User> = {
  code: 200,
  message: "查询成功",
  data: { id: 1, name: "张三" }
};
  • 泛型工具类型（Built-in Generic Utilities）

    TypeScript 内置了多个常用泛型工具类型，基于泛型实现 “类型转换”，无需手动编写复杂类型，常见的有：

    工具类型作用示例
    Partial<T>将 T 所有属性转为可选Partial<User> → { id?: number; name?: string }
    Required<T>将 T 所有可选属性转为必选Required<Partial<User>> → User
    Readonly<T>将 T 所有属性转为只读Readonly<User> → { readonly id: number; ... }
    Pick<T, K>从 T 中挑选部分属性 K（K 是联合类型）Pick<User, "id"> → { id: number }
    Omit<T, K>从 T 中排除部分属性 KOmit<User, "name"> → { id: number }
    Exclude<T, U>从 T 中排除能赋值给 U 的类型`Exclude<stringnumber, string>→number`
    Extract<T, U>从 T 中提取能赋值给 U 的类型`Extract<stringnumber, string>→string`
    ReturnType<T>提取函数 T 的返回值类型ReturnType<() => string> → string

    常用工具类型实战

    // 基础类型
interface User {
  id: number;
  name: string;
  age?: number; // 可选属性
}

// 1. Partial：所有属性可选
type PartialUser = Partial<User>;
// 等价于：{ id?: number; name?: string; age?: number }

// 2. Pick：挑选 id 和 name 属性
type PickUser = Pick<User, "id" | "name">;
// 等价于：{ id: number; name: string }

// 3. Omit：排除 age 属性
type OmitUser = Omit<User, "age">;
// 等价于：{ id: number; name: string }

// 4. ReturnType：提取函数返回值类型
function getUser(): User {
  return { id: 1, name: "张三" };
}
type UserReturnType = ReturnType<typeof getUser>; // 等价于 User

// 5. Readonly：所有属性只读
type ReadonlyUser = Readonly<User>;
const user: ReadonlyUser = { id: 1, name: "张三" };
// user.id = 2; // 报错：Cannot assign to 'id' because it is a read-only property
  • 泛型+交叉/联合类型

    结合之前学的交叉类型（&）和联合类型（|），实现更灵活的类型组合。

    例如：合并类型

    // 泛型函数：合并多个对象，返回交叉类型
function mergeMulti<T extends object[]>(...objs: T): T[number] {
  return Object.assign({}, ...objs);
}

// 合并 User + Contact + Admin，返回 User & Contact & Admin
const merged = mergeMulti(
  { id: 1, name: "张三" },
  { phone: "13800138000" },
  { role: "admin" }
);
console.log(merged.id, merged.phone, merged.role); // 均合法

    例如：泛型+联合类型（多类型可选）

    // 泛型约束：T 只能是 string 或 number
function formatValue<T extends string | number>(value: T): string {
  if (typeof value === "string") {
    return value.toUpperCase();
  }
  return value.toFixed(2);
}

formatValue("hello"); // 合法：返回 "HELLO"
formatValue(123.456); // 合法：返回 "123.46"
// formatValue(true); // 报错：Type 'boolean' is not assignable to type 'string | number'
  • 条件类型（Conditional Types）

    泛型结合条件表达式（T extends U ? X : Y），实现 “类型筛选 / 转换”，是类型编程的核心。

    语法：type 类型名<T> = T extends U ? X : Y（若 T 能赋值给 U，则为 X 类型，否则为 Y 类型）。

    例如：基础条件类型

    // 条件类型：若 T 是 string，返回 number；否则返回 boolean
type StringToNumber<T> = T extends string ? number : boolean;

type A = StringToNumber<string>; // A: number
type B = StringToNumber<number>; // B: boolean
type C = StringToNumber<boolean>; // C: boolean

    例如：分布式条件类型（对联合类型自动分发）

    当 T 是联合类型时，条件类型会自动对每个成员单独计算，再合并为联合类型。

    // 条件类型：提取 T 中 string 类型的成员
type ExtractString<T> = T extends string ? T : never;

// T 是联合类型 string | number | boolean
type D = ExtractString<string | number | boolean>; // D: string（只保留 string 成员）

// 等价于：ExtractString<string> | ExtractString<number> | ExtractString<boolean>
// → string | never | never → string（never 会被联合类型忽略）

    例如：实现自定义工具类型（模仿Partual）

    // 自定义 Partial：遍历 T 的所有属性 K，转为可选属性
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

type PartialUser = MyPartial<User>;
// 等价于 TypeScript 内置的 Partial<User>
  • 映射类型（Mapped Types）

    通过遍历泛型类型 T 的属性（[K in keyof T]），批量转换属性类型，是实现工具类型的基础（如 Partial、Readonly 均基于映射类型）。

    例如：自定义映射类型（批量修改属性类型）

    // 映射类型：将 T 的所有属性转为 string 类型
type ToString<T> = {
  [K in keyof T]: string; // K 遍历 T 的所有属性，属性类型转为 string
};

interface User {
  id: number;
  name: string;
  age?: number;
}

type StringUser = ToString<User>;
// 等价于：{ id: string; name: string; age?: string }

    例如：映射类型+条件类型（更灵活的转换）

    // 映射类型：可选属性保留，必选属性转为只读
type ReadonlyRequired<T> = {
  [K in keyof T]-?: T[K] extends Required<T>[K] ? Readonly<T[K]> : T[K];
  // -?：移除可选修饰符（确保属性存在）
  // Required<T>[K]：判断属性是否为必选
};

type ReadonlyUser = ReadonlyRequired<User>;
// 等价于：{ readonly id: number; readonly name: string; age?: number }

泛型 vs any

特性泛型（Generics）any（任意类型）
类型安全编译时检测类型错误，保持类型关联关闭类型检查，运行时可能报错
类型复用一套代码支持多种类型，无需重复编写需为不同类型编写单独代码（或放弃类型检查）
类型推导支持类型推断，无需显式指定类型无类型推断，返回值始终是 any
代码可读性明确类型约束，代码意图清晰类型模糊，无法判断变量实际类型
