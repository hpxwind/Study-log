模块，包，commonJS

模块：

模块是 具有独立功能的代码单元，通常对应一个文件（如 .js 文件）。它通过 “导出” 暴露部分功能，通过 “导入” 使用其他模块的功能，从而实现：

  • 代码拆分（将大程序拆分为小文件）
  • 作用域隔离（模块内的变量 / 函数默认不污染全局作用域）
  • 复用复用（一个模块可被多个地方处导入使用）
// 模块 A：math.js（负责数学计算）
const add = (a, b) => a + b;
const multiply = (a, b) => a * b;

// 导出模块内容
module.exports = { add, multiply };
// 模块 B：app.js（使用模块 A 的功能）
// 导入模块 A
const math = require('./math.js');

// 使用导入的功能
console.log(math.add(2, 3)); // 输出 5
console.log(math.multiply(2, 3)); // 输出 6

包：

包是 一组相关模块的集合，通常包含多个模块文件、配置文件和说明文档，用于解决特定问题（如 lodash 是处理数组 / 对象的工具包）。

简单说：包 = 多个模块 + 描述文件（package.json）。

lodash/                  # 包目录
├── package.json         # 包描述文件
├── index.js             # 入口文件（指定默认导出）
├── array/               # 数组相关模块
│   ├── chunk.js
│   └── filter.js
└── object/              # 对象相关模块
    ├── assign.js
    └── keys.js
package.json 内容
{
  "name": "lodash",        // 包名
  "version": "4.17.21",    // 版本号
  "main": "index.js",      // 入口文件（导入包时默认加载的文件）
  "dependencies": {}       // 依赖的其他包
}

CommonJS规范

CommonJS 是 JavaScript 模块化规范，主要为了解决 JavaScript 缺乏模块化机制的问题，最初设计用于 Node.js 环境（服务器端），后来也被用于浏览器端（需工具转译）。

核心目标：定义模块的 导出（暴露） 和 导入（使用） 方式，让代码可以在不同文件间安全复用。

导出对象：

// user.js
const name = "张三";
const age = 20;

// 导出多个内容（对象形式）
module.exports = {
  name,
  age,
  sayHello: () => console.log("Hello!")
};

导出单个对象

// utils.js
module.exports = (a, b) => a + b; // 导出一个函数

exports简写

// log.js
exports.info = (msg) => console.log("Info:", msg);
exports.error = (msg) => console.log("Error:", msg);
// 等价于：module.exports.info = ...

导入对象：

// 导入 user.js
const user = require('./user.js'); // 路径可省略 .js

console.log(user.name); // 张三
user.sayHello(); // Hello!

导入单个：

// 导入 utils.js（导出的是一个函数）
const add = require('./utils');
console.log(add(2, 3)); // 5

导入包：

// 导入第三方包（npm 安装的 lodash）
const _ = require('lodash'); // 无需写路径，Node.js 会自动查找 node_modules 目录
_.chunk([1,2,3,4], 2); // [[1,2], [3,4]]

（3）模块加载规则

  • 相对路径：导入本地模块时，路径以 ./（当前目录）或 ../（上级目录）开头，如 require('./user')。
  • 绝对路径：从根目录开始的完整路径，如 require('/project/utils')（较少使用）。
  • 包名：导入第三方包时，直接写包名（如 require('lodash')），Node.js 会从 node_modules 目录查找。
  • 自动查找：如果导入的是目录，Node.js 会自动查找该目录下的 package.json 中的 main 字段指定的入口文件；若没有 package.json，则默认加载 index.js。

CommonJS 的特点

  • 同步加载：require() 是同步执行的（导入完成后才继续执行后续代码），适合服务器端（文件存在本地，加载快）。
  • 运行时加载：模块在代码运行时才被加载和执行。
  • 缓存机制：模块被第一次 require 后会被缓存，后续导入不会重复执行模块代码（提高性能）。
