RAG简介

Retrieval-Augmented Generation

模型不是仅凭记忆“硬背”，而是先查阅我们提供的参考资料，再结合自身能力组织答案

  • 解决三类痛点
    • 实时性问题：模型训练数据存在截止日期，无法获取最新信息。
    • 隐私与合规挑战：企业敏感数据无法上传给第三方模型进行微调。
    • 领域知识局限：通用模型对医疗、法律、工业等垂直领域理解不足，容易产生术语混淆甚至事实性错误。
  • 工作流程 & 标准流程

    RAG遵循经典的 "索引 → 检索 → 增强 → 生成" 四步逻辑

    1. 索引（预处理阶段）：将你的私有文档（如PDF、Word、网页）切割成小块，通过嵌入模型（Embedding Model） 将其转换为数值向量，并存入向量数据库，建立语义索引库。
    1. 检索：用户提问时，将问题也转换为向量，在向量数据库中通过相似度算法找到最相关的“K个”文本片段（如Top-3或Top-5）。
    1. 增强：将检索到的相关片段与用户原始问题，按照预设的提示词模板（Prompt Template） 组合成一条包含“参考资料”的完整指令。
    1. 生成：大语言模型接收指令，基于提供的参考资料和自身推理能力生成最终答案。

    其主要分为两条线

    • 离线处理：向私有知识库（向量存储）源源不断添加私有知识文档。
      • 向知识库添加来自未来的知识文档（基于模型训练完成时间）
      • 向模型添加私有知识文档
      • 给出模型参考资料，规避模型幻觉
    • 在线处理：用户提问会先基于私有知识库做检索，获取参考资料，同步组装新提示词询问大模型获得结果

数学与Python知识扩展

  • 向量

    向量 是计算机能够理解、比较和搜索我们人类语言和知识的一种数字化方式

    在RAG的语境下，我们处理的是文本向量（Text Embedding）。它的核心思想是：通过一个特殊的“嵌入模型”，将一个词、一句话甚至一整篇文章，转换成一个由上百甚至上千个小数组成的向量。

    • 关键特性

      这个向量不仅仅是随机的数字，它“嵌入”了文本的语义信息。

      • 语义相似的词汇或句子，它们的向量在数学空间中的距离非常近。
      • 语义不同的词汇或句子，它们的向量则相距甚远。

      举个例子：

      想象一个简化到三维的向量空间（现实中通常是数百维）。经过模型转换后：

      • “国王”这个词的向量可能是 [0.8, 0.2, 0.1]
      • “王后”这个词的向量可能是 [0.7, 0.3, 0.2]
      • “苹果”（水果）这个词的向量可能是 [-0.5, 0.9, -0.1]
      • “橘子”这个词的向量可能是 [-0.4, 0.8, -0.2]

      你会发现：

      1. “国王”和“王后”的向量很接近（语义相近）。
      1. “苹果”和“橘子”的向量很接近（语义相近）。
      1. “国王”和“苹果”的向量距离很远（语义无关）。

      甚至，向量还能捕捉到关系：“国王”向量 - “男人”向量 + “女人”向量 ≈ “王后”向量。

  • 余弦相似度算法

    Cosine Similarity 是衡量两个向量之间语义相似读最常用，有效的算法。

    核心思想是忽略路径的绝对长度，只关注他们的方向是否一致。（方向越一致，两个文本的语义越接近）

    • 理解

      1. 余弦相似度的直观理解

      想象有两个向量从原点出发，射向高维空间（例如 768 维空间）中的两个点。

      • 方向完全相同：夹角为 0°，余弦相似度 = 1 → 语义完全一致（比如“苹果好吃”和“苹果很好吃”）。
      • 方向垂直：夹角为 90°，余弦相似度 = 0 → 语义毫无关系（比如“苹果好吃”和“明天天气”）。
      • 方向完全相反：夹角为 180°，余弦相似度 = -1 → 语义对立（在文本中很少见，例如“好”与“坏”的向量可能呈现这种关系）。

      核心观点：只关心方向和夹角，不关心向量的“长短”（即文本的篇幅、数字的绝对大小）。

    • 公式

      给定两个向量 A 和 B（对应查询向量和文档向量），余弦相似度的公式为：

      • 分子：点积，表示两向量在对应维度上的乘积之和。
      • 分母：两个向量的模长（即 L2 范数）的乘积，作用是将每个向量归一化为单位向量，从而消除长度影响。

      计算步骤示例（假设 3 维向量，现实中常为 768 或 1536 维）：

      维度查询向量 A文档向量 B
      10.50.4
      20.80.7
      30.10.2
      1. 计算点积：0.5×0.4+0.8×0.7+0.1×0.2=0.20+0.56+0.02=0.780.5×0.4+0.8×0.7+0.1×0.2=0.20+0.56+0.02=0.78
      1. 计算 A 的模：sqrt(0.5^2+0.8^2+0.1^2)≈0.949
      1. 计算 B 的模：sqrt(0.4^2+0.7^2+0.2^2)≈0.831
      1. 计算余弦相似度：0.78/(0.949×0.831)≈0.99 → 两个向量几乎平行，语义高度相似。
    • 为什么 RAG 选择余弦相似度，而不是欧氏距离或曼哈顿距离
      比较维度余弦相似度欧氏距离在 RAG 中的影响
      关注点向量方向的差异向量端点之间的绝对距离文本向量中，方向代表语义主题；长度受文本长度、用词强度等因素影响（同一语义的长文档和短文档长度差异大）
      对长度的鲁棒性✅ 自动归一化，不受文本长度影响❌ 长文本向量幅度大，即使语义相同也会产生很大欧氏距离RAG 中常需要比较短查询（几个词）和长文档段（几百个词），余弦相似度更公平
      值范围[-1, 1]，可直接解释为相似程度[0, ∞)，无固定上限，不方便直接做阈值比较余弦相似度 0.9 表示非常相似，容易设定检索阈值（例如 score > 0.7）
      计算效率需要计算点积和模，比欧氏距离略多两步，但现代向量索引库已将优化做到几乎无差别计算两个模平方和，稍简单差异可忽略，工程上优先选余弦相似度

      结论：在 RAG 中，我们关心的是“用户问题和哪些文档块意思最接近”，而不是“哪些文档块的向量绝对长度最短”。因此，余弦相似度是天然的选择。

    • 实践中如何应用余弦相似度进行检索
      1. 离线索引：所有文档块向量已存储在向量数据库中，数据库会预先计算每个向量的模长（L2 范数）并存储，避免每次查询时重复计算。
      1. 在线查询：
        • 用户查询 → 嵌入模型 → 查询向量 qq。
        • 向量数据库使用相似度搜索算法（如 HNSW、IVF）快速找出与 qq 余弦相似度最高的 Top-K 个文档块向量。
        • 实际计算时往往采用 “余弦距离 = 1 - 余弦相似度”，距离越小表示越相似，这样符合“最近邻”的直觉。
      1. 返回结果：数据库返回相似度分数（通常已转换为 0~1 之间的值，1 表示完全相同）以及对应的原始文本，供大模型生成答案。
    • 代码实现（python）
      import numpy as np

def get_dot(vec_a, vec_b):
    """计算两个向量的点积"""
    # 维度相同才能计算
    if len(vec_a) != len(vec_b):
        raise ValueError("vec_a and vec_b must have the same length")
    dot_sum = 0
    for a, b in zip(vec_a, vec_b):
        dot_sum += a * b
    return dot_sum


def get_norm(vec):
    """计算单个向量的模长"""
    sum_square = 0
    for v in vec:
        sum_square += v**2
    return np.sqrt(sum_square)


def cosine_similarity(vec_a, vec_b):
    """余弦相似度计算"""
    res = get_dot(vec_a, vec_b) / (get_norm(vec_a) * get_norm(vec_b))
    return res

if __name__ == "__main__":
    vec_a = np.array([1, 2, 3])
    vec_b = np.array([4, 5, 6])
    res = cosine_similarity(vec_a, vec_b)
    print(res)

LangChain简介

LangChain是一个开源的软件开发框架，专为简化基于大型语言模型的应用程序构建而设计。

是一个编排层，通过提供标准化的接口和模块化的“乐高积木”组件，让开发者能够轻松地将不同的AI模型、数据源和工具连接组合起来

解决传统大模型三个问题：

  1. 失忆问题：通过内置的Memory模块，管理对话上下文，让模型能够拥有“短期记忆”
  1. 连接孤立数据：通过Document Loaders和Vector Stores等组件，让模型能接入和处理公司内部的实时数据。
  1. 赋予动手能力：通过Tools和Agents，让模型能够自主调用API、搜索引擎等外部工具来执行任务，而不仅仅是文本问答。
