详解

一、AOP 是什么

AOP（面向切面编程），是对 OOP（面向对象）的补充。

  • OOP：纵向继承、封装、多态，解决类与类之间的纵向复用
  • AOP：横向抽取公共通用逻辑，在不修改原有业务代码前提下，增强方法功能

典型使用场景

日志记录、接口耗时统计、权限校验、事务管理、异常统一处理、参数校验、接口防重复提交、敏感词过滤

核心思想

把通用横切逻辑抽成切面，在目标方法执行前/后/异常时自动切入执行，解耦业务与非业务代码。

二、AOP 核心术语

  • 解释（spring中出现过）
    1. 连接点 JoinPoint
      程序中可以被切入的位置，Spring AOP 仅支持方法执行作为连接点（所有方法都是潜在连接点）。
    1. 切入点 Pointcut
      筛选匹配的连接点，定义哪些方法需要被增强，通过表达式匹配。
    1. 通知 Advice
      切面具体要执行的逻辑，一共 5 种类型：
      通知类型执行时机
      @Before目标方法执行之前
      @AfterReturning方法正常执行完毕、有返回值后
      @AfterThrowing方法抛出异常时执行
      @After方法最终执行之后（无论正常/异常都会执行，类似 finally）
      @Around环绕通知，包裹目标方法，可手动控制方法执行、前后自定义逻辑、修改入参返回值
    1. 切面 Aspect
      一个类，包含 Pointcut 切入点 + Advice 通知 的整体载体。
    1. 目标对象 Target
      被 AOP 代理增强的原始业务对象。
    1. 代理 Proxy
      Spring 自动创建代理对象，调用时走代理执行切面逻辑 + 原方法逻辑
      • 接口优先：JDK 动态代理（实现接口）
      • 类直接代理：CGLIB 动态代理（SpringBoot2.x 默认开启）
    1. 织入 Weaving
      将切面代码动态植入目标方法的过程，Spring 运行期动态织入。

三、SpringBoot 快速集成 AOP

1. 引入依赖

<!-- aop 依赖 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>

2. 开启 AOP（默认自动开启，可显式配置）

启动类注解（不加也生效）

@EnableAspectJAutoProxy(proxyTargetClass = true) 强制使用 CGLIB 代理

四、切入点表达式 execution（最常用）

语法：execution(访问修饰符 返回值 包.类.方法(参数))

通配符说明

  • *：匹配任意单个内容
  • ..：匹配多级包 / 任意多个参数

常用示例

  • code
    // 1. 匹配 com.service 下所有类所有方法
execution(* com.service.*.*(..))

// 2. 匹配 com.service 及其子包所有类所有方法
execution(* com.service..*.*(..))

// 3. 匹配指定类所有方法
execution(* com.service.UserService.*(..))

// 4. 匹配指定方法
execution(* com.service.UserService.login(..))

// 5. 任意返回值、public方法
execution(public * com..*.*(..))

其他切入点表达式

  • within(包.类)：限定匹配某个类/包内所有方法
  • @annotation(注解全类名)：匹配加了指定自定义注解的方法（最常用做精准切入）

五、五种通知完整代码示例

步骤1：编写切面类

  • code
    import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.springframework.stereotype.Component;

@Aspect
@Component // 交给Spring管理
public class LogAspect {

    // 定义公共切入点
    @Pointcut("execution(* com.demo.service.*.*(..))")
    public void pointCut(){}

    // 前置通知
    @Before("pointCut()")
    public void before(JoinPoint joinPoint){
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        System.out.println("【前置通知】方法:"+methodName+"，参数:"+args);
    }

    // 后置返回通知（正常返回才执行，可接收返回值）
    @AfterReturning(value = "pointCut()", returning = "result")
    public void afterReturn(JoinPoint joinPoint, Object result){
        System.out.println("【返回通知】返回值："+result);
    }

    // 异常通知（方法报错才进）
    @AfterThrowing(value = "pointCut()", throwing = "ex")
    public void afterThrow(JoinPoint joinPoint, Exception ex){
        System.out.println("【异常通知】异常信息："+ex.getMessage());
    }

    // 最终通知（无论正常异常必执行）
    @After("pointCut()")
    public void after(JoinPoint joinPoint){
        System.out.println("【最终通知】方法执行结束");
    }

    // 环绕通知（功能最强，可手动控制执行）
    @Around("pointCut()")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        System.out.println("【环绕前置】");
        long start = System.currentTimeMillis();

        // 执行目标方法
        Object res = pjp.proceed();

        long cost = System.currentTimeMillis() - start;
        System.out.println("【环绕后置】耗时："+cost+"ms");
        return res;
    }
}

执行顺序

正常执行：
环绕前置 → @Before → 目标方法 → @AfterReturning → @After → 环绕后置

异常执行：
环绕前置 → @Before → 目标方法报错 → @AfterThrowing → @After → 环绕后置

JoinPoint / ProceedingJoinPoint 常用方法

  • code
    joinPoint.getSignature().getName()        // 获取方法名
joinPoint.getTarget()                    // 获取原始目标对象
joinPoint.getArgs()                      // 获取方法入参
joinPoint.getSignature().getDeclaringTypeName() // 获取类全名
proceedingJoinPoint.proceed()            // 执行目标方法（Around专属）

六、注解式切点（精准切入，推荐）

1. 自定义注解

  • code
    @Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface LogRecord {
    String value() default "";
}

2. 切面匹配该注解

  • code
    @Aspect
@Component
public class AnnotationAspect {

    // 匹配加了 @LogRecord 的方法
    @Pointcut("@annotation(com.demo.annotation.LogRecord)")
    public void annoPoint(){}

    @Around("annoPoint()")
    public Object around(ProceedingJoinPoint pjp) throws Throwable{
        // 获取注解属性
        MethodSignature signature = (MethodSignature) pjp.getSignature();
        LogRecord annotation = signature.getMethod().getAnnotation(LogRecord.class);
        System.out.println("注解备注："+annotation.value());
        return pjp.proceed();
    }
}

3. 使用

  • code
    @Service
public class UserService {
    @LogRecord("用户登录操作")
    public String login(String username){
        return "success";
    }
}

七、AOP 代理选择原理

  1. JDK 动态代理
    • 要求：目标类必须实现接口
    • 代理对象实现原有接口，重写接口方法完成增强
  1. CGLIB 代理
    • 目标类没有实现接口时自动使用
    • 生成目标类的子类，重写父类方法实现增强
    • SpringBoot 2.x 默认 proxyTargetClass=true，优先 CGLIB

手动强制配置（application.yml）

spring:
  aop:
    proxy-target-class: true # true CGLIB，false JDK

八、常见坑与注意事项

  1. 内部调用 AOP 失效
    同一个类 this.xxx() 调用本类方法不会触发切面，原因：this 是原始对象，不是代理对象
    解决：从容器获取代理对象调用，或拆分到不同类
  1. private / static 方法无法切入
    Spring AOP 基于动态代理，只能增强 public 实例方法
  1. 切面顺序控制 @Order(n)
    数值越小优先级越高，前置通知先执行；后置通知后执行
    @Aspect
@Component
@Order(1)
public class Aspect1 {}
  1. 事务注解 @Transactional 底层就是 AOP
    通过AOP环绕通知，方法前开启事务，正常提交、异常回滚

九、实战场景：接口耗时统计完整案例

  • code
    @Aspect
@Component
@Order(1)
public class TimeAspect {
    @Pointcut("execution(* com.demo.controller.*.*(..))")
    public void cut(){}

    @Around("cut()")
    public Object timeCount(ProceedingJoinPoint pjp) throws Throwable{
        long start = System.currentTimeMillis();
        Object result = pjp.proceed();
        long time = System.currentTimeMillis() - start;
        String method = pjp.getSignature().toString();
        System.out.printf("方法：%s，执行耗时：%d ms%n",method,time);
        return result;
    }
}
