    分层对应关系：

    分层对应框架对应包位置核心职责
    表现层（Web）SpringMVCcontroller接收请求、参数处理、页面跳转、数据转发
    业务层Springservice + service.impl业务逻辑处理、事务管理、依赖解耦
    持久层（Dao）MyBatismapper（接口 + XML 映射文件）执行 SQL、操作 MySQL 数据库
    模型通用实体entity封装数据表数据，前后层数据载体
  • 执行流程

    web测试流程：

    1. 浏览器发起带参数的 GET 请求，发送至 Tomcat 服务器
    1. Tomcat 读取web.xml配置，由 **DispatcherServlet（SpringMVC 前端控制器）** 拦截全部请求
    1. SpringMVC 根据@RequestMapping("/findUser")匹配到UserController对应的处理方法
    1. Controller 接收前端传递的id参数，调用UserService中的业务查询方法
    1. UserServiceImpl通过@Autowired自动注入UserMapper代理对象，调用 Mapper 查询方法
    1. 程序读取UserMapper.xml中编写的 SQL 语句，向mytest1数据库的ssm_demo表发起查询
    1. MySQL 执行 SQL，查询出对应数据并返回结果集
    1. MyBatis 自动把查询结果封装为User实体类对象，逐层向上回传：Mapper → Service → Controller
    1. Controller 将封装好的User对象存入 Model 数据域，指定跳转视图名show
    1. SpringMVC 视图解析器根据配置前缀/WEB-INF/jsp/、后缀.jsp，拼接得到真实页面路径/WEB-INF/jsp/show.jsp
    1. JSP 页面通过 EL 表达式取出 Model 中的用户数据渲染展示
    1. 最终生成 HTML 页面，响应返回给浏览器完成展示

    main测试流程：

    1. 运行com.Hpc.test.Main的 main 方法
    1. 加载spring.xml配置文件，初始化 Spring 父 IoC 容器
    1. Spring 自动扫描包，创建UserServiceImpl、UserMapper代理对象并完成依赖注入
    1. 从容器获取UserService对象，调用查询方法getUserById(1)
    1. 调用 Mapper 接口，执行UserMapper.xml内 SQL 查询ssm_demo表
    1. 数据库返回数据，MyBatis 封装为 User 对象
    1. 控制台打印对象信息，关闭 Spring 容器，程序结束

  • mapper.UserMapper.java

    MyBatis 数据访问接口，定义查询数据库的方法；配合 UserMapper.xml 实现根据 ID 查询用户，供 Service 层调用操作数据库。

    package com.Hpc.mapper;
import com.Hpc.entity.User;
import org.apache.ibatis.annotations.Param;
/**
 * Mapper接口：对应ssm_demo表的数据操作
 * 规范：方法名、参数、返回值需要和UserMapper.xml映射文件一一对应
 */
public interface UserMapper {
    /**
     * 根据用户id查询一条用户数据
     * @Param("uid")：给形参起别名uid，在Mapper.xml中使用#{uid}接收该参数
     * @param uid 要查询的用户主键id
     * @return 查询成功返回封装好的User对象，无数据返回null
     */
    User selectById(@Param("uid") Integer uid);
    // 查询全部用户
    List<User> selectAll();
    // 新增用户
    int insert(User user);
    // 修改用户
    int update(User user);
    // 删除用户
    int deleteById(@Param("uid") Integer uid);
}

    </update>

  • service.impl.UserServiceImpl.java

    业务层接口实现类，完成具体查询逻辑；交由 Spring 容器管理，自动注入 Mapper 对象，调用数据库查询方法。

    package com.Hpc.service.impl;
import com.Hpc.entity.User;
import com.Hpc.mapper.UserMapper;
import com.Hpc.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

// @Service：将当前类交给Spring IoC容器创建并管理，成为Bean对象
@Service
public class UserServiceImpl implements UserService {

    // @Autowired：Spring自动从容器中匹配UserMapper代理对象，完成依赖注入，无需手动new
    @Autowired
    private UserMapper userMapper;

    /**
     * 实现业务接口查询方法
     * @param id 用户主键编号
     * @return 封装好的用户实体对象
     */
    @Override
    public User getUserById(Integer id) {
        // 调用持久层Mapper方法，执行数据库查询，向上返回查询结果
        return userMapper.selectById(id);
    }
    @Override
    public List<User> findAll() {
        return userMapper.selectAll();
    }

    @Override
    public void addUser(User user) {
        userMapper.insert(user);
    }

    @Override
    public void editUser(User user) {
        userMapper.update(user);
    }

    @Override
    public void delUser(Integer id) {
        userMapper.deleteById(id);
    }
}
  • controller.UserController.java

    SpringMVC 表现层控制器，接收前端浏览器请求，调用业务层查询数据，把数据存入域对象后跳转页面展示。

    package com.Hpc.controller;
import com.Hpc.entity.User;
import com.Hpc.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import java.util.List;

@Controller
@RequestMapping("/user")
public class UserController {
    @Autowired
    private UserService userService;

    // 1. 跳转到列表页，查询全部
    @RequestMapping("/list")
    public String list(Model model){
        List<User> userList = userService.findAll();
        model.addAttribute("list", userList);
        return "list";
    }

    // 2. 跳转新增页面
    @RequestMapping("/toAdd")
    public String toAdd(){
        return "add";
    }
    // 执行新增提交
    @RequestMapping("/add")
    public String add(User user){
        userService.addUser(user);
        return "redirect:/user/list";
    }

    // 3. 跳转修改页面（回显数据）
    @RequestMapping("/toEdit")
    public String toEdit(@RequestParam Integer id, Model model){
        User user = userService.getUserById(id);
        model.addAttribute("user", user);
        return "edit";
    }
    // 提交修改
    @RequestMapping("/update")
    public String update(User user){
        userService.editUser(user);
        return "redirect:/user/list";
    }

    // 4. 删除
    @RequestMapping("/delete")
    public String delete(@RequestParam Integer id){
        userService.delUser(id);
        return "redirect:/user/list";
    }

    // 原有单个查询
    @RequestMapping("/findUser")
    public String findUser(@RequestParam Integer id, Model model){
        User user = userService.getUserById(id);
        model.addAttribute("user",user);
        return "show";
    }
}

    spring.xml 是 Spring 父容器核心配置文件，加载数据库配置、创建 C3P0 连接池、整合 MyBatis、扫描 Service 与 Mapper，完成业务层、持久层 Bean 自动创建与依赖管理。

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="
http://www.springframework.org/schema/beans
https://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/context
https://www.springframework.org/schema/context/spring-context.xsd">

    <!-- 加载类路径下db.properties，读取数据库账号、地址等配置，${key}取值使用 -->
    <context:property-placeholder location="classpath:db.properties"/>

    <!-- 自动扫描com.Hpc.service包，把@Service注解类注册为Spring Bean，交由容器管理 -->
    <context:component-scan base-package="com.Hpc.service"/>

    <!-- 配置C3P0数据库连接池，id=dataSource作为数据源唯一标识 -->
    <bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
        <!-- 驱动类，从properties读取配置 -->
        <property name="driverClass" value="${jdbc.driver}"/>
        <!-- 数据库连接地址 -->
        <property name="jdbcUrl" value="${jdbc.url}"/>
        <!-- 数据库用户名 -->
        <property name="user" value="${jdbc.username}"/>
        <!-- 数据库密码 -->
        <property name="password" value="${jdbc.password}"/>
    </bean>

    <!-- MyBatis整合核心：SqlSessionFactory工厂，构建MyBatis会话对象 -->
    <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <!-- 注入上面配置好的数据库连接池 -->
        <property name="dataSource" ref="dataSource"/>
        <!-- 给entity包下实体类设置别名，Mapper.xml写类名即可，不用写全限定名 -->
        <property name="typeAliasesPackage" value="com.Hpc.entity"/>
    </bean>

    <!-- 自动扫描Mapper接口包，自动生成Mapper代理对象，注入到Spring容器，供Service注入使用 -->
    <bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
        <property name="basePackage" value="com.Hpc.mapper"/>
    </bean>
</beans>

  • web.xml

    web.xml 是 Web 项目部署描述文件，配置编码过滤器、Spring 父子容器加载、SpringMVC 前端控制器，是 SSM 项目 Web 入口配置。

    <?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee shturl.cc/7tc6PjX3uzawPyYLKwpb4DSkKOgqlJL6M9WJrMq9"
         version="4.0">

    <!-- 全局编码过滤器：统一所有请求编码为UTF-8，解决POST中文乱码 -->
    <filter>
        <filter-name>encodingFilter</filter-name>
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
        <init-param>
            <param-name>encoding</param-name>
            <param-value>UTF-8</param-value>
        </init-param>
        <init-param>
            <param-name>forceEncoding</param-name>
            <param-value>true</param-value>
        </init-param>
    </filter>
    <!-- 拦截所有访问路径，全部执行编码过滤 -->
    <filter-mapping>
        <filter-name>encodingFilter</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

    <!-- 监听器：项目启动时创建Spring父容器，加载spring.xml（管理Service、Mapper、数据源） -->
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>
    <!-- 指定Spring配置文件位置 -->
    <context-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:spring.xml</param-value>
    </context-param>

    <!-- SpringMVC前端控制器：全局拦截所有请求，创建SpringMVC子容器 -->
    <servlet>
        <servlet-name>dispatcherServlet</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <!-- 指定MVC配置文件，扫描Controller -->
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:springmvc.xml</param-value>
        </init-param>
        <!-- 项目启动立刻初始化Servlet，不是第一次访问才创建 -->
        <load-on-startup>1</load-on-startup>
    </servlet>
    <!-- 匹配所有URL，交由DispatcherServlet分发处理 -->
    <servlet-mapping>
        <servlet-name>dispatcherServlet</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>

</web-app>
