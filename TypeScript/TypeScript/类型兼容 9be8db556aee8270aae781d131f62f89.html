类型兼容

类型兼容性是指 “一个类型能否赋值给另一个类型”，核心遵循 “结构化类型系统”（Structural Typing）—— 类型的兼容性仅由其 “结构（属性 / 方法）” 决定，而非名义（如类 / 接口的名称）。

TS 兼容性的核心是 “鸭子类型”：“如果一个东西看起来像鸭子、走起来像鸭子、叫起来像鸭子，那它就是鸭子”。

即：若类型 A 包含类型 B 的所有属性 / 方法（且类型兼容），则 A 是 B 的子类型，A 可赋值给 B（反之不成立）

// 示例：结构决定兼容性
interface Duck {
  quack(): void;
  walk(): void;
}

// 类型 A 包含 Duck 的所有结构
const myDuck = {
  quack: () => console.log("嘎嘎"),
  walk: () => console.log("走路"),
  fly: () => console.log("飞行"), // 额外属性不影响兼容性
};

// 合法：myDuck 结构满足 Duck，可赋值给 Duck 类型
let duck: Duck = myDuck;

基础的兼容性问题：

  • 原始类型
    • 字面量类型可赋值给基础类型（如 "a" 可赋值给 string）；
    • 子类型可赋值给父类型（如 number 可赋值给 any，但 any 不可反向赋值）；
    • 无隐式类型转换（如 number 不可赋值给 string）。
    // 合法：字面量类型 → 基础类型
let str: string = "hello";
let strLiteral: "hello" = "hello";
str = strLiteral; // ✅

// 合法：子类型 → 父类型
let anyVal: any = 100;
let num: number = anyVal; // ✅（any 可赋值给任意类型）

// 非法：无隐式转换
let n: number = 10;
// let s: string = n; // ❌（number 不可赋值给 string）
  • 函数类型

    函数兼容性需满足 3 个条件（参数宽松，返回值严格）：

    维度兼容性规则
    参数个数目标函数的参数个数 ≤ 源函数的参数个数（少参数可赋值给多参数）
    参数类型源函数参数类型 ≤ 目标函数参数类型（逆变，后文详解）
    返回值类型源函数返回值类型 ≥ 目标函数返回值类型（协变）

    // 示例1：参数个数兼容（少 → 多）
type F1 = (a: number) => void;
type F2 = (a: number, b: string) => void;

let f1: F1 = (a) => console.log(a);
let f2: F2 = f1; // ✅（F1 参数更少，可赋值给 F2）

// 示例2：参数类型逆变
type F3 = (a: Animal) => void;
type F4 = (a: Dog) => void; // Dog 是 Animal 的子类型
let f4: F4 = (a) => console.log(a.name);
let f3: F3 = f4; // ✅（参数类型：Dog → Animal，逆变兼容）

// 示例3：返回值协变
type F5 = () => Animal;
type F6 = () => Dog;
let f6: F6 = () => ({ name: "旺财" });
let f5: F5 = f6; // ✅（返回值类型：Dog → Animal，协变兼容）
  • 数组/元组类型

    数组：若元素类型兼容，则数组类型兼容（如 number[] 可赋值给 any[]）；

    元组：长度和元素类型需完全匹配（严格模式下），但元组可赋值给数组。

    // 数组兼容
let numArr: number[] = [1, 2, 3];
let anyArr: any[] = numArr; // ✅

// 元组兼容
let tuple: [number, string] = [1, "a"];
let arr: (number | string)[] = tuple; // ✅（元组 → 数组）
// let tuple2: [number, string] = [1]; // ❌（长度不匹配）

复杂类型兼容性：

  • 对象/接口类型

    核心规则：目标类型的所有属性必须在源类型中存在，且类型兼容（源类型可包含额外属性）。

    interface User {
  name: string;
  age: number;
}

interface Admin {
  name: string;
  age: number;
  role: string; // 额外属性
}

// 合法：Admin 包含 User 的所有属性，可赋值给 User
let user: User = { name: "张三", age: 25, role: "admin" }; // ✅
let admin: Admin = user; // ❌（User 缺少 role 属性）

// 注意：字面量赋值时会检查“额外属性”（防止拼写错误）
// let user2: User = { name: "张三", age: 25, role: "admin" }; // ❌（字面量不允许额外属性）
// 解决：先赋值给变量，再赋值（绕开字面量检查）
const temp = { name: "张三", age: 25, role: "admin" };
let user3: User = temp; // ✅
  • 类类型

    类的兼容性仅比较 实例属性 / 方法，忽略静态成员和构造函数；若类有 private/protected 成员，则仅当两个类是 “同一个类 / 父子类” 时兼容。

    class Person {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
}

class Student {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
}

// 合法：实例结构相同，即使类名不同
let p: Person = new Student("李四"); // ✅

// 含 private 成员的类：仅父子类兼容
class Parent {
  private id: number;
  constructor(id: number) {
    this.id = id;
  }
}

class Child extends Parent {}
class Other {
  private id: number;
  constructor(id: number) {
    this.id = id;
  }
}

let parent: Parent = new Child(1); // ✅（父子类）
// let parent2: Parent = new Other(1); // ❌（private 成员来自不同类）
  • 泛型类型

    泛型的兼容性取决于 “泛型参数是否被使用”：

    • 若泛型类型未使用参数（如 Empty<T>），则忽略参数，仅比较结构；
    • 若泛型类型使用参数（如 Box<T>），则参数类型需兼容。
    // 示例1：泛型参数未被使用
interface Empty<T> {}
let e1: Empty<number> = {};
let e2: Empty<string> = e1; // ✅（参数未使用，忽略）

// 示例2：泛型参数被使用
interface Box<T> {
  value: T;
}
let b1: Box<number> = { value: 10 };
// let b2: Box<string> = b1; // ❌（number ≠ string）

协议（Covariance）和逆变（Contravariance）

这是 TS 类型兼容性的核心概念，主要影响函数参数 / 返回值、泛型的兼容性：

概念含义示例
协变子类型 → 父类型（正向）Dog[] → Animal[]
逆变父类型 → 子类型（反向）(a: Animal) → (a: Dog)
不变仅自身类型兼容[number, string] 仅兼容自身
  • 协变（返回值/数组）
    class Animal {}
class Dog extends Animal {}

// 返回值协变
type F = () => Animal;
let f: F = () => new Dog(); // ✅（Dog → Animal）

// 数组合变
let animalArr: Animal[] = [new Dog()]; // ✅
  • 逆变（函数参数）
    type Handler = (a: Animal) => void;
let handler: Handler = (a: Dog) => console.log(a); // ✅（Animal → Dog，逆变）
  • TS 配置：strictFunctionTypes
    • 默认开启 strictFunctionTypes：函数参数严格逆变；
    • 关闭后：函数参数双向协变（宽松模式，不推荐）。

特殊类型兼容性：

  • any/unknown
    • any：可赋值给任意类型，任意类型也可赋值给 any（完全兼容）；
    • unknown：任意类型可赋值给 unknown，但 unknown 不可赋值给其他类型（需类型守卫）。
    let a: any = 10;
let n: number = a; // ✅
a = "hello"; // ✅

let u: unknown = 10;
// let n2: number = u; // ❌
if (typeof u === "number") {
  let n2: number = u; // ✅（类型守卫后兼容）
}
  • never

    never 是所有类型的子类型，可赋值给任意类型，但无类型可赋值给 never（除自身）。

    let n: never;
let num: number = n; // ✅
// n = 10; // ❌
  • void
    • void 函数可赋值给返回任意类型的函数（忽略返回值）；
    • 非 void 函数不可赋值给 void 函数（严格模式下）。
    type F = () => void;
let f: F = () => 10; // ✅（忽略返回值）

type F2 = () => number;
// let f2: F2 = () => {}; // ❌（void ≠ number）

兼容性常见陷阱

  • 字面量额外属性检查

    对象字面量赋值时，会严格检查 “无额外属性”（变量赋值则不检查）：

    interface User {
  name: string;
}

// 非法：字面量有额外属性
// let user: User = { name: "张三", age: 25 }; // ❌

// 合法：先赋值给变量，绕开检查
const temp = { name: "张三", age: 25 };
let user: User = temp; // ✅
  • 函数参数的 “可选 vs 必选”

    可选参数可兼容必选参数（宽松模式），但严格模式下不允许：

    type F1 = (a: number) => void;
type F2 = (a?: number) => void;

let f1: F1 = (a) => console.log(a);
let f2: F2 = f1; // ✅（必选 → 可选）
// let f1: F1 = f2; // ❌（可选 → 必选，严格模式下报错）
