watch监听

在 Vue 中，watch是一个用于监听响应式数据变化的配置选项。当被监听的数据（如data、computed中的属性）发生改变时，watch会自动触发指定的回调函数，用于执行自定义逻辑（如异步操作、复杂数据处理等）。

  • 基础用法

    watch的核心是 “监听数据变化并执行操作”，基本语法如下：

    new Vue({
  data() {
    return {
      message: "Hello",
      user: { name: "张三", age: 20 },
      list: [1, 2, 3]
    };
  },
  watch: {
    // 1. 监听简单数据类型（字符串、数字等）
    message(newVal, oldVal) {
      // newVal：变化后的值
      // oldVal：变化前的值
      console.log(`message从${oldVal}变成了${newVal}`);
    },

    // 2. 监听对象的某个属性（需用字符串形式）
    "user.name"(newVal, oldVal) {
      console.log(`用户名从${oldVal}变成了${newVal}`);
    },

    // 3. 监听数组（直接监听数组本身）
    list(newVal, oldVal) {
      console.log("数组发生了变化", newVal);
    }
  }
});
  • 高级配置（对象形式）

    watch除了简写函数，还可以用对象形式配置更多参数，满足复杂需求：

    watch: {
  message: {
    handler(newVal, oldVal) { // 核心回调函数，必选
      console.log(`message变化了：${oldVal} → ${newVal}`);
    },
    immediate: true, // 可选：初始化时立即执行一次handler（默认false）
    deep: false // 可选：是否深度监听（针对对象/数组，默认false）
  }
}

    参数：

    1、handler

    必选，数据变化时执行的回调函数，接收newVal（新值）和oldVal（旧值）两个参数。

    2、immediate

    • 默认false：数据首次初始化时不执行handler，仅在数据变化时执行。
    • 设置true：页面加载时立即执行一次handler（适合初始化时需要基于初始值做处理的场景）。
    watch: {
  message: {
    handler(val) { console.log("当前message：", val); },
    immediate: true // 初始化时就会打印"当前message：Hello"
  }
}

    3、deep

    • 默认false：仅监听数据的 “引用变化”（如对象被重新赋值），不监听内部属性变化。
    • 设置true：深度监听对象 / 数组的内部属性变化（如user.age修改时触发）。
    data() { return { user: { name: "张三", age: 20 } }; },
watch: {
  user: {
    handler(newVal) { console.log("用户信息变化：", newVal); },
    deep: true // 监听user内部所有属性的变化
  }
}
// 当执行 this.user.age = 21 时，会触发handler
  • 监听复杂数据的注意事项

    1、监听对象

    若不设置deep: true，直接修改对象的属性（如this.user.age = 21）不会触发watch，因为对象的引用未变。

    解决方案：

    • 用deep: true深度监听（适合需要监听所有内部属性的场景）。
    • 直接监听对象的具体属性（如"user.age"），无需deep（更高效）。

    watch: {
  "user.age"(newVal) { // 直接监听对象的某个属性，更高效
    console.log("年龄变化：", newVal);
  }
}

    2、监听数组

    对于数组，watch默认可以监听通过变异方法（push、pop、splice等）修改的变化，以及数组被重新赋值的变化。

    但直接修改数组索引或长度（如this.list[0] = 100）无法被watch监测到，需用Vue.set或splice触发响应式更新：

    // 错误：直接修改索引，watch不触发
this.list[0] = 100;

// 正确：用Vue.set触发更新
this.$set(this.list, 0, 100);

// 正确：用splice触发更新
this.list.splice(0, 1, 100);// 错误：直接修改索引，watch不触发
this.list[0] = 100;

// 正确：用Vue.set触发更新
this.$set(this.list, 0, 100);

// 正确：用splice触发更新
this.list.splice(0, 1, 100);
  • watch与computed的区别
    特性watchcomputed
    核心作用监听数据变化，执行副作用操作（如异步请求、DOM 操作）依赖数据计算衍生值，侧重 “计算”
    缓存机制无缓存，数据变化就触发有缓存，依赖不变则复用结果
    适用场景异步操作（如数据变化后请求接口）、复杂逻辑简单计算（如拼接字符串、过滤列表）
    返回值无返回值，靠副作用生效必须有返回值，用于模板渲染或其他逻辑
  • 适用场景

    1、异步操作：数据变化后发起网络请求（如搜索框输入变化时，延迟发送搜索请求）。

    data() { return { searchKey: "" }; },
watch: {
  searchKey(newVal) {
    // 延迟500ms发送请求，避免输入频繁触发
    clearTimeout(this.timer);
    this.timer = setTimeout(() => {
      console.log("发送搜索请求：", newVal);
    }, 500);
  }
}

    2、数据变化时的复杂逻辑：如数据变化后需要修改 DOM、操作第三方库等。

    3、监听路由变化：在组件中监听$route，实现路由变化时的逻辑（如页面切换时滚动到顶部）。

    watch: {
  $route() {
    window.scrollTo(0, 0); // 路由变化时滚动到顶部
  }
}
