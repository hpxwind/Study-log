面向对象

  • this指针

    this指针，谁调用我，this就指向谁，

    在对象中，指向调用它的那一级对象

    在事件中，指向调用它的事件

    面向对象语言中 this 表示当前对象的一个引用。

但在 JavaScript 中 this 不是固定不变的，它会随着执行环境的改变而改变。

在方法中，this 表示该方法所属的对象。
如果单独使用，this 表示全局对象。
在函数中，this 表示全局对象。
在函数中，在严格模式下，this 是未定义的(undefined)。
在事件中，this 表示接收事件的元素。
类似 call() 和 apply() 方法可以将 this 引用到任何对象。
-----------------------------------------------------------------------------------------
// this 表示 person 对象
var person = {
  firstName: "John",
  lastName : "Doe",
  id       : 5566,
  fullName : function() {
    return this.firstName + " " + this.lastName;
  }
};
-----------------------------------------------------------------------------------------
单独使用 this，则它指向全局(Global)对象。
在浏览器中，window 就是该全局对象为 [object Window]:
var x = this;
-----------------------------------------------------------------------------------------
严格模式下，如果单独使用，this 也是指向全局(Global)对象。
"use strict";
var x = this;
-----------------------------------------------------------------------------------------
在函数中，函数的所属者默认绑定到 this 上。
在浏览器中，window 就是该全局对象为 [object Window]:
function myFunction() {
  return this;
}
-----------------------------------------------------------------------------------------
严格模式下函数是没有绑定到 this 上，这时候 this 是 undefined。
"use strict";
function myFunction() {
  return this;
}
-----------------------------------------------------------------------------------------
在 HTML 事件句柄中，this 指向了接收事件的 HTML 元素：
<button onclick="this.style.display='none'">
点我后我就消失了
</button>
-----------------------------------------------------------------------------------------
下面实例中，this 是 person 对象，person 对象是函数的所有者：

var person = {
  firstName  : "John",
  lastName   : "Doe",
  id         : 5566,
  myFunction : function() {
    return this;
  }
};

显式函数绑定
	在 JavaScript 中函数也是对象，对象则有方法，apply 和 call 就是函数对象的方法。这两个方法异常强大，他们允许切换函数执行的上下文环境（context），即 this 绑定的对象。
	在下面实例中，当我们使用 person2 作为参数来调用 person1.fullName 方法时, this 将指向 person2, 即便它是 person1 的方法：
var person1 = {
  fullName: function() {
    return this.firstName + " " + this.lastName;
  }
}
var person2 = {
  firstName:"John",
  lastName: "Doe",
}
person1.fullName.call(person2);  // 返回 "John Doe"

  • 原型（ES5中类的实现方式）
    • ES5中类的基本结构：构造函数+原型
      // 1. 创建构造函数 (相当于类的构造器)
function Person(name, age) {
    // 2. 实例属性
    this.name = name;
    this.age = age;
    
    // 3. 实例方法 (不推荐，每个实例都会创建新方法)
    this.greet = function() {
        return "Hello, I'm " + this.name;
    };
}

// 4. 原型方法 (推荐，所有实例共享)
Person.prototype.introduce = function() {
    return "My name is " + this.name + ", I'm " + this.age;
};

// 5. 静态方法 (直接添加到构造函数)
Person.species = function() {
    return "Homo sapiens";
};

// 创建实例
var john = new Person("John", 30);
console.log(john.greet());     // "Hello, I'm John"
console.log(john.introduce()); // "My name is John, I'm 30"
console.log(Person.species()); // "Homo sapiens"
    • 原型链中的继承
      // 父类
function Animal(name) {
    this.name = name;
}
Animal.prototype.eat = function() {
    return this.name + " is eating";
};

// 子类
function Dog(name, breed) {
    // 1. 调用父类构造函数
    // this 指向当前Dog实例，相当于ES6中的super(name)
    Animal.call(this, name);
    this.breed = breed;
}

// 2. 设置原型链
// 这相当于让 Dog.prototype 继承自 Animal.prototype
// 这里使用赋值符号=，Dog.prototype包括构造函数在内的所有都与Animal.prototype相同，
// 但实际上只是想要继承
Dog.prototype = Object.create(Animal.prototype);

// 3. 修复构造函数指向
Dog.prototype.constructor = Dog;

// 4. 添加子类方法
Dog.prototype.bark = function() {
    return "Woof! I'm a " + this.breed;
};

// 创建实例
var rex = new Dog("Rex", "Labrador");
console.log(rex.eat());  // "Rex is eating" (继承自Animal)
console.log(rex.bark()); // "Woof! I'm a Labrador" (子类方法)

      原型链关系图：

      rex (Dog实例)
  ├── __proto__ → Dog.prototype
        ├── constructor → Dog
        ├── bark() 
        └── __proto__ → Animal.prototype
               ├── constructor → Animal
               └── eat()
                    └── __proto__ → Object.prototype
    • 实现私有变量（闭包）
      function Counter() {
    // 私有变量 (通过闭包实现)
    var count = 0;
    
    // 公共方法
    this.increment = function() {
        return ++count;
    };
    
    this.getValue = function() {
        return count;
    };
}

var counter = new Counter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.count);       // undefined (无法直接访问)
    • Getter/Setter 实现

      封装与访问器属性

      function Rectangle(width, height) {
		// 私有变量实现闭包
    var _width = width;
    var _height = height;
    
    // 定义只读 area 属性
    Object.defineProperty(this, "area", {
        get: function() {
            return _width * _height;
        },
        enumerable: true
    });
    
    Object.defineProperty(this, "width", {
        get: function() { return _width; },
        set: function(value) { 
            if(value > 0) _width = value; 
        },
        enumerable: true
    });
}

var rect = new Rectangle(10, 5);
console.log(rect.area); // 50
rect.width = 20;
console.log(rect.area); // 100
  • ES6中的类

    与其他语言类似,结构：

    class Person {
  // 构造函数（初始化实例的方法）
  constructor(name, age) {
    this.name = name; // 实例属性
    this.age = age;
  }
每个类都有一个 constructor 方法，用于初始化实例。
当通过 new 创建实例时，自动调用该方法。
若未显式定义，默认会有一个空的构造函数。
------------
  // 实例方法（定义在原型上）
  sayHello() {
    console.log(`Hello, I'm ${this.name}, ${this.age} years old`);
  }

定义在类体中的方法（不含 static）会成为实例的原型方法，所有实例共享。
方法中通过 this 访问实例属性。
------------
  // 静态方法（属于类本身，用static修饰）
  static create(name, age) {
    return new Person(name, age); // 静态方法通常用于创建实例或工具函数
  }
用 static 修饰的方法属于类本身，而非实例。
只能通过类名调用（如 Person.create()），不能通过实例调用。
常用于工具函数或实例创建工厂。

    getter 和 setter

    class Person {
  constructor(name) {
    this._name = name; // 用下划线表示私有属性（约定）
  }

  // getter：读取 name 时触发
  get name() {
    return this._name.toUpperCase(); // 返回大写形式
  }

  // setter：设置 name 时触发
  set name(newName) {
    this._name = newName.trim(); // 去除首尾空格
  }
}

const p = new Person("alice");
console.log(p.name); // 输出：ALICE（触发getter）

p.name = "  bob  "; // 触发setter
console.log(p.name); // 输出：BOB
