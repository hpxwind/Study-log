mocha和断言

Mocha 是 JavaScript 中最流行的测试框架之一，用于组织和运行测试用例；而断言（Assertion） 是验证代码行为是否符合预期的工具（通常由第三方库提供）。两者配合使用可以构建完整的自动化测试体系。

  • Mocha核心作用与特性

    Mocha 本身不提供断言功能，主要负责：

    • 定义测试结构（测试套件、测试用例）
    • 执行测试并生成报告
    • 支持同步 / 异步代码测试
    • 提供钩子函数（控制测试流程）
  • 安装
    # 局部安装（推荐）
npm install mocha --save-dev

# 全局安装（方便命令行直接运行）
npm install mocha -g

  • 基础使用
    // 测试套件：描述一个功能模块
describe('加法函数测试', function() {
  // 测试用例：具体场景
  it('1 + 2 应该等于 3', function() {
    // 这里需要断言库验证结果（下文会讲）
  });

  // 嵌套测试套件
  describe('边界情况', function() {
    it('0 + 0 应该等于 0', function() {
      // 断言逻辑
    });
  });
});
    {
  "scripts": {
    "test": "mocha test/**/*.test.js"  // 匹配 test 目录下所有 .test.js 文件
  }
}

    运行测试：

    npm test
  • 异步测试支持

    Mocha 对异步代码（如回调、Promise、async/await）有原生支持，避免测试提前结束。

    1、回调函数

    通过在测试用例中传入 done 参数，调用 done() 标识异步结束：

    it('异步读取文件应该成功', function(done) {
  fs.readFile('test.txt', 'utf8', function(err, data) {
    if (err) done(err);  // 失败：传递错误
    else {
      // 断言逻辑...
      done();  // 成功：调用 done()
    }
  });
});

    2、Promise

    直接返回 Promise，Mocha 会等待其状态变更：

    it('Promise 应该返回成功结果', function() {
  return new Promise((resolve) => {
    setTimeout(() => {
      // 断言逻辑...
      resolve();  // 成功
    }, 1000);
  });
});

    3、async/await（推荐）

    it('async/await 测试', async function() {
  const result = await someAsyncFunction();  // 等待异步操作
  // 断言逻辑...
});
  • 钩子函数

    用于在测试前/后执行初始化或清理操作

    describe('用户管理模块', function() {
  let db;

  // 所有测试用例执行前运行（只执行一次）
  before(function() {
    db = connectToDatabase();  // 连接数据库
  });

  // 每个测试用例执行前运行
  beforeEach(function() {
    db.clear();  // 清空测试数据
  });

  it('新增用户应该成功', function() {
    // 使用 db 操作...
  });

  // 所有测试用例执行后运行（只执行一次）
  after(function() {
    db.disconnect();  // 断开数据库连接
  });
});
  • 高级用法与配置
    # 监视文件变化，自动重新运行测试
mocha --watch

# 递归查找所有测试文件
mocha --recursive

# 指定报告格式（如 spec、dot、nyan 等）
mocha --reporter nyan

# 只运行匹配的测试用例（通过正则）
mocha --grep "加法"

    配置文件

    创建 mocha.config.js 或 .mocharc.json 统一配置

    // mocha.config.js
module.exports = {
  spec: 'test/**/*.test.js',  // 测试文件路径
  reporter: 'spec',           // 报告格式
  watch: false,               // 是否监视
  timeout: 5000               // 测试超时时间（毫秒）
};

断言库

断言库用于验证代码执行结果是否符合预期（如 “结果应该等于 3”“数组应该包含某个元素”）。Mocha 可搭配任意断言库，常见的有：

  • 内置assert模块（基础）
    const assert = require('assert');

it('1 + 2 应该等于 3', function() {
  assert.equal(1 + 2, 3);  // 相等则通过，否则抛出错误
});

    常用 API：assert.equal(a, b)（宽松相等）、assert.strictEqual(a, b)（严格相等）、assert.ok(condition)（验证条件为真）等。

  • Chai（最流行，支持多种风格）
    npm install chai --save-dev

    1、expect风格（推荐，直观灵活）

    const { expect } = require('chai');

it('测试示例', function() {
  const result = 1 + 2;
  expect(result).to.equal(3);  // 等于 3
  expect([1, 2, 3]).to.include(2);  // 数组包含 2
  expect({ name: 'foo' }).to.have.property('name', 'foo');  // 对象有属性 name 且值为 foo
  expect(true).to.be.true;  // 布尔值为 true
});

    2、should风格（链式语法， 需要先扩展Object.prototype）

    const { should } = require('chai');
should();  // 初始化，扩展 Object.prototype

it('测试示例', function() {
  const result = 1 + 2;
  result.should.equal(3);
  [1, 2, 3].should.include(2);
});

    3、assert风格（类似内置的，但内容更丰富）

    const { assert } = require('chai');

it('测试示例', function() {
  assert.equal(1 + 2, 3);
  assert.include([1, 2, 3], 2);
});
  • should.js（轻量，仅支持should风格）
    npm install should --save-dev
    const should = require('should');

it('测试示例', function() {
  (1 + 2).should.be.exactly(3);
  [1, 2, 3].should.containEql(2);
});
