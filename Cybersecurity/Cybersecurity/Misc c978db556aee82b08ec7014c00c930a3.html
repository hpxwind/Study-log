    • 文件附加

      大部分文件有其固定的文件结构，常见的图片格式如PNG，JPG等都是由一系列特定的数据块组成的。

      以PNG格式为例： 由IHDR（文件头数据块）、PLTE（调色板数据块）、IDAT（图像数据块）、IEND（图像结束数据） 四种标准数据块和一些辅助数据块组成。IHDR、IDAT、IEND 为必须部分，每个数据块由Length（长度）、Chunk Type Code（数据块类型码）、Chunk Data（数据块数据）和CRC（循环冗余校验码）四部分组成。

      一般，固定格式的文件都有固定格式的开头，有些文件还有固定的结尾，而相应格式的查看器一般都只解析从者一文件头到这一文件尾之间的内容。

    • LSB

      LSB：Least Significant Bit（最低有效位）。大多数PNG图像中，每个像素都由R、G、B三原色组成（有的图片还包括A通道：表示透明度），每种颜色一般用8位数据表示（0x00~0xFF），简单的小幅度修改人眼难以分辨。

      例如RGB(255,0,0)与RGB(254,0,0)仅差一位，就可以隐藏一位信息，而一个RGB通道由三原色组成，就可以隐藏三位信息。

      • 工具

        检测LSB隐写痕迹的常用工具是Stegsolve，它还可以查看不同的通道，对图片进行XOR对比操作。

        对于PNG和BMP图片中的LSB等常见的隐写方式，也可以通过zsteg工具进行自动化识别和提取。

    • 盲水印

      平时听到得到音乐就是一段在时域上不断震动的波，

      一段乐谱就可以看作一段音乐在频域中的表示。

      同理，图片是在空间域上的信号表示，而图像灰度变化强烈的情况，也可以视为图像的频率。

      要把时域或空域中表示的信号转换到频域中，要用到傅里叶变换，一般使用的是二维快速傅里叶变换算法（FFT2D）

      得到图片的频域图片后，将水印编码后随机分布到各频率，然后与原图的频域进行叠加，将叠加水印的频谱进行傅里叶变换，即可得到添加了盲水印的图片。

      这种操作相当于往原来的图片信号中加入了噪声，这些噪声遍布全图，在空域中不容易对图片造成破坏。

      要提取水印，只需要把原图和带水印的图在频域中相减，然后根据原来的水印编码方式进行编码，即可提取出水印：

      • 工具

        如果某一题中可以获取两张几乎相同的图片，且进行空域上的变换（XOR等）和提取像素点之间的差异都无法获取有效信息时，就要考虑是否位盲水印的可能性。

    • 伪加密

      在 ZIP 文件中，每个文件的数据区与核心目录区都有通用标记位（Flags）

      核心目录区通用标记位距离核心目录区头50 4B 01 02 偏移8字节，其本身占两个字节，最低位表示该文件是否被加密。

      将最低为改为 1，改解压文件会提示输入密码，但实际未设置密码，所以输入什么都是错误的。

      类似的文件头处的通用标记位距离文件头50 4B 03 04 的偏移为6字节，其本身占2字节，最低位表示这个文件是否加密。1 为加密。

      RAR中也有类似的，RAR4中表加密的文件快中HeadFlags（距离文件头3字节位置），从低位起3bit。

      注意修改标志位后，文件块头本身的CRC（HEAD_CRC）会发生变化，需要重新计算。

    • ZIP的明文攻击

      ZIP的加密算法下，设置的密码被转换成3个4字节的key，再用这个key加密所有文件。所以只需要知道被压缩文件中的一系列连续的明文字节（或一整个明文文件），就可以恢复3个key，从而解密文件。

      再此之前，

      1、需要确定加密ZIP使用的压缩工具版本与算法，010Editor查看，在核心目录区的通用标志位找到相关信息，或是查看文件属性，使用zipinfo（linux）命令查看,

      4-5 个字节记录了解压缩文件所需的最低支持的 ZIP 规范版本

      6-7 个字节记录通用标志位

      8-9 个字节记录压缩包所用到的压缩方式

      2、之后需要将已知明文以同样方式进行压缩，但不加入任何密码。对于完整明文的情况，如果方法正确的话，最终压缩后的大小应该差12个字节

      • 协议过滤

        http        // 仅显示HTTP协议包
dns         // 仅显示DNS协议包
tcp         // 仅显示TCP协议包

        IP地址过滤

        ip.src == 192.168.1.1          // 源IP为192.168.1.1
ip.dst == 192.168.1.100        // 目标IP为192.168.1.100
ip.addr == 192.168.1.0/24      // IP在192.168.1.0/24网段

        端口过滤

        tcp.port == 80                // TCP端口80（HTTP）
udp.port == 53                // UDP端口53（DNS）
tcp.srcport == 443            // TCP源端口443（HTTPS）

        逻辑运算符

        and / &&                      // 与关系
or  / ||                      // 或关系
not / !                       // 非关系
        http && ip.src == 192.168.1.1        // 源IP为192.168.1.1的HTTP包
dns || icmp                          // DNS或ICMP包
not tcp.port == 22                   // 非SSH（22端口）的TCP包

        HTTP过滤

        http.request.method == "POST"        // POST请求
http.host == "example.com"           // 主机名为example.com
http.content_type == "application/json"  // JSON格式请求
http.response.code == 404            // HTTP 404响应

        DNS过滤

        dns.qry.name == "google.com"         // 查询google.com的DNS请求
dns.resp.type == 1                   // A记录（IP地址）响应

        TCP/UDP过滤

        tcp.flags.syn == 1                   // SYN包（TCP连接建立）
tcp.flags.fin == 1                   // FIN包（TCP连接关闭）
tcp.window_size < 1024               // 窗口大小小于1024的TCP包
udp.length > 512                     // UDP包长度大于512字节

        字段包含过滤

        http contains "password"             // HTTP包中包含"password"字符串
data.data matches ".*[0-9]{4}.*"     // 数据字段匹配正则表达式（包含4位数字）

        比较运算符

        ==, !=, >, <, >=, <=                // 等于、不等于、大于等
        frame.time > "2023-01-01 12:00:00"  // 时间在2023年1月1日中午12点之后的包
ip.len >= 1500                      // IP包长度大于等于1500字节

        查找特定IP的HTTP请求：

        ip.addr == 192.168.1.1 && http      // 192.168.1.1的所有HTTP通信

        过滤SSH会话

        tcp.port == 22 && ip.src == 10.0.0.5  // 源IP为10.0.0.5的SSH流量

        分析DNS查询

        dns.qry.type == 1 || dns.qry.type == 28  // A记录（1）或AAAA记录（28）的DNS查询

        找出大流量TCP连接

        tcp.stream > 1000 && ip.len > 1000  // 流ID大于1000且包长度大于1000的TCP流

        选择一个流量包：“作为过滤器应用”s

      • FAT16

        FAT（File Allocation Table），是文件分配表。即用这张表来记录文件所在位置。

        FAT文件系统中，有两个数据单位，扇区和簇

        扇区：一般为512字节

        簇：由若干个扇区组成，是存取数据的最小单位。

        只要文件按照正确顺序去读取簇，就可以获取文件内容

        用16bit表示一个簇。

        基本结构：DBR扇区，FAT表1，FAT表2，根目录和数据区。

        1、DBR扇区是第一个扇区（偏移为0），包括一个引导程序 和 称为BPB的本分区参数记录，记录着本分区的起始扇区，结束扇区，文件系统等。

        2、FAT表1和FAT表2

        是保存簇状态的地方。第0~1个字节表示第0个簇，第2~3个字节表示第1个簇。

        其中FAT2是FAT1的备份。

        0x200h开始为FAT表1，

        首先从目录信息中读取得知它的第一个簇号为0x08

        此时读200h+8h*2=210h处信息，得到09 00（小端序：9h）为下一个簇号，

        此时读200h+9h*2=212h处信息，以此类推。。。。

        最后得到该文件8h，9h，Ah，Bh四个簇组成，依次读取即可。

  • DctSvd

    Dct（离散余弦变换）：

    将图像从空间域转换到频域，把图像的像素值分解为不同频率的预先分量。

    低频分量：对应图像的整体轮廓（人眼敏感）；

    高频分量：对应细节和噪声。

    在图像处理中：

    分块DCT：将图片分割为8x8或16x16的子块，逐个进行DCT。

    频域修改：在频域中修改系数（嵌入水印），可逆。

    Svd（奇异值分解）

    对矩阵（如灰度图像的像素矩阵A）进行分解：A = UΣV^T

    U和V：正交矩阵（左 / 右奇异向量，反映图像的结构特征）。

    Σ：对角矩阵（奇异值，按从大到小排列，反映对应奇异向量的重要性）。

    在图像处理中：

    图像压缩：保留前k个最大奇异值（能量主要集中在前几个），近似还原图像。

    水印嵌入：修改奇异值（调整参数），利用其稳定性。
