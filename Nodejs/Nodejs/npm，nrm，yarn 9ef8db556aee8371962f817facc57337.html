npm，nrm，yarn

  • 什么是npm?
    • 包管理工具：管理项目中使用的第三方库（如 React、Vue、lodash 等）。
    • 命令行工具：通过终端命令操作（如 npm install、npm run）。
    • 生态系统：包含海量开源包，解决各种开发需求（UI 组件、工具函数、构建工具等）。
  • package.json文件

    项目的 “说明书”，记录项目基本信息和依赖关系，位于项目根目录，必需文件。

    {
  "name": "my-project",       // 项目名称（小写，无空格）
  "version": "1.0.0",         // 版本号（遵循语义化版本：MAJOR.MINOR.PATCH）
  "description": "我的第一个项目", // 项目描述
  "main": "index.js",         // 入口文件
  "scripts": {                // 自定义脚本命令
    "start": "node index.js",
    "test": "jest"
  },
  "dependencies": {           // 生产环境依赖（项目运行必需）
    "lodash": "^4.17.21"
  },
  "devDependencies": {        // 开发环境依赖（仅开发时需要，如测试工具）
    "eslint": "^8.56.0"
  },
  "author": "Your Name",      // 作者
  "license": "MIT"            // 开源许可证
}
    • 生产依赖（dependencies）：项目运行时必须的包（如 React 核心库）。
    • 开发依赖（devDependencies）：仅开发阶段使用的包（如代码检查工具 ESLint、测试工具 Jest）。
    • 全局依赖：安装在系统全局的工具（如 vue-cli、nodemon），可在任何项目使用。

    npm 包版本遵循 MAJOR.MINOR.PATCH 格式（如 4.17.21）：

    • MAJOR（主版本）：不兼容的 API 变更（如 4.x → 5.x）。
    • MINOR（次版本）：向后兼容的功能新增（如 4.17.x → 4.18.x）。
    • PATCH（补丁版本）：向后兼容的问题修复（如 4.17.20 → 4.17.21）。

    版本范围符号：

    • ^1.2.3：允许次版本和补丁版本更新（≥1.2.3 且 <2.0.0）
    • ~1.2.3：仅允许补丁版本更新（≥1.2.3 且 <1.3.0）
    • 1.2.3：固定版本（必须精确匹配）
    • ：任意版本
  • npm
    • 1、项目初始化（npm init）

      创建 package.json 文件，初始化项目：

      # 交互式创建（逐步提示输入项目信息）
npm init

# 快速创建（使用默认值，无需交互）
npm init -y  # 或 npm init --yes
    • 2、安装依赖（npm install）
      # 简写：npm i 包名
npm install lodash  # 安装最新版本
npm install react@18.2.0  # 安装指定版本
npm install axios@^1.0.0  # 安装符合版本范围的版本

      安装后，包会被下载到 node_modules 目录，并添加到 package.json 的 dependencies 中。

      安装开发依赖：

      # 简写：npm i -D 包名
npm install eslint --save-dev  # 仅开发环境使用

      安装后，包会添加到 package.json 的 devDependencies 中。

      安装全局依赖：

      # 简写：npm i -g 包名
npm install nodemon -g  # 全局安装 nodemon（用于自动重启 Node.js 服务）

      全局包安装路径：

      • Windows：C:\Users\用户名\AppData\Roaming\npm\node_modules
      • macOS/Linux：/usr/local/lib/node_modules 或 ~/.npm-global/lib/node_modules

      安装 package.json 中的所有依赖：

      克隆项目或迁移环境时，根据 package.json 安装所有依赖：

      npm install  # 安装 dependencies 和 devDependencies 中的所有包
    • 3、卸载依赖（npm uninstall）
      # 卸载生产依赖（同时从 package.json 中移除）
npm uninstall lodash  # 简写：npm r lodash

# 卸载开发依赖
npm uninstall eslint --save-dev  # 简写：npm r -D eslint

# 卸载全局依赖
npm uninstall nodemon -g  # 简写：npm r -g nodemon
    • 4、查看依赖信息
      # 查看已安装的依赖（项目内）
npm list  # 或 npm ls，显示依赖树
npm list --depth=0  # 只显示直接依赖（不展开子依赖）

# 查看全局安装的依赖
npm list -g --depth=0

# 查看某个包的详细信息（版本、描述、依赖等）
npm info lodash  # 或 npm view lodash
    • 5、脚本明令

      执行 package.json 中 scripts 字段定义的命令：

      # 运行 start 脚本（start 是特殊命令，可省略 run）
npm start  # 等价于 npm run start

# 运行自定义脚本（如 test、build）
npm run test
npm run build

# 示例：如果 scripts 中有 "dev": "webpack serve"
npm run dev  # 启动开发服务器

      常用场景：启动开发服务器、构建项目、运行测试、代码检查等。

    • 6、更新依赖
      # 检查可更新的依赖
npm outdated  # 列出所有有新版本的依赖

# 更新指定依赖到最新版本（符合版本范围）
npm update lodash  # 简写：npm up lodash

# 更新到最新版本（可能突破版本范围，谨慎使用）
npm install lodash@latest

      注意：npm update 只会更新到版本范围允许的最新版本（如 ^1.2.3 最多更到 1.x.x），不会跨主版本更新。

    • 7、其他实用命令
      # 清除 npm 缓存（解决安装异常时）
npm cache clean --force

# 查看 npm 配置（如镜像源）
npm config list

# 设置 npm 镜像源（加速下载，如淘宝镜像）
npm config set registry https://registry.npmmirror.com/

# 恢复官方镜像源
npm config set registry https://registry.npmjs.org/
    • node_modules与package-lock.json

      1. node_modules 目录

      • 存放所有安装的依赖包（包括直接依赖和间接依赖）。
      • 通常体积较大，不要提交到 Git（在 .gitignore 中添加 node_modules/）。

      2. package-lock.json 文件

      • 自动生成的锁定文件，精确记录所有依赖的版本号和安装路径。
      • 作用：
        • 确保团队成员或部署环境安装的依赖版本完全一致（解决 “在我电脑上能运行” 问题）。
        • 加速依赖安装（无需重新计算版本范围）。
      • 注意：必须提交到 Git，不要手动修改。
    • npm高级用法
      • 1、发布自己的包到npm
        # 1. 注册 npm 账号（官网：https://www.npmjs.com/）
# 2. 终端登录
npm login  # 输入用户名、密码、邮箱

# 3. 发布包（确保 package.json 中的 name 未被占用）
npm publish

# 4. 更新包（需先修改 version 字段）
npm publish

# 5. 撤销发布（发布后 24 小时内）
npm unpublish 包名@版本号 --force
      • 2、使用.npmrc配置文件

        在项目根目录创建 .npmrc 文件，自定义 npm 配置（优先级高于全局配置）：

        # 设置镜像源
registry=https://registry.npmmirror.com/

# 保存依赖时使用精确版本（而非 ^ 或 ~）
save-exact=true
      • 3、安装指定镜像包

        临时使用其他镜像安装包：

        npm install lodash --registry=https://registry.npmmirror.com/
    • 常见问题与解决
      • 1、安装包时权限错误（macOS/Linux）

        原因：没有 node_modules 或全局目录的写入权限。

        # 方案1：使用 sudo（不推荐，可能导致权限混乱）
sudo npm install -g 包名

# 方案2：修改 npm 全局安装路径（推荐）
npm config set prefix ~/.npm-global
# 然后将 ~/.npm-global/bin 添加到环境变量 PATH 中
      • 2、依赖冲突或安装失败

        尝试删除 node_modules 和 package-lock.json 后重新安装：

        rm -rf node_modules package-lock.json  # Windows: rmdir /s/q node_modules & del package-lock.json
npm install

      • 3、全局安装的包无法使用

        原因：全局安装路径未添加到系统环境变量 PATH 中。

        解决：

        1. 查看全局安装路径：npm config get prefix
        1. 将该路径下的 bin 目录添加到 PATH 中。

  • nrm

    nrm（Node Registry Manager）是一个 npm 镜像源管理工具，它可以帮助开发者快速切换、管理和测试不同的 npm 镜像源，解决因镜像源速度慢或不稳定导致的包安装问题。

    nrm 的作用就是：简化镜像源的切换和管理流程，无需手动执行 npm config set registry <url> 命令，通过简单的指令即可完成镜像源的切换、查看、测试等操作。

    • nrm安装
      # 全局安装 nrm
npm install -g nrm
安装完成后，验证是否安装成功：
nrm --version  # 查看版本号，如显示版本则安装成功
    • 常用命令
      • 1、查看所有可用镜像源

        列出 nrm 内置的所有镜像源，以及当前正在使用的镜像源（带 * 标记）：

        nrm ls

        输出例如：

        * npm -------- https://registry.npmjs.org/
  yarn ------- https://registry.yarnpkg.com/
  tencent ---- https://mirrors.cloud.tencent.com/npm/
  cnpm ------- https://r.cnpmjs.org/
  taobao ----- https://registry.npmmirror.com/
  npmMirror -- https://skimdb.npmjs.com/registry/
  --------------------
* 表示当前正在使用的镜像源（示例中是官方 npm 源）
内置镜像源包括：官方 npm、yarn、腾讯、cnpm、淘宝等（国内源推荐 taobao 或 tencent）
      • 2、切换镜像源
        切换到指定的镜像源，例如切换到淘宝镜像：
nrm use taobao
执行后输出：
Registry has been set to: https://registry.npmmirror.com/
切换后，后续执行 npm install 等命令时，会从新的镜像源下载包。
      • 3、测试镜像源速度
        测试指定镜像源的响应速度（单位：毫秒），帮助选择最快的镜像源：
# 测试单个镜像源（如淘宝源）
nrm test taobao

# 测试所有镜像源（不指定名称）
nrm test
-----------------------------------
测试单个源案例：
taobao ---- 123ms  # 响应时间越短，速度越快
----------------------------------
测试所有源的输出示例：
npm ---- 897ms
yarn --- 786ms
tencent - 156ms
cnpm --- 210ms
taobao - 123ms  # 淘宝源速度最快
npmMirror -- 987ms
      • 4、添加自定义镜像源

        如果内置镜像源不满足需求（如公司内部私有镜像源），可以手动添加：

        # 格式：nrm add <源名称> <源URL>
nrm add mycompany https://npm.mycompany.com/registry/

        添加后，可通过 nrm ls 查看是否添加成功。

      • 5、删除自定义镜像源

        删除已添加的自定义镜像源（内置镜像源无法删除）：

        nrm del mycompany
      • 6、查看当前镜像源
        nrm current
输出示例：
https://registry.npmmirror.com/  # 当前使用淘宝源
    • nrm与npm配置的关系

      nrm 本质上是对 npm 配置的封装，它的所有操作最终都会反映到 npm 的配置中。例如：

      • 执行 nrm use taobao 等价于 npm config set registry https://registry.npmmirror.com/
      • 执行 nrm current 等价于 npm config get registry

      可以通过 npm config list 命令验证镜像源是否切换成功：

      npm config list  # 查看 npm 配置，包含 registry 字段
    • 常见问题

      执行 nrm 命令提示 “命令未找到”

      • 原因：nrm 未全局安装，或安装路径未添加到系统环境变量。
      • 解决：
        1. 重新全局安装：npm install -g nrm
        1. 检查全局安装路径：npm config get prefix
        1. 将该路径添加到系统环境变量 PATH 中。

      2. 切换镜像源后安装包仍缓慢

      • 原因：镜像源可能同步不及时，或本地缓存问题。
      • 解决：
        1. 测试其他镜像源速度：nrm test，切换到更快的源
        1. 清除 npm 缓存：npm cache clean --force

      3. 私有镜像源认证问题

      • 原因：公司私有镜像源可能需要账号密码或 token 认证。
      • 解决：通过 npm 配置添加认证信息：
      # 方式1：用户名+密码
npm login --registry=https://npm.mycompany.com/

# 方式2：设置 token（根据私有源要求）
npm config set //npm.mycompany.com/:_authToken=你的token

  • yarn

    Yarn 是 Facebook 推出的一款高效的 JavaScript 包管理工具，旨在解决 npm（v3 及之前版本）的性能和安全性问题。它与 npm 功能类似，但在安装速度、依赖一致性和缓存机制上有显著优化，目前已成为前端开发的主流包管理工具之一。

    • 优势

      相比传统 npm，Yarn 的核心优势体现在：

      1. 速度更快采用并行安装机制（同时下载多个包）和离线缓存（已安装过的包无需重复下载），安装速度提升 2-3 倍。
      1. 依赖一致性通过 yarn.lock 文件精确锁定依赖版本和安装结构，确保不同环境安装的依赖完全一致（解决 "在我电脑上能运行" 问题）。
      1. 安全性更高安装前会校验包的完整性，避免恶意代码或损坏的包被安装。
      1. 丰富的命令集提供更简洁的命令（如 yarn add 替代 npm install）和额外功能（如工作区、插件系统）。
    • 安装

      1、npm

      npm install -g yarn  # 全局安装 Yarn

      2、官方安装

      curl -o- -L https://yarnpkg.com/install.sh | bash

      3、验证安装

      yarn --version  # 显示版本号即安装成功（如 1.22.19）
    • 核心机制

      1 package.json

      与 npm 一致，记录项目基本信息（名称、版本）和依赖关系，是项目的必备文件。

      2 yarn.lock

      自动生成的锁定文件，精确记录所有依赖的版本、下载地址和校验信息，必须提交到 Git，确保团队成员和部署环境的依赖一致性。

      3 缓存机制

      Yarn 会将下载的包缓存到本地目录（如 ~/.yarn/cache），再次安装相同包时直接从缓存读取，无需重复下载，大幅提升安装速度。

    • 常用命令
      • 1、项目初始化
        yarn init  # 交互式创建（逐步提示输入信息）
yarn init -y  # 快速创建（使用默认值，无需交互）
      • 2、安装依赖

        安装生产依赖

        # 安装最新版本
yarn add lodash  # 等价于 npm install lodash

# 安装指定版本
yarn add react@18.2.0  # 等价于 npm install react@18.2.0

# 安装特定范围版本
yarn add axios@^1.0.0  # 允许次版本更新

        安装后，依赖会添加到 package.json 的 dependencies 中。

        安装开发依赖

        # 简写：yarn add -D
yarn add eslint --dev  # 等价于 npm install eslint --save-dev

        安装全局依赖

        yarn global add nodemon  # 全局安装 nodemon（等价于 npm install -g nodemon）

        全局安装路径可通过 yarn global dir 查看。

        4、安装package.json中的所有依赖

        yarn  # 等价于 npm install，安装所有 dependencies 和 devDependencies
yarn install  # 完整写法，与上一条命令效果一致
      • 3、卸载依赖

      • 4、更新依赖

    • 高级特性

      1 工作区（Workspaces）

      用于管理多包项目（如 Monorepo），可在一个仓库中管理多个关联包，共享依赖，简化开发流程。

      配置方式（在 package.json 中）：

      {
  "private": true,  // 必须设为 private
  "workspaces": [
    "packages/*"  // 指定包所在目录
  ]
}

      2 离线模式（-offline）

      在无网络环境下，使用本地缓存安装依赖：

      yarn install --offline  # 仅从缓存安装，不联网

      3 选择性安装（-production）

      仅安装生产环境依赖（忽略 devDependencies）：

      yarn install --production  # 等价于 npm install --production
    • 常见问题与解决

      1 全局安装的包无法使用

      • 原因：全局安装路径未添加到系统环境变量 PATH。
      • 解决：
        1. 查看全局安装路径：yarn global dir
        1. 将路径添加到系统 PATH 中（如 Windows 需添加 C:\Users\用户名\AppData\Local\Yarn\bin）。

      2 依赖安装冲突

      • 解决：删除 node_modules、yarn.lock 后重新安装：
      rm -rf node_modules yarn.lock  # Windows: rmdir /s/q node_modules & del yarn.lock
yarn install

      3 镜像源切换后速度仍慢

      • 解决：测试镜像源速度，切换到更快的源：
      # 推荐国内镜像源
yarn config set registry https://registry.npmmirror.com/  # 淘宝镜像
yarn config set registry https://mirrors.cloud.tencent.com/npm/  # 腾讯镜像
