原生方法与技术

  • hasOwnProperty(key)
    对象.hasOwnProperty(属性名);
    • 参数 属性名：要检测的属性（字符串或 Symbol 类型）。
    • 返回值：true（自身包含该属性）/ false（不包含，或属性来自原型链）。
    // 定义一个对象
const person = {
  name: "张三",
  age: 25
};

// 检测自身属性
console.log(person.hasOwnProperty("name")); // true（name 是自身属性）
console.log(person.hasOwnProperty("age"));  // true（age 是自身属性）

// 检测不存在的属性
console.log(person.hasOwnProperty("gender")); // false

// 检测原型链上的属性（如 toString 是 Object 原型的方法）
console.log(person.hasOwnProperty("toString")); // false（toString 来自原型链）
  • Object.prototype.hasOwnProperty.call(currentObj, key)

    安全检查对象自身是否包含指定属性的标准写法，核心作用是避免原型链干扰，精准判断属性是否属于对象 “自己”（而非继承自原型）。

    • 为什么需要 call？直接用 currentObj.hasOwnProperty(key) 不行吗？

      直接调用 currentObj.hasOwnProperty(key) 存在 2 个风险，而 call 能解决这些问题：

      风险 1：对象自身重写了 hasOwnProperty 方法

      如果对象自己定义了名为 hasOwnProperty 的属性（覆盖了原型上的方法），直接调用会失效，甚至报错：

      const badObj = {
  name: "Bob",
  // 重写了 hasOwnProperty，变成普通属性（值为 123）
  hasOwnProperty: 123
};

// 错误：badObj.hasOwnProperty 是数字，不是函数，调用会报错
badObj.hasOwnProperty("name"); // Uncaught TypeError: badObj.hasOwnProperty is not a function

      风险 2：对象的原型链被修改，hasOwnProperty 丢失

      如果对象的原型被设置为 null（或原型链上没有 Object.prototype），对象会失去 hasOwnProperty 方法：

      // 创建一个原型为 null 的对象（完全没有继承任何方法）
const noProtoObj = Object.create(null);
noProtoObj.age = 20;

// 错误：noProtoObj 没有 hasOwnProperty 方法
noProtoObj.hasOwnProperty("age"); // Uncaught TypeError: noProtoObj.hasOwnProperty is not a function

      用 call 解决：强制绑定 this 到目标对象

      Object.prototype.hasOwnProperty.call(currentObj, key) 的核心逻辑是：调用 Object.prototype 上的原始 hasOwnProperty 方法，并强制让该方法的 this 指向 currentObj。

      这样无论 currentObj 自身是否重写 hasOwnProperty、是否有原型，都能安全调用原始方法：

  • 函数闭包
    // 有返回值，且返回值是复杂类型，而且要赋值给外面的变量。
function test() {
    var name = "zhangsun"
    var obj={
        a:1,
        b:2
    }
    return obj 
}
var res = test() // name被回收了， obj被赋值给res，执行空间被保留
res = null // 回收obj 执行空间被回收

// 闭包：
// 函数返回一个函数，被外界所引用，
// 这个内部函数就不会被销毁回收
// 内部函数所用到的外部函数的变量也不会被销毁

function outer() {
    var name = "zhangjian"
    return function inner () {
        return name+"12321"
    }
}

var func = outer()
console.log(func());

// 优点：让临时变量永驻内存
// 缺点：内存泄漏（解决：func=null）

// 封装
function FetchContainer(url) {
    return function (path) {
        return fetch(url+path)
    }
}

var fetcha = FetchContainer("https://api.githsub.com")
fetcha("/aaaaaaa").then(res => res.json()).then(data => console.log(data))
fetcha("/bbbbbbb").then(res => res.json()).then(data => console.log(data))

  • 防抖（debounce）

    防抖（debounce） 高阶函数。在调用防抖函数后，提供的函数应该在一定延迟 t 后被调用。

    如果在 t 毫秒内再次调用防抖函数，提供的函数的执行应该被取消并重置计时器。

    /**
 * @param {Function} fn
 * @param {number} t milliseconds
 * @return {Function}
 */
var debounce = function(fn, t) {
    var timer = null // 设置定时器
    return function(...args) {
		    // 及时清除定时器
        if (timer) clearTimeout(timer)
        // 设置正常定时器
        timer = setTimeout(()=>{
		        // 确保指针正确指向
            fn.apply(this, args)
            // 执行完毕，清除定时器
            timer = null
        }, t)
    }
};
