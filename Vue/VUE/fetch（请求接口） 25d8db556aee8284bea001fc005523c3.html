fetch（请求接口）

在 Vue 中，fetch 是一种用于发起网络请求的原生 JavaScript API（并非 Vue 专属），常用于从后端接口获取数据或提交数据。它基于 Promise 设计，语法简洁，替代了传统的XMLHttpRequest，在 Vue 项目中被广泛用于异步数据交互。

  • 基础语法
    fetch(url, options)
  .then(response => {
    // 处理响应（如解析为JSON、文本等）
    return response.json(); // 解析为JSON格式
  })
  .then(data => {
    // 处理解析后的数据
    console.log(data);
  })
  .catch(error => {
    // 捕获错误（如网络错误）
    console.error('请求失败:', error);
  });
    • url：必选，请求的接口地址（字符串）。
    • options：可选，配置对象，包含请求方法、 headers、body 等。
  • 在Vue中的典型用法

    在 Vue 组件中，fetch 通常用于生命周期钩子（如created、mounted）或事件方法（如按钮点击）中，用于获取数据并更新组件状态。

    获取数据：

    <template>
  <div>
    <ul>
      <li v-for="user in users" :key="user.id">
        {{ user.name }} - {{ user.email }}
      </li>
    </ul>
    <p v-if="loading">加载中...</p>
    <p v-if="error">{{ error }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      users: [], // 存储接口返回的用户数据
      loading: false, // 加载状态
      error: '' // 错误信息
    };
  },
  created() {
    // 组件创建后发起请求（适合初始化数据）
    this.fetchUsers();
  },
  methods: {
    fetchUsers() {
      this.loading = true; // 开始加载
      this.error = ''; // 清空错误

      // 发起GET请求
      fetch('https://api.example.com/users')
        .then(response => {
          // 检查HTTP响应状态（200-299为成功）
          if (!response.ok) {
            throw new Error(`HTTP错误，状态码: ${response.status}`);
          }
          // 解析响应为JSON（返回Promise）
          return response.json();
        })
        .then(data => {
          this.users = data; // 更新数据
          this.loading = false; // 结束加载
        })
        .catch(error => {
          this.error = error.message; // 捕获错误
          this.loading = false; // 结束加载
        });
    }
  }
};
</script>
  • options配置

    fetch 的第二个参数options用于配置请求细节，常用属性如下：

    1、请求方法（method）

    默认是GET，常用值：GET，POST，PUT，DELETE等

    fetch('/api/user', {
  method: 'POST' // 发送POST请求
});

    2、请求头（headers）

    设置请求头信息（如Content-Type）：

    fetch('/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json', // 表示请求体是JSON格式
    'Authorization': 'Bearer ' + token // 携带令牌（如JWT）
  }
});

    3、请求体（body）

    发送数据（仅用于POST、PUT等方法），需与Content-Type匹配：

    JSON数据：需要JSON.stringify()转换位字符串

    fetch('/api/user', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: '张三', age: 20 }) // JSON格式
});

    表单数据：用FormData对象

    const formData = new FormData();
formData.append('name', '张三');
formData.append('avatar', file); // 可上传文件

fetch('/api/upload', {
  method: 'POST',
  body: formData // 表单数据（自动设置Content-Type为multipart/form-data）
});

    4、其他常用配置

    credentials：控制是否写道Cookie（跨域请求时候）

    fetch('https://other-domain.com/api', {
  credentials: 'include' // 跨域请求携带Cookie
});
  • async/await简化代码

    fetch 返回 Promise，可配合async/await使代码更简洁（避免嵌套then）：

    methods: {
  async fetchUsers() { // 声明为async函数
    try {
      this.loading = true;
      const response = await fetch('https://api.example.com/users');
      
      if (!response.ok) {
        throw new Error(`HTTP错误，状态码: ${response.status}`);
      }
      
      const data = await response.json(); // 等待JSON解析
      this.users = data;
    } catch (error) {
      this.error = error.message;
    } finally {
      this.loading = false; // 无论成功失败，都结束加载
    }
  }
}
  • 注意事项
    1. 错误处理：fetch 仅在网络错误（如断网）时触发catch，对于 HTTP 错误状态（如 404、500）不会直接报错，需手动通过response.ok（true表示 200-299 状态）判断。
    1. 跨域问题：若请求跨域接口，需后端配置CORS（跨域资源共享），否则会被浏览器拦截。
    1. 兼容性：现代浏览器（Chrome、Firefox 等）支持fetch，但 IE 完全不支持，如需兼容可引入whatwg-fetch等 polyfill。
    1. 与 Axios 的区别：
      • fetch 是原生 API，无需额外安装；Axios 是第三方库，需引入（npm install axios）。
      • Axios 自动解析 JSON 响应，内置拦截器（请求 / 响应拦截），而fetch需手动处理。
      • 在 Vue 项目中，两者均可使用，根据需求选择（简单场景用fetch，复杂场景用Axios）。
  • fetch-get/post/put/delete写法
    • get：获取数据
      <template>
  <div>
    <button @click="getData">获取数据</button>
    <p>{{ result }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      result: null
    };
  },
  methods: {
    async getData() {
      try {
        // 1. 定义URL（可拼接查询参数，如?id=1&name=test）
        const url = 'https://api.example.com/data?id=1';
        
        // 2. 发送GET请求（默认method为GET，可省略）
        const response = await fetch(url, {
          method: 'GET', // 可选，默认就是GET
          headers: {
            'Content-Type': 'application/json', // 告诉服务器期望接收JSON
            // 可选：携带令牌（如JWT）
            'Authorization': 'Bearer ' + localStorage.getItem('token')
          },
          credentials: 'include' // 跨域请求时携带Cookie（如需）
        });

        // 3. 检查响应状态（200-299为成功）
        if (!response.ok) {
          throw new Error(`请求失败：${response.status}`);
        }

        // 4. 解析响应数据（根据实际返回格式选择json()/text()等）
        const data = await response.json();
        this.result = data; // 更新组件数据

      } catch (error) {
        console.error('GET请求错误：', error.message);
      }
    }
  }
};
</script>
    • post：提交数据

      向服务器提交新数据（如创建用户、提交表单），数据放在body中。

      headers:{

      “Content-Type”:”application/x-www-form-urlencoded”

      },

      body:”name=kkk&age=100”

      <template>
  <div>
    <button @click="submitData">提交数据</button>
  </div>
</template>

<script>
export default {
  methods: {
    async submitData() {
      try {
        // 1. 定义请求数据
        const postData = {
          name: '张三',
          age: 20,
          email: 'zhangsan@example.com'
        };

        // 2. 发送POST请求
        const response = await fetch('https://api.example.com/users', {
          method: 'POST', // 必须指定为POST
          headers: {
            'Content-Type': 'application/json', // 声明请求体为JSON格式
            'Authorization': 'Bearer ' + localStorage.getItem('token')
          },
          body: JSON.stringify(postData), // 数据必须转为JSON字符串
          credentials: 'include'
        });

        if (!response.ok) {
          throw new Error(`提交失败：${response.status}`);
        }

        const result = await response.json();
        console.log('创建成功：', result);

      } catch (error) {
        console.error('POST请求错误：', error.message);
      }
    }
  }
};
</script>
    • put：更新数据

      全量更新服务器资源（如替换整条数据），需指定资源 ID，数据放在body中。

      <template>
  <div>
    <button @click="updateData">更新数据</button>
  </div>
</template>

<script>
export default {
  methods: {
    async updateData() {
      try {
        const userId = 1; // 要更新的资源ID
        const updateData = {
          name: '张三（更新后）',
          age: 21 // 全量更新，需包含所有必填字段
        };

        const response = await fetch(`https://api.example.com/users/${userId}`, {
          method: 'PUT', // 必须指定为PUT
          headers: {
            'Content-Type': 'application/json',
            'Authorization': 'Bearer ' + localStorage.getItem('token')
          },
          body: JSON.stringify(updateData), // 全量更新的数据
          credentials: 'include'
        });

        if (!response.ok) {
          throw new Error(`更新失败：${response.status}`);
        }

        const result = await response.json();
        console.log('更新成功：', result);

      } catch (error) {
        console.error('PUT请求错误：', error.message);
      }
    }
  }
};
</script>
    • delete：删除数据

      删除服务器资源，需指定资源 ID，通常无请求体（或仅传递少量参数）。

      <template>
  <div>
    <button @click="deleteData">删除数据</button>
  </div>
</template>

<script>
export default {
  methods: {
    async deleteData() {
      try {
        const userId = 1; // 要删除的资源ID

        const response = await fetch(`https://api.example.com/users/${userId}`, {
          method: 'DELETE', // 必须指定为DELETE
          headers: {
            'Authorization': 'Bearer ' + localStorage.getItem('token') // 通常需要权限验证
          },
          credentials: 'include'
        });

        if (!response.ok) {
          throw new Error(`删除失败：${response.status}`);
        }

        // 有些接口删除成功后返回空，用text()解析
        const result = await response.text();
        console.log('删除成功：', result || '无返回数据');

      } catch (error) {
        console.error('DELETE请求错误：', error.message);
      }
    }
  }
};
</script>
    请求类型用途核心配置差异数据位置
    GET获取数据method: 'GET'（可省略）URL 查询参数
    POST创建数据method: 'POST'body（JSON 字符串）
    PUT全量更新数据method: 'PUT'body（全量数据）
    DELETE删除数据method: 'DELETE'通常在 URL 路径中
    • 注意事项
      1. 响应处理：
        • 用response.json()解析 JSON 格式响应（最常用）。
        • 用response.text()解析文本响应。
        • 用response.blob()解析二进制响应（如图片）。
      1. 错误处理：
        • fetch仅在网络错误（如断网）时触发catch，HTTP 错误状态（404、500 等）需通过response.ok手动判断。
      1. 跨域请求：
        • 需后端配置 CORS，否则请求会被浏览器拦截。
        • 跨域携带 Cookie 需设置credentials: 'include'。
      1. Content-Type：
        • 发送 JSON 数据时必须设置'Content-Type': 'application/json'，并将body转为 JSON 字符串。
        • 发送表单数据（含文件）时用FormData，无需设置Content-Type（浏览器自动处理）。
