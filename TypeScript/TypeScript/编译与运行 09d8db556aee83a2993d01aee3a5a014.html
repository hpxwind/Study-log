编译与运行

  • 手动编译
    1. 环境准备

    安装TypeScript和nodejs

    # 全局安装（推荐）
npm install -g typescript
# 验证安装成功（查看版本）
tsc -v  # 输出如 Version 5.3.3 即为成功
    1. 创建TS文件

    新建ex.ts文件, 写入简单代码

    // hello.ts
function sayHello(name: string): string {
  return `Hello, ${name}!`;
}

const userName: string = "TypeScript";
console.log(sayHello(userName));
    1. 编译TS为JS
    tsc ex.tsc

    编译后会生成hello.js文件

    // hello.js
function sayHello(name) {
  return `Hello, ${name}!`;
}
const userName = "TypeScript";
console.log(sayHello(userName));
    1. 运行JS文件

    通过nodejs运行编译后的js文件

  • tsconfig.json
    1. 生成tsconfig.json

    在项目根目录执行

    tsc --init # 生成默认配置文件
    1. 关键配置项解析

    tsconfig.json核心字段，这个是简化了的

    {
  "compilerOptions": {
    "target": "ES6",        // 编译目标 JS 版本（ES3/ES5/ES6/ES2020 等）
    "module": "CommonJS",   // 模块系统（CommonJS/ESModule/AMD 等）
    "outDir": "./dist",     // 编译后的 JS 文件输出目录（默认和 TS 同目录）
    "rootDir": "./src",     // TS 源文件根目录（指定后仅编译该目录下的文件）
    "strict": true,         // 开启严格类型检查（必开！TS 核心价值）
    "esModuleInterop": true,// 兼容 CommonJS 和 ESModule 模块
    "skipLibCheck": true,   // 跳过第三方库（如 node_modules）的类型检查
    "sourceMap": true       // 生成 sourceMap 文件，方便调试 TS 源码
  },
  "include": ["./src/**/*"], // 要编译的文件（**/* 表示所有子目录）
  "exclude": ["node_modules"] // 排除编译的文件/目录
}
    1. 基于配置文件编译

    假设项目结构：

    project/
├─ src/
│  └─ hello.ts  # TS 源文件
└─ tsconfig.json

    执行编译命令：

    tsc  # 编译 src 下所有 TS 文件，输出到 dist 目录

    编译后生成 dist/hello.js 运行

    node dist/hello.js
    1. 常用编译命令 & 参数
    命令 / 参数作用
    tsc读取 tsconfig.json 编译所有匹配文件
    tsc hello.ts编译单个 TS 文件
    tsc --watch / tsc -w监听 TS 文件变化，自动重新编译（开发时常用）
    tsc --noEmit仅做类型检查，不生成 JS 文件（快速验证代码类型是否合法）
    tsc --target ES5 hello.ts覆盖配置文件，指定编译目标为 ES5
  • 简化编译+运行流程
    1. 使用 ts-node (直接运行TS)

    ts-node可以跳过手动编译，直接运行TS文件（底层是自动编译+运行）

    # 安装 ts-node（全局/本地均可）
npm install -g ts-node
# 直接运行 TS 文件
ts-node src/hello.ts
# 输出：Hello, TypeScript!

    开发时候监听文件并自动运行：

    ts-node-dev --respawn src/hello.ts  
# 需要先安装 ts-node-dev：npm install -g ts-node-dev
    1. 集成搭配npm脚本

    在 package.json 中配置脚本，统一命令

    {
  "scripts": {
    "build": "tsc",                // 编译 TS 到 dist
    "start": "node dist/hello.js", // 运行编译后的 JS
    "dev": "ts-node-dev src/hello.ts" // 开发模式（监听+自动运行）
  }
}

    执行脚本：

    npm run build  # 编译
npm run start  # 运行
npm run dev    # 开发模式
