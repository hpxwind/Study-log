索引查询类型

索引查询类型（又称「索引访问类型」）是 TypeScript 类型系统中提取已有类型中子类型的核心工具，通过「索引语法」从对象、数组、元组或联合类型中获取指定属性 / 元素的类型。其核心价值是复用已有类型的子结构、实现类型动态推导、提升类型关联性与安全性，也是映射类型、泛型工具类型的基础依赖。

索引查询类型的本质是：通过「索引表达式」访问目标类型（T）中对应索引的类型，类比 JavaScript 中通过「对象。属性」「数组 [下标]」访问值，索引查询类型是「类型层面的访问操作」。

  • 基本语法
    T[K]
    • T：目标类型（可以是对象类型、数组类型、元组类型、联合类型等）。
    • K：索引表达式（必须是 T 中有效索引的类型，如对象的属性名类型、数组的数字索引类型）。
    • 结果：T 中索引为 K 对应的类型。

  • 核心逻辑

    索引查询的核心是「类型层面的查找」：

    • 若 T 是对象类型，K 需是 T 的属性名类型（keyof T 的子集），返回该属性的类型；
    • 若 T 是数组 / 元组类型，K 可以是 number（数组的数字索引）或具体数字（元组的位置索引），返回对应元素的类型；
    • 若 T 是联合类型，K 需是所有联合成员共有的属性名类型，返回所有成员对应属性的类型联合。
  • 访问对象类型的属性类型

    最常用场景：从接口 / 对象类型中提取单个或多个属性的类型。

    1.提取单个属性类型

    // 源对象类型
interface User {
  name: string;
  age: number;
  address: {
    province: string;
    city: string;
  };
  isActive?: boolean; // 可选属性
}

// 提取单个属性的类型：User["name"] → string
type UserNameType = User["name"]; // string

// 提取可选属性的类型：自动包含 undefined
type IsActiveType = User["isActive"]; // boolean | undefined

// 提取嵌套对象的属性类型：User["address"] → 嵌套对象类型
type AddressType = User["address"]; // { province: string; city: string }

    2.提取多个属性的类型（联合索引）

    当 K 是「属性名联合类型」时，返回多个属性类型的联合：

    // 提取 "name" 和 "age" 的类型：string | number
type NameOrAgeType = User["name" | "age"]; // string | number

// 结合 keyof T：提取所有属性的类型联合
type AllUserPropTypes = User[keyof User]; 
// string | number | { province: string; city: string } | boolean | undefined

    3.通过类型别名作为索引

    索引表达式 K 可复用类型别名，提升灵活性：

    type TargetProp = "name" | "address";
// 提取 TargetProp 对应的属性类型：string | { province: string; city: string }
type TargetPropTypes = User[TargetProp];
  • 访问数组/元组类型的元素类型

    数组和元组的索引查询核心是「数字索引」的应用，需区分数组（动态长度）和元组（固定长度）的差异。

    1.数组类型：提取元素类型

    数组类型的索引是 number（所有数字下标共享同一元素类型），因此用 T[number] 提取元素类型：

    // 数组类型
type NumberArray = number[];
type StringArray = Array<string>;

// 提取数组元素类型：NumberArray[number] → number
type NumType = NumberArray[number]; // number

// 提取泛型数组的元素类型：StringArray[number] → string
type StrType = StringArray[number]; // string

// 复杂数组：提取嵌套类型
type UserArray = User[];
type UserArrayItem = UserArray[number]; // 等价于 User 类型

    2.元组类型：提取指定位置/所有元素类型

    元组是「固定长度、元素类型可能不同」的数组，支持两种索引方式：

    • 具体数字索引：提取对应位置的元素类型；
    • number 索引：提取所有元素类型的联合。
    // 元组类型
type Point = [number, string, boolean]; // 索引 0: number, 1: string, 2: boolean

// 提取指定位置的元素类型：Point[0] → number
type XType = Point[0]; // number
type YType = Point[1]; // string

// 提取所有元素类型的联合：Point[number] → number | string | boolean
type PointItemTypes = Point[number]; // number | string | boolean

// 错误示例：元组索引不能超出长度（TypeScript 会报错）
type InvalidIndex = Point[3]; // 报错：索引 3 超出元组长度 3

  • 访问联合类型的属性类型

    若 T 是联合类型，K 必须是「所有联合成员共有的属性名」，返回所有成员对应属性的类型联合（遵循「分配律」）。

    // 联合类型（共享 id 和 name 属性）
type Student = { id: number; name: string; grade: number };
type Teacher = { id: number; name: string; subject: string };
type Person = Student | Teacher;

// 提取共有属性 id 的类型：number（所有成员的 id 都是 number）
type PersonIdType = Person["id"]; // number

// 提取共有属性 name 的类型：string（所有成员的 name 都是 string）
type PersonNameType = Person["name"]; // string

// 提取非共有属性：报错（grade 不是 Teacher 的属性，subject 不是 Student 的属性）
type InvalidProp = Person["grade"]; // 报错：Property 'grade' does not exist on type 'Person'
  • 嵌套索引访问（深层提取）

    支持多层索引链式访问，从嵌套结构中提取深层子类型：

    // 从 User → address → city 提取类型：string
type UserCityType = User["address"]["city"]; // string

// 结合数组和对象：提取数组中对象的深层属性
type UserArrayCityType = UserArray[number]["address"]["city"]; // string

// 结合元组和对象：提取元组中指定位置对象的属性
type PointTuple = [User, Student];
type TupleFirstUserAge = PointTuple[0]["age"]; // number（元组第 0 项是 User，提取 age）
  • 结合泛型：动态提取属性类型

    通过泛型参数接收「属性名」，实现通用的属性类型提取工具，提升类型复用性。

    // 泛型工具：提取 T 中 K 属性的类型（K 必须是 T 的属性名）
type GetPropType<T, K extends keyof T> = T[K];

// 使用：提取 User 的 name 属性类型
type UserName = GetPropType<User, "name">; // string

// 提取 User 的 address 属性类型
type UserAddress = GetPropType<User, "address">; // { province: string; city: string }

// 提取数组的元素类型（K 为 number）
type ArrayItem<T extends any[]> = GetPropType<T, number>;
type NumArrayItem = ArrayItem<number[]>; // number

  • 结合条件类型：过滤/转换属性类型

    通过条件类型判断索引对应的类型，实现「类型过滤」或「动态转换」。

    // 工具类型：提取 T 中类型为 string 的属性名
type StringPropNames<T> = {
  [K in keyof T]: T[K] extends string ? K : never;
}[keyof T]; // 索引访问：提取所有非 never 的属性名（never 会被自动过滤）

// 使用：获取 User 中类型为 string 的属性名
type UserStringProps = StringPropNames<User>; // "name" | "address.province"？不，实际是 "name"（address 是对象类型）
// 修正：User 中 string 类型的属性是 name、address.province、address.city？
// 上述工具仅处理浅层属性，深层需递归，结合索引访问可实现：

// 递归提取所有 string 类型的属性名（浅层+深层）
type DeepStringPropNames<T> = T extends object 
  ? {
      [K in keyof T]: 
        T[K] extends string ? K : 
        T[K] extends object ? `${string & K}.${DeepStringPropNames<T[K]>}` : 
        never;
    }[keyof T]
  : never;

// 使用：提取 User 中所有 string 类型的属性名（含深层）
type UserDeepStringProps = DeepStringPropNames<User>; // "name" | "address.province" | "address.city"
  • 结合映射类型：遍历+提取（呼应映射类型知识点）

    映射类型的核心是「遍历属性」，而索引查询类型（T[K]）是「提取属性类型」的关键，二者是天然的组合（之前映射类型中的 [K in keyof T]: T[K] 正是此组合）。

    // 映射类型 + 索引访问：将 T 中所有属性转为其类型的数组
type PropToArr<T> = {
  [K in keyof T]: T[K][]; // 索引访问 T[K]，转为数组类型
};

// 使用：User 的所有属性转为对应类型的数组
type UserPropArr = PropToArr<User>;
// {
//   name: string[];
//   age: number[];
//   address: { province: string; city: string }[];
//   isActive?: (boolean | undefined)[];
// }
  • 访问基本类型的原型属性类型

    keyof 可获取基本类型（如 string、number）的原型属性名，结合索引访问可提取其原型方法 / 属性的类型。

    // 提取 string 类型的 length 属性类型：number
type StringLengthType = string["length"]; // number

// 提取 string 类型的 slice 方法类型：(start?: number | undefined, end?: number | undefined) => string
type StringSliceType = string["slice"]; 

// 提取 number 类型的 toFixed 方法类型：(fractionDigits?: number | undefined) => string
type NumberToFixedType = number["toFixed"];
  • 提取API响应的字段类型

    后端返回的复杂响应类型中，提取前端所需的特定字段类型，避免重复定义。

    // API 响应类型
interface ApiResponse {
  code: number;
  message: string;
  data: {
    userList: User[];
    total: number;
  };
}

// 提取 data 字段类型
type ResponseData = ApiResponse["data"]; // { userList: User[]; total: number }

// 提取 userList 字段类型
type UserList = ApiResponse["data"]["userList"]; // User[]

// 提取 userList 中单个用户的类型
type ApiUser = UserList[number]; // User
  • 实现类型安全的属性访问函数

    通过泛型 + 索引查询，封装通用的属性访问函数，确保访问的属性存在且类型正确。

    // 类型安全的属性访问函数：获取 obj 中 key 对应的属性值
function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

// 使用：
const user: User = { name: "张三", age: 20, address: { province: "北京", city: "北京" } };

const userName = getProp(user, "name"); // string（类型自动推导）
const userAge = getProp(user, "age"); // number
const userCity = getProp(user, "address").city; // string

// 错误用法：访问不存在的属性，TypeScript 报错
const userGender = getProp(user, "gender"); // 报错：Argument of type 'gender' is not assignable to parameter of type 'keyof User'
  • 复用嵌套/数组子类型，减少重复定义

    当类型结构复杂时，通过索引访问提取子类型，避免重复书写嵌套结构。

    // 无需重复定义 Address 类型，从 User 中提取
type Address = User["address"]; // { province: string; city: string }

// 无需重复定义 User 数组元素类型，从数组类型中提取
type UserFromArray = (User[])["number"]; // User
  • 配合类型守卫实现动态类型判断

    结合索引访问和类型守卫，判断对象属性的类型，提升代码安全性。

    // 判断 obj 的 key 属性是否为 string 类型
function isPropString<T, K extends keyof T>(obj: T, key: K): obj is T & { [P in K]: string } {
  return typeof obj[key] === "string";
}

// 使用：
if (isPropString(user, "name")) {
  user.name.toUpperCase(); // 安全：name 被推导为 string 类型
}
  • 注意点
    1. 索引必须是目标类型的有效索引K 必须是 keyof T 的子集（或 number 对于数组），否则 TypeScript 会报错。❌ 错误示例：User["gender"]（gender 不是 User 的属性）；✅ 正确示例：User["name"]（name 是 User 的属性）。
    1. 联合类型的索引访问遵循分配律若 T 是联合类型，T[K] 会对每个联合成员执行索引访问，最终返回类型联合。示例：(Student | Teacher)["id"] → Student["id"] | Teacher["id"] → number | number → number。
    1. 可选属性的索引访问会包含 undefined若属性是可选的（?），索引访问的结果会自动添加 undefined 类型（即使属性本身没有显式声明）。示例：User["isActive"] → boolean | undefined（而非单纯 boolean）。
    1. 元组的索引不能超出长度元组是固定长度的，使用超出长度的数字索引会报错（数组无此限制，因数组长度动态）。❌ 错误示例：[number, string][2]（元组长度为 2，索引 2 无效）。
    1. keyof 对基本类型的处理基本类型（string、number）的 keyof 会返回其原型属性 / 方法的名称（如 length、slice），因此可通过索引访问其原型类型，但需注意实际用途（通常用于工具类型封装）。
