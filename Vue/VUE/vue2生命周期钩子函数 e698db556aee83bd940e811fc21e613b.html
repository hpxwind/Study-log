vue2生命周期钩子函数

Vue2 的生命周期可分为 4 个核心阶段：创建阶段、挂载阶段、更新阶段、销毁阶段，每个阶段包含 2 个关键钩子，共 8 个核心钩子。此外还有针对keep-alive组件的特殊钩子。

beforeCreate → created → beforeMount → mounted → 
[当数据更新时：beforeUpdate → updated] → 
[当实例销毁时：beforeDestroy → destroyed]
  • 创建阶段：实例初始化(内存中)

    阶段目标：Vue 实例在内存中被创建，初始化数据观测、事件机制等，但尚未关联 DOM。

    1、beforeCreate()

    执行时机：Vue初始化之后（new Vue() 被调用），数据观测（data）和事件配置之前触发

    可访问的内容：

    • 无法访问data中的数据（如this.myname为undefined）
    • 无法访问methods中的方法
    • 无法访问 DOM（this.$el为undefined）

    几乎不用，因为此时实例尚未准备好任何核心数据。

    2、created() 请求接口一般

    执行时机：Vue 实例创建完成后触发（数据观测、属性和方法的运算、事件回调已配置完成）。

    可访问的内容：

    • 可以访问data中的数据（如this.myname）
    • 可以调用methods中的方法
    • 仍无法访问 DOM（this.$el为undefined，因为模板还未渲染）

    常见用途：

    • 初始化数据（如从本地存储读取数据：this.myname = localStorage.getItem('name')）
    • 发起初始网络请求（如获取用户信息）
    • 绑定自定义事件（非 DOM 事件）
  • 挂载阶段：实例与DOM关联

    阶段目标：将 Vue 实例的模板渲染为真实 DOM，并挂载到页面中（绑定到el选项指定的 DOM 元素）。

    1、beforeMount()

    执行时机：模板编译完成（虚拟 DOM 已生成），但尚未挂载到真实 DOM 上时触发。

    执行时机：

    • 可以访问data和methods
    • this.$el已存在（但为虚拟 DOM，未渲染到页面）

    极少使用，可在 DOM 挂载前做最后准备（如修改数据，会被合并到首次渲染中）。

    2、mounted() 获取dom

    执行时机：Vue 实例的模板已成功挂载到真实 DOM 上（el被新创建的vm.$el替换），页面可见时触发。

    可访问内容：

    • 可以访问data、methods
    • 可以访问真实 DOM（this.$el为真实 DOM 元素）
    • 可以通过this.$refs获取 DOM 元素

    常见用途：

    • 执行 DOM 操作（如初始化第三方插件：地图、图表等，需要真实 DOM）
    • 发起依赖 DOM 的网络请求（如根据 DOM 尺寸加载数据）
    • 绑定 DOM 事件（如监听滚动、 resize 事件）
  • 更新阶段：数据变化导致DOM更新

    阶段目标：当响应式数据（data或computed）发生变化时，Vue 会重新渲染 DOM，此阶段的钩子用于监控更新过程。

    1、beforeUpdate()

    执行时机：响应式数据发生变化后，DOM 更新前触发（虚拟 DOM 重新渲染，但尚未应用到真实 DOM）。

    可访问的内容：

    • 可以获取更新前的 DOM 状态（如旧的输入框值）
    • 可以访问更新后的数据（this.myname已变为新值）

    常见用途：获取更新前的 DOM 信息（如滚动位置），用于更新后恢复状态。

    2、update()

    执行时机：响应式数据变化导致 DOM 更新完成后触发（真实 DOM 已更新）。

    可访问内容：

    • 可以获取更新后的 DOM 状态（如新的输入框值）
    • 可以访问更新后的数据

    常见用途：处理更新后的 DOM 逻辑（如根据新 DOM 尺寸调整布局）。

    注意避免在updated()中修改数据（可能导致无限循环更新）。

  • 销毁阶段：实例被销毁

    阶段目标：Vue 实例被销毁，释放资源（移除事件监听、解绑数据等）。

    1、beforeDestroy()

    执行时机：Vue 实例销毁前触发（实例仍完全可用）。

    可访问的内容：

    • 可以访问所有data、methods和 DOM
    • 实例尚未被销毁，仍能正常操作

    常见用途：

    • 清理资源（如清除定时器clearInterval(timer)）
    • 解绑自定义事件（如this.$off('eventName')）
    • 取消网络请求（如 Axios 的cancelToken）

    2、destroyed()

    执行时机：Vue 实例销毁后触发（所有事件监听被移除，子实例被销毁，与 DOM 的关联被切断）。

    可访问的内容：

    • 虽然仍能访问data和methods，但修改数据不会触发更新
    • DOM 可能仍存在，但已与 Vue 实例解绑

    常见用途：做最终的资源清理（如释放内存缓存）。

  • 特殊钩子（针对keep-alive组件）

    当组件被<keep-alive>包裹时，会触发以下 2 个钩子（用于缓存组件状态）：

    • activated()：组件被激活（从缓存中取出显示）时触发。
    • deactivated()：组件被缓存（隐藏）时触发。

    用途：在组件激活时重新发起请求，或在缓存时保存临时状态。

  • 生命周期钩子的核心原则
    1. 自动触发：钩子函数由 Vue 自动调用，无需手动执行。
    1. this 指向：所有钩子函数中的this均指向当前 Vue 实例。
    1. 执行次数：
      • beforeCreate、created、beforeMount、mounted 只执行一次（实例创建到挂载完成）。
      • beforeUpdate、updated 每次响应式数据变化时都会触发（可能多次）。
      • beforeDestroy、destroyed 只执行一次（实例销毁时）。
钩子典型场景
created初始化数据、发起初始网络请求
mountedDOM 操作、初始化第三方插件（依赖真实 DOM）
beforeUpdate保存更新前的 DOM 状态（如滚动位置）
updated处理更新后的 DOM 逻辑（如调整布局）
beforeDestroy清理定时器、解绑事件、取消网络请求
