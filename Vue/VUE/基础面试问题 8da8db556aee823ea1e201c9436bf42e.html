基础面试问题

1. vue3中的setup与data，methods有什么关系，有冲突以谁为主，可以互相读取吗？

  1. 关系
    Vue3 兼容选项式 API（data、methods）和组合式 API（setup），二者可在同一组件混用；setup 执行时机早于 data、methods。
  1. 同名冲突优先级
    出现同名变量 / 方法时，setup 优先级更高，会覆盖 data、methods。
  1. 互相读取
  • 普通 setup 函数：内部无 this，执行初期无法直接访问 data/methods，可借助 getCurrentInstance 或生命周期访问；setup 通过 return 暴露的内容，可被 methods 通过 this 读取调用。
  • <script setup> 语法糖：需用 getCurrentInstance 读取选项式数据 / 方法；选项式 API 想要访问它内部内容，必须通过 defineExpose 主动暴露。

2. ref为什么也可以处理数组与对象，更改对象整体的时候ref和reactive都可以使用吗，为什么，两者的区别以及什么时候使用最好？

  1. ref 为何能处理对象 / 数组
    ref 会将所有值包装为 { value: 原值 }，若内部是对象 / 数组，底层会自动调用 reactive 做 Proxy 代理，因此支持引用类型。
  1. 整体替换对象
  • ref：可以直接整体赋值，响应式正常；
  • reactive：不建议整体重赋值，会丢失 Proxy 代理、失效，仅能修改内部属性 / 元素。
  1. 核心区别
  • 取值赋值：ref 脚本内需 .value，模板省略；reactive 直接访问。
  • 支持类型：ref 兼容基本类型、引用类型；reactive 仅支持对象、数组等引用类型。
  • 底层：ref 结合 Object.defineProperty+Proxy；reactive 纯 Proxy。
  • 解构：ref 解构保留响应式；reactive 直接解构会丢失响应。

使用原则：

1. 需要基本类型的响应式数据，必须使用ref
2. 需要响应式对象，层级不深，ref，reactive都可以
3. 需要响应式对象，且层级较深，推荐使用reactive

3. computed计算属性可以直接修改ref或reactive的值吗，方法如何修改computed中的值?

  1. 只读computed只能读取ref/reactive，不能直接赋值修改；
  1. 可写computed配置get、set，set内部修改源ref/reactive数据；
  1. 场景：单纯派生数据用只读，需要双向改动源数据时开启set。
    const num = ref(10)
const double = computed({
  get() {
    return num.value * 2
  },
  set(val) {
    num.value = val / 2 // 在set里改原始响应式变量
  }
})
// 修改
double.value = 20
// 无set double.value = 100 报错
console.log(num.value) // 10

4. watch的五种情况，如何解除watch监听，watch监听的四种类型？

一、watch四种监听类型

  1. 监听单个ref基本值
  1. 监听ref包裹的对象
  1. 监听reactive对象整个数据
  1. 监听reactive对象内部某个属性

二、五种常用配置情况

  1. 基础监听（默认只改触发）
  1. immediate:true 初始化立刻执行一次
  1. deep:true 深度监听嵌套对象
  1. 带新旧参数(newVal,oldVal)
  1. 多源监听，数组传多个数据源

三、解除监听

watch调用后会返回停止函数，执行即可销毁

const stopWatch = watch(num,()=>{})
stopWatch()

5. v-if 和 v-show的区别？

  • 渲染机制
    v-if：条件不成立时不渲染 DOM 节点；v-show：节点始终渲染，靠 CSS display:none 隐藏。
  • 性能开销
    频繁切换用 v-show（仅改样式，开销小）；初始大概率不显示用 v-if（减少初始 DOM 渲染）。
  • 配套指令
    v-if 可搭配 v-else/v-else-if；v-show 无分支语法。
  • 惰性加载
    v-if 为惰性，条件为真才创建组件 / 节点；v-show 不管真假都会初始化。

6. vue文件共有三种暴露方式分别是什么，怎么实现？

  1. 选项式 export default {}
    直接导出配置对象，基础写法
export default { data, methods, mounted }
  1. setup语法糖 <script setup>
    标签加setup，变量方法直接模板使用，无需return
<script setup>
const msg = '123'
</script>
  1. 普通setup函数
    export default中写setup，需return暴露变量方法
export default {
  setup() {
    const num = 1
    return { num }
  }
}

7. <header/>和<header></header>有什么区别？

  1. 自闭合<header/>：Vue/模板规范，单标签简写，无内部内容
  1. 双标签<header></header>：可包裹子元素、插槽内容
  1. 本质渲染DOM无差异，HTML原生header不能自闭合，仅模板语法简写

8. 路由传递参数的两种方式和细节上的注意点？

1. params 传参（路径占位）

写法：/user/:id 匹配 $route.params.id
注意：刷新页面参数会丢失；必须配置路由占位；push对象时name必填，path不能和params同用。

2. query 传参（拼接url问号）

写法：/detail?id=1 匹配 $route.query.id
注意：参数展示在地址栏，刷新不丢失；path/name均可搭配；适合传递公开少量数据。

精简取舍

隐私、短标识用params；不怕暴露、需持久用query；复杂大数据建议存pinia/localStorage。

9. replace模式是什么，还有什么模式，原理是什么？

两种跳转模式

  1. push（默认）
    原理：新增一条历史记录，可回退上一页。
  1. replace
    原理：替换当前历史记录，无新增，回退跳过本页面。

使用：$router.push() / $router.replace()

10. 自定义ref（customRef()）中的track()和trigger()的含义，以及运作原理？customRef() 有什么用处？

一、track()、trigger()含义与原理

  1. track()：收集依赖；读取数据时调用，记录用到该ref的模板/副作用函数。
  1. trigger()：派发更新；修改数据时调用，执行所有收集到的依赖，触发页面更新。
    原理：手动接管响应式追踪更新流程，替代原生ref自动依赖收集。

二、customRef作用

自定义响应式逻辑，可拦截get/set；常用于防抖节流、数据格式化、异步取值等场景。

11. Vue3中的非兼容性改变全部有哪些？

  1. 全局API改用createApp，移除Vue.prototype，用app.config.globalProperties
  1. 选项式this行为变化，setup无this
  1. v-model重写，多v-model绑定，.sync移除
  1. 生命周期部分更名（destroyed→unmounted、beforeDestroy→beforeUnmount）
  1. 过滤器filter移除，改用计算属性/方法
  1. 插槽slot语法变更，具名slot v-slot简写#
  1. 组件根节点不再唯一，支持多根Fragment
  1. 事件API $on/$off/$once移除，改用mitt
  1. 渲染函数h参数、VNode结构调整
  1. 过渡transition类名前缀调整，v-if过渡行为改动
