SSRF

>>>SSRF(Server-Side Request Forgery)，服务器端请求伪造，利用漏洞伪造服务器端发起请求，从而突破客户端获取不到数据的限制。

>>>在服务器对外发起请求获取数据的时候 ，未能做好目标地址的过滤与限制。

>>>服务端必须存在向外发送网络请求的能力。

>>>SSRF类型：1、回显SSRF。2、盲SSRF

>>>在线网络上，AI可以自动生成SSRF

  • SSRF漏洞危害

    主要用来进行内网探测存活主机，和开放端口，信息收集。

    1、可以对外网服务器所在的内网进行端口扫描，获取一些服务的banner信息；

    2、有时候可以攻击运行在内或本地的应用程序（比如溢出）；比如通过ssrf给内网的redis写定时任务，实现反弹shell

    3、对内网web应用进行指纹识别；

    4、攻击内外网的web应用，主要是使用get参数就可以实现的攻击（struts2，sql注入）；

    5、利用file协议赌气本地文件等；

    6、各种伪协议进行探测：http,file,dict,ftp,gopher等

  • 攻击原理以及流程

    1、侦测向外发送请求的能力

    通过探测内网服务，端口，远程执行代码，攻击内网web应用，读取文件等。

    →向存在SSRF漏洞的Web服务器发送恶意请求

    →Web服务器向位于防火墙后面的受害者服务器发出请求

    →受害者服务器把响应之后的数据传递回web服务器

    →web服务器会将数据返回给攻击者机器

  • PHP中产生SSRF漏洞的函数

    其实每个语言中，可以向网络发起请求的模块，函数等都有可能存在风险。

    1、file_get_contents()

    读取文件内容，支持通过URL赌气网络资源（需要allow_url_fopen=On）

    如果将用户1可控的URL直接传入该函数，攻击者可构造恶意URL让服务器发起非预期请求。

    <?php
$url = $_GET['url'];
$content = file_get_contents($url); // 攻击者可以传入 http://127.0.0.1/secret 等
?>

    2、fopen()

    打开文件或URL资源（需要allow_url_fopen=On）

    与file_get_contents()类似

    <?php
$url = $_GET['url'];
$handle = fopen($url, 'r'); // 可被滥用访问内网或本地文件
?>

    3、curl_exec()

    通过cURL库发送网络请求，支持多种协议

    若未限制协议和目标地址，攻击者可利用CURLOPT_URL构造内网IP，本地文件路径。

    <?php
$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, $_GET['url']); // 用户可控URL
curl_exec($ch); // 可访问 http://192.168.1.1 等内网地址
curl_close($ch);
?>

    4、fsockopen()

    打开网络连接，可手动构造HTTP等协议请求

    用户控制主机名和端口时，可让服务器连接内网服务

    <?php
$host = $_GET['host'];
$port = $_GET['port'];
$fp = fsockopen($host, $port); // 连接攻击者指定的内网主机和端口
?>

    5、stream_socket_client()

    创建网络流套接字连接，支持多种协议

    与fsockopen()类似

    <?php
$socket = stream_socket_client("tcp://{$_GET['host']}:{$_GET['port']}");
?>

    6、框架/库中的HTTP客户端

    许多 PHP 框架（如 Laravel、Symfony）或第三方库（如 Guzzle）提供的 HTTP 客户端，若未正确过滤用户输入的 URL，同样会导致 SSRF：

    <?php
// Guzzle 示例（未过滤URL）
$client = new GuzzleHttp\Client();
$response = $client->get($_GET['url']); // 风险：用户可控URL
?>
  • redis未授权访问漏洞

    redis时告诉缓存型数据库（类似还有mongdb），主要用户缓存一些频繁使用的数据来缓解数据库压力。

    redis通过内存储存数据，数据库通过磁盘存储。

    • redis安装
      1、安装 redis 依赖
Redis 是基于 C 语言编写的，因此首先需要安装 Redis 所需要的 gcc 依赖
yum install -y gcc tcl

2、将我们下载的 redis-6.2.13.tar.gz 上传到服务器上，然后解压，我直接上传到了 /root 目录下面
tar -xzf redis-6.2.13.tar.gz

3、进入 redis 目录，运行编译命令进行编译安装
cd redis-6.2.13
make && make install
默认的安装路径是在 /usr/local/bin 目录下，查看目录下的内容确保安装成功

4、可以直接启动 redis 了
redis-server
redis-server redis.conf # 指定配置文件启动，配置文件默认位置在我们解压的目录中，cd /root/redis-6.2.13 目录中
5、停止 redis 进程
ps -ef | grep redis
kill -9 pid 即可

6、修改配置文件
#先备份配置文件
cp redis.conf redis.conf.bak
vim redis.conf
修改如下两个配置
#允许访问的地址，默认是 127.0.0.1，会导致只能在本地访问。修改为 0.0.0.0 则可以在任意 IP 访问，生产环境不要设置为 0.0.0.0
bind 0.0.0.0
#守护进程，修改为 yes 后即可后台运行
daemonize yes
#设置后访问 Redis 必须输入密码
#注意：这个密码不用配置，要的就是没有密码直接可以连接 redis 服务，这就是未授权访问漏洞
#requirepass 123456

7、启动 redis
redis-server redis.conf
查看是否运行成功
ps -ef | grep redis

8、为了方便控制 redis，我们可以把 redis 添加为服务，就可以通过 systemctl 来控制 redis 了
第二部分
一、首先，新建一个系统服务文件
vim /etc/systemd/system/redis.service
二、写入如下内容
[Unit]
Description=redis-server
After=network.target

[Service]
Type=forking
ExecStart=/usr/local/bin/redis-server/root/redis-6.2.13/redis.conf #配置文件所在路径
PrivateTmp=true

[Install]
WantedBy=multi-user.target
#三、重载系统服务
systemctl daemon-reload
#四、需要重启主机
reboot
第三部分

9、然后可以用下面这组命令来操作 redis
#启动
systemctl start redis
#停止
systemctl stop redis
#重启
systemctl restart redis
#查看状态
systemctl status redis
#最后设置 redis 开机自启
systemctl enable redis

10、测试连接 redis 服务端，安装完 redis 之后，默认会安装一个本地的客户端 redis-cli，可以通过这个客户端连接操作服务端
redis-cli #直接回车就能连接本地的 redis 数据库服务端，默认是没有密码的
redis-cli -h 远程 ip 地址 #可连接远程 redis 服务端
    • redis基本操作

      redis默认端口为6379

      添加数据：set key value
获取数据：get key
删除数据：del key
查看有哪些key：key *
清空所有数据：flusha11 # (慎用)

      redis配置操作

      config set dir /home/test                # 设置工作目录
config set dbfilename redis.rdb          # 设置备份备份文件名
config get dir                           # 检查工作目录是否设置成功
config get dbfilename                    # 检查备份文件名是否设置成功
save                                     # 进行一次备份操作

    redis安装后默认绑定0.0.0.0:6379（允许所有网络访问），并且默认未启用认证机制（requirepass未配置），运行身份可能为root。

    如果管理员未修改默认配置，攻击者可以通过redis-cli或其他工具直接连接redis服务器，执行set，config，slaveof等命令，进行篡改，写入，窃取数据。（可以实现反弹shell，webshell等操作）
