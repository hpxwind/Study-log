CSRF

>>> 跨站请求伪造（Cross-site request forgery）/XSRF

>>> 1、用户在目标网站上处于登录状态。2、存在CSRF

csrf的攻击建立在浏览器与web服务器的会话之中。

欺骗用户访问URL

  • 原理与步骤

    1、用户登录网站A，浏览器验证成功，返回cookie

    2、用户又进入了不安全站点或网页B，B中有一段不安全代码，自动向A发送请求

    3、对A用户进行非法操作

    • 步骤：（表单必查、有框必查）

      1、确定目的，

      确保用户处于登录状态，

      2、捕获伪造请求，

      3、构造恶意连接，（POC）

      判断有无token或者referer表单，cookie是否带有SameSite属性

      → 存在，1、采用删除参数方法判断是否影响结果，不影响就是假的参数。

      2、token是否可以自行构造，还是随机的。

      3、SameSite属性设置，见下

      4、发送给目标，然后触发（需要存在有效凭证，目标网站存在CSRF漏洞）

      用户处于登录就说明存在登录凭证，在构造恶意连接时需要此凭证，
如果在一个没有登录过的浏览器上注入，会卡在登录界面

      SameSite属性 :追踪csrf，限制csrf攻击
SameSite值：
1、Strict：完全进制第三方Cookie，即只有当前网页的URL与请求目标一致时，才会带上Cookie。
2、Lax：大多数情况也不发送第三方Cookie，但导航到目标网址的GET请求除外。
3、None：Chrome将Lax变为默认值，网站可以选择显式关闭SameSite属性，设置为None。但必须同时设置Secure属性，否则无效。
  • 类型

    站外攻击：从其他站点B向目标A发起恶意请求。

    站内攻击：站点A存在漏洞，被植入恶意代码，用户执行存在漏洞的功能时触发。
