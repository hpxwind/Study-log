命名路由

命名路由是通过名称来标识路由的方式，而不是通过路径。它为路由提供一个唯一的名称，使得导航更加灵活和可维护。

  • 基础语法
    import Vue from 'vue'
import VueRouter from 'vue-router'

Vue.use(VueRouter)

const routes = [
  // 🔹 未命名路由（传统方式）
  {
    path: '/',
    component: Home
  },
  
  // 🔹 命名路由
  {
    path: '/',
    name: 'Home',        // 添加 name 属性
    component: () => import('@/views/Home.vue')
  },
  
  {
    path: '/about',
    name: 'About',
    component: () => import('@/views/About.vue'),
    meta: { title: '关于我们' }
  },
  
  {
    path: '/user/:id',
    name: 'UserProfile',  // 有意义的名称
    component: () => import('@/views/UserProfile.vue'),
    props: true
  },
  
  {
    path: '/contact',
    name: 'ContactUs',   // 描述性名称
    component: () => import('@/views/Contact.vue')
  }
]

const router = new VueRouter({
  mode: 'history',
  routes
})

export default router
  • 声明式导航中的命名路由
    <!-- 方式1：字符串路径（传统） -->
<router-link to="/" class="nav-link">首页</router-link>
<router-link to="/about" class="nav-link">关于</router-link>
    <!-- 方式2：命名路由（推荐） -->
<router-link :to="{ name: 'Home' }" class="nav-link">

    1. 基本编程式导航
    <template>
  <div class="user-navigation">

    1. 复杂的导航逻辑
    // 导航辅助函数
export class NavigationHelper {
  // 根据用户角色导航到不同页面
  static navigateByUserRole(role, additionalParams = {}) {
    const routes = {
      admin: { name: 'AdminDashboard', params: additionalParams },
      user: { name: 'UserDashboard', params: additionalParams },
      guest: { name: 'GuestWelcome', params: additionalParams }
    }
    
    const routeConfig = routes[role] || routes.guest
    return this.$router.push(routeConfig)
  }
  
  // 带权限检查的导航
  static async navigateWithAuthCheck(routeName, params = {}) {
    try {
      // 检查认证状态
      const isAuthenticated = await this.checkAuthentication()
      
      if (isAuthenticated) {
        this.$router.push({ name: routeName, params })
      } else {
        // 重定向到登录页，并保存目标路由
        this.$router.push({ 
          name: 'Login', 
          query: { redirect: routeName, ...params } 
        })
      }
    } catch (error) {
      console.error('导航错误:', error)
      this.$router.push({ name: 'Error' })
    }
  }
  
  // 条件导航
  static conditionalNavigate(condition, successRoute, failureRoute) {
    if (condition) {
      this.$router.push({ name: successRoute.name, params: successRoute.params })
    } else {
      this.$router.push({ name: failureRoute.name, params: failureRoute.params })
    }
  }
}

// 在 Vue 组件中使用
export default {
  methods: {
    async navigateToDashboard() {
      await NavigationHelper.navigateWithAuthCheck('UserDashboard', { userId: this.userId })
    },
    
    navigateBasedOnRole() {
      NavigationHelper.navigateByUserRole(this.userRole, { timestamp: Date.now() })
    }
  }
}
  • 命名路由的高级用法
    1. 路由别名的命名路由
    const routes = [
  {
    path: '/home',
    name: 'Home',
    component: Home,
    alias: ['/welcome', '/index']  // 别名路径
  },
  
  {
    path: '/about-us',
    name: 'About',
    component: About,
    alias: '/about'  // 单个别名
  },
  
  {
    path: '/products/:category',
    name: 'ProductCategory',
    component: ProductCategory,
    alias: [
      '/items/:category',
      '/goods/:category'
    ]
  }
]
    1. 命名视图的命名路由
    const routes = [
  {
    path: '/layout',
    name: 'ComplexLayout',
    components: {  // 注意是 components（复数）
      default: MainContent,     // 默认视图
      header: PageHeader,       // 命名视图
      sidebar: PageSidebar,
      footer: PageFooter
    },
    children: [
      {
        path: 'dashboard',
        name: 'DashboardLayout',
        components: {
          default: Dashboard,
          header: DashboardHeader,
          sidebar: DashboardSidebar
        }
      }
    ]
  }
]
