计算属性

计算属性（Computed Properties） 是用于处理复杂数据逻辑的一种方式，它基于响应式数据（data 或其他计算属性）派生新数据，并且具有缓存特性，能显著提升性能。

当模板中需要对数据进行复杂处理（如拼接、过滤、计算）时，直接在模板中写逻辑会导致：

  • 模板臃肿，可读性差；
  • 重复计算（模板重新渲染时会重复执行逻辑），性能损耗。

计算属性的出现就是为了解决这些问题：它将复杂逻辑封装，且只在依赖数据变化时重新计算。

计算属性定义在组件的 computed 选项中，每个计算属性是一个函数（默认是 getter 函数），返回处理后的结果。在模板中直接使用计算属性名（无需加括号）。

  • 简单计算
    <template>
  <div>
    <p>原始价格：{{ price }}</p>
    <p>含税价格（10%）：{{ taxPrice }}</p> <!-- 直接使用计算属性名 -->
  </div>
</template>

<script>
export default {
  data() {
    return {
      price: 100 // 原始价格
    }
  },
  computed: {
    // 计算属性：返回含税价格
    taxPrice() {
      console.log('计算含税价格...'); // 验证缓存特性
      return this.price * 1.1;
    }
  }
}
</script>

    效果：

    • 模板中通过 {{ taxPrice }} 直接使用计算结果；
    • 当 price 不变时，多次访问 taxPrice 只会执行一次计算（缓存生效）。
  • 过滤数据
    <template>
  <div>
    <input v-model="keyword" placeholder="搜索商品...">
    <ul>
      <!-- 渲染过滤后的列表 -->
      <li v-for="item in filteredProducts" :key="item.id">
        {{ item.name }}
      </li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      keyword: '',
      products: [
        { id: 1, name: 'iPhone' },
        { id: 2, name: 'iPad' },
        { id: 3, name: 'MacBook' }
      ]
    }
  },
  computed: {
    // 计算属性：根据关键词过滤商品
    filteredProducts() {
      return this.products.filter(item => 
        item.name.toLowerCase().includes(this.keyword.toLowerCase())
      );
    }
  }
}
</script>

    逻辑：

    • filteredProducts 依赖 keyword 和 products；
    • 当 keyword 变化时，自动重新过滤并返回新列表；
    • 若 keyword 不变，多次访问会直接返回缓存结果。
  • 核心特性：缓存机制

    计算属性的关键优势是缓存，它会记住依赖的响应式数据，只有当依赖变化时才会重新计算。这与方法（methods） 形成鲜明对比：

    特性计算属性（computed）方法（methods）
    调用方式模板中直接用名称（{{ prop }}）模板中需加括号（{{ method() }}）
    执行时机依赖变化时才重新计算每次模板渲染都会重新执行
    适用场景基于已有数据派生新数据（同步逻辑）处理业务逻辑（如事件处理）

    有无缓存对比：

    <template>
  <div>
    <!-- 计算属性：只在 count 变化时重新计算 -->
    <p>computed: {{ computedNum }}</p>
    <!-- 方法：每次渲染都会执行 -->
    <p>method: {{ methodNum() }}</p>
    <button @click="count++">+1</button>
  </div>
</template>

<script>
export default {
  data() { return { count: 0 } },
  computed: {
    computedNum() {
      console.log('computed 执行了');
      return this.count * 2;
    }
  },
  methods: {
    methodNum() {
      console.log('method 执行了');
      return this.count * 2;
    }
  }
}
</script>

    结果：

    • 初始渲染：两者都执行一次；
    • 点击按钮（count 变化）：两者都重新执行；
    • 不点击按钮（count 不变），刷新页面或触发其他数据更新导致模板重渲染时：
      • computedNum 不会重新执行（用缓存）；
      • methodNum() 会重新执行。
  • 计算机属性的setter(可写计算属性)

    默认情况下，计算属性是只读的（只有 getter）。若需要 “通过修改计算属性反向更新依赖数据”，可定义 set 方法。

    <template>
  <div>
    <p>姓：{{ firstName }}</p>
    <p>名：{{ lastName }}</p>
    <p>全名：{{ fullName }}</p>
    <input v-model="fullName" placeholder="输入全名">
  </div>
</template>

<script>
export default {
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    fullName: {
      // getter：根据 firstname 和 lastname 计算全名
      get() {
        return this.firstName + this.lastName;
      },
      // setter：当 fullName 被修改时，更新 firstname 和 lastname
      set(newValue) {
        // 假设输入格式为“姓+名”（如“李四”）
        const arr = newValue.split('');
        this.firstName = arr[0] || '';
        this.lastName = arr.slice(1).join('') || '';
      }
    }
  }
}
</script>

    效果：

    • 输入框修改 fullName 时，set 方法被触发，自动拆分并更新 firstName 和 lastName；
    • 修改 firstName 或 lastName 时，get 方法触发，fullName 自动更新。
  • 计算属性 vs watch

    Vue 中的 watch 用于监听数据变化并执行副作用（如异步操作），与计算属性的区别如下：

    场景推荐使用原因
    基于已有数据派生新数据计算属性代码更简洁，自带缓存
    数据变化时执行异步操作watch计算属性不支持异步（无法返回异步结果）
    复杂的副作用逻辑watch计算属性适合纯数据计算，副作用用 watch
    <template>
  <div>
    <input v-model="userId" placeholder="输入用户ID">
    <p>用户名：{{ userName }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      userId: '',
      userName: ''
    }
  },
  watch: {
    // 监听 userId 变化，执行异步请求
    userId(newVal) {
      if (newVal) {
        // 模拟接口请求
        setTimeout(() => {
          this.userName = `用户${newVal}`; // 异步更新数据
        }, 500);
      } else {
        this.userName = '';
      }
    }
  }
}
</script>

    说明：此处用 watch 更合适，因为需要在 userId 变化时执行异步操作（接口请求），而计算属性无法处理异步逻辑。

  • vue3中的计算属性(组合式API)

    在 Vue3 组合式 API 中，计算属性通过 computed 函数创建，返回一个响应式的 ref 对象，使用时需通过 .value 访问（模板中可省略）。

    <template>
  <p>原始值：{{ count }}</p>
  <p>计算值：{{ doubleCount }}</p>
</template>

<script setup>
import { ref, computed } from 'vue';

const count = ref(0);
// 定义计算属性（getter）
const doubleCount = computed(() => count.value * 2);

// 可写计算属性
const fullName = computed({
  get: () => firstName.value + lastName.value,
  set: (newVal) => { /* 更新逻辑 */ }
});
</script>
  • 特点与场景

    计算属性是 Vue 中处理数据派生的最佳实践，核心特点：

    1. 依赖追踪：自动追踪依赖的响应式数据；
    1. 缓存机制：依赖不变时直接返回缓存结果，提升性能；
    1. 简洁性：将复杂逻辑从模板中抽离，提高代码可读性。

    使用场景：

    • 数据格式化（如日期、价格）；
    • 列表过滤 / 排序；
    • 多数据组合计算（如全名 = 姓 + 名）。

    避免过度使用：计算属性适合同步逻辑，异步操作或复杂副作用优先用 watch。
