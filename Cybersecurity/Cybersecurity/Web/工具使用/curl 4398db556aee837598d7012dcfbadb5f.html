curl

curl 是一款功能强大的命令行工具，用于在网络上传输数据，支持 HTTP、HTTPS、FTP、SFTP 等多种协议，广泛用于接口测试、文件下载、数据提交等场景。以下是其核心用法详解，涵盖基础到高级功能：

  • 1、发送get请求(默认方式)
    # 基础用法：获取网页内容
curl https://www.example.com

# 带参数的 GET 请求（参数用 ?key=value&key2=value2 拼接）
curl "https://api.example.com/user?id=1&name=test"
  • 2、get常用参数

    -i：显示响应头 + 响应体（默认只显示响应体）。

    curl -i https://www.example.com  # 查看状态码、Content-Type 等头信息

    -I：只显示响应头（不显示响应体）。

    curl -I https://www.example.com  # 快速查看状态码（如 200、404）

    -L：跟随重定向（如果服务器返回 301/302 跳转，自动跟随）。

    curl -L http://example.com  # 自动跳转到 https 版本（若服务器配置了重定向）
  • 3、发送post请求（提交数据）

    1、提交表单application/x-www-form-urlencoded

    最常见的 POST 格式，数据用 key=value&key2=value2 表示：

    # 提交表单：id=123&name=test
curl -X POST -d "id=123&name=test" "https://api.example.com/submit"

    2、提交json数据application/json

    # Linux/macOS（用单引号包裹 JSON，避免双引号转义问题）
curl -X POST -H "Content-Type: application/json" -d '{"id":123, "name":"test"}' "https://api.example.com/submit"

# Windows cmd（双引号需用两个双引号转义："" 表示一个 "）
curl -X POST -H "Content-Type: application/json" -d "{""id"":123, ""name"":""test""}" "https://api.example.com/submit"
  • 4、post选项说明

    X POST：指定请求方法为 POST（必须）。

    d（或 -data）：携带 POST 数据（支持多组 d 拼接，如 d "a=1" -d "b=2"）。

    H（或 -header）：设置请求头（如 Content-Type 告诉服务器数据格式）。

  • 5、文件下载

    curl 可直接下载文件到本地，常用 -o 或 -O 选项：

    -o <文件名>：指定保存的文件名（自定义名称）。

    curl -o example.html https://www.example.com  # 下载网页保存为 example.html

    -O：按服务器原文件名保存（需 URL 包含文件名）。

    curl -O https://example.com/file.zip  # 保存为 file.zip

    高级下载选项：

    -C -：断点续传（下载中断后，继续从断点下载）。

    curl -C - -O https://example.com/large-file.iso

    -L：跟随重定向下载（适用于链接跳转后的文件）。

  • 6、文件上传 FTP/HTTP

    1、http上传文件multipart/form-data 格式

    用 -F 模拟表单上传文件（name 为表单字段名，@ 后接本地文件路径）：

    # 上传本地 image.png 文件，表单字段名为 "file"
curl -X POST -F "file=@./image.png" "https://api.example.com/upload"

    2、ftp上传文件

    需提供 FTP 服务器用户名和密码（式：ftp://user:pass@host/path）：

    # 上传 local.txt 到 FTP 服务器的 /remote/path 目录
curl -T ./local.txt "ftp://user:password@ftp.example.com/remote/path/"
  • 7、高级功能

    1、设置请求头（-H）

    自定义请求头信息（如认证令牌、User-Agent 等）：

    # 携带 Bearer 令牌（常见于 API 认证）
curl -H "Authorization: Bearer abc123" "https://api.example.com/data"

# 模拟浏览器 User-Agent（避免服务器识别为爬虫）
curl -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/100.0.0.0" "https://example.com"

    2、认证（登录验证）

    1、基本认证（Basic Auth）

    用 -u 传入用户名和密码（格式：user:pass）：

    # 访问需要 Basic Auth 的接口
curl -u "username:password" "https://api.example.com/secure"

    2、摘要认证（Digest Auth）

    用 --digest 选项（适用于更安全的摘要认证）：

    curl --digest -u "username:password" "https://api.example.com/secure"

    3、使用代理

    通过 -x（或 --proxy）指定代理服务器（支持 HTTP/HTTPS/SOCKS 代理）：

    # 使用 HTTP 代理（格式：代理IP:端口）
curl -x "http://127.0.0.1:8080" "https://example.com"

# 使用 SOCKS5 代理
curl -x "socks5://127.0.0.1:1080" "https://example.com"

    4、超时设置

    避免请求无限等待，用 -m 设置总超时时间（单位：秒）：

    curl -m 10 "https://example.com"  # 10秒内未完成则超时退出

    5、查看详细请求过程（调试用）

    用 -v（ verbose）显示完整请求过程（包括 DNS 解析、TCP 连接、请求 / 响应细节）：

    curl -v "https://www.example.com"  # 调试接口时常用，排查连接/参数问题
  • 编码问题

    url编码可以在Windows/Linux/macOS中都能稳定解析

    {"x":"wllm"}的 URL 编码为%7B%22x%22%3A%22wllm%22%7D，命令如下：

curl -X POST -d "id=wllmNB" "http://node7.anna.nssctf.cn:22571/index.php?json=%7B%22x%22%3A%22wllm%22%7D"
