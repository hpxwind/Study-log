Spring

Spring全家桶：Spring MVC, Spring Boot, Spring Cloud, Spring Data JPA, Spring Security, Spring Ai

项目开发基于Spring框架，由Spring框架搭建项目的基础环境，在此环境上再添加其他的业务架构

Spring不是业务层框架，它是构建业务层框架的框架

Spring提供的是底层的容器，该容器来构建业务层框架

业务框架本质上也是由各个对象组成的，Spring提供的容器就是管理这些对象的

Spring两大核心组件：Ioc + AOP

    package com.southwind.test.configuration;

import com.southwind.test.entity.Address;
import com.southwind.test.entity.User;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

/**
 * Spring 配置类
 * 作用 = 替代原来的 spring.xml 配置文件
 */
// 标记这是一个 Spring 配置类
@Configuration
public class BeanConfiguration {

    /**
     * 从 Spring 容器中自动找到 Address 对象
     * 并赋值给当前属性 → 依赖注入

    @Autowired
    private Address address;  // 注意：你之前拼写错误，这里已修正

    /**
     * 创建 User 对象，并交给 Spring 管理
     * 方法名 user() = bean 的 id
     * 相当于 XML 中的 <bean id="user" class="...User"/>
     */
    @Bean
    public User user() {
        // 1. 创建 User 对象
        User user = new User();

        user.setPwd("123");
        user.setName("zhangsan");
        
        // 3. 把 Spring 注入的 Address 对象 设置给 User
        user.setAddress(address);
        
        // 4. 将创建并赋值好的 User 对象交给 Spring 容器
        return user;
    }
}

Spring AOP

APO（Aspect Oriented Programming）: 面向切面编程
OOP（Object Oriented Programming）: 面向对象编程

OOP将程序中参与模块都抽象成对象，然后通过对象之间的相互调用完成特定功能，AOP是对OOP的一种补充，是在另一个维度上抽象出对象，具体是指程序运行时动态将非业务切入到业务代码中，实现了业务代码和非业务代码的解耦合。

对于如下代码：

@Override
public int add(int a, int b) {
    System.out.println("parmas:" + a + "," + b); // 业务代码
    int res = a + b; // 功能代码
    System.out.println("result:" + res); // 业务代码
    return res;
}

OOP可以将这个代码解耦合，将业务代码和功能代码分开

对于每个方法进行结构拆分

业务和切面最后要融合到一起输出，最终通过代理融合（通过IoC）
AOP基于IoC， 将业务对象和切面对象分别注入到IoC中，同时业务对象只写业务代码，切面对象只写非业务代码，从而在代码层面实现业务代码和非业务代码的解耦合，通过IoC再将业务代码和非业务代码整合到一起即可，开发时解耦，运行时合并。

如何整合？ 通过代理对象，将业务对象注入到IoC中，将切面对象也注入到IoC中，由IoC负责根据业务对象和切面对象创建出一个动态代理对象，分别取完成业务代码和非业务代码的执行。
1、业务对象和切面必须注入到IoC中
2、切面对象需要天机器切面注释以及动态代理注释

    package com.southwind.test.aop;

import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.AfterReturning;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.springframework.context.annotation.EnableAspectJAutoProxy;
import org.springframework.stereotype.Component;

import java.util.Arrays;

/**
 * 日志切面类
 * 作用：在不修改业务代码的前提下，给计算方法统一添加【日志功能】
 */
// 1. 把这个类交给 Spring 管理（必须加，否则 Spring 不识别）
@Component
// 2. 声明这是一个【切面类】（AOP核心注解）
@Aspect
// 3. 开启 AspectJ 自动代理，让 AOP 注解生效（Spring AOP 开关）
@EnableAspectJAutoProxy
public class LoggerAspect {

    /**
     * 前置通知：目标方法【执行之前】运行
     * 作用：打印方法名 + 传入的参数
     *
     * @Before 前置通知注解
     * execution(...) 切点表达式：表示要拦截哪些方法
     * 解释：拦截 com.southwind.test.aop.CalImpl 类下的 所有public int返回值的方法
     */
    @Before("execution(public int com.southwind.test.aop.CalImpl.*(..))")
    public void before(JoinPoint joinPoint) {
        // JoinPoint：连接点对象，封装了【目标方法】的所有信息

        // 1. 获取目标方法的名字（如 add、sub、mul、div）
        String name = joinPoint.getSignature().getName();

        // 2. 获取目标方法的参数，转成字符串
        String args = Arrays.toString(joinPoint.getArgs());

        // 3. 打印日志：方法名 + 参数
        System.out.println(name + "'s params: " + args);
    }

    /**
     * 返回后通知：目标方法【正常执行完毕、返回结果后】运行
     * 作用：打印方法名 + 返回结果
     *
     * @AfterReturning
     *  value：切点表达式（和上面一样，拦截同一批方法）
     *  returning = "result"：把方法返回值赋值给参数 result
     */
    @AfterReturning(
            value="execution(public int com.southwind.test.aop.CalImpl.*(..))",
            returning = "result")
    public void afterReturning(JoinPoint joinPoint,  Object result) {
        // 1. 获取目标方法名
        String name = joinPoint.getSignature().getName();

        // 2. 打印日志：方法名 + 返回结果
        System.out.println(name + " result:" + result);
    }
}

Spring MVC

Spring MVC（业务框架）对JavaWeb开发流程进行了封装，不需要再通过Servlet的方式来完成代码的开发，无论时代码量还是参数的接收，返回值等各个方面都做了优化。基于Spring 框架（基于IoC容器）

MVC 只使用一个Servle作为控制器（Controller）进行中间调度，把用户操作汇总，集中查询请求，再返回，例如：

请求→web.xml 分发→springmvc.xml 解析规则→Controller 执行业务

  1. web.xml:项目全局入口配置，Tomcat 优先加载
    • web.xml

      注册前端控制器，拦截所有请求，绑定 SpringMVC 配置文件

      浏览器发请求 → /拦截 → 交给 DispatcherServlet → 读取 springmvc.xml

      <!-- 注册SpringMVC核心调度器 -->
<servlet>
  <servlet-name>springmvc</servlet-name>
  <!-- 框架核心类，所有请求必经此类 -->
  <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
  <!-- 指定加载springmvc配置文件 -->
  <init-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:springmvc.xml</param-value>
  </init-param>
</servlet>

<!-- 映射：拦截浏览器所有访问请求 -->
<servlet-mapping>
  <servlet-name>springmvc</servlet-name>
  <url-pattern>/</url-pattern>
</servlet-mapping>

  1. 用户请求整个流程
    1. 客户端浏览器发起请求，访问地址形如http://localhost:8080/项目名/index。
    1. 服务器读取web.xml配置，匹配到拦截规则/，将所有请求交给 SpringMVC 核心调度器DispatcherServlet处理。
    1. 调度器根据配置参数，加载读取springmvc.xml框架配置文件。
    1. 配置文件执行包扫描，检索到com.southwind.test包下标注@Controller的控制器类，并将其交由 Spring 容器管理。
    1. 框架通过注解驱动功能，匹配请求路径与控制器中@RequestMapping("/index")映射地址，定位到对应处理方法。
    1. 执行 index 方法，控制台打印日志，方法执行完毕返回视图名称index。
    1. 视图解析器按照前缀、后缀配置，拼接得到真实页面路径/index.jsp。
    1. 服务器跳转至对应 jsp 页面，最终页面内容响应返回给浏览器展示。

Spring MVC会自动对request中的参数进行解析，类型转换，开发者只需要在方法定义声明需要的参数类型格式，Spring MVC会自动对参数进行解析和转换。

  • MyBites梳理流程

    1. 核心配置文件 config.xml（MyBatis 的 “大脑”）

    它包含数据库连接信息（url、driver、username、password）、事务配置、日志等全局设置。

    图里它有两个关键箭头：

    1. 箭头 1 → DB（数据库）：config.xml 里的 <environments> 配置，直接决定了 MyBatis 怎么连接到你的 MySQL。
    1. 箭头 2 → MyBatis 核心容器：它会把自己的配置加载到 MyBatis 里，告诉 MyBatis 去哪里找 Mapper.xml。

    2. Mapper.xml（SQL 语句的 “载体”）

    它里面写了具体的增删改查 SQL，比如你写的 insert into sys_news ...。

    图里它的箭头 → Mapper 代理：MyBatis 会把 XML 里的 SQL 和 Mapper 接口绑定起来，让代理对象知道 “调用这个方法时，要执行哪条 SQL”。

    3. Mapper 接口（NewsMapper.java）

    这是一个纯 Java 接口，只有方法声明，没有实现，比如 int add(News news);。

    图里它的箭头 → Mapper 代理：你调用 session.getMapper(NewsMapper.class) 时，MyBatis 会动态生成一个代理对象，帮你实现这个接口。

    4. Mapper 代理（MyBatis 给你生成的 “干活的人”）

    这是 MyBatis 动态生成的实现类，你看不见它的源码，但它是真正干活的：

    • 接收你调用接口方法时传的参数（比如 News 对象）；
    • 找到 Mapper.xml 里对应的 SQL；
    • 把参数塞进 SQL 里；
    • 用 config.xml 里的数据库连接，把 SQL 发给数据库执行；
    • 把数据库返回的结果（比如受影响行数）再返回给你。

  • 用户触发流程
    1. 加载 config.xml：你的 Test 类先读取 config.xml，MyBatis 拿到数据库配置和 Mapper.xml 路径。
    1. 构建 SqlSessionFactory：工厂对象创建好，里面包含了所有配置信息。
    1. 打开 SqlSession：拿到数据库会话，相当于 JDBC 的 Connection。
    1. 获取 Mapper 代理对象：session.getMapper(NewsMapper.class) 让 MyBatis 生成代理，把 NewsMapper.java 和 NewsMapper.xml 绑定。
    1. 调用接口方法 mapper.add(news)：代理对象拿到你的 News 对象，找到 XML 里的 insert SQL，把参数拼进去，发给数据库执行。
    1. 提交事务 session.commit()：事务提交，数据真正写入数据库。

import com.southwind.test.Department;
import com.southwind.test.Employee;
import org.apache.ibatis.annotations.Many;
import org.apache.ibatis.annotations.Result;
import org.apache.ibatis.annotations.Results;
import org.apache.ibatis.annotations.Select;

import java.util.List;

/**
 * 部门 Mapper 接口
 * 负责：单表查询 + 一对多关联查询
 */
public interface DepartmentMapper {

    // ======================== 方法 1：根据ID查部门（给【一对一】调用） ========================
    /**
     * 作用：根据部门ID 查询 单个部门信息
     * 场景：EmployeeMapper 中【一对一查询员工+所属部门】时，会调用这个方法
     *
     * 为什么必须写 @Results？
     * 因为：数据库字段名 和 Java实体属性名 不一致！
     * 数据库：dept_name    实体：deptName
     * 数据库：create_time  实体：createTime
     * MyBatis 默认不会自动驼峰转换 → 不写映射就会是 null！
     */
    @Select("select * from department where id = #{deptId}")
    @Results({
            // 数据库列名 dept_name  → 映射到 → 实体属性 deptName
            @Result(column = "dept_name", property="deptName"),
            // 数据库列名 create_time → 映射到 → 实体属性 createTime
            @Result(column = "create_time", property = "createTime")
    })
    Department getDeptById(Integer deptId);

    // ======================== 方法 2：根据ID查部门 + 旗下所有员工（一对多） ========================
    /**
     * 作用：查询 1个部门 + 该部门下所有员工（一对多查询）
     * 一个部门 → 多个员工 → 所以用 @Many
     */
    @Select("select * from department where id = #{deptId}")
    @Results({
            // 主键映射（必须加 id=true，提高效率）
            @Result(id = true, column = "id", property = "id"),

            // 普通字段映射：数据库列 → 实体属性
            @Result(column = "dept_name", property = "deptName"),
            @Result(column = "location", property = "location"),
            @Result(column = "manager", property = "manager"),
            @Result(column = "remark", property = "remark"),
            @Result(column = "create_time", property = "createTime"),

            /**
             * 一对多 核心映射！
             * property = "employeeList"：Department实体中的集合属性名
             * column = "id"：把当前部门表的 id 列的值，当作参数传给子查询
             * many = @Many：表示一对多（返回多个结果 → 集合）
             * select：指定调用哪个 Mapper 的哪个方法
             *
             * 执行流程：
             * 1. 先查 department 得到部门 id=1
             * 2. 自动调用 EmployeeMapper.getEmpByDeptId(1)
             * 3. 把查到的员工列表 → 塞进 Department.employeeList
             */
            @Result(
                    property = "employeeList",
                    column = "id",
                    many = @Many(select = "com.southwind.test.mapper.EmployeeMapper.getEmpByDeptId")
            )
    })
    Department getDeptAndEmp(Integer deptId);

    // ======================== 方法 3：根据部门ID查所有员工（给【一对多】调用） ========================
    /**
     * 作用：根据部门ID，查询该部门下所有员工
     * 场景：被上面的【一对多查询】调用
     * 这里没有 @Results 为什么？
     * 因为 employee 表的字段和实体属性 已经在 EmployeeMapper 里映射好了！
     */
    @Select("select * from employee where dept_id = #{deptId}")
    List<Employee> getEmpByDeptId(Integer deptId);
}

 * 员工表 Mapper
 * 功能：
 * 1. 根据员工ID查询员工 + 所属部门（一对一关联查询）

     * 多个员工 ← 对应 → 一个部门
     * 员工 → 部门 是【一对一】关系
     */

    @Results({
        // ======================================
        // 主键映射：id=true 表示这是主键字段
        // column = 数据库列名
        // property = 实体类属性名
        // ======================================

        // 数据库列 emp_name → 映射到实体属性 empName
        @Result(column = "emp_name", property = "empName"),

        // 数据库列 age → 映射到实体属性 age
        @Result(column = "age", property = "age"),

        // 数据库列 gender → 映射到实体属性 gender
        @Result(column = "gender", property = "gender"),

        // 数据库列 phone → 映射到实体属性 phone
        @Result(column = "phone", property = "phone"),

        // 数据库列 dept_id → 映射到实体属性 deptId
        @Result(column = "dept_id", property = "deptId"),

        // 数据库列 address → 映射到实体属性 address
        @Result(column = "address", property = "address"),

        // 数据库列 create_time → 映射到实体属性 createTime
        @Result(column = "create_time", property = "createTime"),

        // ======================================
        // 【一对一 关 联 查 询 核 心】
        // 作用：查询员工的同时，自动查询所属部门
        // ======================================
        @Result(
            // property = "department"
            // 表示：把查询到的部门对象，赋值给 Employee 类中的 department 属性
            property = "department",

            // column = "dept_id"

                select = "com.southwind.test.mapper.DepartmentMapper.getDeptById"

MyBatis延迟加载

(Lazy Loading 懒加载)是针对多表关联查询的核心优化机制：查询主表数据时，不会立刻加载关联表数据；只有当代码主动访问关联对象 / 集合的属性时，才会额外执行 SQL 加载关联数据，可以降低内存与IO开销。

  • 立即加载与懒加载区别

    立即加载：查询主对象时，一次性通过联表查询 / 多条 SQL 把主表 + 所有关联表数据全部加载。

    • 特点：一次查询拿到所有数据，后续访问关联属性无需再查库
    • 适用：每次查询主数据必然要使用关联数据

    懒加载：查询主对象时只查主表，关联数据先不加载；访问关联属性时才触发二次查询

    • 特点：按需加载，减少无效查询
    • 适用：大部分场景不需要使用关联数据，仅少数场景需要

  • MyBatis 延迟加载基于 CGLIB 动态代理 实现：
    • 开启懒加载后，MyBatis 不会直接返回原生实体对象，而是返回代理对象；
    • 当代码调用关联属性的 getter 方法时，代理对象拦截请求；
    • 拦截后自动执行预定义的关联查询 SQL，加载数据并注入到关联属性中；
    • 后续再次访问该关联属性，不会重复查库（数据已缓存）。
强制要求：实体类必须提供getter/setter方法(@Data)，否则代理无法拦截，懒加载直接失效。

全局核心配置以及解释：

  • mybatis-config.xml

    延迟加载的总开关与行为控制，全部配置在<settings>标签内，共 3 个核心参数。

    <configuration>
    <settings>
        <!-- 1. 全局延迟加载总开关：默认 false（关闭），true 开启全局懒加载 -->
        <setting name="lazyLoadingEnabled" value="true"/>

        <!-- 2. 积极懒加载：默认 true（老版本默认行为） -->
        <setting name="aggressiveLazyLoading" value="false"/>

        <!-- 3. 触发懒加载的方法列表：默认 equals,clone,hashCode,toString -->
        <setting name="lazyLoadTriggerMethods" value="equals,clone,hashCode"/>
    </settings>
</configuration>

    lazyLoadingEnabled

    • true：全局所有 association/collection 关联统一开启延迟加载；
    • false（默认）：全局关闭，所有关联立即加载。

    aggressiveLazyLoading（重点）

    • true（默认）：积极懒加载。只要调用主对象的任意方法，就会一次性加载所有关联数据，失去 “按需加载” 意义；
    • false（推荐配置）：按需懒加载。只有真正访问关联对象的属性 /getter，才会触发加载。

    lazyLoadTriggerMethods指定哪些方法会强制触发延迟加载（不受 aggressiveLazyLoading=false 限制）。

    • 默认值：equals,clone,hashCode,toString
    • 场景：即使只打印主对象 System.out.println(user)（调用 toString()），也会加载所有关联数据；
    • 优化：可移除 toString，避免打印对象时误触发懒加载。

    association（一对一）、collection（一对多）标签支持 fetchType 属性，局部单独控制当前关联的加载方式，优先级高于全局配置：

    • fetchType="lazy"：当前关联延迟加载；
    • fetchType="eager"（默认）：当前关联立即加载。

MyBatis缓存

如何提升运行效率？实用缓存，减少Java程序和数据库的交互次数，从而提升程序的运行效率。

当查询出数据后，将数据保存到缓存中，当下一次需要实用同样数据的时候，直接从缓存中取出数据即可。

一级缓存

默认开启，无需手动配置，在同一个sqlsession中缓存

  • 详细
    • 所属范围：SqlSession 级别（一次数据库会话）
    • 默认状态：默认开启，全局开关无法关闭，只能清空 / 失效
    • 存储位置：内存，和当前 SqlSession 绑定
    • 生命周期：
      1. 打开 SqlSession → 执行查询，结果存入一级缓存
      1. 同会话内，相同 SQL + 相同参数 再次查询，走缓存
      1. SqlSession 关闭 / 提交 / 回滚 → 一级缓存自动清空

    执行流程：

    • 第一次查询：查数据库 → 结果放入一级缓存
    • 同 SqlSession、同 SQL、同参数：直接读缓存，不访问数据库

    失效场景：满足任意一条，缓存清空，下次重新查库：

    1. 执行 增 / 删 / 改（INSERT/UPDATE/DELETE） 操作（不管操作哪张表）
    1. 手动调用 sqlSession.clearCache() 清空缓存
    1. 会话关闭：sqlSession.close()
    1. 不同 SqlSession 之间，一级缓存互不共享
    public static void main(String[] args) throws IOException {
    InputStream is = Resources.getResourceAsStream("mybatis-config.xml");
    SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(is);
    // 同一个 SqlSession
    SqlSession session = factory.openSession();
    UserMapper mapper = session.getMapper(UserMapper.class);

    // 第1次查询：访问数据库，存入一级缓存
    User user1 = mapper.getUserById(1);
    System.out.println(user1);

    // 第2次查询：同SqlSession、同SQL参数 → 走一级缓存，不查库
    User user2 = mapper.getUserById(1);
    System.out.println(user2);

    // 手动清空缓存
    session.clearCache();

    // 第3次查询：缓存已清空 → 重新访问数据库
    User user3 = mapper.getUserById(1);
    System.out.println(user3);

    session.close();
}

二级缓存

Mapper 级别，手动开启，在一级缓存失效后会话关闭，将数据写入二级缓存

  • 详细
    • 所属范围：Mapper 命名空间（namespace） 级别
    • 默认状态：默认关闭，需要手动开启
    • 生命周期：跨 SqlSession，多个会话共享缓存
    • 存储位置：内存，作用域是整个 Mapper XML 对应的命名空间
    • 数据流转：数据库 → 一级缓存 → 会话关闭后数据写入二级缓存

    失效场景:

    • 当前 namespace 下执行 增 / 删 / 改 操作 → 清空该命名空间下所有二级缓存
    • 手动 sqlSession.clearCache()：只清空一级缓存，不影响二级缓存
    • 不同 namespace（不同 Mapper）缓存相互隔离，互不干扰
  • 开启流程
    1. 全局配置开启二级缓存总开关

      在 mybatis-config.xml 的 <settings> 中添加：

      <settings>
    <!-- 开启二级缓存总开关，默认 true（高版本MyBatis默认开启，建议显式写上） -->
    <setting name="cacheEnabled" value="true"/>
</settings>
    1. 在Mapper XML中开启当前命名空间缓存

      在你的 UserMapper.xml 根标签下添加 <cache/> 标签：

      <mapper namespace="com.southwind.test.mapper.UserMapper">
    <!-- 开启当前 namespace 的二级缓存 -->
    <cache/>

    <!-- 下方原有 resultMap、SQL 不变 -->
</mapper>

      <cache/> 默认配置说明：

      • 使用 LRU 淘汰策略
      • 缓存可读写
      • 刷新间隔、缓存大小使用默认值
    1. 实体类实现序列化接口

      二级缓存会将对象序列化存储，实体必须实现 java.io.Serializable

      // User.java

    private String username;

    1. 自定义规则（自定义）

eviction：缓存淘汰策略
  LRU：最近最少使用（默认）
  FIFO：先进先出
flushInterval：缓存刷新间隔(毫秒)，为空则不自动刷新

    flushInterval="60000"
    size="512"
    readOnly="true"/>

    SELECT id, username FROM user WHERE id = #{id}
</select>
  • 执行流程
    • 第一个 SqlSession 查询 → 查库 → 数据存入一级缓存
    • SqlSession.close() 关闭会话 → 一级缓存数据刷入二级缓存
    • 新的 SqlSession 执行相同查询 → 优先从二级缓存取数据，不查库

    演示：

    public static void main(String[] args) throws IOException {
    InputStream is = Resources.getResourceAsStream("mybatis-config.xml");
    SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(is);

    // 会话1
    SqlSession session1 = factory.openSession();
    UserMapper mapper1 = session1.getMapper(UserMapper.class);
    User user1 = mapper1.getUserById(1);
    System.out.println(user1);
    session1.close(); // 关闭会话，数据写入二级缓存

    // 会话2（全新 SqlSession）
    SqlSession session2 = factory.openSession();
    UserMapper mapper2 = session2.getMapper(UserMapper.class);
    // 跨会话，直接读取二级缓存，不执行SQL
    User user2 = mapper2.getUserById(1);
    System.out.println(user2);
    session2.close();
}

    test判断表达式，支持OGNL表达式

    • 判空：username ≠ null and username ≠ ‘’
    • 数值判断：id ≠ null

    多条件查询id

    <select id="findUser" resultType="com.southwind.test.User">
    SELECT id, username FROM user
    WHERE 1=1
    <!-- 如果 id 有值，拼接 id 查询条件 -->
    <if test="id != null">
        AND id = #{id}
    </if>
    <!-- 如果 username 非空，拼接模糊查询 -->
    <if test="username != null and username != ''">
        AND username LIKE CONCAT('%',#{username},'%')
    </if>
</select>

    测试场景

    • 不传参数：执行 SELECT id,username FROM user WHERE 1=1，查询全表
    • 只传 id=1：拼接 AND id = 1
    • 只传 username='hpc'：拼接模糊查询
    问题：手动写 WHERE 1=1 是为了避免多个 if 同时不生效，出现 WHERE 关键字悬空语法错误。
  • <where>只能处理where关键字

    替代手动 WHERE 1=1，自动识别条件、自动剔除多余的 AND / OR，语法更优雅。

    规则：

    1. 只有内部 <if> 条件成立，才会生成 WHERE 关键字
    1. 自动移除条件开头多余的 AND / OR

    对if中的进行改写

    <select id="findUser" resultType="com.southwind.test.User">
    SELECT id, username FROM user
    <where>
        <if test="id != null">
            AND id = #{id}
        </if>
        <if test="username != null and username != ''">
            AND username LIKE CONCAT('%',#{username},'%')
        </if>
    </where>
</select>
    • 无任何条件：不生成 WHERE，SQL 正常
    • 有条件：自动拼接 WHERE + 条件，去掉开头多余 AND
  • <trim>自定义截取/拼接

    通用格式化标签，可替代 where、set，自定义前缀、后缀、移除指定前缀 / 后缀字符

    属性说明
    prefix整体添加前缀（如 WHERE、SET）
    prefixOverrides移除开头指定字符（如 AND、OR）
    suffix整体添加后缀
    suffixOverrides移除结尾指定字符（如 , 逗号）

    用trim实现where效果

    <select id="findUser" resultType="com.southwind.test.User">
    SELECT id, username FROM user
    <trim prefix="WHERE" prefixOverrides="AND|OR">
        <if test="id != null">
            AND id = #{id}
        </if>
        <if test="username != null and username != ''">
            AND username LIKE CONCAT('%',#{username},'%')
        </if>
    </trim>
</select>
  • <set>动态更新字段

    用于update语句，动态拼接更新字段，自动移除末尾多余逗号,

    场景：只更新传入的非空字段，其余字段保持不变

    案例，动态更新用户：

    <update id="updateUser">
    UPDATE user
    <set>
        <if test="username != null and username != ''">
            username = #{username},
        </if>
    </set>
    WHERE id = #{id}
</update>

    逻辑：

    1. username 非空 → 拼接 username = ?,
    1. <set> 自动删除最后面多余的逗号，避免 SQL 语法报错

    等价trim写法:

    <update id="updateUser">
    UPDATE user
    <trim prefix="SET" suffixOverrides=",">
        <if test="username != null and username != ''">
            username = #{username},
        </if>
    </trim>
    WHERE id = #{id}
</update>
  • <choose> <when> <otherwise>多分支判断

    等价 Java 中switch-case-default，多选一，只会执行一个分支.

    场景：优先级查询（有 id 查 id，有用户名查用户名，都没有查全部）

    <choose>
    <when test="条件1">SQL片段</when>
    <when test="条件2">SQL片段</when>
    <otherwise>都不满足时执行</otherwise>
</choose>

    案例：

    <select id="findUser" resultType="com.southwind.test.User">
    SELECT id, username FROM user
    <where>
        <choose>
            <when test="id != null">
                id = #{id}
            </when>
            <when test="username != null and username != ''">
                username LIKE CONCAT('%',#{username},'%')
            </when>
            <!-- 都不传参数，查询所有 -->
            <otherwise>
                1=1
            </otherwise>
        </choose>
    </where>
</select>

    执行规则：从上到下匹配，匹配到第一个满足条件的分支就结束.

  • <foreach>遍历集合（批量操作核心）

    遍历数组、List 集合，常用于 IN 查询、批量新增、批量删除

    属性说明
    collection集合类型：List 写 list / 加了 @Param 写参数名
    item遍历后单个元素别名
    open遍历整体开头拼接字符
    close遍历整体结尾拼接字符
    separator元素之间的分隔符
    1. 场景1：IN 查询（根据多个 id 查询）
      <select id="findUserByIds" resultType="com.southwind.test.User">
    SELECT id, username FROM user
    WHERE id IN
    <foreach collection="ids" item="uid" open="(" close=")" separator=",">
        #{uid}
    </foreach>
</select>

      最终拼接 SQL：WHERE id IN (1,2,3)

    1. 场景2：批量删除（最常用）

      接口方法：

      int batchDelete(@Param("ids") List<Integer> ids);

      XML：

      <delete id="batchDelete">
    DELETE FROM user WHERE id IN
    <foreach collection="ids" item="uid" open="(" close=")" separator=",">
        #{uid}
    </foreach>
</delete>
  • <bind>绑定变量

    自定义变量，解决数据库模糊查询兼容问题（不同数据库拼接 % 语法不同），统一写法

    示例：模糊查询通用写法

    <select id="findUser" resultType="com.southwind.test.User">
    <!-- 绑定变量 pattern，拼接前后 % -->
    <bind name="pattern" value="'%' + username + '%'"/>
    SELECT id, username FROM user
    <where>
        <if test="username != null and username != ''">
            username LIKE #{pattern}
        </if>
    </where>
</select>

    优势：不再使用 CONCAT，适配 MySQL、Oracle 等多种数据库
