面向对象

类和对象

面向对象最基础的元素是对象，对象从何而来？通过类来创建，即：类→对象

类是所有对象的抽象模板，对象是类的一个具体实例

我们编写的代码都是类，静态资源，程序运行时候，JVM会加载类，运行类中代码，从而产生对象

对象的存在是一种瞬间状态，程序执行期间对象是存在的，程序一旦结束对象就不存在了，而类是一直存在的，

类中包含两部分内容，用来描述对象的特征

  • 静态特征 颜色，重量，编号….
  • 动态特征 带有动作执行的操作 走路，跑….

静态特征在类中定义为属性（成员变量）

动态特征在类中定义为方法

构造函数

类是创建对象的，对象是通过构造函数来创建的，类中必须有至少一个构造函数 ，每个类中都有一个默认的无参数构造函数

构造函数不需要返回值，方法名必须与类名相同，可以有参数列表，访问权限修饰符不能用private ，因为private 只等通过内部创建对象，如果为private 外部无法创建对象（也有private 的情况）

如果声明了一个有参数构造，默认的无参数构造会被覆盖，无参数构造需要手动再创建

无参数构造与有参数构造的区别？

无参数构造只能创建对象，无法给对象的属性赋值

有参数构造可以连同创建 + 属性赋值一起完成

public class Student {
    public int id;
    public String name;
    // 无参构造
    public Student() {}
    // 有参数构造
    public Student(int id, String name) {
        this.id = id;
        this.name = name;
    }

    // 方法重写
    @Override
    public  String toString() {
        return "Student{" + "id=" + id + ", name=" + name + "}";
    }
}


public class Main {
  public static void main(String[] args) {
      // 无参数构造
      Student s = new Student();
      s.id = 1;
      s.name = "zs";
      System.out.println(s);
      // 有参数构造
      Student s1 = new Student(2, "ww");
      System.out.println(s1);
  }
}
结果：
Student{id=1, name=zs}
Student{id=2, name=ww}

this 关键子

this用来指代当前类的实例化对象，通过 this 可以调用当前类的属性和方法 ，比如有参数构造中，通过 this 将外部传递来的值赋给当前实例化对象、

this 除了可以在类中访问属性也可以在类中调用方法，类中的方法分两种：构造方法和普通方法。用this调用这两种方法的语法也不同

  • 调用构造方法 this(参数列表)
  • 调用普通函数 this.方法名(参数列表)

不能再普通方法中使用this来调用构造函数

不能再构造函数中用this调用构造函数，因为会产生递归，构造函数调用会创建对象，会占用内存，导致内存溢出。

而普通方法可以递归，因为普通方法递归不会创建对象，因此不会占用内存。

static 关键字

static表示静态或者全局，可以用来修饰成员变量和成员方法以及代码块，

成员变量和成员方法，必须通过对象来访问吗，必须依赖于对象

而使用static 修饰的成员变量和成员方法独立于任何一个实例对象，访问时候不依赖任何一个对象，可以直接通过类来访问

类 → 对象 → 方法

类 → 方法

被static修饰的变量叫做静态变量，被static修饰的方法叫静态方法

多个对象共享，内存中只有一份

public class User {
    public static String name;
    public static void show() {
        System.out.println("这是一个输出类");
    }
}
public class Main {
    public static void main(String[] args) {
        User.name = "张三"; // 可以直接调用， 而不用创建对象
        User.show();
    }
}

因为static下的属性和方法内存为同一块，为共享的，一旦更改，所有的都会改变

public class Main {
    public static void main(String[] args) {
        User user1 = new User();
        user1.name = "张三";
        user1.show(); // 张三

        User user2 = new User();
        user2.name = "李四";
        user2.show(); // 李四，
        user1.show(); //  李四，因为同一块地址被user2的李四覆盖，user1的也变为李四
    }
}

而实例中每开辟一个对象就实例化一个属性，所以下面的代码最终输出为1，总共创建了实例化了十个m，而每个m都有自己的id

而如果private int id;变为private static int id; 结果变为10，因为id是公共的

public class Main {
    private int id;
    public static void main(String[] args) {
        Main m = null;
        for (int i = 0; i < 10; i++) {
            m = new Main();
            m.id++;
        }
        System.out.println(m.id); // 1
    }
}

使用static修饰的静态方法中不能使用this关键字，不能访问类的实例变量和实例方法，可以直接访问类的静态变量和静态方法，如果要访问实例变量和实例方法，必须先实例化该类的对象，通过对象去访问。

因为静态方法已经属于类了，不能再方法中调用类，因此无法使用this关键字（静态不能访问非静态的，静态指的是类，this指的是对象）

方法重载（@Override）

java重载是java代码复用的一种重要方式，指的是两个方法之间的一种关系，要求：

  • 在同一个类中
  • 方法名相同
  • 参数列表不同（个数或数据类型不同）
  • 与返回值和访问权限修饰符无关
@Override

成员变量和局部变量

两个变量之间除了数据类型，变量名，变量值不同之外，变量的作用域范围也是不同的。

变量的作用域是指在程序中可以通过变量名来访问该变量的范围

变量的作用域由变量被声明时候所在位置决定

根据不同的作用域可以将变量分为成员变量和局部变量

如果一个变量在方法中声明，则该变量为局部变量

如果一个变量在方法外，类中声明，则该变量为成员变量

public class Student {
    // 成员变量
    public int id;

    public void p1() {
        // 局部变量
        int num  = 1;
    }
}

成员变量的作用域是整个类，类中的任意一个方法都可以访问该变量，而局部变量的作用域仅在该方法中，其他局部变量无法调用

局部变量没有默认值，成员变量有默认值（在没有人为赋值的时候有默认值）

封装

为什么要有封装？

为了保证数据的安全性

将类的信息全部封装到内部，让外部无法直接访问，确保数据的安全

  1. 将属性私有化（private）
  1. 通过间接的方法让外部访问内部的私有属性
public class Student {
    // 成员变量私有化
    private String name;
    private int age;

    // 间接方法访问私有属性
    public void setAge(int age) {
        if (age < 0) { // 封装中可以判断违规
            System.out.println("年龄不合规");
            this.age = 18;
        }
        this.age = age;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "student" + "{name=" + name +", age=" + age +"}";
    }
}

Object类

Object是java中全部类的共同父类，实现代码复用

Object中定义了一些所有类都需要使用到的方法，所有类直接继承使用

  • 列举
    1. protected Object clone()
创建并返回此对象的一个副本。类必须实现 Cloneable接口。

2. boolean equals(Object obj)
指示其他某个对象是否与此对象“相等”。用于逻辑比较，默认比较引用地址。

3. protected void finalize()
当垃圾回收器确定不存在对该对象的更多引用时，由对象的垃圾回收器调用此方法（已不推荐使用）。

4. Class<?> getClass()
返回此对象的运行时类（Class对象）。

5. int hashCode()
返回该对象的哈希码值。通常与 equals方法一同重写。

6. String toString()
返回该对象的字符串表示。默认格式为：类名@十六进制哈希码。

7. void notify()
唤醒在此对象监视器上等待的单个线程。

8. void notifyAll()
唤醒在此对象监视器上等待的所有线程。

9. void wait()
导致当前线程等待，直到另一个线程调用此对象的 notify()或 notifyAll()方法。

10. void wait(long timeout)
导致当前线程等待，直到另一个线程调用此对象的 notify()或 notifyAll()方法，或者超过指定的时间量（毫秒）。

11. void wait(long timeout, int nanos)
导致当前线程等待，直到另一个线程调用此对象的 notify()或 notifyAll()方法，或者其他某个线程中断当前线程，或者已超过某个实际时间量（毫秒+纳秒）。
此外，还有一组与内存排序相关的方法（自Java 9+引入，由java.lang.invoke.VarHandle提供类似功能）：

12. public final native int getAddress() (已废弃/内部使用)
（注：严格来说，现代JDK中一些底层或内部方法未被列为标准API。核心公共API方法即为上述11个。）

二级父类：Character，Number，Boolean

Number子类：Byte，Short，Integer，Long，Float，Double

实际开发中对于包装类的使用就是将基本数据类型转换为包装类，或者将包装类转换为基本数据类型

装箱和拆箱

装箱：将基本类型转换为包装类

public class Main {
    public static void main(String[] args) {
        byte b = 1;
        Byte byt = new Byte(b);
        short s = 2;
        Short sho = new Short(s);
        int i=3;
        Integer integer = new Integer(i);
        long l = 4;
        Long lon = new Long(l);
        float f = 4.4f;
        Float flo = new Float(f);
        double d = 22.2f;
        Double dou = new Double(d);
        char c = 'a';
        Character cha = new Character(c);
        boolean o = true;
        Boolean boo = new Boolean(o);
    }
}

拆箱：将包装类转换为基本类型

public class Main {
    public static void main(String[] args) {
        byte b = 1;
        Byte byt = new Byte(b);
        byte b1 = byt.byteValue();

        short s = 2;
        Short sho = new Short(s);
        short s1 = sho.shortValue();

        int i=3;
        Integer integer = new Integer(i);
        int i1 = integer.intValue();

        long l = 4;
        Long lon = new Long(l);
        long l1 = lon.longValue();

        float f = 4.4f;
        Float flo = new Float(f);
        double d = 22.2f;
        Double dou = new Double(d);
        char c = 'a';
        Character cha = new Character(c);
        boolean o = true;
        Boolean boo = new Boolean(o);
    }
}

有些时候无法直接使用int，等基本类型，所以需要了解包装类

异常

java中错误可以分为两类：

  • 编译时报错，一般指语法错误
  • 运行时报错，语法没问题，可以正常通过编译，但是运行时报错
java中一切皆是对象，错误也是对象

java提供了一组类，来表示各种各样的运行时错误

  • ArithmeticException: 数学异常
  • ClassNotFoundException: 类未定义异常
  • IllegalArgumentException: 参数格式异常
  • ArrayIndexOutOfBoundsException: 数组下标越界异常
  • NullPointerException: 空指针异常
  • NoSuchMethodException: 方法未定义异常
  • NumberFormatException: 将其他数据类型转为数值类型的不匹配异常
public class Main {
    public static void main(String[] args) throws Exception{
        System.out.println(Class.forName("23132"));

    public static void test(Object i) {
        Integer i1 = null;

try-catch-finally

try: 监听可能会抛出异常的代码，一旦出现错误，JDK会自动创建一个错误对应的异常对象，抛出该异常对象
catch: 用来捕获JDK创建的异常对象，进行后续的处理
finally: 关键字，无论是否抛出异常，finally代码块中的程序一定会执行
try {
} catch (Exception e) {
} finally {
}
public class Main {
    public static void main(String[] args) throws Exception{
        try {
            System.out.println(Class.forName("23132"));
        } catch (Exception e) {
            System.out.println(e);
        }
    }
    public static void test(Object i) {
        Integer i1 = null;
        System.out.println(i1);
    }
}
结果：
java.lang.ClassNotFoundException: 23132

业务代码出错的概率比较大时，一般需要加上try-catch来进行异常处理，如果代码出错概率较小，不用加

public class Main {
    public static void main(String[] args) throws Exception{
        try {
            int num = 10/0;
            System.out.println(123);
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("finally...");
        }
        System.out.println("out...");
    }
}

结果：
java.lang.ArithmeticException: / by zero
	at Main.main(Main.java:6)
finally...
out...

没有输出123，报错后，try中出错位置之后的代码不再执行，try-catch后和finally中的代码会正常执行，

return 可以直接中断，但再finally中的代码会在终端前执行

public class Main {
    public static void main(String[] args) throws Exception{
        System.out.println(test());
    }
    public static int test() {
        try {
            System.out.println("try..");
            return 10;
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("finally...");
            return 20;
        }
    }
}

结果：
try..
finally...
20

即使try中有return， 但是再执行 try中return之前，会优先执行finally中的代码

throw和throws

throw和throws时java在处理异常时使用的两个关键字，都是用来抛出异常的，但是使用方式和表示的含义完全不同。

java中抛出异常的方式有三种：

  • try-catch: 是一种防范机制，代码可能会出现异常，如果抛出则捕获即可，如果不抛出则继续执行
  • throw: 开发者主动（明确异常类型）创建一个错误并抛出throw new Exceprion()
  • throws: 标注方法的，用来描述该方法可能会抛出的异常
throw主动抛出对象，throws标注一个方法可能会抛出某些异常
public class Main {
    public static void main(String[] args) throws Exception {
        try {
            test();
        } catch (NumberFormatException e) {
            throw new RuntimeException(e);
        }
    }
    public static void test() throws  NumberFormatException {
        String str = "Java";
        int num = Integer.parseInt(str);
    }
}

自定义异常

定义一个方法，对传入的参数进行++操作，并返回结果，同时要求参数必须是整数类型，如果传入的参数不是整数类型则抛出异常

NumberException.java : 一个自定义异常类

public class NumberException extends Exception{
    public NumberException(String message) {
        super(message);
    }
}
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        try {
            int a = add(1);
            System.out.println(a);
        } catch (NumberException e) {
            throw new RuntimeException(e);
        }
    }

    public int add(Object o) throws NumberException {
        // 判断类型
        if (!(o instanceof Integer)) {
            // 抛出异常
            // 异常信息是传入的参数不是Integer
            throw new NumberException("传入的参数不是Integer");
        } else  {
            int num = (int) o;
            return ++num;
        }
    }
}
