Application.properties中MyBatis和MySql配置

  • 完整配置
    # ===================== 1. MySQL 数据库连接配置 =====================
# 数据库驱动（MySQL8.0+必须用 com.mysql.cj.jdbc.Driver）
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# 连接地址说明：
# serverTimezone=Asia/Shanghai 时区，解决时差、插入时间偏差
# useUnicode=true&characterEncoding=utf-8 编码
# useSSL=false 开发关闭SSL，生产按需开启
# allowPublicKeyRetrieval=true 解决公钥检索报错
spring.datasource.url=jdbc:mysql://localhost:3306/test_db?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf-8&useSSL=false&allowPublicKeyRetrieval=true

# 数据库账号密码
spring.datasource.username=root
spring.datasource.password=123456

# 数据源类型（SpringBoot默认Hikari，性能最优）
spring.datasource.type=com.zaxxer.hikari.HikariDataSource

# ===================== 2. Hikari 连接池配置（默认连接池） =====================
# 最小空闲连接
spring.datasource.hikari.minimum-idle=5
# 最大连接数
spring.datasource.hikari.maximum-pool-size=15
# 空闲连接超时时间（毫秒）
spring.datasource.hikari.idle-timeout=300000
# 连接最大生命周期
spring.datasource.hikari.max-lifetime=1800000
# 获取连接超时
spring.datasource.hikari.connection-timeout=20000

# ===================== 3. MyBatis 核心配置 =====================
# mapper.xml 文件存放路径，多个用逗号分隔
mybatis.mapper-locations=classpath:mapper/**/*.xml

# 实体类别名包，不用写全类名，直接写类名
mybatis.type-aliases-package=com.example.demo.entity

# 开启MyBatis日志，打印执行SQL（开发必备）
mybatis.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl

# 开启驼峰命名自动转换：数据库 user_name → 实体 userName
mybatis.configuration.map-underscore-to-camel-case=true

# 关闭自动驼峰默认值（可选）
# mybatis.configuration.map-underscore-to-camel-case=false

# 全局开启延迟加载
mybatis.configuration.lazy-loading-enabled=true
# 按需延迟加载
mybatis.configuration.aggressive-lazy-loading=false

# ===================== 4. MyBatis 全局配置文件（可选） =====================
# 如果有 mybatis-config.xml 全局配置文件，指定路径
# mybatis.config-location=classpath:mybatis-config.xml

# ===================== 5. 分页插件 PageHelper 配置（常用） =====================
# pagehelper分页方言适配mysql
pagehelper.helper-dialect=mysql
# 合理化分页，页码<=1查第一页，超过总页数查最后一页
pagehelper.reasonable=true
# 支持参数分页
pagehelper.support-methods-arguments=true

1、MySQL 数据源四要素（必须配置）

① driver-class-name 驱动类

  • MySQL 5.x：com.mysql.jdbc.Driver
  • MySQL 8.0+ 强制：com.mysql.cj.jdbc.Driver
高版本驱动不带cj会启动报错

② url 连接地址参数详解

jdbc:mysql://IP:端口/数据库名?参数1&参数2&参数3

参数作用
serverTimezone=Asia/Shanghai设置东八区，解决插入时间比实际晚8小时、时区异常报错
useUnicode=true&characterEncoding=utf-8指定字符集，防止中文乱码
useSSL=false开发环境关闭SSL安全校验；生产环境建议改为true
allowPublicKeyRetrieval=true解决 Public Key Retrieval is not allowed 连接报错

③ username / password

数据库登录账号密码，不要写中文、空格。

④ type 数据源类型

SpringBoot2.x 默认 HikariDataSource（速度优于Druid）
如果想用Druid：spring.datasource.type=com.alibaba.druid.pool.DruidDataSource

2、Hikari 连接池参数详解

# 最小空闲连接：常驻池子最少连接数
spring.datasource.hikari.minimum-idle=5
# 最大连接总数：数据库并发上限
spring.datasource.hikari.maximum-pool-size=15
# 空闲连接多久销毁（5分钟）
spring.datasource.hikari.idle-timeout=300000
# 单个连接最长存活时间（30分钟）
spring.datasource.hikari.max-lifetime=1800000
# 拿连接超时时间（20秒拿不到连接抛异常）
spring.datasource.hikari.connection-timeout=20000

3、MyBatis 核心配置（重中之重）

（1）mybatis.mapper-locations

指定 Mapper XML 文件位置

# classpath:resources下mapper文件夹所有xml
mybatis.mapper-locations=classpath:mapper/**/*.xml
  • * 代表递归所有子文件夹
  • 配置错误会报：Invalid bound statement (not found)

（2）mybatis.type-aliases-package

实体类别名包
例：包 com.example.demo.entity 下有 User 实体

  • 不配置：XML中写 resultType="com.example.demo.entity.User"
  • 配置后：直接简写 resultType="User"

（3）驼峰自动转换（高频实用）

mybatis.configuration.map-underscore-to-camel-case=true

数据库字段 user_name → 实体属性 userName，不用手动写 <column> 映射。

（4）SQL日志打印（调试必备）

mybatis.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl

控制台自动输出：执行SQL、参数、影响行数，方便排错。

（5）mybatis.config-location（可选）

如果你单独创建 mybatis-config.xml 存放 setting、typeHandler、插件等，才需要配置：

mybatis.config-location=classpath:mybatis-config.xml
大部分场景直接在 application.properties 配置即可，不需要单独xml。

4、PageHelper 分页插件配置（常用拓展）

pagehelper.helper-dialect=mysql
pagehelper.reasonable=true
pagehelper.support-methods-arguments=true
  • reasonable=true：页码容错，pageNum<=1自动查第一页，超出总页数查最后一页

五、常见报错&解决方案

  1. Invalid bound statement (not found)
    • mapper-locations路径写错 / 接口方法名和xml id不一致 / 没加@MapperScan
  1. 时区异常 The server time zone value...
    • url 补上 serverTimezone=Asia/Shanghai
  1. Public Key Retrieval is not allowed
    • url 追加 allowPublicKeyRetrieval=true
  1. 数据库字段下划线无法映射实体属性
    • 开启 map-underscore-to-camel-case=true
