对象类型

TypeScript（TS）的对象类型是对 JavaScript 对象的结构化类型约束，核心是明确对象的属性名、属性类型、可选属性、只读属性，以及方法的类型签名

  • 直接字面量定义（内联类型，适合一次性使用）

    直接在变量 / 参数后用 { 键: 类型, ... } 标注对象结构，简单直观

    // 变量标注
const user: {
  name: string; // 必选属性：字符串类型
  age: number;  // 必选属性：数字类型
  isAdmin?: boolean; // 可选属性：布尔类型（加 ? 标记）
  sayHello: () => string; // 方法：无参数，返回字符串
} = {
  name: "张三",
  age: 20,
  sayHello: () => "Hello!",
  // isAdmin 可选，可省略
};

// 函数参数标注
function printUser(user: { name: string; age: number }): void {
  console.log(`${user.name}, ${user.age}岁`);
}
printUser({ name: "李四", age: 25 }); // 合法
  • 类型别名（type）：复用对象类型

    用 type 定义对象类型的别名，适合多次复用同一结构

    // 定义对象类型别名
type User = {
  name: string;
  age: number;
  isAdmin?: boolean; // 可选属性
  hobbies: string[]; // 数组属性：字符串数组
  address: { // 嵌套对象类型
    province: string;
    city: string;
  };
  // 带参数的方法
  greet: (msg: string) => string;
};

// 使用类型别名
const user1: User = {
  name: "王五",
  age: 30,
  hobbies: ["篮球", "读书"],
  address: {
    province: "广东",
    city: "深圳",
  },
  greet: (msg) => `${msg}, 我是${user1.name}`,
};
  • 接口（interface）：扩展 / 继承对象类型

    interface 专门用于定义对象类型，支持扩展（extends） 和合并，适合团队协作或需要扩展的场景

    // 基础接口
interface User {
  name: string;
  age: number;
}

// 接口扩展（继承）
interface AdminUser extends User {
  role: "admin" | "super_admin"; // 字面量类型：仅限两个值
  manage: (userId: number) => void; // 方法：接收数字，无返回值
}

// 使用扩展后的接口
const admin: AdminUser = {
  name: "管理员",
  age: 28,
  role: "admin",
  manage: (userId) => console.log(`管理用户${userId}`),
};

// 接口合并（TS 特有：同名接口自动合并）
interface User {
  email?: string; // 新增可选属性
}
// 此时 User 包含：name、age、email
const user2: User = { name: "赵六", age: 22, email: "zhaoliu@test.com" };
  • 可选属性

    用 ? 标记属性为 “可选”，表示该属性可存在也可不存在，类型为 指定类型 | undefined

    type Product = {
  id: number;
  name: string;
  price: number;
  desc?: string; // 可选属性：string | undefined
};

const product1: Product = { id: 1, name: "手机", price: 2999 }; // 合法（省略 desc）
const product2: Product = { id: 2, name: "电脑", price: 5999, desc: "高性能" }; // 合法
  • 只读属性

    用 readonly 标记属性为 “只读”，初始化后不可修改，类似常量（const）但仅针对对象属性

    type Config = {
  readonly appName: string; // 只读属性
  version: string;
};

const config: Config = { appName: "TS_APP", version: "1.0.0" };
config.version = "1.0.1"; // 合法：普通属性可修改
// config.appName = "NEW_APP"; // 报错：只读属性不可修改
  • 索引签名：处理动态属性名
    1. 字符串索引签名
    // 字符串索引：键为 string，值为 number
type ScoreMap = {
  [key: string]: number; // 索引签名
  math: number; // 可显式声明固定属性（需符合索引类型）
};

const scores: ScoreMap = {
  math: 90,
  english: 85, // 动态属性：符合索引签名
  chinese: 95,
};
scores["history"] = 88; // 合法：动态添加属性
    1. 数字索引签名

    适用于属性名为数字的场景（如类数组对象）

    // 数字索引：键为 number，值为 string
type StringArray = {
  [index: number]: string;
  length: number; // 固定属性：需符合索引类型（number → string？不，length 是 number，需兼容）
};

// 正确写法：索引签名可兼容多类型（用联合类型）
type StringArray = {
  [index: number]: string;
  length: number; // 合法：number 是索引值类型的子类型（TS 特殊规则）
};
const arr: StringArray = { 0: "a", 1: "b", length: 2 };
  • 方法的类型定义
    type Person = {
  name: string;
  // 方式1：箭头函数形式（更直观）
  greet: (msg: string) => string;
  // 方式2：方法简写形式（更贴近 JS 写法）
  walk(distance: number): void;
};

const person: Person = {
  name: "小明",
  greet: (msg) => `${msg}, 我是${person.name}`,
  walk: (distance) => console.log(`走了${distance}米`),
};
  • 交叉类型（&）：合并多个对象类型

    用 & 将多个对象类型合并为一个，包含所有类型的属性（类似 “并集”）

    type BasicInfo = { name: string; age: number };
type ContactInfo = { phone: string; email?: string };

// 交叉类型：同时包含 BasicInfo 和 ContactInfo 的属性
type UserInfo = BasicInfo & ContactInfo;

const userInfo: UserInfo = {
  name: "小红",
  age: 24,
  phone: "13800138000",
};
  • 类型断言：覆盖对象类型推断

    当 TS 推断的类型不符合实际，用 as 进行类型断言（仅告诉 TS 类型，不改变运行时）

    // TS 默认推断为 { name: string; age: number; }
const obj = { name: "小李", age: 26 };

// 断言为包含 extra 属性的类型（需确保运行时存在该属性，否则可能报错）
const extendedObj = obj as { name: string; age: number; extra: string };
// 注意：仅临时覆盖类型，运行时 extendedObj.extra 为 undefined
  • 可选链（?.）+ 非空断言（!）：处理可选属性

    结合 TS 语法糖，安全访问对象的可选 / 嵌套属性

    type User = {
  name: string;
  address?: {
    city: string;
  };
};

const user: User = { name: "小张" };

// 可选链：address 不存在时返回 undefined，不报错
console.log(user.address?.city); // undefined

// 非空断言：告诉 TS 该属性一定存在（需确保，否则运行时报错）
const user2: User = { name: "小刘", address: { city: "北京" } };
console.log(user2.address!.city); // 北京
  • 类型守卫：缩小对象类型范围

    通过条件判断缩小对象的类型范围，TS 会自动推导更精确的类型

    type Animal = {
  type: "cat" | "dog";
  name: string;
  // 猫特有属性
  meow?: () => void;
  // 狗特有属性
  bark?: () => void;
};

function interact(animal: Animal): void {
  if (animal.type === "cat") {
    // TS 推导：animal 是 { type: "cat", ..., meow?: () => void }
    animal.meow?.(); // 安全调用
  } else {
    // TS 推导：animal 是 { type: "dog", ..., bark?: () => void }
    animal.bark?.();
  }
}

type vs interface:

特性type（类型别名）interface（接口）
定义对象类型✅ 支持✅ 支持（专为对象设计）
扩展 / 继承用交叉类型（&）用 extends 关键字，更直观
合并同名定义❌ 不支持（会报错）✅ 支持（自动合并）
定义非对象类型✅ 支持（如联合类型、字面量类型）❌ 仅支持对象 / 函数类型
映射类型✅ 更常用（如 Partial、Readonly）✅ 支持，但不如 type 灵活

误区：

  1. 对象字面量的 “多余属性检查”：直接传递对象字面量时，TS 会严格检查是否有多余属性；但赋值给变量后则不会。
type User = { name: string; age: number };
// 报错：多余属性 email
// printUser({ name: "张三", age: 20, email: "zhangsan@test.com" });

// 合法：先赋值给变量，TS 不检查多余属性
const user = { name: "张三", age: 20, email: "zhangsan@test.com" };
printUser(user);

2. 索引签名与固定属性的兼容性：固定属性的类型必须兼容索引签名的类型。

// 报错：name 是 string，索引签名要求值为 number
// type BadMap = { [key: string]: number; name: string };
  1. readonly 仅约束类型，不影响运行时：编译为 JS 后，readonly 会被移除，运行时仍可修改（仅 TS 编译期检查）。
