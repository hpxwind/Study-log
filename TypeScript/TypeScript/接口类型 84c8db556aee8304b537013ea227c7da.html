接口类型

接口（Interface） 是用于约束对象、函数、类等结构的核心类型工具，核心作用是定义 “契约” —— 明确指定数据 / 代码的结构规范，是 TS 实现类型抽象、复用和扩展的关键特性

接口仅存在于编译阶段

  • 对象接口
    // 定义接口：描述 User 对象的结构
interface User {
  // 必选属性：字符串类型
  name: string;
  // 必选属性：数字类型
  age: number;
  // 可选属性：加 ? 标记，类型为 string | undefined
  email?: string;
  // 只读属性：加 readonly，初始化后不可修改
  readonly id: number;
}

// 符合接口的对象（合法）
const user1: User = {
  id: 1001,
  name: "张三",
  age: 25,
  // email 可选，可省略
};

// 错误示例：缺少必选属性 age（TS 编译报错）
// const user2: User = { id: 1002, name: "李四" };

// 错误示例：修改只读属性（TS 编译报错）
// user1.id = 1003;
  • 嵌套接口

    接口可嵌套定义，描述多层级的对象结构

    // 嵌套接口：地址结构
interface Address {
  province: string;
  city: string;
  detail?: string;
}

// 主接口：包含嵌套接口
interface User {
  name: string;
  age: number;
  address: Address; // 嵌套接口作为属性类型
}

const user: User = {
  name: "李四",
  age: 30,
  address: {
    province: "浙江",
    city: "杭州",
  },
};
  • 索引签名：处理动态属性

    当对象的属性名是动态的（不确定具体键名），用索引签名描述

    // 字符串索引签名：键为 string，值为 number（适用于任意字符串键）
interface ScoreMap {
  [key: string]: number;
  // 可显式声明固定属性（需符合索引值类型）
  math: number;
}

const scores: ScoreMap = {
  math: 90,
  english: 85, // 动态属性，符合索引签名
  chinese: 95,
};

// 数字索引签名：键为 number，值为 string（适用于类数组对象）
interface StringArray {
  [index: number]: string;
  length: number; // 固定属性（TS 允许 number 类型，兼容数字索引）
}

const arr: StringArray = { 0: "a", 1: "b", length: 2 };
  • 接口集成（extends）

    通过 extends 实现接口复用和扩展，支持单继承 / 多继承，比 type 的交叉类型（&）更直观。

    1. 单集成
    // 基础接口
interface Person {
  name: string;
  age: number;
}

// 继承 Person，扩展新属性
interface Student extends Person {
  studentId: number;
  grade: string;
}

// Student 包含：name、age、studentId、grade
const student: Student = {
  name: "王五",
  age: 18,
  studentId: 2024001,
  grade: "高一",
};
    1. 多继承

    继承多个接口，合并所有属性：

    interface Contact {
  phone: string;
  email: string;
}

// 同时继承 Person 和 Contact
interface Teacher extends Person, Contact {
  subject: string;
}

// Teacher 包含：name、age（Person） + phone、email（Contact） + subject
const teacher: Teacher = {
  name: "赵六",
  age: 35,
  phone: "13800138000",
  email: "zhaoliu@school.com",
  subject: "数学",
};
  • 接口合并（声明合并）

    TS 特有特性：同名接口会自动合并，适合拆分复杂接口定义（如分文件定义同一接口）。

    // 第一次定义 User 接口
interface User {
  name: string;
  age: number;
}

// 第二次定义同名接口，自动合并
interface User {
  email?: string;
  greet: () => string;
}

// 合并后的 User 包含：name、age、email、greet
const user: User = {
  name: "孙七",
  age: 28,
  greet: () => "Hello!",
};

    ⚠️ 注意：合并的接口中，同名属性的类型必须兼容，否则报错：

    // 错误示例：同名属性类型冲突
// interface User { age: number; }
// interface User { age: string; } // TS 报错：属性 age 类型不兼容
  • 接口描述函数类型

    接口不仅能约束对象，还能定义函数的参数和返回值类型（替代 type 定义函数类型）：

    // 定义函数接口：接收两个 number，返回 number
interface CalculateFunc {
  (a: number, b: number): number;
}

// 实现函数（符合接口规范）
const add: CalculateFunc = (x, y) => x + y;
const subtract: CalculateFunc = (a, b) => a - b;

add(1, 2); // 3

  • 接口描述类类型（implements）

    接口可定义类的 “实现规范”，类通过 implements 实现接口，强制类满足接口的结构约束

    // 定义接口：约束类必须包含的属性和方法
interface Animal {
  name: string;
  makeSound(): void;
}

// 类实现接口（必须实现所有属性和方法）
class Dog implements Animal {
  name: string; // 必须有 name 属性
  constructor(name: string) {
    this.name = name;
  }
  // 必须实现 makeSound 方法
  makeSound(): void {
    console.log(`${this.name} 汪汪叫`);
  }
}

const dog = new Dog("旺财");
dog.makeSound(); // 旺财 汪汪叫

    类实现多个接口

    interface Runable {
  run(): void;
}

// 同时实现 Animal 和 Runable
class Cat implements Animal, Runable {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
  makeSound(): void {
    console.log(`${this.name} 喵喵叫`);
  }
  run(): void {
    console.log(`${this.name} 跑起来了`);
  }
}
  • 泛型接口

    结合泛型，让接口支持“动态类型”，提升复用性

    // 泛型接口：T 为类型参数，可自定义
interface Result<T> {
  code: number;
  msg: string;
  data: T; // 动态类型
}

// 使用泛型接口：指定 T 为 User 类型
type UserResult = Result<User>;
const userResult: UserResult = {
  code: 200,
  msg: "成功",
  data: { name: "周八", age: 30 },
};

// 使用泛型接口：指定 T 为数组类型
type ListResult = Result<string[]>;
const listResult: ListResult = {
  code: 200,
  msg: "成功",
  data: ["a", "b", "c"],
};

接口 vs 类型别名（type）

特性接口（interface）类型别名（type）
核心用途定义对象 / 类 / 函数的 “结构契约”定义任意类型（对象、联合、字面量等）
继承支持 extends（单 / 多继承）需用交叉类型 & 实现 “合并”
声明合并支持同名接口自动合并不支持（同名 type 会报错）
泛型支持支持（泛型接口）支持（泛型类型）
定义非对象类型不支持（仅对象 / 函数 / 类）支持（如 type Num = number、`type Union = stringnumber`）
映射类型支持，但不如 type 灵活更适合映射类型（如 Partial<T>）
