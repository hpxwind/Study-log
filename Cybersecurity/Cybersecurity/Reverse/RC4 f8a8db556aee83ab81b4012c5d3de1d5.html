RC4

RC4是一种对称流密码，加密和机密使用相同的密钥和流程，核心包括：

密钥调度算法（KSA）和伪随机数生成算法（PRGA）

  • 1、密钥调度算法KSA:初始化S盒

    KSA 的作用是用输入密钥（Key）初始化一个 256 字节的状态数组 S（称为 “S 盒”），步骤如下：

    • 初始化 S 为 [0, 1, 2, ..., 255]；
    • 用密钥对 S 进行打乱（置换），公式为：
    j = 0
for i from 0 to 255:
    j = (j + S[i] + Key[i mod keylen]) mod 256
    swap S[i] and S[j]

    其中 keylen 是密钥长度（字节），i mod keylen 确保密钥循环使用。

  • 2、伪随机数生成算法PRGA:生成密钥流

    PRGA 利用初始化后的 S 盒生成与明文等长的 “密钥流”（Keystream），然后通过 明文与密钥流异或 得到密文（解密时，密文与同一密钥流异或还原明文）。步骤如下：

    • 初始化索引 i=0，j=0；
    • 每次生成一个密钥流字节：
    i = (i + 1) mod 256
j = (j + S[i]) mod 256
swap S[i] and S[j]
t = (S[i] + S[j]) mod 256
keystream_byte = S[t]
    • 加密：ciphertext_byte = plaintext_byte XOR keystream_byte
    • 解密：plaintext_byte = ciphertext_byte XOR keystream_byte
  • 逆向中识别RC4的核心特征

    1. 代码特征（关键逻辑）

    • S 盒初始化：存在一个 256 字节数组的初始化（通常是 0~255 顺序填充）；
    • KSA 阶段双重循环：外层循环 i=0~255，内层计算 j 并交换 S[i] 与 S[j]，且 j 的计算依赖 S[i] 和密钥（j = (j + S[i] + Key[i%keylen]) % 256）；
    • PRGA 阶段循环：存在 i 和 j 的更新（i=(i+1)%256，j=(j+S[i])%256），交换 S[i] 与 S[j]，并通过 t=(S[i]+S[j])%256 取 S[t] 作为密钥流字节；
    • 异或操作：生成的密钥流字节与明文 / 密文进行异或（RC4 加密的核心运算）。

    2. 数据特征

    • 密钥长度可变（通常 1~256 字节），但 KSA 中会通过 i mod keylen 循环使用；
    • 加密 / 解密使用同一套逻辑（仅输入为明文 / 密文的区别）；
    • 密钥流与明文长度相同，且异或结果即为密文（密文与明文的熵值接近，无固定块大小，区别于分组密码）。
  • 逆向分析RC4的关键

    遇到RC4核心是提取密钥，还原加密/解密流程

    1. 定位 RC4 函数

    通过上述代码特征（256 字节数组、双重循环、异或操作）在反汇编代码中定位 RC4 加密函数。例如：

    • 在 IDA Pro 或 Ghidra 中，搜索 “256 字节数组初始化”（如 memset 或循环赋值 0~255）；
    • 查找包含 i 和 j 两个索引变量的循环，且存在数组元素交换；
    • 跟踪函数输入参数（通常包括：密钥、密钥长度、明文 / 密文、数据长度）。

    2. 提取密钥

    密钥是 RC4 逆向的核心，常见提取方式：

    • 硬编码密钥：若密钥直接写在程序中（如字符串常量），可通过反汇编直接读取（例如在数据段中查找疑似密钥的字节序列）；
    • 动态获取密钥：若密钥是动态生成（如从配置文件、网络请求中读取），可通过调试工具（如 x64dbg）在 KSA 阶段拦截密钥数据（跟踪 Key 指针指向的内存）；
    • 密钥长度推断：通过 KSA 中 i mod keylen 的运算（如 i % 16 说明密钥长度为 16 字节）。
