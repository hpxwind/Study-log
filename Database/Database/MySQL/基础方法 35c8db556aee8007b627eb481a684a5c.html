  • sleep()

    SLEEP() 是一个用于暂停查询执行指定时间的函数,模拟延迟、测试并发或控制执行节奏。

    SLEEP(seconds)
参数：seconds 为等待时间，支持小数（如 0.5 表示 500 毫秒）。
返回值：成功返回 0，若被中断返回 1。
    -- 暂停2秒后执行查询
SELECT SLEEP(2);

-- 在事务中插入延迟
INSERT INTO logs (message) VALUES ('开始操作');
DO SLEEP(3); -- DO 语句用于执行函数但不返回结果
INSERT INTO logs (message) VALUES ('延迟3秒后操作');
  • group_concat()

    >>> 聚合函数，用于将分组中的多行数据合并为单个字符串。

    它在数据报表，嵌套查询和数据导出等场景中非常使用。

    >>> 语法;

    GROUP_CONCAT(
    [DISTINCT] expr1 [, expr2 ...]
    [ORDER BY expr [ASC|DESC]]
    [SEPARATOR str_val]
)
    • 参数说明：
      • DISTINCT：去重。
      • ORDER BY：定义合并顺序。
      • SEPARATOR：指定分隔符（默认逗号 ,）。

    >>> 用法：

    1、合并单列数据

    -- 将统一部门的姓名合并为字符串
select department_id, group_concat(name) as employees
from employees
group by department_id;

-- 结果示例：
-- department_id | employees
-- --------------+------------------
-- 1             | Alice,Bob,Charlie
-- 2             | Dave,Eve

    2、指定分割符

    -- 使用 "||" 作为分隔符
select 
		department_id, 
		group_concat(name order by salary desc separator ' || ') as employees
from employees
group by department_id;

-- 结果示例：
-- department_id | employees
-- --------------+------------------
-- 1             | Bob || Alice || Charlie

    3、合并多列数据

    select 
		department_id,
		group_concat(concat(name, ':', salary)) as emp_info
from employees
group by department_id;

-- 结果示例：
-- department_id | emp_info
-- --------------+-----------------------
-- 1             | Alice:5000,Bob:6000

    5、结合distinct去重

    -- 获取每个订单包含的唯一产品类别
SELECT 
    order_id,
    GROUP_CONCAT(DISTINCT category) AS categories
FROM order_items
GROUP BY order_id;

    突破长度限制：默认group_concat被限制长度为group_concat_max_len=1024字节，可以临时修改：

    set session group_concat_max_len=1024*1024; -- 1MB
-- 再配置文件中可以永久修改
[mysql]
group_concat_max_len = 1048576

    6、嵌套使用

    -- 先按部门分组，再按角色合并
select 
		company_id,
		group_concat(
				concat(
					department_name, ':',
					group_concat(role separator '|')
				)
				separator ';'
		) as dept_roles
from departments
join employees using (department_id)
group by company_id;

-- 结果示例：
-- company_id | dept_roles
-- -----------+---------------------------------
-- 1          | HR:Manager|Recruiter;IT:Developer|Admin

  • updatexml()

    UPDATEXML() 是一个用于修改 XML 数据的函数，主要作用是更新 XML 文档中指定节点的值.

    UPDATEXML() 是一个用于修改 XML 数据的函数，主要作用是更新 XML 文档中指定节点的值.
xml_target：需要修改的 XML 字符串或 XML 类型数据。
xpath_expr：XPath 表达式，用于定位需要修改的节点（必须是有效的 XPath 路径）。
new_xml：替换目标节点的新 XML 内容（字符串形式）。

    例如：

    <bookstore>
  <book id="1"><title>SQL 基础</title><price>39.9</price></book>
  <book id="2"><title>XML 教程</title><price>45.0</price></book>
</bookstore>

--------------------
将id=‘1’的书的price修改为49.9 ：
SELECT UPDATEXML(
  '<bookstore>
    <book id="1"><title>SQL 基础</title><price>39.9</price></book>
    <book id="2"><title>XML 教程</title><price>45.0</price></book>
  </bookstore>',
  '/bookstore/book[@id="1"]/price',  -- XPath 定位目标节点
  '<price>49.9</price>'  -- 新内容
) AS updated_xml;

------------
结果：
<bookstore>
  <book id="1"><title>SQL 基础</title><price>49.9</price></book>
  <book id="2"><title>XML 教程</title><price>45.0</price></book>
</bookstore>

  • extractvalue()

    extractvalue() 是 SQL 中用于从 XML 数据中提取指定节点值的函数，广泛应用于支持 XML 类型的数据库。

    通过 XPath 表达式定位 XML 节点，并并返回，并返回节点的文本内容。

    extractvalue(xml_target, xpath_expr)
xml_target：需要解析的 XML 字符串或 XML 类型数据（源 XML）。
xpath_expr：有效的 XPath 表达式，用于定位要提取的节点。

    例如：

    <students>
  <student id="1"><name>张三</name><age>20</age><major>计算机</major></student>
  <student id="2"><name>李四</name><age>22</age><major>数学</major></student>
</students>

---------------
提取id=‘1’的学生姓名：
SELECT extractvalue(
  '<students>
    <student id="1"><name>张三</name><age>20</age></student>
    <student id="2"><name>李四</name><age>22</age></student>
  </students>',
  '/students/student[@id="1"]/name'  -- XPath 定位 name 节点
) AS student_name;

---------------------
student_name
------------
张三
