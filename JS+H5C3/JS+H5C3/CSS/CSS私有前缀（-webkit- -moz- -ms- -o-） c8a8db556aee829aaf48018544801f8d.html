CSS私有前缀（-webkit-/-moz-/-ms-/-o-）

私有前缀是浏览器厂商给 CSS 属性 / 规则加的「专属标识」，格式为 -[厂商标识]-[属性名]，例如：

webkit-border-radius（WebKit 内核浏览器专属）

moz-box-shadow（Firefox 专属）

    -webkit-Apple、GoogleWebKit / BlinkSafari（macOS/iOS）、Chrome、Edge（80+）、Opera、Brave、iOS WebView仍需使用（Safari/iOS 兼容性依赖）
    -moz-MozillaGeckoFirefox（火狐）部分特性仍需（如 -moz-appearance）
    -ms-MicrosoftTrident（旧）IE 6-11、Edge（18- 旧版）基本淘汰（仅兼容 IE 旧版时使用）
    -o-Opera SoftwarePresto（旧）Opera（12- 旧版）

  • 书写顺序：前缀在前，标准在后

    浏览器会优先识别「带前缀的属性」，若不支持则 fallback 到标准属性，最终标准属性会覆盖前缀属性（保证现代浏览器用标准实现）。

    ❌ 错误写法（标准在前，前缀在后，会被覆盖）：

    .box {
  border-radius: 8px; /* 标准在前，-webkit- 会覆盖它（Safari 中） */
  -webkit-border-radius: 8px;
}

    ✅ 正确写法（前缀在前，标准在后）：

    .box {
  -webkit-border-radius: 8px; /* Safari/Chrome 旧版 */
  -moz-border-radius: 8px;    /* Firefox 旧版 */
  -ms-border-radius: 8px;     /* IE 旧版 */
  border-radius: 8px;         /* 标准属性（现代浏览器） */
}
  • 需要添加，不盲目堆砌

    不需要给所有属性加全 4 个前缀，根据目标浏览器选择：

    • 只兼容 Safari/Chrome/Edge：仅加 webkit- + 标准；
    • 只兼容 Firefox：仅加 moz- + 标准；
    • 兼容 IE 旧版：加 ms- + 标准。
  • 避免依赖私有特性

    不要使用「仅存在于私有前缀中，未被标准化」的属性（如 -webkit-marquee 滚动文字），这类属性可能被浏览器后续移除，导致兼容性问题。

自动处理工具

手动编写容易出错，开发中使用工具自动生成，如：Autoprefixer

Autoprefixer CSS online

  1. 优先使用标准属性：现代项目（不兼容 IE 旧版）尽量直接用标准 CSS 属性，减少前缀依赖；
  1. 必要时兼容前缀：仅在需要支持 Safari/iOS 或旧浏览器时，添加对应前缀（重点是 webkit-）；
  1. 工具自动化：用 Autoprefixer 或 Tailwind 自动处理前缀，避免手动编写错误；
  1. 查询兼容性：不确定属性是否需要前缀时，用 caniuse.com 查询（输入属性名，查看各浏览器支持情况）；
  1. 渐进增强：以标准属性为核心，前缀为补充，保证旧浏览器有基本效果，现代浏览器有更好体验。
