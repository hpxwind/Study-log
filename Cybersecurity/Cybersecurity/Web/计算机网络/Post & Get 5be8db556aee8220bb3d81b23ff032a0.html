Post & Get

1. 核心功能与应用场景

特性GET 请求POST 请求
核心用途从服务器获取资源（如网页、API 数据）。向服务器提交数据（如表单提交、文件上传）。
幂等性多次相同请求不影响结果（幂等）。例如：多次刷新网页不会重复创建数据。非幂等：多次提交可能产生重复数据（如多次提交表单可能创建多条记录）。
典型场景- 搜索页面（参数包含搜索关键词）- API 数据查询（如获取用户信息）。- 注册 / 登录表单提交- 上传文件- 创建 / 更新数据库记录。

2. 数据传输方式

特性GET 请求POST 请求
参数位置参数附加在 URL 的查询字符串中（?key1=value1&key2=value2）。参数放在 HTTP 请求体（body）中，不显示在 URL 中。
可见性参数直接暴露在 URL 中，例如：https://example.com/search?q=keyword。参数不可见（但仍可通过工具如浏览器开发者工具查看）。
数据类型只能传输文本（URL 编码后）。支持多种数据类型（如表单数据、JSON、二进制文件等）。
编码方式URL 编码（如空格会被转换为 %20，中文会被转换为 %E4%B8%AD）。常见编码：- application/x-www-form-urlencoded（默认）- multipart/form-data（文件上传）- application/json（JSON 数据）。

3. 安全性

特性GET 请求POST 请求
参数暴露参数暴露在 URL 中，存在以下风险：- 敏感信息（如密码）直接可见。- 可能被浏览器历史记录、服务器日志记录。参数不在 URL 中，但仍需防范中间人攻击（需配合 HTTPS）。
适用场景不适合传输敏感数据（如密码、信用卡号）。更适合传输敏感数据，但需注意防范 CSRF 攻击（如使用验证码、CSRF Token）。
缓存性请求可能被浏览器缓存（如图片、静态资源）。默认不缓存（但可通过设置响应头强制缓存）。

4. 请求长度限制

特性GET 请求POST 请求
URL 长度受浏览器和服务器限制（通常约 2KB - 8KB）。不同浏览器限制不同：- Chrome：约 2MB- Firefox：约 65KB。理论上无限制（实际受服务器配置限制，如 PHP 的 post_max_size）。
数据量适合小数据量传输（如搜索关键词）。适合大数据量（如上传文件、批量数据）。

5. 性能与幂等性

特性GET 请求POST 请求
请求速度通常更快（无请求体，参数直接拼接在 URL 中）。略慢（需封装请求体）。
幂等性幂等：多次相同请求不会产生副作用（如查询数据）。非幂等：多次提交可能产生重复操作（如创建多条记录）。
缓存策略可利用缓存减少服务器负载（如频繁访问的 API）。需谨慎使用缓存（如表单提交后不应缓存结果）。
