Sass

Sass（Syntactically Awesome Style Sheets）是一种CSS 预处理器，它扩展了 CSS 的语法，增加了变量、嵌套、混合、继承等高级特性，让 CSS 的编写更高效、可维护性更强。Sass 最终会被编译为普通 CSS 文件，供浏览器识别。

  • 优势

    相比原生 CSS，Sass 的核心优势体现在以下几点：

    1. 变量（Variables）：将重复使用的样式值（如颜色、字体、尺寸）定义为变量，一处修改，多处生效。
    1. 嵌套（Nesting）：支持选择器嵌套，清晰反映 HTML 结构，避免重复书写父选择器。
    1. 混合（Mixins）：封装可复用的样式片段（如浏览器前缀、复杂布局），通过@include调用，减少代码冗余。
    1. 继承（Inheritance）：通过@extend实现样式继承，复用已有类的样式，生成更精简的 CSS。
    1. 模块化（Modules）：将样式拆分到多个文件，通过@use和@forward导入组合，便于大型项目管理。
    1. 运算与函数：支持数学运算（如width: 100% - 20px）和内置函数（如颜色处理、字符串操作）。
  • 两种语法格式
    格式扩展名语法特点示例代码
    Sass.sass缩进式语法，无大括号{}和分号;，依靠缩进区分代码块body\n color: red\n font-size: 16px
    SCSS.scss类 CSS 语法，保留大括号和分号，兼容原生 CSSbody { color: red; font-size: 16px; }

    推荐使用 SCSS 格式：更接近 CSS 写法，学习成本低，且兼容所有原生 CSS 代码（直接复制粘贴即可运行）。

  • 环境搭建

    方式一：

    安装Sass（依赖Node.js环境）

    npm install -g sass  # 全局安装Sass

    编译命令

    # 单文件编译：将 input.scss 编译为 output.css
sass input.scss output.css

# 实时监听（开发常用）：文件变化时自动编译
sass --watch input.scss:output.css

# 监听文件夹：src目录下的.scss文件编译到dist目录
sass --watch src/:dist/
    • -style compressed：生成压缩后的 CSS（无空格和注释）
    • -source-map：生成 Source Map 文件，方便调试

    方式二：

    在 Webpack、Vite、Gulp 等构建工具中集成 Sass，自动处理编译过程：

    ·Vite项目（无需额外配置，天生支持Sass）：

    npm create vite@latest my-project  # 创建Vite项目
cd my-project
npm install

    直接创建.scss文件并在组件中引入即可。

    ·Webpack项目：

    需要安装sass和sass-loader：

    npm install sass sass-loader --save-dev
  • Sass核心特性
    • 1、变量（Variables）

      作用：存储重复使用的样式值，方便统一维护。

      语法：$变量名: 值;（变量名以$开头）

      // 定义变量
$primary-color: #0d6efd;  // 主色调（Bootstrap蓝）
$base-font-size: 16px;    // 基础字体大小
$border-radius: 4px;      // 边框圆角

// 使用变量
.btn {
  color: white;
  background-color: $primary-color;  // 引用变量
  font-size: $base-font-size;
  border-radius: $border-radius;
}

.card {
  border: 1px solid #ddd;
  border-radius: $border-radius;  // 复用变量
}

      编译css后：

      .btn {
  color: white;
  background-color: #0d6efd;
  font-size: 16px;
  border-radius: 4px;
}

.card {
  border: 1px solid #ddd;
  border-radius: 4px;
}

      变量作用域：

      • 局部变量：在选择器或代码块内定义，仅在该范围内有效
      • 全局变量：在最外层定义，或通过!global声明为全局
      $global-color: red;  // 全局变量

.box {
  $local-color: blue;  // 局部变量
  color: $local-color;
  background: $global-color;
}

.footer {
  color: $global-color;  // 可访问全局变量
  // color: $local-color;  // 错误：无法访问局部变量
}
    • 2、嵌套（Nesting）

      作用：通过嵌套反映 HTML 结构，减少代码冗余，提高可读性。

      选择器嵌套：

      // HTML结构：
// <nav class="navbar">
//   <ul class="menu">
//     <li><a href="#">首页</a></li>
//   </ul>
// </nav>

.navbar {
  background: #333;
  padding: 1rem;

  .menu {  // 嵌套：等价于 .navbar .menu
    list-style: none;
    margin: 0;
    padding: 0;

    li {  // 嵌套：等价于 .navbar .menu li
      display: inline-block;
      margin-right: 1rem;

      a {  // 嵌套：等价于 .navbar .menu li a
        color: white;
        text-decoration: none;

        &:hover {  // & 代表父选择器（此处为 a）
          color: #ffc107;
        }
      }
    }
  }
}

      编译后css：

      .navbar {
  background: #333;
  padding: 1rem;
}

.navbar .menu {
  list-style: none;
  margin: 0;
  padding: 0;
}

.navbar .menu li {
  display: inline-block;
  margin-right: 1rem;
}

.navbar .menu li a {
  color: white;
  text-decoration: none;
}

.navbar .menu li a:hover {  // & 被替换为 a
  color: #ffc107;
}

      属性嵌套：

      对于具有相同前缀的 CSS 属性（如font-*、border-*），可简化写法：

      .box {
  font: {  // 属性嵌套
    family: Arial, sans-serif;
    size: 16px;
    weight: bold;
  }

  border: {
    width: 1px;
    style: solid;
    color: #ddd;
  }
}

      编译后的css：

      .box {
  font-family: Arial, sans-serif;
  font-size: 16px;
  font-weight: bold;
  border-width: 1px;
  border-style: solid;
  border-color: #ddd;
}
    • 3、混合（Mixins）

      作用：封装可复用的样式片段（尤其是带参数的复杂样式），通过@include调用。

      1、基础混合

      // 定义混合（无参数）
@mixin clearfix {  // @mixin 混合名
  &::after {
    content: "";
    display: table;
    clear: both;
  }
}

// 使用混合
.container {
  @include clearfix;  // @include 调用混合
  width: 100%;
}

.card {
  @include clearfix;  // 复用清除浮动样式
  border: 1px solid #eee;
}

      编译后css：

      .container::after {
  content: "";
  display: table;
  clear: both;
}

.container {
  width: 100%;
}

.card::after {
  content: "";
  display: table;
  clear: both;
}

.card {
  border: 1px solid #eee;
}

      带参数的混合：

      // 定义带参数的混合（可设置默认值）
@mixin button-style($bg-color: #0d6efd, $text-color: white) {
  padding: 0.5rem 1rem;
  border: none;
  border-radius: 4px;
  background-color: $bg-color;
  color: $text-color;
  cursor: pointer;

  &:hover {
    opacity: 0.9;
  }
}

// 使用混合（传递参数）
.btn-primary {
  @include button-style;  // 使用默认参数
}

.btn-danger {
  @include button-style(#dc3545, white);  // 传递自定义参数
}

.btn-warning {
  @include button-style($text-color: #333, $bg-color: #ffc107);  // 按名称传参
}
    • 4、继承（Inheritance）

      作用：通过@extend让一个选择器继承另一个选择器的所有样式，生成更精简的 CSS。

      // 基础样式类
.base-btn {
  padding: 0.5rem 1rem;
  border-radius: 4px;
  border: none;
  cursor: pointer;
}

// 继承基础样式
.btn-primary {
  @extend .base-btn;  // 继承 .base-btn 的所有样式
  background-color: #0d6efd;
  color: white;
}

.btn-secondary {
  @extend .base-btn;
  background-color: #6c757d;
  color: white;
}

      编译后的css:

      .base-btn, .btn-primary, .btn-secondary {  // 合并选择器，减少冗余
  padding: 0.5rem 1rem;
  border-radius: 4px;
  border: none;
  cursor: pointer;
}

.btn-primary {
  background-color: #0d6efd;
  color: white;
}

.btn-secondary {
  background-color: #6c757d;
  color: white;
}

      注意：@extend 与 @mixin 的区别：

      • @mixin 适合封装带参数的动态样式，会复制代码到调用处
      • @extend 适合复用静态样式，会合并选择器，生成更精简的 CSS
    • 5、模块化（Modules）

      作用：将样式拆分为多个文件（模块），通过@use和@forward管理依赖，避免全局命名冲突。

      1、文件命名规范

      模块文件通常以_开头（如_variables.scss），表示这是一个局部文件（不会被单独编译为 CSS）。

      2、@use: 导入并使用模块

      // _variables.scss（局部文件）
$primary: #0d6efd;
$secondary: #6c757d;

// _mixins.scss（局部文件）
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

// main.scss（主文件）
@use 'variables';  // 导入 variables 模块（省略 .scss 和 _）
@use 'mixins';

.header {
  background-color: variables.$primary;  // 通过 模块名.$变量名 访问
  @include mixins.flex-center;  // 通过 模块名.混合名 调用
  height: 60px;
}

      编译后的css:

      .header {
  background-color: #0d6efd;
  display: flex;
  align-items: center;
  justify-content: center;
  height: 60px;
}

      3、别名与命名空间：

      // 给模块起别名（简化调用）
@use 'variables' as v;
@use 'mixins' as m;

.footer {
  color: v.$secondary;  // 使用别名访问
  @include m.flex-center;
}

      4、@forward：转发模块（公共API）

      用于创建 “入口模块”，统一对外暴露多个子模块：

      // themes/_dark.scss
$bg-color: #333;
$text-color: white;

// themes/_light.scss
$bg-color: #fff;
$text-color: #333;

// themes/_index.scss（入口模块）
@forward 'dark';  // 转发 dark 模块
@forward 'light'; // 转发 light 模块

// main.scss
@use 'themes' as t;  // 只需导入入口模块

.dark-mode {
  background: t.$bg-color;  // 访问 dark 模块的变量
}
  • 运算与函数
    • 1、数学运算

      支持+、-、*、/、%运算：

      $base-width: 1000px;
$gutter: 20px;

.container {
  width: $base-width;
}

.content {
  width: $base-width - 200px;  // 减法
  margin-left: $gutter / 2;    // 除法
}

.sidebar {
  width: 200px - $gutter;      // 运算组合
}

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  // 内置函数 repeat
  gap: $gutter;
}
    • 2、颜色函数

      常用颜色函数：

      • lighten($color, $amount)：提亮颜色
      • darken($color, $amount)：加深颜色
      • saturate($color, $amount)：提高饱和度
      • desaturate($color, $amount)：降低饱和度
      • rgba($color, $alpha)：设置透明度

      Sass 提供丰富的颜色处理函数：

      $primary: #0d6efd;

.btn {
  background: $primary;
}

.btn:hover {
  background: darken($primary, 10%);  // 颜色加深10%
}

.btn-outline {
  color: $primary;
  border: 1px solid lighten($primary, 20%);  // 颜色变浅20%
}

.alert {
  background: rgba($primary, 0.2);  // 调整透明度
}

    • 3、控制语句

      Sass 支持条件判断和循环，实现更复杂的样式逻辑（主要用于混合和函数中）。

      1、条件判断（@if/@else）

      @mixin theme($mode: light) {
  @if $mode == dark {
    background: #333;
    color: white;
  } @else if $mode == light {
    background: #fff;
    color: #333;
  } @else {
    background: #f5f5f5;
    color: #666;
  }
}

.card {
  @include theme(dark);  // 调用时指定模式
}

.footer {
  @include theme;  // 使用默认模式（light）
}

      2、循环（@for/@each）

      // @for 循环（生成网格列）
@for $i from 1 through 6 {
  .col-#{$i} {  // #{} 是插值语法，用于拼接变量和字符串
    width: 100% / 6 * $i;
  }
}

// @each 循环（处理列表）
$colors: primary #0d6efd, secondary #6c757d, danger #dc3545;

@each $name, $color in $colors {
  .btn-#{$name} {
    background-color: $color;
  }
}

      编译后的css:

      .col-1 { width: 16.66667%; }
.col-2 { width: 33.33333%; }
/* ... 直到 col-6 */

.btn-primary { background-color: #0d6efd; }
.btn-secondary { background-color: #6c757d; }
.btn-danger { background-color: #dc3545; }
  • 注意
    1. 合理组织文件结构：
      scss/
├── _variables.scss    // 变量定义
├── _mixins.scss       // 混合定义
├── _buttons.scss      // 按钮组件
├── _cards.scss        // 卡片组件
└── main.scss          // 主文件（导入所有模块）
    1. 避免过度嵌套：嵌套深度建议不超过 3 层，否则会生成冗余的 CSS 选择器（如.a .b .c .d）。
    1. 优先使用@use而非@import：@use是 Sass 现代语法（替代旧的@import），支持命名空间，避免变量冲突。
    1. 利用注释分类：使用/* */（会保留到 CSS）和//（仅在 Sass 中可见）区分注释，方便维护：
      // 变量：颜色系统（仅Sass可见）
$primary: #0d6efd;

/* 基础按钮样式（会编译到CSS）
-------------------------------------------- */
.btn { ... }
    1. 结合 CSS 变量：Sass 变量（编译时生效）和 CSS 变量（运行时生效）结合使用，兼顾灵活性和动态性：
      // 定义Sass变量
$primary: #0d6efd;

:root {
  --primary: #{$primary};  // 转为CSS变量
}

.btn {
  background: var(--primary);  // 使用CSS变量
}
