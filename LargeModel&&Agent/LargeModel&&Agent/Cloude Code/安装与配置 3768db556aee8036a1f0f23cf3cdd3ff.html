安装

Powershell

irm https://claude.ai/install.ps1 | iex

如果不行

# 创建目录
New-Item -ItemType Directory -Path $env:USERPROFILE\.local\bin -Force
# 配置代理（7890）
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
# 下载CLI程序
Invoke-WebRequest -Uri "https://downloads.claude.ai/claude-code-releases/2.1.162/win32-x64/claude.exe" -OutFile "$env:USERPROFILE\.local\bin\claude.exe"

或者直接下载对应claude.exe，手动添加

# 先检查exe在不在
Test-Path $env:USERPROFILE\.local\bin\claude.exe
# 先直接运行 exe
& "$env:USERPROFILE\.local\bin\claude.exe" --version

添加环境

$env:PATH += ";$env:USERPROFILE\.local\bin"
claude --version

在json文件中添加绕过验证

找到C盘下，claude的json配置文件，路径为: C:\Users\username\.claude.json ,添加如下代码：

"hasCompletedOnboarding":true

再次执行claude命令可以正常启动了，但是无法使用（没有付费，模型用不了），我们接入国产大模型

选择阿里云百练

流程如下：

  1. 去官网注册账号，创建API-Key
  1. 修改环境变量（cmd环境）
    • 设置Key：setx ANTHROPIC_API_KEY "Your Dashscope api key"
    • 设置URL：setx ANTHROPIC_BASE_URL "https://dashscope.aliyuncs.com/apps/anthropic"
    • 设置模型：setx ANTHROPIC_MODEL "glm-5"

    （ps环境）

    # 替换成你的阿里云Dashscope真实KEY sk-xxxx
[Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY","sk-真实密钥","User")
# 阿里云Anthropic兼容接口地址
[Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL","https://dashscope.aliyuncs.com/compatible-mode/v1","User")
[Environment]::SetEnvironmentVariable("ANTHROPIC_MODEL","glm-5","User")
  1. 检测参数是否设置成功（cmd环境）
    • echo %ANTHROPIC_API_KEY%
    • echo %ANTHROPIC_BASE_URL%
    • echo %ANTHROPIC_MODEL%

    （ps环境）

    $env:ANTHROPIC_API_KEY
$env:ANTHROPIC_BASE_URL
$env:ANTHROPIC_MODEL
claude

报错解决

报错：There's an issue with the selected model (glm-5). It may not exist or you may not have access to it. Run /model to
pick a different model.

报错根源：Claude CLI 客户端本地硬编码校验模型名，不认百炼glm-5，接口侧百炼完全支持 glm-5

# 开启开发环境，放开自定义第三方模型校验（关键）
[Environment]::SetEnvironmentVariable("NODE_ENV","development","User")
# 补充兜底变量绕过内置模型白名单
[Environment]::SetEnvironmentVariable("CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC","1","User")
# 修正百炼CodingPlan专属Anthropic接口地址（你之前地址通用版不支持glm5，必须coding专用url）
[Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL","https://coding.dashscope.aliyuncs.com/apps/anthropic","User")
# 锁定glm-5不变
[Environment]::SetEnvironmentVariable("ANTHROPIC_MODEL","glm-5","User")
[Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY","sk-7408fff38da743f3aff0fd2ec012e017","User")

运行依旧报错

Claude CLI前端硬编码内置官方模型白名单，启动时在校验glm-5不在 Claude 官方模型列表直接拦截报错，实际接口百炼 glm-5 可用，只是客户端本地校验拦截。

  1. 打开路径：C:\Users\username\.claude\settings.json（没有就新建）写入配置（替换 sk 密钥、BASE_URL）
    {
  "env": {
    "ANTHROPIC_BASE_URL": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "ANTHROPIC_AUTH_TOKEN": "sk-7408fff38da743f3aff0fd2ec012e017",
    "ANTHROPIC_MODEL": "glm-5"
  },
  "skipModelValidation": true,
  "hasCompletedOnboarding": true
}
  1. 再修改上层配置： C:\Users\hpc\.claude.json （前面设置过）
    {"hasCompletedOnboarding":true}
  1. 删掉冲突环境

    删除多余 ANTHROPIC_AUTH_TOKEN

    [Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN",$null,"User")
