详细配置

传统 SSM = Spring + SpringMVC + MyBatis
SpringBoot 自动装配，不用写大量 XML 配置，一键整合三者，是现在企业主流写法。

一、整体架构对应

  1. Spring：IoC容器、事务管理、Bean管理、AOP
  1. SpringMVC：Web层，接收请求、参数解析、视图/JSON响应
  1. MyBatis：持久层，数据库CRUD操作

SpringBoot 自动帮你配置 SpringMVC DispatcherServlet、字符编码过滤器、视图解析器、事务管理器等。

二、创建项目 & 引入依赖（Maven）

# MyBatis 配置
mybatis:
  # mapper xml 文件路径
  mapper-locations: classpath:mapper/*.xml
  # 实体类别名包，xml中可直接写类名不用全限定名
  type-aliases-package: com.example.ssm.entity
  configuration:
    # 开启驼峰命名自动映射：user_name -> userName

    # 打印SQL日志
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

# 事务默认自动开启，无需额外配置

五、启动类（核心注解）

  • code
    package com.example.ssm;

import org.mybatis.spring.annotation.MapperScan;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

// 开启SpringBoot自动装配（Spring+SpringMVC自动配置）
@SpringBootApplication
// 扫描Mapper接口，代替每个Mapper上加@Repository
@MapperScan("com.example.ssm.mapper")
public class SsmApplication {
    public static void main(String[] args) {
        SpringApplication.run(SsmApplication.class, args);
    }
}
  • @SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan
  • @MapperScan：批量扫描MyBatis Mapper接口，注册到Spring容器

六、分层代码实现（结合你之前 Lombok 知识点）

1. Entity 实体 User.java（@Builder、@Singular、@Slf4j）

  • entity/User.java
    package com.example.ssm.entity;

import lombok.AllArgsConstructor;
import lombok.Builder;
import lombok.Data;
import lombok.NoArgsConstructor;
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private Long id;
    private String username;
    private Integer age;
    private String email;
}

2. Mapper 接口 UserMapper.java（MyBatis）

  • mapper/UserMapper.java
    package com.example.ssm.mapper;

import com.example.ssm.entity.User;
import org.apache.ibatis.annotations.Param;

public interface UserMapper {
    User selectById(@Param("id") Long id);
    int insert(User user);
}

3. Mapper 映射文件 resources/mapper/UserMapper.xml

  • resources/mapper/UserMapper.xml
    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
"<http://mybatis.org/dtd/mybatis-3-mapper.dtd>">
<mapper namespace="com.example.ssm.mapper.UserMapper">

    <select id="selectById" resultType="User">
        select id,username,age,email from user where id = #{id}
    </select>

    <insert id="insert">
        insert into user(username,age,email)
        values(#{username},#{age},#{email})
    </insert>
</mapper>

4. Service 业务层（Spring 事务控制）

接口 UserService

  • service/UserService.java
    package com.example.ssm.service;

import com.example.ssm.entity.User;

public interface UserService {
    User getUserById(Long id);
    int addUser(User user);
}

实现类 UserServiceImpl

  • service/impl/UserServiceImpl.java
    package com.example.ssm.service.impl;

import com.example.ssm.entity.User;
import com.example.ssm.mapper.UserMapper;
import com.example.ssm.service.UserService;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import javax.annotation.Resource;

@Service
public class UserServiceImpl implements UserService {

    @Resource
    private UserMapper userMapper;

    @Override
    public User getUserById(Long id) {
        return userMapper.selectById(id);
    }

    @Override
    @Transactional // 声明式事务，Spring提供
    public int addUser(User user) {
        return userMapper.insert(user);
    }
}

5. Controller 控制层（SpringMVC，接收HTTP请求）

  • controller/Controller.java
    package com.example.ssm.controller;

import com.example.ssm.entity.User;
import com.example.ssm.service.UserService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import javax.annotation.Resource;

@Slf4j
@RestController
@RequestMapping("/user")
public class UserController {

    @Resource
    private UserService userService;

    @GetMapping("/{id}")
    public User getById(@PathVariable Long id) {
        log.info("查询用户id:{}", id);
        return userService.getUserById(id);
    }
}

七、SSM 各司其职总结

  1. Spring
    • @Service/@Controller/@Mapper 管理所有Bean，IoC依赖注入
    • @Transactional 声明式事务、AOP切面
  1. SpringMVC
    • @RestController 接收前端请求
    • 参数绑定、全局异常、JSON返回、拦截器
  1. MyBatis
    • Mapper接口+XML，操作数据库，ORM映射
    • @MapperScan 整合进Spring容器

八、配套你前面知识点拓展

  1. MyBatis逆向工程 MBG：自动生成 Entity、Mapper、Mapper.xml，不用手写表映射
  1. Lombok：@Data @Builder @Slf4j @Singular 简化实体冗余代码
  1. 事务场景：多表更新加 @Transactional 保证原子性

九、传统SSM vs SpringBoot整合SSM区别

  1. 原生SSM需要：spring.xml + springmvc.xml + mybatis.xml + web.xml 一大堆配置
  1. SpringBoot SSM：零XML配置，自动装配，仅yml配置+启动类，开发效率极高
