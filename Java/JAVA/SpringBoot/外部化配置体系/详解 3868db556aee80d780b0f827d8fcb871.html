详解

一、YAML 基础概述

1. 全称与定位

YAML = YAML Ain’t Markup Language（YAML 不是标记语言）

  • 诞生：2001 年，专为配置文件设计
  • 后缀：.yaml / .yml 两种都通用，无本质区别
  • 对比 XML / JSON：
    • JSON：紧凑、适合程序交互，嵌套多可读性差
    • XML：标签冗余、体积大
    • YAML：缩进结构、极简、可读性极强，配置文件首选（Docker、K8s、SpringBoot、GitLab CI、Ansible、Prometheus 全部标配）

2. 核心设计思想

  1. 用缩进表示层级关系（不能用 Tab，只能空格）
  1. 大量省略括号、引号、逗号
  1. 天然兼容 JSON，YAML 可以直接转 JSON

3. 基础硬性语法规则（重中之重，踩坑最多）

  1. 缩进必须是空格，禁止 Tab 制表符
    行业惯例：2 个空格缩进一级，也可用 4 空格，整个文件缩进规则必须统一
  1. 冒号 : 后面必须跟一个空格，格式 key: value
    ❌ name:张三
    ✅ name: 张三
  1. 短横线 后面必须加空格，代表数组元素
  1. # 是单行注释，不支持多行注释
  1. 大小写敏感：Name 和 name 是两个不同键
  1. 字符串默认可以不加引号；特殊字符场景才需要单/双引号
  1. 不能用 \ 做转义（双引号除外）

二、YAML 五种核心数据类型（逐类详解+示例）

类型1：标量（单个值：字符串、数字、布尔、null、日期）

1. 字符串 String

三种写法：

# 写法1：默认不写引号（最常用）
username: zhangsan
address: 北京市朝阳区

# 写法2：单引号 ' '
# 特点：原样输出，不会解析转义字符 \n \t
str1: 'hello\nworld'

# 写法3：双引号 " "
# 特点：识别转义字符 \n换行、\t制表符、\"引号
str2: "hello\nworld"

# 字符串包含冒号/井号必须加引号
special: "<http://www.xxx.com>"
note: "内容#带井号的文字"

2. 数字（整数、浮点数）

age: 25          # 整数
price: 99.99     # 浮点数
big_num: 1000000

3. 布尔值 true / false

enable: true
is_delete: false
# 大小写都识别 True/TRUE / False/FALSE
switch: TRUE

4. 空值 null

三种等价写法：

val1: null
val2: ~
val3:

5. 日期时间（原生支持）

create_date: 2026-06-21
time: 2026-06-21T15:30:00

类型2：对象（键值对，映射 Map / Dictionary）

用缩进表示子属性层级

# 方式一：多行缩进写法（最常用）
user:
  name: 李四
  age: 22
  gender: 男

# 方式二：行内紧凑写法（JSON 风格）
user: {name: 李四, age: 22, gender: 男}

对应 JSON：

{"user":{"name":"李四","age":22,"gender":"男"}}

类型3：数组 / 列表 List（序列 Sequence）

写法1：短横线 开头（最常用）

hobbies:
  - 篮球
  - 游泳
  - 看书

写法2：行内数组 []

hobbies: [篮球, 游泳, 看书]

对象数组（数组元素是对象，K8s 高频写法）

students:
  - name: 小明
    score: 90
  - name: 小红
    score: 95

类型4：多行字符串（| 与 > 两大符号，极易混淆）

1. | 保留换行（原样换行）

content: |
  第一行文字
  第二行文字
  第三行文字

实际值：第一行文字\n第二行文字\n第三行文字\n

2. > 折叠换行（换行转为空格，末尾保留一个换行）

content: >
  第一行文字
  第二行文字
  第三行文字

实际值：第一行文字 第二行文字 第三行文字\n

后缀 去除末尾换行

|-、>- 会删除字符串最后的换行符

text: |-
  aaa
  bbb
# 结果：aaa\nbbb（末尾无换行）

类型5：锚点 & 引用 * 、合并 <<（高级复用，K8s 高频）

解决重复配置冗余问题

  1. &别名：定义锚点（模板）
  1. 别名：引用锚点
  1. <<: *别名：合并对象所有键

示例1：基础锚点复用

# 定义锚点 base_config
base_config: &base
  host: 127.0.0.1
  port: 3306

mysql:
  <<: *base    # 继承 base 所有配置
  database: test_db

redis:
  <<: *base
  port: 6379

等价展开后：

mysql:
  host: 127.0.0.1
  port: 3306
  database: test_db

示例2：数组锚点

default_tags: &tags
  - dev
  - test

server1:
  tags: *tags
server2:
  tags: *tags

三、引号深度详解（单引号 vs 双引号 vs 无引号）

写法转义字符用途
不加引号不解析转义 \n普通字符串首选
单引号 ' '完全原样，\ 只是普通字符包含 \、特殊符号、避免自动类型转换
双引号 " "识别 \n \t \" \\ 转义需要换行、转义场景

特殊场景必须加引号：

  1. 字符串以 [ { ' " 开头
  1. 包含 # : 等符号
  1. 字符串是 true/false/null，想要强制存为字符串而非布尔/空值
# 错误：会识别成布尔
flag: true
# 强制字符串
flag: "true"

四、YAML 特殊语法

1. 多行键（问号 ? 复杂键名）

键名包含空格、冒号、特殊字符时使用：

? user name
: 张三

? <http://demo.com>
: 网站地址

2. 分段 -- 分割多个 YAML 文档

一个文件存放多个独立配置，K8s 批量创建资源常用

---
apiVersion: v1
kind: Pod
metadata:
  name: pod1

---
apiVersion: v1
kind: Pod
metadata:
  name: pod2

... 可选，代表文档结束标记。

3. 类型强制转换 !!

手动指定数据类型，防止自动推断：

num_str: !!str 123    # 强制字符串 "123"
bool_str: !!str true
int_val: !!int "456"

常用类型：!!str、!!int、!!float、!!bool、!!null

五、常见易错坑（避坑手册，必看）

  1. 混用 Tab 和空格 → 解析直接报错，编辑器设置全部用空格
  1. key:值 冒号后面没空格 → 语法错误
  1. 元素 短横线后面没空格 → 数组解析失败
  1. 缩进层级混乱，上下缩进空格数量不一致
  1. 多行字符串 | > 用反混淆
  1. 字符串是 true/false/null/数字 不加引号，被自动转成非字符串类型
  1. 不能写多行注释，# 只能单行
  1. YAML 不允许末尾多余逗号（JSON 允许）

六、YAML 与 JSON 互相转换示例

JSON

{
  "project": "yaml-demo",
  "version": "1.0",
  "author": {
    "name": "老王",
    "age": 28
  },
  "tags": ["java", "yaml", "config"]
}

等价 YAML

project: yaml-demo
version: 1.0
author:
  name: 老王
  age: 28
tags:
  - java
  - yaml
  - config

七、主流编程语言解析 YAML 实操示例

1. Python（依赖 pyyaml）

安装：pip install pyyaml

import yaml

# 读取 yaml 文件
with open("config.yml", "r", encoding="utf-8") as f:
    data = yaml.safe_load(f)

print(data)

# 写入 yaml
data = {"name": "test", "list": [1,2,3]}
with open("out.yml", "w", encoding="utf-8") as f:
    yaml.dump(data, f, sort_keys=False, allow_unicode=True)

2. Java（SnakeYAML / SpringBoot 内置解析）

SpringBoot application.yml 自动加载，底层 SnakeYAML 解析；
手动依赖 SnakeYAML 即可序列化反序列化。

3. Go：gopkg.in/yaml.v3

4. JavaScript/NodeJS：js-yaml
