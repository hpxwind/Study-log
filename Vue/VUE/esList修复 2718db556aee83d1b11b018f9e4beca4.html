esList修复

ESLint插件用于自动查找语法中的格式问题

要让 ESLint 自动纠正代码中的语法错误和风格问题，需要结合 VS Code 配置 和 ESLint 规则 实现

  • 确保依赖和插件的安装

    确保已安装 ESLint 及相关解析器（如 Vue 项目需要 vue-eslint-parser、@babel/eslint-parser 等）：

    # 安装核心依赖
npm install eslint --save-dev
# Vue 项目额外安装
npm install vue-eslint-parser @babel/eslint-parser eslint-plugin-vue --save-dev

    VS Code 插件：安装官方 ESLint 插件（Microsoft 发布），用于在编辑器中集成 ESLint 功能。

  • 配置ESLint自动修复规则

    在 ESLint 配置文件（.eslintrc.js 或 .eslintrc.json）中，确保需要自动修复的规则被启用。

    大部分 代码风格类规则（如缩进、引号、分号）支持自动修复

    逻辑错误类规则（如未定义变量）无法自动修复

    示例.eslintrc.js：

    module.exports = {
  root: true,
  env: { node: true },
  extends: [
    'plugin:vue/essential', // Vue 基础规则
    'eslint:recommended'    // ESLint 推荐规则
  ],
  parserOptions: {
    parser: '@babel/eslint-parser'
  },
  rules: {
    // 支持自动修复的规则示例
    'indent': ['error', 2], // 缩进 2 空格（自动修复）
    'quotes': ['error', 'single'], // 单引号（自动修复）
    'semi': ['error', 'always'], // 强制分号（自动修复）
    'no-trailing-spaces': 'error', // 禁止行尾空格（自动修复）
    'space-before-function-paren': ['error', 'never'], // 函数括号前无空格（自动修复）
    // 其他规则...
  }
};
  • 配置VS Code自动保存时候修复

    通过 VS Code 的设置，实现 保存文件时自动触发 ESLint 修复

    1. 打开 VS Code 配置文件：
      • 方式 1：按 Ctrl+,（或 Cmd+,）打开设置，搜索 codeActionsOnSave。
      • 方式 2：在项目根目录创建 .vscode/settings.json（仅作用于当前项目）。
    1. 添加自动修复配置：
    // .vscode/settings.json
{
  // 保存时自动修复 ESLint 问题
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  // 确保 ESLint 对 Vue 文件生效
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    "vue"
  ]
}
  • 手动触发修复（可选）

    如果需要手动修复当前文件或项目：

    1. 修复当前文件：
      • 按 Ctrl+Shift+P（或 Cmd+Shift+P）打开命令面板。
      • 输入 ESLint: Fix all auto-fixable Problems 并执行。
    1. 修复整个项目：在 package.json 中添加脚本：
    {
  "scripts": {
    "lint:fix": "eslint --fix src/" // 修复 src 目录下的所有文件
  }
}

    执行命令：

    npm run lint:fix
