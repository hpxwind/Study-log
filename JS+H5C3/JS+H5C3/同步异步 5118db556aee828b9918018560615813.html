同步异步

  • 同步与异步
    程序的运行是同步的（同步不意味着所有步骤同时运行，而是指步骤在一个控制流序列中按顺序执行）。
而异步的概念则是不保证同步的概念，也就是说，一个异步过程的执行将不再与原有的序列有顺序关系。
简单来理解就是：同步按你的代码顺序执行，异步不按照代码顺序执行，异步的执行效率更高。

    回调函数
	回调函数就是一个函数，它是在我们启动一个异步任务的时候就告诉它：等你完成了这个任务之后要干什么。这样一来主线程几乎不用关心异步任务的状态了，他自己会善始善终。
如：程序等待3秒后自动执行 print 函数。
setTimeout(print, 3000);
//setTimeout 就是一个消耗时间较长（3 秒）的过程，它的第一个参数是个回调函数，第二个参数是毫秒数，这个函数执行之后会产生一个子线程，子线程会等待 3 秒，然后执行回调函数 "print"
可以简写两个函数如下：
setTimeout(function () {
    document.getElementById("demo").innerHTML="RUNOOB!";
}, 3000);
-----------------------------------------------------------------------------------------
注意：既然 setTimeout 会在子线程中等待 3 秒，在 setTimeout 函数执行之后主线程并没有停止，所以：
setTimeout(function () {
    document.getElementById("demo1").innerHTML="RUNOOB-1!";  // 三秒后子线程执行
}, 3000);
document.getElementById("demo2").innerHTML="RUNOOB-2!";      // 主线程先执行
-----------------------------------------------------------------------------------------

  • 回调地狱

    当多个异步操作需要按顺序执行时，嵌套多层回调函数导致代码可读性差、维护困难的现象。

    JavaScript 是单线程语言，异步操作（如网络请求、定时器）不会阻塞主线程，而是通过回调函数在操作完成后触发后续逻辑。

    当多个异步操作存在依赖关系（如 “先请求 A，再用 A 的结果请求 B，再用 B 的结果请求 C”）时，就需要嵌套回调，层数越多，代码越像 “金字塔”，形成回调地狱。

    假设需要实现以下需求：

    1. 先获取用户 ID（异步）
    1. 用用户 ID 获取用户信息（异步）
    1. 用用户信息获取用户订单（异步）

    用传统回调函数实现会是这样：

    // 模拟异步请求函数
function getUserID(callback) {
  setTimeout(() => {
    const id = 1001;
    callback(id); // 第一个回调：获取ID后执行
  }, 1000);
}

function getUserInfo(id, callback) {
  setTimeout(() => {
    const info = { id: id, name: "Alice" };
    callback(info); // 第二个回调：获取信息后执行
  }, 1000);
}

function getOrders(info, callback) {
  setTimeout(() => {
    const orders = [{ id: 1, product: "手机" }, { id: 2, product: "电脑" }];
    callback(orders); // 第三个回调：获取订单后执行
  }, 1000);
}

// 按顺序执行异步操作（回调嵌套）
getUserID((id) => {
  console.log("用户ID：", id);
  getUserInfo(id, (info) => {
    console.log("用户信息：", info);
    getOrders(info, (orders) => {
      console.log("用户订单：", orders);
      // 若还有更多步骤，会继续嵌套...
    });
  });
});

  • Promise+then()

    本质上是一个容器，保存着某个未来才会结束的事件（通常是异步操作）的结果。

    状态：Promise 有三种状态，且状态一旦改变就不可逆：

    pending（进行中）：初始状态，既不是成功也不是失败。

    fulfilled（已成功）：异步操作完成并返回结果。

    rejected（已失败）：异步操作出错或被拒绝。

    作用：将异步操作以同步操作的流程表达出来，避免回调嵌套；提供统一的接口，让异步操作更容易控制。

    • 创建promise实例

      使用 new Promise() 构造函数创建 Promise 实例，接收一个函数作为参数，该函数有两个参数 resolve 和 reject（均为函数）：

      resolve：将状态从 pending 改为 fulfilled，并将异步操作的结果作为参数传递出去。

      reject：将状态从 pending 改为 rejected，并将错误信息作为参数传递出去。

      const promise = new Promise((resolve, reject) => {
  // 模拟异步操作（如网络请求、定时器）
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve("操作成功！"); // 成功时调用 resolve
    } else {
      reject(new Error("操作失败！")); // 失败时调用 reject
    }
  }, 1000);
});
    • 处理promise结果

      通过 then() 处理成功的结果，catch() 处理失败的原因：

      then()：接收两个可选参数，分别是 fulfilled 和 rejected 状态的回调函数。

      catch()：专门处理 rejected 状态的回调，等价于 then(null, onRejected)。

      promise
  .then((result) => {
    // 处理成功的结果（resolve 传递的值）
    console.log(result); // 输出：操作成功！
  })
  .catch((error) => {
    // 处理失败的原因（reject 传递的错误）
    console.error(error.message); // 输出：操作失败！（若 success 为 false）
  });

      finally()：无论 Promise 状态是 fulfilled 还是 rejected，都会执行（ES2018 新增），常用于清理操作（如关闭加载动画）。

      new Promise((resolve, reject) => {
  reject(new Error("主动报错"));
})
  .then((res) => {
    console.log(res); // 不会执行
  })
  .catch((err) => {
    console.error("捕获错误：", err.message); // 输出：捕获错误：主动报错
  })
  .finally(() => {
    console.log("无论成功失败都会执行"); // 输出：无论成功失败都会执行
  });

      then()、catch()、finally() 中的回调函数都是异步执行的，会在同步代码执行完后再执行。

    • promise其他方法

      1、Promise.race()

      Promise.race() 接收一个 Promise 数组，只要有一个 Promise 状态改变（成功或失败），就立即返回该结果。

      const promise1 = new Promise((resolve) => setTimeout(() => resolve(1), 1000));
const promise2 = new Promise((resolve) => setTimeout(() => resolve(2), 500));

Promise.race([promise1, promise2])
  .then((result) => {
    console.log(result); // 输出：2（因为 promise2 更快完成）
  });

      2、Promise.all()

      Promise.all() 接收一个 Promise 数组，当所有 Promise 都成功时，返回一个包含所有结果的数组；只要有一个失败，就立即返回该失败的原因。

      const promise1 = Promise.resolve(1);
const promise2 = Promise.resolve(2);
const promise3 = Promise.resolve(3);

Promise.all([promise1, promise2, promise3])
  .then((results) => {
    console.log(results); // 输出：[1, 2, 3]（所有成功）
  })
  .catch((err) => {
    console.error(err); // 若有一个失败，输出该错误
  });
  • Promise 常见问题

    1. 避免 Promise 嵌套

    2. 总是处理拒绝情况

    忘记处理 Promise 的拒绝会导致"未捕获的 Promise 拒绝"错误。总是使用 .catch() 或 try/catch（在 async/await 中）来处理错误。

    3. Promise 不是可取消的

    一旦创建，Promise 就无法取消。如果需要取消功能，可以考虑使用 AbortController 或其他模式。

    4. 回答常见的问题（FAQ）

    Q: then、catch 和 finally 序列能否顺序颠倒？

    A: 可以，效果完全一样。但不建议这样做，最好按 then-catch-finally 的顺序编写程序。

    Q: 除了 then 块以外，其它两种块能否多次使用？

    A: 可以，finally 与 then 一样会按顺序执行，但是 catch 块只会执行第一个，除非 catch 块里有异常。所以最好只安排一个 catch 和 finally 块。

    Q: then 块如何中断？

    A: then 块默认会向下顺序执行，return 是不能中断的，可以通过 throw 来跳转至 catch 实现中断。

    Q: 什么时候适合用 Promise 而不是传统回调函数？

    A: 当需要多次顺序执行异步操作的时候，例如，如果想通过异步方法先后检测用户名和密码，需要先异步检测用户名，然后再异步检测密码的情况下就很适合 Promise。

    Q: Promise 是一种将异步转换为同步的方法吗？

    A: 完全不是。Promise 只不过是一种更良好的编程风格。

    Q: 什么时候我们需要再写一个 then 而不是在当前的 then 接着编程？

    A: 当你又需要调用一个异步任务的时候。
