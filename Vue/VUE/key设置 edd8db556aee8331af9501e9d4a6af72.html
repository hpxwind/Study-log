key设置

在 Vue 中，key 是一个特殊的属性，主要用于辅助 Vue 的虚拟 DOM（Virtual DOM）实现高效的节点更新。它的核心作用是标识节点的唯一性，帮助 Vue 识别哪些节点发生了变化，从而避免不必要的 DOM 操作，提升渲染性能。

  • key的工作原理

    Vue 的虚拟 DOM 采用「diff 算法」对比新旧节点，核心逻辑是同级节点对比。当对比列表节点时，key 是关键依据：

    1. 无 key 时：Vue 会默认通过「节点顺序」对比，尝试复用尽可能多的节点。这可能导致：
      • 节点被错误复用（即使内容不同），引发状态混乱（如表单输入值错乱）。
      • 当列表排序、新增 / 删除元素时，大量节点被销毁重建，性能低下。
    1. 有 key 时：Vue 会根据 key 的值精准匹配新旧节点：
      • 若 key 相同，认为是「同一个节点」，仅更新差异内容（如文本、属性）。
      • 若 key 不同，直接销毁旧节点并创建新节点。
  • v-for 循环中必须设置 key

    这是 key 最常见的场景。推荐使用唯一标识（如 ID）作为 key，而非索引（index）。

    如果设置索引值（index）若删除列表中间元素，后续元素的索引会变化，导致 Vue 误判节点「已改变」，引发输入框值错乱（复用了错误的节点状态）。

    <template>
  <ul>
    <!-- 用索引作为 key，当列表排序/删除时会出问题 -->
    <li v-for="(item, index) in list" :key="index">
      {{ item.name }} <input type="text">
    </li>
  </ul>
</template>

    使用ID

    即使列表排序或增删元素，key 仍能精准匹配节点，避免状态复用错误。

    <template>
  <ul>
    <!-- 用唯一 ID 作为 key，稳定且唯一 -->
    <li v-for="item in list" :key="item.id">
      {{ item.name }} <input type="text">
    </li>
  </ul>
</template>

<script>
export default {
  data() {
    return {
      list: [
        { id: 1, name: 'A' },
        { id: 2, name: 'B' },
        { id: 3, name: 'C' }
      ]
    }
  }
}
</script>
  • 强制元素/组件重新渲染

    当需要强制替换一个元素（而非更新）时，可通过改变 key 的值触发重新渲染，例如：

    • 重置组件状态（避免复用旧实例）。
    • 触发过渡动画（旧节点销毁、新节点创建时执行过渡）。
    <template>
  <div>
    <!-- 点击按钮时，key 变化，强制重新创建组件 -->
    <MyComponent :key="componentKey" />
    <button @click="resetComponent">重置组件</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      componentKey: 0 // 初始 key
    }
  },
  methods: {
    resetComponent() {
      this.componentKey += 1; // 改变 key，触发重新渲染
    }
  }
}
</script>
  • 注意事项
    1. 唯一性：同一父节点下的子节点 key 必须唯一（不同父节点下可重复），否则会导致渲染错误。
    1. 稳定性：key 的值应保持稳定（如 ID），避免使用动态生成的临时值（如 Math.random()），否则会导致节点频繁销毁重建，性能下降。
    1. 不滥用：非列表场景（如单个元素）无需设置 key，多余的 key 会增加 diff 算法的比对成本。
    1. Vue2 与 Vue3 的差异：
      • Vue2 中，key 若为 undefined，会默认使用索引作为隐式 key。
      • Vue3 中，v-for 必须显式设置 key，否则会报警告，且不再默认使用索引。
