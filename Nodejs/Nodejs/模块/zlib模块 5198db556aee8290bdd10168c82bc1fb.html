zlib模块

Node.js 的 zlib 模块是用于数据压缩和解压缩的内置模块，支持多种压缩算法（如 gzip、deflate、brotli 等），广泛应用于处理 HTTP 压缩响应、文件压缩、数据传输优化等场景。

  • 核心功能
    • 数据压缩 / 解压缩：对字符串、Buffer 或流（Stream）进行压缩和解压缩
    • HTTP 压缩：配合 http 模块实现 gzip 等压缩响应（减少传输体积）
    • 文件压缩：压缩日志文件、静态资源等
    • 数据传输优化：在网络传输中减小数据体积，提升效率
  • 常用压缩算法
    算法压缩方法解压缩方法特点
    gzipzlib.createGzip()zlib.createGunzip()常用压缩格式，压缩率较高
    deflatezlib.createDeflate()zlib.createInflate()HTTP 常用压缩算法，与 gzip 类似
    deflateRawzlib.createDeflateRaw()zlib.createInflateRaw()无头部信息的 deflate，体积略小
    brotlizlib.createBrotliCompress()zlib.createBrotliDecompress()较新算法，压缩率优于 gzip（Node.js v11+ 支持）
  • 基础压缩/解压方法（同步方法）

    适用于小型数据（一次性加载到内存）:

    const zlib = require('zlib');
const data = '这是一段需要压缩的文本数据，用于测试 zlib 模块的基本功能';

// 压缩（gzip 算法）
const compressed = zlib.gzipSync(data);
console.log('压缩后长度：', compressed.length);

// 解压缩
const decompressed = zlib.gunzipSync(compressed).toString();
console.log('解压缩内容：', decompressed);

    同步方法还有：deflateSync()/inflateSync()、brotliCompressSync()/brotliDecompressSync() 等。

  • 流式压缩/解压缩（推荐）

    对于大文件或流数据（如 HTTP 响应、大文件读写），应使用流式方法（非阻塞，内存友好）：

    const fs = require('fs');
const zlib = require('zlib');

// 创建可读流（源文件）→ 压缩流 → 可写流（压缩后的文件）
fs.createReadStream('large-file.txt')
  .pipe(zlib.createGzip()) // 压缩
  .pipe(fs.createWriteStream('large-file.txt.gz'))
  .on('finish', () => {
    console.log('文件压缩完成');
  });

    解压缩：

    fs.createReadStream('large-file.txt.gz')
  .pipe(zlib.createGunzip()) // 解压缩
  .pipe(fs.createWriteStream('large-file-restored.txt'))
  .on('finish', () => {
    console.log('文件解压缩完成');
  });
  • http响应压缩
    const http = require('http');
const zlib = require('zlib');
const fs = require('fs');

http.createServer((req, res) => {
  // 读取文件内容
  const raw = fs.createReadStream('large-response.txt');
  
  // 检查客户端是否支持压缩（Accept-Encoding 头）
  const acceptEncoding = req.headers['accept-encoding'];
  
  // 如果客户端支持 gzip 压缩
  if (acceptEncoding && acceptEncoding.includes('gzip')) {
    res.writeHead(200, { 'Content-Encoding': 'gzip' });
    raw.pipe(zlib.createGzip()).pipe(res); // 压缩后返回
  } else {
    res.writeHead(200);
    raw.pipe(res); // 不压缩直接返回
  }
}).listen(3000);
  • 压缩选项配置
    // gzip 压缩选项
const gzipOptions = {
  level: 6, // 压缩级别（1-9，默认6，级别越高压缩率越好但速度越慢）
  windowBits: 15 // 窗口大小（影响压缩率）
};

// 使用选项压缩
fs.createReadStream('file.txt')
  .pipe(zlib.createGzip(gzipOptions))
  .pipe(fs.createWriteStream('file.txt.gz'));

    Brotli 算法的压缩选项更丰富（如 quality 控制压缩质量）：

    const brotliOptions = {
  quality: 6 // 压缩质量（0-11，默认6）
};

zlib.brotliCompressSync(data, brotliOptions);
  • 事件与错误处理

    压缩 / 解压缩流会触发常见的流事件，需处理错误：

    const gzip = zlib.createGzip();

gzip.on('error', (err) => {
  console.error('压缩错误：', err);
});

gzip.on('end', () => {
  console.log('压缩完成');
});

// 处理流数据
fs.createReadStream('file.txt').pipe(gzip).pipe(fs.createWriteStream('file.txt.gz'));
