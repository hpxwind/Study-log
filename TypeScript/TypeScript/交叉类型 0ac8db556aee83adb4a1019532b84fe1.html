交叉类型

交叉类型（& 符号）是 TypeScript 中用于合并多个类型的核心特性，它表示 “同时具备所有参与合并类型的属性和方法”，本质是 “类型的与（AND）关系”（对比联合类型 | 是 “类型的或（OR）关系”）。

  • 语法定义
    type 交叉类型名 = 类型A & 类型B & 类型C;
    • 含义：交叉类型名 的变量必须同时满足类型 A、类型 B、类型 C 的所有约束（拥有所有类型的属性 / 方法）。
    • 关键：交叉类型是 “合并” 而非 “选择”，不存在 “二选一” 的情况。
  • 合并接口/类型别名

    合并多个接口或类型别名的属性，实例必须包含所有类型的全部属性（包括必选和可选）。

    // 定义两个基础类型
interface User {
  id: number;
  name: string;
}

interface Contact {
  phone: string;
  email?: string; // 可选属性
}

// 交叉类型：同时具备 User 和 Contact 的所有属性
type UserWithContact = User & Contact;

// 合法实例：包含 User 的所有必选属性 + Contact 的必选属性（可选属性可省略）
const user1: UserWithContact = {
  id: 1,
  name: "张三",
  phone: "13800138000"
};

// 合法实例：包含所有属性（包括可选属性）
const user2: UserWithContact = {
  id: 2,
  name: "李四",
  phone: "13900139000",
  email: "lisi@example.com"
};

// 非法实例：缺少 User 的必选属性 name → 报错
const user3: UserWithContact = {
  id: 3,
  phone: "13700137000" // 报错：Property 'name' is missing
};
  • 合并基本类型

    应用于基本类型（string、number、boolean 等）时，只有当所有类型完全一致时，结果才是该类型；否则结果为 never（无任何值能满足所有基本类型约束）

    // 1. 所有类型一致 → 结果为该类型
type StrAndStr = string & string;
let a: StrAndStr = "hello"; // 合法
// let b: StrAndStr = 123; // 报错：Type 'number' is not assignable to type 'string'

// 2. 类型不一致 → 结果为 never（无值能同时是 string 和 number）
type StrAndNum = string & number;
// let c: StrAndNum = "abc"; // 报错：Type 'string' is not assignable to type 'never'
// let d: StrAndNum = 123; // 报错：Type 'number' is not assignable to type 'never'

// 3. 布尔值交叉 → 结果为 never（无值能同时是 true 和 false）
type TrueAndFalse = true & false; // 类型为 never
  • 合并函数类型

    交叉类型可合并多个函数类型，结果是一个 “同时满足所有函数签名” 的函数 —— 即参数列表取交集（兼容所有函数的参数），返回值取交集。

    // 函数类型1：接收 string 参数，返回 number
type Fn1 = (a: string) => number;
// 函数类型2：接收 string | number 参数，返回 string
type Fn2 = (a: string | number) => string;

// 交叉类型：同时满足 Fn1 和 Fn2 的函数
type CombinedFn = Fn1 & Fn2;

// 合法实现：
// - 参数 a 必须兼容两个函数的参数（Fn1 要求 string，Fn2 要求 string|number → 取交集 string）
// - 返回值必须兼容两个函数的返回值（number & string → never？不，实际是“返回值需同时满足两个类型”，即 never，所以这里用断言绕开演示）
const fn: CombinedFn = (a: string) => {
  return 123 as number & string; // 需要同时满足是字符串和数字
};

// 调用：参数必须是 string（兼容 Fn1 的约束）
fn("hello"); // 合法
// fn(123); // 报错：Argument of type 'number' is not assignable to parameter of type 'string'
    注意：函数交叉类型实际用得较少，
更多用于 “函数重载” 或 “混合函数与对象属性”（如下方混入场景）。
  • 交叉类型+联合类型

    注意优先级

    交叉类型的优先级高于联合类型，可通过括号改变优先级，实现 “更精细的类型约束”。

    // 需求：类型是 (User 且 Admin) 或 (Contact 且 Guest)
interface Admin {
  role: "admin";
}

interface Guest {
  role: "guest";
}

// 错误写法：优先级问题 → (User 且 (Admin 或 Contact)) 且 Guest → 不符合需求
// type MixedType = User & Admin | Contact & Guest;

// 正确写法：用括号明确优先级
type MixedType = (User & Admin) | (Contact & Guest);

// 合法实例1：User + Admin
const adminUser: MixedType = {
  id: 1,
  name: "管理员",
  role: "admin"
};

// 合法实例2：Contact + Guest
const guestContact: MixedType = {
  phone: "13800138000",
  role: "guest"
};
  • 交叉类型处理可选属性与冲突属性

    1.可选属性合并

    若多个类型有同名的可选属性，合并后仍为可选属性；若有一个类型的属性是必选，合并后为必选属性（“必选” 优先级更高）。

    interface A {
  x?: number;
  y: string;
}

interface B {
  x: number; // 必选
  z?: boolean;
}

type C = A & B;
// C 的类型等价于：{ x: number; y: string; z?: boolean }
const c: C = {
  x: 1, // 必选（因 B 中 x 是必选）
  y: "hello"
};

    2.冲突属性合并

    若多个类型有同名但不同类型的属性，合并后该属性的类型为 “所有类型的交叉”（可能是 never）。

    interface D {
  x: string;
}

interface E {
  x: number;
}

type F = D & E;
// F 的 x 属性类型是 string & number → never
const f: F = {
  // x: "abc" → 报错：Type 'string' is not assignable to type 'never'
  // x: 123 → 报错：Type 'number' is not assignable to type 'never'
};

  • 应用：混入（Mixin）模式

    交叉类型是实现 “混入（Mixin）” 的核心 —— 将多个类 / 对象的功能合并到一个类 / 对象中，实现代码复用。

    // 定义两个混入函数（给类添加额外方法）
function withLog<T extends Constructor>(Base: T) {
  return class extends Base {
    log() {
      console.log(`Log: ${JSON.stringify(this)}`);
    }
  };
}

function withDelete<T extends Constructor>(Base: T) {
  return class extends Base {
    delete() {
      console.log(`Deleted: ${this.id}`);
    }
  };
}

// 基础构造函数类型（约束混入的类必须有 id 属性）
type Constructor = new (...args: any[]) => { id: number };

// 基础类
class BaseEntity {
  constructor(public id: number) {}
}

// 交叉类型合并混入：同时具备 BaseEntity + withLog + withDelete 的功能
const Entity = withLog(withDelete(BaseEntity));
type EntityType = InstanceType<typeof Entity>; // 提取实例类型

// 使用
const entity = new Entity(1);
entity.log(); // 调用 withLog 的方法
entity.delete(); // 调用 withDelete 的方法
console.log(entity.id); // 调用 BaseEntity 的属性

交叉类型 vs 联合类型

| 特性 | 交叉类型（&） | 联合类型（|）

|| ------------------- | ---------------------------------------- | ---------------------------------------- ||

逻辑关系 | 与（AND）：同时满足所有类型 | 或（OR）：满足任意一个类型即可 ||

实例化要求 | 必须包含所有类型的全部属性 / 方法 | 只需包含任意一个类型的属性 / 方法 ||

基本类型合并结果 | 类型一致则为该类型，否则 never | 所有类型的集合（如 string | number） ||

属性冲突处理 | 属性类型为交叉（可能是 never） | 属性类型为联合（如 string | number） ||

典型场景 | 混入、扩展类型、合并接口 | 多类型可选（如函数参数多类型） |

type A = { x: number };
type B = { y: string };

// 交叉类型：必须有 x 和 y
type Intersection = A & B;
const inter: Intersection = { x: 1, y: "a" }; // 合法

// 联合类型：有 x 或 y 即可
type Union = A | B;
const union1: Union = { x: 1 }; // 合法
const union2: Union = { y: "a" }; // 合法
const union3: Union = { x: 1, y: "a" }; // 也合法（兼容所有类型）
