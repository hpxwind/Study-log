axios（请求接口）

xios 是一个基于 Promise 的 HTTP 客户端，主要用于浏览器和 Node.js 环境中发送 HTTP 请求。它具有简洁的 API、拦截请求 / 响应、转换数据、取消请求等强大功能，是前端开发中最常用的请求工具之一，尤其在 Vue 项目中被广泛使用

  • 核心特性
    1. 支持 Promise API：可配合 async/await 语法，让异步请求代码更简洁。
    1. 拦截请求和响应：能在请求发送前或响应返回后统一处理（如添加 token、处理错误）。
    1. 转换请求 / 响应数据：自动转换 JSON 数据，也可自定义转换规则。
    1. 取消请求：支持取消正在进行的请求（如页面跳转时取消未完成的请求）。
    1. 自动转换请求头：根据请求数据自动设置 Content-Type（如 application/json）。
    1. 客户端防御 XSRF：支持设置 XSRF 令牌，提升安全性。
  • 基本请求示例
    import axios from 'axios';

// 1. GET 请求（获取数据）
axios.get('/api/user', {
  params: { id: 1 } // URL 参数（会拼接到 URL 后：/api/user?id=1）
})
.then(response => { 
  console.log('请求成功', response.data); // 响应数据在 response.data 中
})
.catch(error => {
  console.log('请求失败', error);
});

// 2. POST 请求（提交数据）
axios.post('/api/user', {
  name: '张三',
  age: 20
})
.then(response => {
  console.log('提交成功', response.data);
})
.catch(error => {
  console.log('提交失败', error);
});

// 3. 通用请求方法（支持所有 HTTP 方法）
axios({
  method: 'put', // 请求方法：get/post/put/delete 等
  url: '/api/user/1', // 请求 URL
  data: { name: '李四' } // 请求体数据（post/put 等方法用）
})
.then(response => { ... })
.catch(error => { ... });
  • 配合async/await使用（推荐）
    async function getUser() {
  try {
    // 等待请求完成，直接获取响应数据
    const response = await axios.get('/api/user', { params: { id: 1 } });
    console.log('用户数据', response.data);
    return response.data;
  } catch (error) {
    console.log('请求失败', error);
    throw error; // 向上层抛出错误
  }
}
  • 在vue中axios的使用

    在 Vue 项目中，通常会对 Axios 进行全局配置或封装，以便在所有组件中统一使用，减少重复代码。

    • 1、全局配置

      通过 axios.defaults 设置全局默认配置（如基础 URL、超时时间等），避免每个请求重复配置。

      // main.js
import Vue from 'vue';
import axios from 'axios';
import App from './App.vue';

// 全局配置
axios.defaults.baseURL = 'https://api.example.com'; // 基础 URL（所有请求会自动拼接在前面）
axios.defaults.timeout = 5000; // 超时时间（5秒）
axios.defaults.headers.common['Authorization'] = 'Bearer ' + localStorage.getItem('token'); // 全局请求头（如 token）
axios.defaults.headers.post['Content-Type'] = 'application/json'; // POST 请求默认 Content-Type

// 将 axios 挂载到 Vue 原型上，方便组件中使用
Vue.prototype.$axios = axios;

new Vue({
  render: h => h(App)
}).$mount('#app');

      在组件中使用：

      <template>
  <div>用户列表</div>
</template>

<script>
export default {
  mounted() {
    // 通过 this.$axios 调用（已包含全局配置）
    this.getUserList();
  },
  methods: {
    async getUserList() {
      try {
        const response = await this.$axios.get('/users'); // 实际请求 URL：https://api.example.com/users
        console.log('用户列表', response.data);
      } catch (error) {
        console.log('获取失败', error);
      }
    }
  }
};
</script>
    • 2、拦截器（请求/响应拦截）

      拦截器用于在请求发送前或响应返回后做统一处理（如添加 token、处理 401 错误），是 Vue 项目中必备的功能。

      创建拦截器（推荐封装为单独文件）：

      新建 src/utils/request.js：

      import axios from 'axios';

// 创建 axios 实例（可自定义配置，避免污染全局 axios）
const request = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 5000
});

// 1. 请求拦截器（请求发送前执行）
request.interceptors.request.use(
  config => {
    // 示例：添加 token 到请求头
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = 'Bearer ' + token;
    }
    return config; // 必须返回 config，否则请求会被拦截
  },
  error => {
    // 请求出错（如网络错误）
    return Promise.reject(error);
  }
);

// 2. 响应拦截器（响应返回后执行）
request.interceptors.response.use(
  response => {
    // 只返回响应数据（简化使用）
    return response.data;
  },
  error => {
    // 处理错误状态码
    if (error.response) {
      switch (error.response.status) {
        case 401:
          // 401：未授权（token 过期或无效），跳转登录页
          localStorage.removeItem('token');
          window.location.href = '/login';
          break;
        case 404:
          console.log('接口不存在');
          break;
        case 500:
          console.log('服务器错误');
          break;
      }
    }
    return Promise.reject(error);
  }
);

export default request; // 导出封装后的实例

      在 Vue 组件中使用封装的 request：

      <template>
  <div>用户详情</div>
</template>

<script>
import request from '@/utils/request'; // 引入封装的请求工具

export default {
  methods: {
    async getUserDetail(id) {
      try {
        // 直接获取响应数据（响应拦截器已处理 response.data）
        const data = await request.get('/user', { params: { id } });
        console.log('用户详情', data);
      } catch (error) {
        console.log('错误', error);
      }
    }
  }
};
</script>
    • 3、处理跨域问题（Vue开发环境）

      在 Vue 开发中，若前端（如 http://localhost:8080）与后端接口（如 https://api.example.com）不在同一域名，会出现跨域错误。可通过 Vue CLI 的代理配置解决。

      配置 vue.config.js：

      module.exports = {
  devServer: {
    proxy: {
      // 匹配所有以 /api 开头的请求
      '/api': {
        target: 'https://api.example.com', // 目标服务器地址
        changeOrigin: true, // 开启跨域（伪装请求来源为目标服务器）
        pathRewrite: {
          '^/api': '' // 去掉 URL 中的 /api 前缀（如 /api/user → 实际请求 /user）
        }
      }
    }
  }
};

      组件中使用：

      // 请求 URL 写 /api/user，会被代理到 https://api.example.com/user
request.get('/api/user', { params: { id: 1 } });
    • 4、vue3中使用axios

      Vue 3 中取消了 Vue.prototype 挂载方式，推荐使用 Provide/Inject 或直接在组件中导入使用。

      示例（Vue 3 + Composition API）：

      <template>
  <div>{{ user.name }}</div>
</template>

<script setup>
import { ref, onMounted } from 'vue';
import request from '@/utils/request';

const user = ref({});

onMounted(async () => {
  try {
    const data = await request.get('/user/1');
    user.value = data;
  } catch (error) {
    console.log(error);
  }
});
</script>
  • 常见问题与解决

    1、POST请求参数格式错误

    若后端需要 form-data 格式（如文件上传），需手动设置请求头并转换数据：

    import qs from 'qs'; // 需安装 qs：npm install qs

// 方式1：使用 qs 转换为 form-data 格式
request.post('/upload', qs.stringify({ name: '张三' }), {
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' }
});

// 方式2：使用 FormData（适合文件上传）
const formData = new FormData();
formData.append('name', '张三');
formData.append('file', file); // file 为文件对象
request.post('/upload', formData, {
  headers: { 'Content-Type': 'multipart/form-data' }
});

    2、取消请求

    用于取消未完成的请求（如切换标签页时）：

    import axios from 'axios';

// 创建取消令牌
const source = axios.CancelToken.source();

// 发送请求时关联令牌
request.get('/user', { cancelToken: source.token })
  .catch(error => {
    if (axios.isCancel(error)) {
      console.log('请求已取消', error.message);
    }
  });

// 取消请求（如在组件销毁时）
source.cancel('用户主动取消请求');

    3、并发请求

    同时发送多个请求，全部完成后再处理：

    async function getMultipleData() {
  const [userRes, listRes] = await Promise.all([
    request.get('/user/1'),
    request.get('/list')
  ]);
  console.log('用户数据', userRes);
  console.log('列表数据', listRes);
}
  • axios二次封装

    在src下创建utils文件夹，定义一个专门接收一个地址来的数据

    这样其他功能文件只需要调用这个文件就可以得到数据了

    src/utils/request.js

    import axios from 'axios'

// axios 二次 封装

// 2. 创建axios对象
const  service = axios.create({
    baseURL: 'https://api.example.com'
})

// 3. 请求拦截器 ==> 前端给后端发送数据 [ 没有到后端 ]
// 拦截器 ==> 在发送请求之前做些什么（ headers给后端传递token ）
service.interceptors.request.use( config => {
    console.log("3")
    return config
}, error => {
    Promise.reject(error)
})


// 4.  相应拦截器 ==> 后端给前端返回数据 [ 前端到后端了 ]
service.interceptors.response.use(
    (response) => {
        console.log("4")
        return response.data
    },
    error => {
        return Promise.reject(new Error(error.response.data))
    }
)

export default service

  • 解耦

    同一个API接口的数据可能被多个功能使用

    在解耦文件中调用二次封装的文件，将不同功能的数据细分，提供一个导出，

    为其他功能文件提供使用

    import request  from '@/utils/request'
// API接口解耦 ==> 模块化
// 因为很多模块都用到了request中的API，所以这里进行功能分解再封装
// 这样是前端接收的时候就方便使用

export function getSliders()  {
    return  request({
        url: '/api/sliders/getSliders'
    })
}

export function course()  {
    return  request({
        url: '/api/sliders/courses'
    })
}
