外部化配置体系

外部化配置是指：将配置信息存储在应用程序代码之外的地方，这样配置信息就可以独立于代码进行管理。这样方便配置的修改，并且修改后不需要重新编译代码，也不需要重新部署项目，只需重启项目。

外部化传递配置的方法，包括但不限于：properties文件，YAML文件，系统环境变量，命令行参数……

外部配置：灵活，易于维护，安全，共享，版本控制

application.properties配置文件加载

application.properties是springboot框架默认配置文件。但不是必须的，对于程序来说提供了一套默认配置（自动配置），如果要改默认配置，需要配置application.properties，其可以放在类路径中，也可以放在项目之外。

springboot启动时候，会按顺序从一下位置加载application.properties配置文件：

  1. file:./config/ ：首先在当前工作目录下的config目录下找
  1. file:./config/application.yml ：如果没有会在当前工作目录下的config目录下application.yml
  1. file:./：如果当前项目config下没有，会在当前工作目录中找
  1. classpath:/config/：如果工作目录中找不到，会从类路径中找，先从/config/目录下
  1. classpath:/：如果类路径/config/下没有，再在类根路径下找
优先级依次1>5，高优先级覆盖低的，所有的配置文件会合并。

如果打包后使用命令行参数（通过--spring.config.location=路径方式指定配置文件）

java -jar xxx.jar --spring.config.location=file:///E:\a\b\application.properties

@Value注解的使用

本质上是<bean id=”springBean” class=”SpringBoot”><property name=”age” value=”20”>

@Value("${key}") // 可以取配置文件里的信息，如果key不存在会报错
@Value("${key: defalut}") // 可以自定义默认值
@Value("${APP_KEY}") // 也可以取操作系统中的环境变量值
@Value可以将application.properties或application.yml文件中的配置信息注入/绑定到java对象的属性上。

配置文件合并

一个项目所有配置文件都集中在application.properties文件中，会导致配置文件不易维护，可以将配置文件编写到不同的文件中，

如：application-mysql.properties专门配置mysql信息，application-redis.properties专门配置redis信息，最后将连个文件合并即可

  • 例如

    application-mysql.properties

    spring.datasource.username=root
spring.datasource.password=123123

对应yml：
spring:
	datasource:
		username: root
		password: 123123

    application-redis.properties

    spring.data.redis.host=localhost
spring.data.redis.port=6379

对应yml：
spring:
	data:
		redis:
			host:localhost
			port:6379

    application.properties合并

    spring.config.import=classpath:application-mysql.properties,classpath:application-redis.properties

对应yml
spring:
  config:
    import:
      - classpath:application-mysql.properties
      - classpath:application-redis.properties

像spring.config.import这样的key是被定义好的，存在一个配置类，被写好了类似
@Value(”${spring.config.import}”)

多环境切换

  1. springboot支持多配置文件切换
  1. 例如有3个配置文件：application-x.properties，application-y.properties，application-z.properties
  1. 在application.properties中配置spring.profiles.active=x则自动启动application-x.properties配置
  1. 也可以通过命令行参数来指定启动哪个 : --spring.profiles.active=x

在springboot中，多环境是指在一个应用程序中支持多种运行环境配置的能力，这通常用于区分开发（development）测试（testing）预生产（staging）和生产（producting）等不同阶段的环境。

开发阶段：application-dev.properties
生产阶段：application-prod.properties
测试阶段：application-test.properties
预生产阶段：application-preprod.properties
  • 案例

    1. 四个配置文件

    ① application.yml（公共全局配置）

    spring:
  application:
    name: demo-project
# 默认激活开发环境
  profiles:
    active: dev

    ② application-dev.yml（开发）

    server:
  port: 8080
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/dev_db
    username: root
    password: 123456

    ③ application-test.yml（测试）

    server:
  port: 8081
spring:
  datasource:
    url: jdbc:mysql://192.168.1.100:3306/test_db
    username: test
    password: test123

    ④ application-prod.yml（生产）

    server:
  port: 80
spring:
  datasource:
    url: jdbc:mysql://192.168.1.200:3306/prod_db
    username: prod
    password: xxxxxx

    2. 三种切换环境方式

    方式 1：配置文件内修改（本地调试最常用）

    修改 application.yml

    spring:
  profiles:
    active: prod

    方式 2：启动参数指定（部署最常用，优先级最高）

    java -jar demo.jar --spring.profiles.active=prod

    方式 3：虚拟机启动参数

    Dspring.profiles.active=test

将配置绑定简单Bean

  1. 使用@Component+@ConfigurationProperties(prefix=”app”)可以将配置文件中的配置一次性绑定到Bean上。
  1. 如果没有前缀，prefix可以省略。
  1. 绑定时，配置文件中的key需要和Bean的属性名对应上，并且给属性提供setter方法。
  1. bean的属性需要时非静态的。
springboot配置文件中的信息除了可以使用@Value注释读取之外，也可以将配置信息一次性赋值给Bean对象的属性。
  • 例如

    application.yml

    user:
  name: 张三
  age: 20
  email: zhangsan@123.com

    编写实体bean

    import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
// 前缀对应yml里的 user
@Data
@ConfigurationProperties(prefix = "user")
public class User {
    private String name;
    private Integer age;
    private String email;

    @Override
    public String toString() {
        return "User{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", email='" + email + '\'' +
                '}';
    }
}

    测试调用

    import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;
import javax.annotation.Resource;

@RestController
public class TestController {

    @Resource
    private User user;

    @GetMapping("/getUser")
    public String getUser(){
        return user.toString();
    }
}

嵌套Bean

一个 @Bean 方法内部，调用本配置类里另一个 @Bean 方法，核心差异完全由proxyBeanMethods 控制。

  • @Configuration（默认 true，Full 模式）
    配置类生成代理，内部调用不走 new，从容器取单例，==为 true。
  • @Configuration (proxyBeanMethods=false，Lite 模式)
    无代理，内部是普通方法调用，多次 new 多实例，==为 false。
  • 最佳写法
    别内部互调，用方法参数注入依赖，两种模式都能保证单例，性能更好。

Configuration注解

  1. 如果是一个配置类的话，建议使用@Configuration注释替代@Component，语义更加明确
  1. 使用@Configuration注释后，输出对象的地址是一个代理对象地址（使用@Component不会产生代理对象），使用代理对象效率比较低，添加proxyBeanMethods=false属性不生成代理对象
  1. proxyBeanMethods=true生成代理，效率低，但可以保证bean是单例，false，效率高，但不保证单例
单例 = 整个 Spring 容器中，这个 Bean 全局只有唯一一个实例对象，
不管获取多少次，拿到的都是同一个内存地址的对象，== 比较结果一定是 true
  • 例子

    实体类

    public class User {
    private String name;
    // 构造、get/toString
    public User(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "User{" + "name='" + name + '\'' + ", hashCode=" + hashCode() + '}';
    }
}

    默认 @Configuration（proxyBeanMethods = true，Full 模式）

    import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration // 默认 proxyBeanMethods = true
public class AppConfig {

    @Bean
    public User user1() {
        System.out.println("创建 user1");
        return new User("张三");
    }

    // 在本配置类内部调用上面的 @Bean 方法
    @Bean
    public User user2() {
        User u = user1();
        System.out.println("user2 中获取的 user1：" + u);
        return u;
    }
}

    测试

    import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class TestConfig {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx = new AnnotationConfigApplicationContext(AppConfig.class);

        User u1 = ctx.getBean("user1", User.class);
        User u2 = ctx.getBean("user2", User.class);

        System.out.println("容器获取user1：" + u1);
        System.out.println("容器获取user2：" + u2);
        System.out.println("u1 == u2 ? " + (u1 == u2));

        // 查看配置类本身是不是代理对象
        AppConfig config = ctx.getBean(AppConfig.class);
        System.out.println("配置类是否为代理：" + config.getClass());
    }
}
结果：
- `AppConfig` 是 **CGLIB 代理子类**
- `user2()` 里调用 `user1()` 不会真正执行方法新建对象，代理拦截直接从容器拿已有单例
- `u1 == u2` 结果为 `true`，两个是同一个对象

主入口类的Enable和Scan

  1. 之前的的写法：@Component+@ConfigurationProperties(Prefix=”app”) 或者 @Configuration+@ConfigurationProperties(prefix=”app”)
  1. 有了如下两个注解，@Component和@Comfiguration可以省略了：
    1. @EnableConfigurationProperties(Bean.class)：手动逐个指定，把括号里 @ConfigurationProperties 类注册进容器、开启配置绑定，配置类不用加 @Component
    1. @ConfigurationPropertiesScan(basePackages=””)：批量扫描指定包，自动发现所有带 @ConfigurationProperties 的类，统一注册，不用逐个声明
这两个都是标注在springboot主入口程序上的，将AppBean纳入IoC容器的管理
  • @EnableConfigurationProperties：手动罗列单个配置类，少量配置、第三方 jar 配置类
  • @ConfigurationPropertiesScan：包扫描批量自动注册，配置类多、统一放配置包

将配置赋值到Bean的Map/List/Array属性上

@ConfigurationProperties可以自动把 yml/properties 配置，绑定到 Bean 的List、数组、Map属性上。

前提：实体类加@ConfigurationProperties(prefix = "xxx")，并被 Spring 识别为配置绑定类。

  • 案例

    yml配置

    demo:
  list:
    - 苹果
    - 香蕉
  arr: [10,20,30]
  map:
    k1: 100
    k2: 200

    绑定实体类

    @ConfigurationProperties(prefix = "demo")
public class DemoProp {
    private List<String> list;
    private Integer[] arr;
    private Map<String,Integer> map;
    // get、set、toString
}
    • 逐个注册：@EnableConfigurationProperties(DemoProp.class)
    • 包扫描批量：@ConfigurationPropertiesScan("所在包")

    结果容器注入 DemoProp 后，属性自动填充配置值：

    • list = ["苹果","香蕉"]
    • arr = [10,20,30]
    • map = {k1=100, k2=200}

    要点：

    • List / 数组：yml 用 单行罗列或 [] 简写
    • Map：yml 直接写 key: value
    • 必须有 setter，反射赋值依赖 set 方法

将配置绑定在第三方对象上

第三方类不能改源码加 @ConfigurationProperties，用 @Bean + @ConfigurationProperties 在配置类里封装绑定。

  • 案例

    application.yml

    third:
  username: root
  password: 123456

    第三方类

    public class ThirdDataSource {
    private String username;
    private String password;
    // getter、setter
}

    配置类绑定

    @Configuration
public class Config {
    @Bean
    @ConfigurationProperties(prefix = "third")
    public ThirdDataSource thirdDataSource(){
        return new ThirdDataSource();
    }
}
    • 不能给第三方类加注解，只能在 @Bean 方法上加 @ConfigurationProperties
    • Spring 创建实例后自动匹配前缀赋值
    • 必须提供 set 方法才能注入属性

指定数据来源

  1. 如果需要加载的配置不是application.properties中的，可以使用@PropertySource来指定数据的来源
  1. @PropertySource(”classpath:a/b/group-info.properties”)
  • 案例

    自定义路径a/b/group-info.properties

    user.name=张三
user.age=18

    配置类绑定

    @ConfigurationProperties(prefix = "user")
@PropertySource("classpath:a/b/group-info.properties") // 指定数据源
public class UserProp {
    private String name;
    private Integer age;
    // get set
}
    • @PropertySource 只认 properties，不支持 yml
    • 搭配 @EnableConfigurationProperties / @ConfigurationPropertiesScan 生效
    • 可写绝对路径、外部文件路径

ImportResource注解

  1. 加载老式 XML 配置文件，把 xml 里定义的 Bean 导入 Spring 容器（SpringBoot 注解时代兼容旧项目用）
  1. 在入口类上使用
  • 案例

    旧配置 beans.xml

    <bean id="user" class="com.demo.User"/>

    启动类使用

    @SpringBootApplication
@ImportResource("classpath:beans.xml")
public class App {
    public static void main(String[] args) {
        SpringApplication.run(App.class, args);
    }
}

    常用写法

    // 单个文件
@ImportResource("classpath:beans.xml")
// 多个文件
@ImportResource(locations = {"classpath:a.xml","classpath:b.xml"})

Environment环境接口

Environment是springboot框架提供的环境对象，程序中可以直接注入Environment，然后调用相关方法获取各种配置信息。

统一获取所有配置属性：yml/properties、系统环境变量、JVM 参数、命令行参数等。

  • 案例与使用

    注入方式

    @Autowired
private Environment env;

    常用方法

    env.getProperty("key");                // 获取字符串
env.getProperty("num", Integer.class); // 指定类型
env.containsProperty("key");            // 判断是否存在
env.getActiveProfiles();                // 获取激活环境

    application.yml

    demo.name=test
取值：env.getProperty("demo.name")
    • 优先级：命令行 > JVM 参数 > 配置文件
    • 可读取配置、环境变量、运行参数
    • 对比：@Value 单个注入，Environment 动态灵活读取
