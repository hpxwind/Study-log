依赖安装

CC-Switch（Tauri2 桌面端 v3.16.1）Windows 完整编译&启动全流程

项目：farion1231/cc-switch，Tauri2+Vite+React+TS，两大报错：Tauri前后端版本不匹配、pnpm阻止esbuild/msw编译脚本，分步从环境→安装→修复→运行→打包。

一、前置环境安装

1. Node.js

安装 Node 20.x LTS（≥20.12，不选22），勾选Add to PATH

node -v
pnpm -v # 要求pnpm≥10，你当前11.5.1可用

2. Rust编译环境（Tauri必需）

  1. 打开管理员PowerShell安装rustup
winget install Rustlang.Rustup
# 或官方脚本
irm <https://astral.sh/uv/install.ps1> | iex
irm <https://win.rustup.rs/x86_64> | iex
  1. 重启终端，校验
rustc -v
cargo -v
  1. Windows系统依赖（VS生成工具）
winget install Microsoft.VisualStudio.BuildTools
# 安装时勾选：使用C++的桌面开发

3. 系统必备依赖

  • WebView2（Win11自带；Win10自动下载，Tauri运行依赖）

二、拉取源码+清理旧文件

# 进入项目目录
cd D:\\CCSwitch\\cc-switch-3.16.1
# 彻底清理旧依赖（解决残留锁文件、版本混乱）
Remove-Item -Recurse -Force node_modules,pnpm-lock.yaml,src-tauri/target,src-tauri/Cargo.lock -ErrorAction SilentlyContinue

三、修复两大核心报错

报错1：Tauri Rust包 ↔ JS包版本不一致

tauri(cargo:2.10.3) <==> @tauri-apps/api(npm:2.11.0)
tauri-plugin-dialog(cargo:2.6.0) <==> @tauri-apps/plugin-dialog(npm:2.7.1)
Tauri规则：主、次版本必须完全一致（2.x.x全部对齐）

方案A：统一降级JS侧依赖到Cargo版本（推荐）

  1. 打开 package.json 修改依赖：
"dependencies": {
  "@tauri-apps/api": "2.10.3",
  "@tauri-apps/plugin-dialog": "2.6.0",
  "@tauri-apps/plugin-process": "2.3.0",
  "@tauri-apps/plugin-store": "2.4.0",
  "@tauri-apps/plugin-updater": "2.9.0"
}
  1. 进入 src-tauri/Cargo.toml，确认：
tauri = "2.10.3"
tauri-plugin-dialog = "2.6.0"

方案B：全量升级Rust侧对齐JS（可选）

cd src-tauri
cargo update tauri tauri-plugin-dialog
cd ..

报错2：pnpm Ignored build scripts: esbuild,msw → 安装终止

pnpm11默认拦截原生模块postinstall脚本，需要放行编译

# 交互式放行esbuild、msw
pnpm approve-builds esbuild msw --all
# 写入配置永久生效（自动添加进pnpm配置）
备选：项目根新建.npmrc
ignore-scripts=false
shamefully-hoist=true

四、正式安装依赖

# 严格根据lock文件安装，禁止浮动版本
pnpm install --frozen-lockfile

安装无报错再往下走。

五、开发调试启动 dev

# 两种启动方式等价
pnpm run dev
# 或原命令
npm run dev

执行逻辑：

  1. pnpm run dev:renderer 启动Vite前端3000端口
  1. Tauri后端Rust编译，挂载前端页面
卡在Waiting for localhost:3000：前端vite启动失败，单独运行pnpm run dev:renderer排查前端报错。

六、生产打包（生成exe安装包/便携包）

pnpm run build
# 打包产物路径：src-tauri/target/release/bundle/msi/ 或 portable

七、备选：不想编译，直接用官方预编译包

Releases下载现成打包好的msi/zip便携版：https://github.com/farion1231/cc-switch/releases

八、常见异常快速修复

  1. Rust编译报错：link.exe找不到 → VS生成工具没装C++桌面组件，重装勾选
  1. esbuild平台二进制错误
pnpm remove esbuild && pnpm add esbuild
  1. 端口3000被占用 → 修改vite配置端口，或关闭占用进程
