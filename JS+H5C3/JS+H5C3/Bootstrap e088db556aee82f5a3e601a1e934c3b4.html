Bootstrap

Bootstrap 是全球最流行的前端开源框架之一，由 Twitter 团队于 2011 年推出，核心目标是帮助开发者快速构建响应式、美观且兼容多设备的网页或 Web 应用。它封装了大量经过优化的 CSS 样式、JavaScript 交互组件和布局工具，让开发者无需从零编写基础代码，专注于业务逻辑实现。

官网：Reboot · Bootstrap v5.3

  • 核心特点（需知道）
    1. 响应式设计（Responsive）基于 “移动优先”（Mobile-First）理念，通过 栅格系统 和 响应式工具类，自动适配手机、平板、电脑等不同屏幕尺寸，无需为不同设备单独开发页面。
    1. 开箱即用（Out-of-the-Box）内置大量预设组件（如按钮、导航栏、卡片、表单等）和样式，直接通过添加特定类名即可使用，极大缩短开发周期。
    1. 兼容性强兼容主流浏览器（Chrome、Firefox、Safari、Edge 等），解决了不同浏览器间的样式和脚本兼容性问题，开发者无需手动编写浏览器前缀（如 webkit-、moz-）。
    1. 可定制化支持通过官方工具（Bootstrap 定制器）或 SCSS 变量修改默认样式（如颜色、字体、间距），满足项目的个性化设计需求。
    1. 轻量且高效核心文件（CSS + JS）经过压缩后体积小（Bootstrap 5 压缩后 CSS 约 20KB，JS 约 15KB），加载速度快；同时提供 “按需引入” 功能，避免冗余代码。
  • 版本选择（很重要）
    版本发布时间核心差异适用场景
    Bootstrap 52021 年1. 移除 jQuery 依赖（原生 JS 实现交互）2. 新增网格系统优化（支持 12 列 + 自动列）3. 增强响应式工具（如 xxl 断点）4. 支持 CSS 变量（自定义属性）推荐新项目使用，尤其是追求轻量、现代的开发场景（如 Vue/React 项目中引入）
    Bootstrap 42018 年1. 依赖 jQuery 和 Popper.js（弹窗定位）2. 引入 “移动优先” 理念3. 支持 Flexbox 布局旧项目维护，或团队熟悉 jQuery 生态的场景
    Bootstrap 32013 年1. 基于 “桌面优先” 理念2. 依赖 jQuery3. 不支持 Flexbox 和 CSS 变量仅用于极旧项目维护，不推荐新项目使用

    注意：本文以 Bootstrap 5（最新稳定版）为例讲解，其是目前生态最活跃、功能最完善的版本。

  • 引入

    使用 Bootstrap 有两种核心方式：CDN 引入（快速上手）和 本地下载（定制化开发）。

    1、CDN：

    无需下载文件，直接通过 CDN 链接引入 Bootstrap 的 CSS 和 JS 文件，适用于快速原型开发或小型项目。

    <!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <!-- 1. 视口设置：开启响应式（必须） -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Bootstrap 5 基础模板</title>
  
  <!-- 2. 引入 Bootstrap CSS（核心样式） -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
  
  <!-- （可选）引入 Bootstrap 图标库（内置 2000+ 图标） -->
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.10.5/font/bootstrap-icons.css">
</head>
<body>

  <!-- 页面内容（示例：一个按钮） -->
  <button type="button" class="btn btn-primary">Bootstrap 按钮</button>

  <!-- 3. 引入 Bootstrap JS（交互组件依赖，如模态框、下拉菜单） -->
  <!-- 注意：Bootstrap 5 无需 jQuery，但部分组件依赖 Popper.js（已包含在下方链接中） -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
    • 视口 meta 标签：<meta name="viewport" ...> 是响应式的 “开关”，必须添加，否则 Bootstrap 无法识别设备屏幕尺寸。
    • bootstrap.bundle.min.js：包含了 Popper.js（用于弹窗、下拉菜单的定位），无需单独引入；若只需基础样式（无交互），可省略 JS 引入。

  • 1、栅格系统（Grid System）：响应式布局的核心

    栅格系统是 Bootstrap 实现响应式的 “骨架”，基于 12 列网格 和 5 个响应式断点，让页面元素在不同屏幕上自动排列。

    • 容器（Container）：栅格系统的外层包裹元素，用于控制内容的宽度和间距，分两种：
      • .container：固定宽度（不同断点下宽度不同，如电脑端约 1140px）；
      • .container-fluid：100% 宽度（占满父容器，无固定宽度）。
    • 行（Row）：栅格的 “行”，必须包裹在容器内，用于包裹 “列”，且会清除浮动（避免列换行错乱）。
    • 列（Column）：栅格的 “列”，必须包裹在行内，通过类名控制列的宽度和响应式行为。
    • 响应式断点
      断点前缀屏幕宽度范围设备类型
      col-<576px超小屏幕（手机竖屏）
      col-sm-≥576px小屏幕（手机横屏、平板竖屏）
      col-md-≥768px中等屏幕（平板横屏、小电脑）
      col-lg-≥992px大屏幕（电脑）
      col-xl-≥1200px超大屏幕（大电脑）
      col-xxl-≥1400px特超大屏幕（超大显示器）
    • 列的使用规则：
      • 列的宽度通过 “前缀 + 数字” 控制，数字范围 1-12（如 col-md-6 表示中等屏幕下占 6 列，即 50% 宽度）；
      • 一行中列的总数字之和建议 ≤12，超过则自动换行；
      • 可给同一列添加多个断点类，实现 “不同屏幕下不同宽度”（如 col-sm-6 col-md-4 表示小屏幕占 6 列，中等屏幕占 4 列）。
      <!-- 容器：固定宽度 -->
<div class="container mt-3"> <!-- mt-3：margin-top 为 3 个单位（Bootstrap 间距工具） -->
  <!-- 行 -->
  <div class="row">
    <!-- 列 1：超小屏占 12 列（全屏），小屏及以上占 6 列（半屏），中等屏及以上占 4 列（1/3 屏） -->
    <div class="col-12 col-sm-6 col-md-4 mb-3"> <!-- mb-3：margin-bottom 3 个单位 -->
      <div class="card"> <!-- 卡片组件（下文讲解） -->
        <div class="card-body">
          <h5 class="card-title">卡片 1</h5>
          <p class="card-text">超小屏全屏，小屏半屏，中屏及以上 1/3 屏</p>
        </div>
      </div>
    </div>

    <!-- 列 2：与列 1 规则相同 -->
    <div class="col-12 col-sm-6 col-md-4 mb-3">
      <div class="card">
        <div class="card-body">
          <h5 class="card-title">卡片 2</h5>
          <p class="card-text">超小屏全屏，小屏半屏，中屏及以上 1/3 屏</p>
        </div>
      </div>
    </div>

    <!-- 列 3：超小屏全屏，小屏及以上占 12 列（全屏），中等屏及以上占 4 列（1/3 屏） -->
    <div class="col-12 col-sm-12 col-md-4 mb-3">
      <div class="card">
        <div class="card-body">
          <h5 class="card-title">卡片 3</h5>
          <p class="card-text">超小屏全屏，小屏全屏，中屏及以上 1/3 屏</p>
        </div>
      </div>
    </div>
  </div>
</div>
    • 栅格系统进阶：列偏移、排序、嵌套
      • 列偏移：通过 offset-* 类让列向右偏移（如 offset-md-2 表示中等屏幕下偏移 2 列）；
      • 列排序：通过 order-* 类调整列的显示顺序（如 order-md-1 表示中等屏幕下列的顺序为 1）；
      • 列嵌套：在列中嵌套新的 “行” 和 “列”，实现更复杂的布局（如在 col-md-6 中嵌套 row 和 col-6）。
  • 2、响应式工具类：控制元素显示/隐藏

    Bootstrap 提供了一套工具类，用于在不同屏幕尺寸下控制元素的 显示（display）、隐藏（hidden） 或 对齐方式，无需手动写 CSS 媒体查询。

    • 常用显示/隐藏工具类
      类名作用（以 md 断点为例）
      d-none所有屏幕下隐藏
      d-md-none中等屏幕及以上隐藏，以下显示
      d-md-block中等屏幕及以上显示，以下隐藏
      d-md-flex中等屏幕及以上使用 Flex 布局
      d-md-inline中等屏幕及以上为行内元素

      案例：不同屏幕下显示不同内容：

      <!-- 手机端显示（<768px），电脑端隐藏（≥768px） -->
<div class="d-block d-md-none">
  欢迎访问手机版网站！
</div>

<!-- 电脑端显示（≥768px），手机端隐藏（<768px） -->
<div class="d-none d-md-block">
  欢迎访问电脑版网站！
</div>
    • 间距工具类（Margin/Padding）

      通过 m-*（margin）和 p-*（padding）控制元素间距，* 代表 0-5 的数字（数字越大，间距越大），同时支持方向控制（如 mt-3 表示 margin-top: 1rem，px-2 表示 padding-left 和 padding-right: 0.5rem）。

      类名前缀方向示例含义
      m-所有方向m-3四个方向 margin 为 3 单位
      mt-上mt-2上 margin 为 2 单位
      mb-下mb-4下 margin 为 4 单位
      ms-左（start）ms-1左 margin 为 1 单位
      me-右（end）me-5右 margin 为 5 单位
      p-所有方向p-3四个方向 padding 为 3 单位

  • 3、常用组件：开箱即用的UI元素

    通过 btn 基础类 + 主题类控制按钮样式，支持不同状态（默认、激活、禁用）和尺寸。

    主题类样式效果示例代码
    btn-primary主色调（默认蓝）<button class="btn btn-primary">主按钮</button>
    btn-secondary次要色调（灰）<button class="btn btn-secondary">次要按钮</button>
    btn-success成功色（绿）<button class="btn btn-success">成功按钮</button>
    btn-danger危险色（红）<button class="btn btn-danger">删除按钮</button>
    btn-outline-*边框样式<button class="btn btn-outline-primary">边框按钮</button>

    尺寸控制：btn-sm（小）、btn-lg（大）、btn-block（全屏宽度）。

  <img src="https://picsum.photos/200/150" class="card-img-top" alt="示例图片">

    <!-- 卡片链接 -->

    <a href="#" class="card-link">收藏</a>

</div>

    • 2、导航栏（Navbar）

      响应式导航组件，在电脑端显示横向菜单，手机端折叠为 “汉堡按钮”（点击展开菜单），是网站顶部导航的常用方案。

      <nav class="navbar navbar-expand-md navbar-dark bg-primary"> <!-- 深色背景+主色调 -->
  <div class="container">
    <!-- 品牌Logo -->
    <a class="navbar-brand" href="#">MySite</a>
    
    <!-- 手机端折叠按钮（仅在 <md 屏幕显示） -->
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarNav">
      <span class="navbar-toggler-icon"></span>
    </button>
    
    <!-- 导航菜单（可折叠） -->
    <div class="collapse navbar-collapse" id="navbarNav">
      <ul class="navbar-nav ms-auto"> <!-- ms-auto：右对齐 -->
        <li class="nav-item">
          <a class="nav-link active" aria-current="page" href="#">首页</a> <!-- active：当前页高亮 -->
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">产品</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">关于我们</a>
        </li>
      </ul>
    </div>
  </div>
</nav>
    • 3、表单（Forms）

      Bootstrap 对表单元素（输入框、下拉框、复选框等）进行了样式优化，支持水平布局、验证状态提示等功能。

      <div class="container mt-4">
  <form>
    <!-- 文本输入框 -->
    <div class="mb-3">
      <label for="username" class="form-label">用户名</label>
      <input type="text" class="form-control" id="username" placeholder="请输入用户名">
    </div>
    
    <!-- 密码输入框 -->
    <div class="mb-3">
      <label for="password" class="form-label">密码</label>
      <input type="password" class="form-control" id="password" placeholder="请输入密码">
    </div>
    
    <!-- 复选框 -->
    <div class="mb-3 form-check">
      <input type="checkbox" class="form-check-input" id="remember">
      <label class="form-check-label" for="remember">记住我</label>
    </div>
    
    <!-- 提交按钮 -->
    <button type="submit" class="btn btn-primary">登录</button>
  </form>
</div>
    • 4、模态框（Modals）

      弹出式对话框，用于显示重要信息、表单或交互操作（如确认删除、用户登录），需要通过 JavaScript 或 data-* 属性触发。

      <div class="container mt-4">
  <!-- 触发按钮：data-bs-toggle="modal" 表示触发模态框，data-bs-target="#myModal" 指向模态框ID -->
  <button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#myModal">
    打开模态框
  </button>

  <!-- 模态框 -->
  <div class="modal fade" id="myModal" tabindex="-1" aria-hidden="true">
    <!-- 模态框对话框 -->
    <div class="modal-dialog">
      <!-- 模态框内容 -->
      <div class="modal-content">
        <!-- 模态框头部 -->
        <div class="modal-header">
          <h5 class="modal-title">模态框标题</h5>
          <!-- 关闭按钮 -->
          <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
        </div>
        <!-- 模态框主体 -->
        <div class="modal-body">
          这是模态框的内容区域，可以放文本、表单或其他元素。
        </div>
        <!-- 模态框底部 -->
        <div class="modal-footer">
          <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">取消</button>
          <button type="button" class="btn btn-primary">确认</button>
        </div>
      </div>
    </div>
  </div>
</div>
  • 4、JavaScript插件：实现交互

    Bootstrap 5 的交互组件（如模态框、下拉菜单、轮播图）依赖内置的 JavaScript 插件，无需手动写 JS 逻辑，通过以下两种方式触发：

    1. data-* 属性触发（推荐）：通过添加 data-bs-toggle、data-bs-target 等属性，无需写 JS 代码即可触发交互（如上文模态框示例）。
    1. JavaScript 代码触发（灵活控制）：通过 Bootstrap 提供的 API 手动控制组件，适用于需要自定义逻辑的场景。
    <!-- 按钮 -->
<button type="button" class="btn btn-primary" id="openModalBtn">通过JS打开模态框</button>

<!-- 模态框（同上，ID为myModal） -->
<div class="modal fade" id="myModal" ...></div>

<!-- JS 代码 -->
<script>
  // 获取按钮和模态框元素
  const openModalBtn = document.getElementById('openModalBtn');
  const myModal = new bootstrap.Modal(document.getElementById('myModal')); // 初始化模态框

  // 按钮点击事件：打开模态框
  openModalBtn.addEventListener('click', function() {
    myModal.show(); // 打开模态框
    // myModal.hide(); // 关闭模态框（可选）
  });
</script>
    插件名称功能描述触发属性示例
    Modal模态框data-bs-toggle="modal"
    Dropdown下拉菜单data-bs-toggle="dropdown"
    Carousel轮播图data-bs-ride="carousel"
    Collapse折叠面板（如导航栏折叠）data-bs-toggle="collapse"
    Tab选项卡切换data-bs-toggle="tab"
