Prompt Engineering

一、什么是 Prompt 工程？

Prompt 工程（提示工程）是指设计和优化输入提示（prompt），以引导大语言模型（LLM，如 GPT-4、通义千问、Claude 等）生成符合预期输出的技术和方法。它本质上是人与 AI 之间的一种“沟通艺术”和“工程技术”，目标是用最简洁、最有效的指令激发出模型的最佳性能。

核心观点：好的 prompt 不是“询问”，而是“编程”——你通过在提示中明确上下文、约束格式、指定推理步骤，来精确控制模型的行为。

二、为什么 Prompt 工程如此重要？

  1. 模型能力与用户意图的鸿沟：LLM 拥有海量知识，但它不知道你具体想要什么格式、风格、详细程度。Prompt 是填补这一鸿沟的桥梁。
  1. 成本与效率：精心设计的 prompt 可以直接获得高质量输出，避免多次迭代、后处理或微调，节省 API 调用成本和开发时间。
  1. 可解释性与可控性：好的 prompt 可以让模型的推理过程透明化（如“思维链”），便于调试和信任。
  1. 任务适配：同一个模型通过不同 prompt 可以完成翻译、摘要、代码生成、角色扮演等截然不同的任务，无需修改模型参数。

三、Prompt 工程的基本原则

原则说明示例
清晰明确避免歧义，使用具体、可度量的指令❌“写一篇好文章” → ✅“写一篇500字的介绍北京故宫的博客，语气亲切，面向中学生”
提供上下文给出背景、角色、约束条件“你是一位资深 Python 工程师，请审查以下代码并指出潜在的性能问题。”
分隔清晰使用标记（如 ###、"""、---）区分指令、输入、示例### 指令：... ### 输入：... ### 输出：
指定输出格式明确要求 JSON、Markdown、列表、表格等“以 JSON 格式输出，包含字段 name, age, city”
给出示例（Few‑shot）提供 1~3 个输入输出对，让模型模仿示例见下文 Few‑shot 部分
分步思考（Chain‑of‑Thought）让模型先写出推理步骤，再给出最终答案“请一步步推理，最后以‘因此答案是：...’结尾”
设置角色为模型分配一个专家角色“你是一位营养学专家，请根据以下食谱给出健康建议。”

四、Prompt 工程的核心技术与高级技巧

1. Zero‑shot Prompting（零样本提示）

直接给出指令，不提供示例。适用于模型已具备的基础任务（翻译、分类等）。

将以下句子翻译成英文：“今天天气很好。”

2. Few‑shot Prompting（少样本提示）

提供几个输入输出示例，让模型学习模式。尤其适合需要格式、风格、特殊逻辑的任务。

将英文短语翻译为中文俚语：
“Break a leg” -> “祝你演出成功”
“Piece of cake” -> “小菜一碟”
“Spill the beans” -> “？？？”

3. Chain‑of‑Thought (CoT) Prompting（思维链提示）

诱导模型生成中间推理步骤，显著提升逻辑、数学、常识推理任务的准确性。

问：一个水池有一个进水口和一个排水口。单独开进水口5小时可注满，单独开排水口10小时可排空。如果同时打开，多少小时可以注满水池？
答：每小时进水 1/5 池，排水 1/10 池，净进水 1/5 - 1/10 = 1/10 池。所以需要 10 小时。因此答案是：10小时。

变种：Self‑consistency CoT（自我一致性）——多次采样 CoT，取多数答案。

4. Tree‑of‑Thoughts (ToT)（思维树）

扩展 CoT，让模型在推理过程中探索多个分支，并自我评估选择最优路径。适合复杂规划、创意写作等任务。

5. ReAct Prompting（Reasoning + Acting）

结合推理和行动，让模型不仅能“想”，还能“调用工具”（如搜索、计算）。常用于 AI Agent。

你是一个可以查询天气的助手。当用户询问天气时，请先思考需要哪个城市，然后调用 `get_weather(city)` 工具。
用户：北京今天需要带伞吗？
思考：北京今天有没有雨？我需要获取北京的天气数据。
行动：get_weather("北京")
观察：北京今天小雨，气温5-10℃。
最终回答：北京今天有小雨，建议带伞。

6. Directional Stimulus Prompting（定向刺激提示）

在 prompt 中包含一个“提示线索”（如关键词或小模板）来引导模型生成特定内容。

7. Automatic Prompt Engineering (APE)

使用算法自动搜索、优化 prompt（如通过梯度下降或 LLM 自己生成候选 prompt）。

8. Meta‑Prompting（元提示）

让模型为自己生成或改进 prompt。例如：“请为‘将产品评论分类为正面或负面’这个任务生成一个有效的 prompt。”

五、Prompt 的结构化设计模式

一个优秀的 prompt 常包含以下模块（可按需组合）：

[角色定义] 你是一位[领域]专家。
[任务描述] 你需要[具体任务]。
[输入格式] 输入会以[格式]给出，内容如下：
[输入]
[约束条件] 请遵守：1. ... 2. ...
[输出格式] 以[格式]输出，例如...
[示例] （Few‑shot 示例）
[思考指令] 请先逐步推理，然后输出最终答案。

六、常见陷阱与误区

陷阱说明改进方法
过度约束太多的规则会限制模型创造力或导致矛盾保留必要约束，信任模型能力
模糊否定“不要写不相关的内容” —— 模型不知道什么是相关正面描述需要的内容
忽略格式错误忘记要求纯 JSON 或 Markdown，模型会加入额外解释明确要求“只输出 JSON，不要有其他文字”
上下文错位把示例放在指令之前/之后导致模型混淆保持顺序：角色→指令→上下文→示例→输入
示例不够代表性示例太简单或太特殊，模型无法泛化选择覆盖不同边缘情况的示例
忘记多轮对话在连续对话中每次都用独立 prompt，丢失记忆使用 RunnableWithMessageHistory 等管理历史
过度依赖 prompt任务过于复杂或需要大量示例 → 应考虑微调评估是否 finetune 更合适

2. 信息抽取

从下面的新闻中抽取：时间、地点、人物、事件。输出 JSON。
新闻：2025年10月1日，中国科学家李明在实验室成功合成了一种新型超导材料。

3. 代码生成

编写一个 Python 函数，输入是一个整数列表，返回排序后的列表，要求使用快速排序算法，并包含类型注解和文档字符串。

4. 创意写作

你是一位科幻作家。请根据以下设定写一段 200 字的故事开头：22 世纪，人类已经移民火星，但地球突然出现一种能吞噬金属的细菌。

5. 对话系统（角色扮演）

你是一位耐心的数学老师，名叫“李老师”。你的任务是用苏格拉底式提问帮助学生解方程。学生问题：老师，2x + 5 = 15，x 等于多少？请一步步提问引导。

6. 结构化数据生成

生成 5 条模拟的用户评论，用于测试电影推荐系统。每条评论包含字段：user_id（整数），movie_name（字符串），rating（1-5 整数），comment（10-30 字）。输出为 JSON 数组。

九、评估 Prompt 效果的方法

  1. 定性评估：人工检查输出的准确性、相关性、格式正确性。
  1. 定量指标：对于分类/抽取任务，计算精确率、召回率、F1；对于生成任务，使用 BLEU、ROUGE、BERTScore 等。
  1. 鲁棒性测试：改变输入微小扰动（如改写、加噪），观察输出稳定性。
  1. 对抗性测试：设计边缘案例或故意误导的输入，检查模型是否保持正确。
  1. 成本与延迟：统计 token 使用量和响应时间，平衡质量与效率。

十、Prompt 工程的未来趋势

  1. 自动化 Prompt 优化：AI 自己设计、A/B 测试、进化 prompt（如 APE, OPRO）。
  1. 多模态 Prompt：结合图像、音频、视频的跨模态提示。
  1. 系统化工程：Prompt 版本管理、CI/CD 测试、注册表（Prompt Hub）将成为 MLOps 的一部分。
  1. 领域专用模板库：医疗、法律、金融等垂直行业会积累经过验证的 prompt patterns。
  1. 交互式 Prompt 调试：可视化工具允许开发者像调试代码一样单步执行 prompt 的推理过程。
  1. 从 Prompt 到微调：对于企业级应用，prompt 工程会与 RAG（检索增强生成）、轻量级微调（LoRA）结合，形成混合策略。

十一、学习与实践建议

  1. 从模板开始：使用 LangChain 的 PromptTemplate 结构化你的提示。
  1. 记录迭代：保存每次修改的 prompt 及对应的输出，建立实验日志。
  1. 反向思考：先手写理想输出，再倒推需要什么样的 prompt 能产生它。
  1. 与人测试：让你同事阅读你的 prompt，看他们是否能准确理解任务要求。
  1. 利用模型自身：要求模型对自己的输出进行评分或解释，发现 prompt 的不足之处。
  1. 关注安全：防止 prompt 注入（利用恶意输入绕过限制），使用边界标记和输入过滤。

总结：Prompt 工程是一门将人类意图高效转换为 LLM 可执行指令的技艺。它融合了语言学、认知科学、软件工程和机器学习的思想。随着大语言模型能力的不断增强，prompt 工程的意义不仅在于“更好地提问”，更在于构建出可靠、可控、可复用的 AI 应用系统。掌握它，是每一位 LLM 开发者的基本功与进阶之路。
