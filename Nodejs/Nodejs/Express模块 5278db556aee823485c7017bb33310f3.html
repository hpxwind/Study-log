Express模块

  • 路由规则

    路由匹配支持特殊字符和正则表达式风格的模式，用于灵活匹配不同的 URL 路径。你提供的这 5 个路由规则，分别使用了不同的特殊符号（\?、:、+、*、()+?），各自对应不同的匹配逻辑。

    • 1、app.get("/ab?cd", (req, res) => { ... })

      规则：匹配「a 开头，b 可选，结尾是 cd」的路径

      • ✅ http://localhost:3000/acd（b 出现 0 次）
      • ✅ http://localhost:3000/abcd（b 出现 1 次）
      • ❌ http://localhost:3000/abbcd（b 出现 2 次，超出 ? 的 “0 或 1 次” 范围）
      • ❌ http://localhost:3000/abxcd（b 后多了 x，破坏 cd 结尾规则）
    • 2、app.get("/ab/:id", (req, res) => { ... })

      规则：匹配「/ab/ 固定前缀 + 动态参数」的路径

      • ✅ http://localhost:3000/ab/123（id=123，数字参数）
      • ✅ http://localhost:3000/ab/user456（id=user456，字符串参数）
      • ✅ http://localhost:3000/ab/abc-def（id=abc-def，含特殊字符参数）
      • ❌ http://localhost:3000/ab/123/456（/ab/123/456 中 123 是 id，但后面多了 /456，路径层级不匹配）
      • ❌ http://localhost:3000/ab?id=123（?id=123 是查询参数，路径核心是 /ab，无 :id 部分）
    • 3、app.get("/ab+cd", (req, res) => { ... })

      规则：匹配「a 开头，b 出现 1 次及以上，结尾是 cd」的路径

      • ✅ http://localhost:3000/abcd（b 出现 1 次）
      • ✅ http://localhost:3000/abbcd（b 出现 2 次）
      • ✅ http://localhost:3000/abbbcd（b 出现 3 次）
      • ❌ http://localhost:3000/acd（b 出现 0 次，不满足 b+ 的 “1 次及以上”）
      • ❌ http://localhost:3000/abxcd（b 后多了 x，破坏 cd 结尾规则）
    • 4、app.get("/ab*cd", (req, res) => { ... })

      规则：匹配「ab 开头、cd 结尾，中间可包含任意内容」的路径

      • ✅ http://localhost:3000/abcd（中间无内容，ab + `` + cd）
      • ✅ http://localhost:3000/ab123cd（中间是数字，ab + 123 + cd）
      • ✅ http://localhost:3000/ab/ef/gcd（中间包含 /，ab + /ef/g + cd）
      • ✅ http://localhost:3000/ab-x+y=zcd（中间是特殊符号，ab + x+y=z + cd）
      • ❌ http://localhost:3000/ab123（缺少 cd 结尾）
      • ❌ http://localhost:3000/123cd（缺少 ab 开头）
    • 5、app.get("/ab(cd)?", (req, res) => { ... })

      规则：匹配「ab 开头，cd 作为整体可选」的路径

      • ✅ http://localhost:3000/ab（分组 cd 出现 0 次）
      • ✅ http://localhost:3000/abcd（分组 cd 出现 1 次）
      • ❌ http://localhost:3000/abbcd（分组 cd 未完整出现，多了 b）
      • ❌ http://localhost:3000/abx（分组 cd 未出现，且多了 x）

    核心差异：

    路由规则特殊符号作用匹配逻辑典型匹配 URL
    /ab?cd? 匹配前面字符 0/1 次匹配 acd 或 abcd/acd、/abcd
    /ab/:id: 动态参数（非 / 字符）匹配 /ab/xxx（xxx 为动态值）/ab/123、/ab/user
    /ab+cd+ 匹配前面字符 1+ 次匹配 abcd、abbcd 等/abcd、/abbbcd
    /ab*cd* 匹配任意中间内容 0+ 次匹配 ab 开头 + 任意内容 + cd/abcd、/ab/ef/cd
    /ab(cd)?() 分组 + ? 可选匹配 /ab 或 /abcd/ab、/abcd
  • 中间件

    中间件本质是一个函数

    • 这个函数具有的特征
      1. 参数固定：接受三个核心参数（或四个，错误处理中间件）
        • req（Request）：请求对象，包含客户端发送的所有信息（如 URL、参数、请求体等）
        • res（Response）：响应对象，用于向客户端返回数据（如设置响应头、发送响应体等）
        • next：回调函数，调用后会将请求传递给「下一个中间件」或「路由处理函数」
        • 错误处理中间件多一个 err 参数：(err, req, res, next)
      1. 功能定位：
        • 执行任意代码（如日志记录、权限验证、数据转换）
        • 修改请求 / 响应对象（如添加自定义属性、解析请求体）
        • 终止请求 - 响应循环（如直接返回错误响应）
        • 调用下一个中间件（通过 next()）
      1. 执行时机：在请求到达路由处理函数之前，或在路由处理函数之后、响应发送之前。

    • 中间件工作流程（管道模型）
      客户端请求 → 中间件1 → 中间件2 → ... → 路由处理函数 → 响应客户端
       ↑                                        ↓
       └────────────────────────────────────────┘
                （若中间件调用 next()）
      • 关键规则：
        • 中间件按「定义顺序」执行（先定义的先执行）
        • 若中间件未调用 next() 且未发送响应，请求会「挂起」（客户端一直等待）
        • 若中间件发送了响应（如 res.send()），则终止流程，后续中间件和路由不会执行
    • 应用级中间件（Application-Level Middleware）

      绑定到 Express 应用实例（app），作用于全局或指定路径的所有请求。

      const express = require('express');
const app = express();

// 1. 无路径：所有请求都会经过
app.use((req, res, next) => {
  console.log('时间:', Date.now()); // 记录请求时间
  next(); // 传递给下一个中间件
});

// 2. 有路径：仅 /user 开头的请求会经过
app.use('/user', (req, res, next) => {
  console.log('访问用户相关接口');
  next();
});

// 3. 特定 HTTP 方法：仅 POST 请求且路径为 /login 时执行
app.post('/login', (req, res, next) => {
  console.log('处理登录请求');
  next();
});

      典型场景：全局日志、跨域处理、请求体解析、全局权限验证。

    • 路由级中间件（Router-Level Middleware）

      绑定到路由实例（express.Router()），仅作用于该路由负责的路径，用于拆分模块化路由逻辑。

      // 创建路由实例
const userRouter = express.Router();

// 路由级中间件：仅作用于 userRouter 处理的请求
userRouter.use((req, res, next) => {
  console.log('用户路由中间件');
  next();
});

// 路由级中间件：仅作用于 userRouter 的 GET /:id 路由
userRouter.get('/:id', (req, res, next) => {
  console.log('检查用户 ID 格式');
  if (!isNaN(req.params.id)) {
    next(); // ID 有效，继续
  } else {
    res.status(400).send('无效 ID'); // 终止流程
  }
});

// 路由处理函数
userRouter.get('/:id', (req, res) => {
  res.send(`用户 ID: ${req.params.id}`);
});

// 将路由实例挂载到应用
app.use('/user', userRouter); // 所有 /user/* 请求会经过 userRouter 中间件

      典型场景：某类路由的专用验证（如用户路由验证用户权限）、业务逻辑预处理。

    • 错误处理中间件（Error-handling Middleware）

      专门处理请求流程中抛出的错误，必须有 4 个参数（err, req, res, next），否则会被视为普通中间件。

      注意放在最后

      // 错误处理中间件（需放在所有路由和中间件之后）
app.use((err, req, res, next) => {
  console.error('错误:', err.stack); // 打印错误堆栈
  // 根据错误类型返回响应
  if (err.type === 'validation') {
    res.status(400).send(`验证错误: ${err.message}`);
  } else {
    res.status(500).send('服务器内部错误');
  }
});

// 触发错误的示例路由
app.get('/error', (req, res, next) => {
  const err = new Error('自定义错误');
  err.type = 'custom';
  next(err); // 传递错误给错误处理中间件
});

      关键特性：

      • 需放在所有路由和中间件的最后，确保捕获所有错误
      • 若调用 next(err)，Express 会跳过普通中间件，直接执行错误处理中间件
    • 内置中间件（Built-in Middleware）

      Express 自带的中间件，无需额外安装，用于处理常见任务。

      常用内置中间件：

      1、express.static(root, [options])：托管静态资源（如 HTML、CSS、图片）

      app.use(express.static('public')); // 访问 /index.html 会返回 public/index.html

      2、express.json([options])：解析 JSON 格式的请求体（Content-Type: application/json）

      app.use(express.json()); // 解析后的数据存放在 req.body

      3、express.urlencoded([options])：解析 URL 编码的请求体（Content-Type: application/x-www-form-urlencoded）

      app.use(express.urlencoded({ extended: true })); // extended: true 支持嵌套对象
    • 第三方中间件（Third-party Middleware）

      由社区开发的中间件，需通过 npm 安装，扩展 Express 功能。

      常用第三方中间件：

      1、morgan：HTTP 请求日志记录

      const morgan = require('morgan');
app.use(morgan('combined')); // 输出详细日志

      2、cors：处理跨域资源共享（CORS）

      const cors = require('cors');
app.use(cors()); // 允许所有跨域请求

      3、cors：处理跨域资源共享（CORS）

      const cors = require('cors');
app.use(cors()); // 允许所有跨域请求

      4、helmet：设置 HTTP 头增强安全性

      const helmet = require('helmet');
app.use(helmet()); // 自动添加安全相关的响应头

      5、multer：处理 multipart/form-data 类型的请求（用于文件上传）

      const multer = require('multer');
const upload = multer({ dest: 'uploads/' });
app.post('/upload', upload.single('file'), (req, res) => {
  res.send('文件上传成功');
});
    • 核心用法与技巧
      • 1、数据传递：req与res.locals

        中间件之间共享数据的两种方式：

        req 对象：在整个请求生命周期内有效，所有中间件和路由都可访问

        // 中间件1：设置数据
app.use((req, res, next) => {
  req.user = { id: 1, name: '张三' };
  next();
});

// 路由：读取数据
app.get('/', (req, res) => {
  res.send(`当前用户: ${req.user.name}`);
});

        res.locals：仅在视图渲染时有效（若使用模板引擎），不会污染 req 对象

        app.use((req, res, next) => {
  res.locals.title = '我的网站'; // 模板中可直接使用 {{ title }}
  next();
});
      • 2、条件分支：控制请求流向

        通过中间件的条件判断，决定请求走不同的处理流程：

        app.use((req, res, next) => {
  if (req.method === 'GET') {
    console.log('处理 GET 请求');
    next(); // 继续执行后续中间件
  } else if (req.method === 'POST') {
    console.log('验证 POST 数据');
    if (req.body.token) {
      next(); // 验证通过
    } else {
      res.status(403).send('缺少 token'); // 终止流程
    }
  } else {
    next('不支持的请求方法'); // 传递错误给错误处理中间件
  }
});
      • 3、异步中间件：处理异步操作

        中间件支持异步逻辑（如数据库查询），需注意正确处理 next() 调用：

        // 异步中间件（使用 async/await）
app.use(async (req, res, next) => {
  try {
    const user = await db.getUser(req.params.id); // 异步查询数据库
    req.user = user;
    next(); // 成功：传递给下一个中间件
  } catch (err) {
    next(err); // 失败：传递错误给错误处理中间件
  }
});
    • logger(’dev’)

      logger是morgan中间件，专门用来记录http请求日志

      ‘dev’是Morgan提供的预定义格式之一，会显示：

      • HTTP 方法 (GET, POST 等)
      • 请求路径
      • 状态码 (200, 404, 500 等)
      • 响应时间
      • 内容长度

      如：

      app.use(logger('dev'));
GET /users 200 15ms - 200
    • express.urlencoded({ extended: false })
      1. 注册中间件：使用 app.use() 注册 express.urlencoded 中间件
      1. 解析表单数据：解析 Content-Type 为 application/x-www-form-urlencoded 的请求体数据
      1. 设置解析选项：通过 { extended: false } 配置解析方式
      • 功能：当客户端通过 HTML 表单提交数据时，数据会以 URL 编码格式发送，此中间件会将这些数据解析并添加到 req.body 对象中
      • extended: false：使用 querystring 库解析 URL 编码的数据，只支持字符串和数组类型的数据
      • extended: true：使用 qs 库解析，支持嵌套对象

      当客户端提交如下表单数据时：

      <form method="POST" action="/users">
  <input type="text" name="username" value="张三">
  <input type="text" name="email" value="zhangsan@example.com">
  <input type="submit">
</form>

      中间件会将数据解析为：

      req.body = {
  username: '张三',
  email: 'zhangsan@example.com'
}

      这样在后续的路由处理函数中就可以直接通过 req.body.username 和 req.body.email 访问表单数据了。

    • express.json()
      1. 注册中间件：使用 app.use() 方法注册解析 JSON 数据的中间件
      1. 解析 JSON 请求体：专门用于解析 Content-Type 为 application/json 的请求体数据
      1. 自动转换数据：将原始的 JSON 字符串请求体解析为 JavaScript 对象

      详细说明

      • express.json() 是 Express 内置的中间件函数
      • 当客户端发送 JSON 格式的请求体时，此中间件会自动解析并将其转换为 JavaScript 对象
      • 解析后的数据会被存储在 req.body 对象中
      • 默认情况下，它只解析最多 100kb 的 JSON 数据

      客户端发送以下POST请求时

      POST /api/users
Content-Type: application/json

{
  "name": "John",
  "age": 30,
  "email": "john@example.com"
}

      经过这个中间件处理后，可以在路由处理函数中通过 req.body 访问数据：

      console.log(req.body.name);  // 输出: John
console.log(req.body.age);   // 输出: 30
console.log(req.body.email); // 输出: john@example.com

      这对于构建 RESTful API 和处理 AJAX 请求非常重要，因为这些场景通常使用 JSON 格式传输数据。

    • cookieParser()
      1. 注册中间件：使用 app.use() 方法注册 Cookie 解析中间件
      1. 解析 Cookie 数据：解析客户端发送的 Cookie 请求头中的数据
      1. 提供访问接口：将解析后的 Cookie 数据存储在 req.cookies 对象中

      详细说明

      • cookieParser() 是一个第三方中间件函数，需要通过 npm install cookie-parser 安装
      • 自动解析 HTTP 请求头中的 Cookie 字段
      • 将解析后的 Cookie 键值对存储在 req.cookies 对象中
      • 支持签名 Cookie（需要提供密钥参数）

      当客户端发送带有Cookie的请求时候

      GET /users
Cookie: sessionId=abc123; username=john; theme=dark

      经过这个中间处理后，可以在路由处理函数中通过req.cokies访问Cookie数据

      console.log(req.cookies.sessionId); // 输出: abc123
console.log(req.cookies.username);  // 输出: john
console.log(req.cookies.theme);     // 输出: dark

      带签名的 Cookie 支持

      如果需要解析签名 Cookie，可以提供密钥：

      app.use(cookieParser('your-secret-key'));

      签名 Cookie 会被解析到 req.signedCookies 对象中。

  • 接收客户端
    • 1、URL路径参数（动态路由参数）

      参数位置：URL路径中，以 :参数名 形式定义（如 /user/:id）

      使用场景：传递资源标识（如用户 ID、商品 ID），属于 URL 的一部分。

      获取方法：req.params

      req.params 是一个对象，包含 URL 路径中动态参数的键值对。

      const express = require('express');
const app = express();

// 定义带动态参数的路由
app.get('/user/:id', (req, res) => {
  // 获取路径参数
  console.log(req.params); // { id: '123' }（当访问 /user/123 时）
  res.send(`用户 ID: ${req.params.id}`);
});

// 多参数示例
app.get('/post/:postId/comment/:commentId', (req, res) => {
  console.log(req.params); // { postId: '456', commentId: '789' }（访问 /post/456/comment/789 时）
  res.send(`文章 ID: ${req.params.postId}, 评论 ID: ${req.params.commentId}`);
});

app.listen(3000);

      特点：

      • 参数名与路由定义中的 :参数名 完全一致。
      • 参数值均为字符串类型（如需数字需手动转换，如 Number(req.params.id)）。
      • 只能匹配单一路径片段（不会跨 / 匹配，如 /user/123/profile 中 id 仅为 123）。
    • 2、查询字符串参数（Query Parameters）

      参数位置：URL中 ? 后面的键值对（如：/search?keyword=js&page=1）

      适用场景：传递筛选条件，分页信息，搜索关键词等非核心路径参数

      获取方法：req.query

      req.query 是一个对象，自动解析 URL 中的查询字符串为键值对。

      app.get('/search', (req, res) => {
  // 获取查询参数
  console.log(req.query); // { keyword: 'js', page: '1' }（访问 /search?keyword=js&page=1 时）
  
  // 提取单个参数
  const keyword = req.query.keyword; // 'js'
  const page = parseInt(req.query.page) || 1; // 转换为数字（默认1）
  
  res.send(`搜索关键词: ${keyword}, 页码: ${page}`);
});

// 特殊情况：参数值为数组
// 访问 /tags?tag=html&tag=css 时
app.get('/tags', (req, res) => {
  console.log(req.query.tag); // ['html', 'css']
  res.send(`标签: ${req.query.tag.join(', ')}`);
});

      特点：

      • 自动解析 ? 后的键值对，支持 & 分隔多个参数。
      • 参数值均为字符串（需手动转换为数字、布尔等类型）。
      • 支持数组形式（如 ?tag=1&tag=2 解析为 { tag: ['1', '2'] }）。
      • 不会影响路由匹配（同一路由不同查询参数会匹配同一处理函数）。
    • 3、请求体参数（Request Body）

      参数位置：HTTP 请求体中（常用于 POST、PUT 等方法）。适用场景：传递大量数据（如表单提交、JSON 数据）。

      获取方法：需配合中间件解析，解析后通过 req.body 获取

      Express 不会自动解析请求体，需根据请求体格式使用对应的中间件：

      • 1、JSON格式请求体（Content-Type： application/json）

        使用内置中间件 express.json() 解析：

        // 启用 JSON 解析中间件（必须放在路由之前）
app.use(express.json());

app.post('/api/user', (req, res) => {
  // 解析后的 JSON 数据在 req.body 中
  console.log(req.body); // { name: '张三', age: 18 }（当请求体为 JSON 时）
  
  const { name, age } = req.body;
  res.send(`创建用户: ${name}, 年龄: ${age}`);
});
      • 2、表单格式请求体（Content-Type: application/x-www-form-urlencoded）

        使用内置中间件 express.urlencoded() 解析：

        // 启用表单解析中间件（extended: true 支持嵌套对象）
app.use(express.urlencoded({ extended: true }));

app.post('/login', (req, res) => {
  // 解析后的表单数据在 req.body 中
  console.log(req.body); // { username: 'admin', password: '123' }（表单提交时）
  
  const { username, password } = req.body;
  res.send(`登录用户: ${username}`);
});
      • 3、多部分表单数据（Content-Type: multipart/form-data）

        用于文件上传，需使用第三方中间件 multer 解析：

        npm install multer  # 安装 multer
        const multer = require('multer');
// 配置上传目录（默认存储为临时文件）
const upload = multer({ dest: 'uploads/' });

// 解析单文件上传（字段名为 'avatar'）
app.post('/upload', upload.single('avatar'), (req, res) => {
  // 文件信息在 req.file 中
  console.log(req.file); // 包含文件名、路径、大小等信息
  // 其他表单字段在 req.body 中
  console.log(req.body); // 非文件字段
  res.send('文件上传成功');
});

// 解析多文件上传（字段名为 'photos'，最多3个文件）
app.post('/uploads', upload.array('photos', 3), (req, res) => {
  console.log(req.files); // 数组形式的文件信息
  res.send('多文件上传成功');
});

        特点：

        • 必须通过中间件解析后才能通过 req.body 获取（否则为 undefined）。
        • 不同 Content-Type 需对应不同中间件（JSON、表单、多部分表单分开处理）。
        • 适用于 POST、PUT、PATCH 等可包含请求体的 HTTP 方法。
    • 4、HTTP请求头参数（Headers）

      参数位置：HTTP 请求头中（如 Authorization、Content-Type 等）。适用场景：传递认证信息、客户端类型、数据格式等元数据。

      获取方法：req.headers

      req.headers 是一个对象，包含所有请求头的键值对（键名自动转为小写）。

      app.get('/api/data', (req, res) => {
  // 获取所有请求头
  console.log(req.headers);
  
  // 提取单个请求头（键名小写）
  const auth = req.headers.authorization; // 获取认证令牌
  const userAgent = req.headers['user-agent']; // 获取客户端信息
  const contentType = req.headers['content-type']; // 获取内容类型
  
  res.send(`Authorization: ${auth || '无'}, User-Agent: ${userAgent}`);
});

      特点：

      • 请求头键名在 req.headers 中自动转为小写（如 Authorization → authorization）。
      • 常见用途：验证 JWT 令牌（Authorization: Bearer <token>）、跨域请求处理等。
    • 5、Cookie与Session参数

      参数位置：请求头中的 Cookie 字段（客户端存储的键值对）。适用场景：身份认证、会话管理（如记住登录状态）。

      获取方法：

      需使用第三方中间件 cookie-parser 解析 Cookie：

      npm install cookie-parser  # 安装
      const cookieParser = require('cookie-parser');
app.use(cookieParser()); // 启用 Cookie 解析

app.get('/profile', (req, res) => {
  // 获取 Cookie
  console.log(req.cookies); // 普通 Cookie 对象
  console.log(req.signedCookies); // 签名 Cookie（需配置密钥）
  
  const userId = req.cookies.userId; // 提取单个 Cookie
  res.send(`当前用户 ID（从 Cookie 获取）: ${userId || '未登录'}`);
});

      扩展：Session

      Session 数据存储在服务器，客户端通过 Cookie 中的 Session ID 关联。需使用 express-session 中间件：

      npm install express-session  # 安装
      const session = require('express-session');
app.use(session({
  secret: 'secret-key', // 用于签名 Session ID 的密钥
  resave: false,
  saveUninitialized: true
}));

app.get('/cart', (req, res) => {
  // Session 数据存储在 req.session 中
  req.session.cart = req.session.cart || []; // 初始化购物车
  res.send(`购物车商品数: ${req.session.cart.length}`);
});
    • 综合实例
      const express = require('express');
const cookieParser = require('cookie-parser');
const app = express();

// 中间件配置（必须在路由前）
app.use(express.json()); // 解析 JSON 请求体
app.use(express.urlencoded({ extended: true })); // 解析表单请求体

    // 路径参数

    // 查询参数

    // 请求体参数（JSON 或表单）

    • 注意
      1. 中间件顺序：解析请求体的中间件（express.json()、express.urlencoded()）必须放在路由之前，否则 req.body 为 undefined。
      1. 参数类型转换：所有参数默认均为字符串类型，需根据需求手动转换（如 Number(req.query.page)）。
      1. 参数验证：客户端传递的参数可能不完整或格式错误，需验证后再使用（可借助 joi、express-validator 等库）。
      1. 安全性：
        • 避免直接使用用户输入的参数拼接 SQL（防注入）。
        • 敏感参数（如密码）需加密处理。
        • 解析多部分表单时，限制文件大小和类型（防止恶意上传）。
      1. 路由匹配优先级：带静态路径的路由（如 /user/profile）比带动态参数的路由（如 /user/:id）优先级高，定义时需注意顺序。
  • 静态资源绑定

    静态资源（HTML、CSS、JS、图片等）的托管是通过内置的 express.static 中间件实现的。它能让客户端直接访问服务器上的静态文件，无需编写额外路由。

    • 基本原理

      express.static(root) 中间件会将 root 参数指定的目录作为静态资源根目录，客户端请求的路径会相对于该目录查找文件。

      例如：

      • 服务器端目录结构：public/index.html
      • 托管：app.use(express.static('public'))
      • 客户端访问：http://localhost:3000/index.html → 实际返回 public/index.html
    • 核心用法 express.static(path)
      const express = require('express');
const app = express();

// 托管静态资源目录（public 目录下的文件可直接访问）
app.use(express.static('public'));

// 可选：添加虚拟路径前缀（访问时需加前缀）
// app.use('/static', express.static('public')); 
// 此时访问 public/index.html 需用 http://localhost:3000/static/index.html

app.listen(3000, () => {
  console.log('服务器运行在 http://localhost:3000');
});
    • 多目录托管

      可同时托管多个静态目录，Express 会按顺序查找文件：

      // 先查 public 目录，找不到再查 assets 目录
app.use(express.static('public'));
app.use(express.static('assets'));
    • 路径规则
      • 客户端请求路径是「相对静态目录的路径」，无需包含静态目录名。
      • 支持目录索引：若请求目录（如 /pages/），会自动查找该目录下的 index.html。
      • 忽略文件名大小写（Windows 系统），但建议保持路径大小写一致。

  • 服务端渲染 SSR

    服务端渲染，后端嵌套模板，后端渲染模板 SSR（后端把页面1组装）

    1、做好静态页面，动态效果

    2、把前端代码提供给后端，后端把静态html以及里面的假数据给删掉，通过模板进行动态生成html的内容

    EJS（Embedded JavaScript）是一种简洁高效的模板引擎，常与 Express 框架配合使用，用于动态生成 HTML 页面。它允许在 HTML 中嵌入 JavaScript 代码，实现数据与视图的结合，非常适合服务端渲染（SSR）场景。

    • EJS 与 Express 结合的核心价值
      • 动态页面生成：通过模板引擎将后端数据（如数据库查询结果）注入 HTML，生成动态内容。
      • 代码复用：通过模板继承、包含（include）功能，复用公共组件（如导航栏、页脚）。
      • 简化开发：语法接近 HTML 和原生 JavaScript，学习成本低，适合快速开发。

    • 安装与配置
      npm install express ejs  # 安装 Express 和 EJS

      在 Express 中指定视图引擎为 EJS，并设置模板文件存放目录（默认是 views 文件夹）：

      const express = require('express');
const app = express();

// 配置 EJS 为视图引擎
app.set('view engine', 'ejs');

// 可选：自定义模板文件目录（默认是项目根目录下的 views 文件夹）
// app.set('views', './templates'); 

app.listen(3000, () => {
  console.log('服务器运行在 http://localhost:3000');
});
    • 创建ejs模板文件

      在项目根目录下创建 views 文件夹，并新建 EJS 模板（文件名以 .ejs 结尾）：

      示例：views/index.ejs

      <!DOCTYPE html>
<html>
<head>
  <title><%= title %></title> <!-- 嵌入变量 -->
</head>
<body>
  <h1><%= message %></h1>
  
  <!-- 嵌入 JavaScript 逻辑 -->
  <% if (user) { %>
    <p>欢迎，<%= user.name %>！</p>
  <% } else { %>
    <p>请登录</p>
  <% } %>
  
  <!-- 循环渲染列表 -->
  <ul>
    <% items.forEach(item => { %>
      <li><%= item %></li>
    <% }) %>
  </ul>
</body>
</html>
    • 在express路由中渲染模板

      使用 res.render(templateName, data) 方法渲染 EJS 模板，第一个参数是模板文件名（不含 .ejs 后缀），第二个参数是传递给模板的数据：

      app.get('/', (req, res) => {
  // 准备要传递给模板的数据
  const data = {
    title: 'EJS 示例',
    message: 'Hello EJS!',
    user: { name: '张三' },
    items: ['苹果', '香蕉', '橙子']
  };
  
  // 渲染 views/index.ejs 模板，并传递数据
  res.render('index', data);
});

      访问效果

      启动服务器后访问 http://localhost:3000，EJS 会自动将数据注入模板，生成最终的 HTML 页面并返回给客户端。

    • ejs核心语法

      EJS 的语法简洁，主要通过 <% ... %> 标签嵌入 JavaScript 代码，常用标签如下：

      标签语法作用示例
      <%= 变量 %>输出变量值（自动转义 HTML 特殊字符）<%= user.name %> → 输出用户名称
      <%- 变量 %>输出变量值（不转义，适合插入 HTML）<%- htmlContent %> → 插入原始 HTML
      <% 代码 %>执行 JavaScript 代码（无输出）<% if (flag) { %> ... <% } %>
      <%# 注释 %>EJS 注释（不会输出到 HTML 中）<%# 这是一条注释 %>
      -%>去除标签后的换行（美化输出格式）<% for (let i=0; i<3; i++) { -%>
    • ejs高级特性（express结合）
      • 1、模板包含（include）

        复用公共组件（如导航栏、页脚），通过 <%- include('路径') %> 引入其他 EJS 模板：

        步骤：

      优势

      1. 简单易学：语法接近 HTML 和原生 JS，开发者上手快。

        // 路由中对用户输入进行转义
const escapeHtml = (str) => str.replace(/[&<>"']/g, char => ({

// 传递转义后的数据给模板
res.render('page', { userInput: escapeHtml(req.query.input) });
      1. 模板缓存：在生产环境中，Express 会自动缓存 EJS 模板（提高性能），开发时可通过 app.disable('view cache') 禁用缓存，避免重启服务器。
      1. 复杂场景局限：对于大型应用，EJS 的模板继承能力较弱，可考虑使用 Pug（ Jade ）或配合 ejs-mate 等扩展。

    前后端分离，BSR（前端中组装页面）

    • app.js： 项目核心配置文件，包含 Express 应用的初始化、中间件配置、路由挂载等

      项目核心配置文件，包含 Express 应用的初始化、中间件配置、路由挂载等：

      var createError = require('http-errors');
var express = require('express');
var path = require('path');
var cookieParser = require('cookie-parser');
var logger = require('morgan');

// 引入路由模块
var indexRouter = require('./routes/index');
var usersRouter = require('./routes/users');

// 创建 Express 应用
var app = express();

// 配置视图引擎（EJS）
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'ejs');

// 注册中间件
app.use(logger('dev')); // 日志中间件（开发环境）
app.use(express.json()); // 解析 JSON 请求体
app.use(express.urlencoded({ extended: false })); // 解析表单请求体
app.use(cookieParser()); // 解析 Cookie
app.use(express.static(path.join(__dirname, 'public'))); // 托管静态资源

// 挂载路由
app.use('/', indexRouter); // 根路径路由
app.use('/users', usersRouter); // /users 路径路由

// 捕获 404 错误并转发到错误处理器
app.use(function(req, res, next) {
  next(createError(404));
});

// 错误处理中间件
app.use(function(err, req, res, next) {
  // 设置 locals，仅在开发环境提供错误信息
  res.locals.message = err.message;
  res.locals.error = req.app.get('env') === 'development' ? err : {};

  // 渲染错误页
  res.status(err.status || 500);
  res.render('error');
});

module.exports = app;

var router = express.Router();

      • routes/users.js 处理 /users 相关请求
        var express = require('express');
var router = express.Router();

// GET 用户列表
router.get('/', function(req, res, next) {
  res.send('用户列表页面');
});

// GET 单个用户（动态参数）
router.get('/:id', function(req, res) {
  res.send(`用户 ID: ${req.params.id}`);

module.exports = router;
      • views/目录 存放 EJS 模板文件
        • index.ejs：首页模板，通过 res.render('index', { title: 'Express' }) 渲染。
        • error.ejs：错误页模板，显示 404 或 500 错误信息。
      • public/目录 静态资源目录，通过 express.static 托管，可直接访问其中的文件

        例如：public/stylesheets/style.css 可通过 http://localhost:3000/stylesheets/style.css 访问。

    • 核心配置

      Express 生成器自动配置了多个实用功能，无需手动设置：

      1. 中间件预设已集成常用中间件：
        • morgan：HTTP 请求日志（开发环境下打印请求信息）。
        • cookie-parser：解析 Cookie 数据（req.cookies）。
        • express.json() 和 express.urlencoded()：解析请求体（req.body）。
        • express.static：托管静态资源。
      1. 视图引擎配置自动设置视图目录（views）和模板引擎（如 EJS），可直接使用 res.render() 渲染模板。
      1. 错误处理预设 404 错误捕获和全局错误处理中间件，开发环境下会显示详细错误堆栈。
      1. 启动脚本package.json 中预设了启动脚本：
        "scripts": {
  "start": "node ./bin/www",
  "dev": "nodemon ./bin/www"  // 手动添加热更新脚本
}
    • 自定义扩展

      生成的项目是基础骨架，可根据需求扩展：

      • 1、添加路由

        新建路由文件（如 routes/posts.js），定义路由后在 app.js 中挂载：

        // 引入新路由
var postsRouter = require('./routes/posts');
// 挂载路由
app.use('/posts', postsRouter);
      • 2、更换模板引擎

        若生成时未指定，可手动修改视图引擎（如改为 Pug）：

        app.set('view engine', 'pug'); // 需先安装：npm install pug
      • 3、添加数据库连接

        在 app.js 中引入数据库模块（如 Mongoose）并连接：

        var mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/myapp')
  .then(() => console.log('MongoDB 连接成功'))
  .catch(err => console.error('连接失败', err));
      • 4、添加跨域支持

        安装 cors 中间件并配置：

        npm install cors
        var cors = require('cors');
app.use(cors()); // 允许所有跨域请求
