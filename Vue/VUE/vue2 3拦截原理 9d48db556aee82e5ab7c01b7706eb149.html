vue2/3拦截原理

  • vue2拦截原理：基于Object.defineProperty

    Vue2 的响应式系统核心是通过 Object.defineProperty 方法，对对象的属性进行拦截，重写其 getter（读取时触发）和 setter（修改时触发）。

    1. 核心流程

    初始化拦截：当数据（如 data 中的对象）初始化时，Vue 会递归遍历对象的所有属性，对每个属性调用 Object.defineProperty，定义 getter 和 setter。

    依赖收集（getter）：当属性被读取时（如模板渲染、计算属性访问），getter 会触发 “依赖收集”，记录当前使用该属性的组件（Watcher）。

    派发更新（setter）：当属性被修改时，setter 会触发 “派发更新”，通知所有依赖该属性的组件重新渲染。

    // Vue2 响应式核心逻辑简化
function defineReactive(obj, key, value) {
  // 递归处理嵌套对象
  observe(value);
  
  // 存储依赖（使用 Dep 类管理）
  const dep = new Dep();
  
  // 拦截属性
  Object.defineProperty(obj, key, {
    get() {
      // 依赖收集：将当前 Watcher 加入 dep
      if (Dep.target) {
        dep.depend();
      }
      return value;
    },
    set(newValue) {
      if (newValue !== value) {
        value = newValue;
        // 派发更新：通知所有依赖重新渲染
        dep.notify();
      }
    }
  });
}

// 递归观测对象
function observe(obj) {
  if (typeof obj !== 'object' || obj === null) {
    return;
  }
  Object.keys(obj).forEach(key => {
    defineReactive(obj, key, obj[key]);
  });
}

    局限:

    由于 Object.defineProperty 是对已有属性的拦截，因此存在以下限制：

    无法监听对象新增属性：若给对象新增属性（如 obj.newKey = 123），不会触发 setter，需用 this.$set(obj, 'newKey', 123) 手动触发。

    无法监听属性删除：删除对象属性（如 delete obj.key）不会触发拦截，需用 this.$delete(obj, 'key')。

    对数组的特殊处理：Object.defineProperty 无法拦截数组的索引修改（如 arr[0] = 1）和 length 变化，因此 Vue2 对数组的 7 个变更方法（push、pop、splice 等）进行了原型重写，通过包装这些方法实现拦截。

  • vue3拦截原理：基础Proxy

    Vue3 改用 Proxy 实现响应式，Proxy 可以创建一个对象的 “代理”，直接拦截对整个对象的操作（而非单个属性），支持更多拦截场景。

    1. 核心流程

    创建代理：对目标对象（如 reactive 传入的对象）创建 Proxy 实例，代理对象会拦截所有对目标对象的操作。

    拦截操作：Proxy 支持 13 种拦截行为（如 get 读取、set 修改、deleteProperty 删除等），覆盖对象和数组的几乎所有操作。

    依赖收集与更新：在 get 拦截中收集依赖，在 set、deleteProperty 等拦截中触发更新，逻辑与 Vue2 类

    // Vue3 响应式核心逻辑简化
function reactive(target) {
  return new Proxy(target, {
    // 拦截属性读取（包括对象属性、数组索引、数组 length 等）
    get(target, key, receiver) {
      const result = Reflect.get(target, key, receiver);
      // 依赖收集：记录当前依赖
      track(target, key);
      // 递归代理嵌套对象（懒代理，访问时才代理）
      if (typeof result === 'object' && result !== null) {
        return reactive(result);
      }
      return result;
    },
    
    // 拦截属性修改
    set(target, key, value, receiver) {
      const oldValue = Reflect.get(target, key, receiver);
      if (oldValue !== value) {
        Reflect.set(target, key, value, receiver);
        // 派发更新：通知依赖
        trigger(target, key);
      }
      return true;
    },
    
    // 拦截属性删除
    deleteProperty(target, key) {
      const hadKey = hasOwn(target, key);
      const result = Reflect.deleteProperty(target, key);
      if (hadKey && result) {
        // 派发更新
        trigger(target, key);
      }
      return result;
    }
  });
}

    优势：

    支持对象新增 / 删除属性：Proxy 能直接拦截 obj.newKey = 123（set）和 delete obj.key（deleteProperty），无需手动调用 $set/$delete。

    原生支持数组操作：能拦截数组索引修改（如 arr[0] = 1）、length 变化（如 arr.length = 0），以及所有数组方法（无需像 Vue2 那样重写原型）。

    懒代理：对嵌套对象的代理是 “按需进行” 的（访问时才代理），而非 Vue2 那样初始化时递归遍历所有属性，性能更优。

    拦截范围更广：除了读写，还能拦截 in 操作符（key in obj）、Object.keys 等，功能更强大。
