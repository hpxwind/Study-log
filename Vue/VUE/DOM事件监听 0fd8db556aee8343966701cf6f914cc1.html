DOM事件监听

在 Vue 中，事件处理器用于监听 DOM 事件（如点击、输入、滚动等）并触发相应逻辑，核心通过 v-on 指令（简写为 @）实现。事件处理器支持多种用法，包括绑定方法、内联语句、传递参数、修饰符等，以下是详细解析：

  • v-on 指令

    将事件触发的逻辑封装在组件方法中，通过方法名绑定，代码更清晰。

    <div id="app">
  <!-- 绑定方法（无参数） -->
  <button @click="handleClick">点击我</button>
  
  <!-- 绑定方法（带参数） -->
  <button @click="handleGreet('Vue2')">打招呼</button>
</div>

<script>
new Vue({
  el: '#app',
  methods: {
    handleClick() {
      console.log('按钮被点击了');
    },
    handleGreet(frame) {
      console.log(`Hello, ${frame}!`);
    }
  }
})
</script>

    示例（Vue3，组合式API）

    <div id="app">
  <button @click="handleClick">点击我</button>
  <button @click="handleGreet('Vue3')">打招呼</button>
</div>

<script>
const { createApp } = Vue;
createApp({
  setup() {
    const handleClick = () => {
      console.log('按钮被点击了');
    };
    const handleGreet = (frame) => {
      console.log(`Hello, ${frame}!`);
    };
    return { handleClick, handleGreet };
  }
}).mount('#app');
</script>
  • 内联语句（简单逻辑）

    直接在模板中写简单逻辑（如修改数据），无需定义方法。

    <div id="app">
  <p>计数：{{ count }}</p>
  <!-- 内联语句：直接修改数据 -->
  <button @click="count++">+1</button>
  <button @click="count--">-1</button>
</div>

<script>
new Vue({
  el: '#app',
  data() { return { count: 0 }; }
})
</script>
  • 事件对象
    属性 / 方法作用
    event.target触发事件的 DOM 元素（如点击的按钮）
    event.type事件类型（如 "click"、"keyup"）
    event.clientX/clientY鼠标点击时相对浏览器窗口的坐标
    event.preventDefault()阻止事件默认行为（如阻止链接跳转）
    event.stopPropagation()阻止事件冒泡（不触发父元素同名事件）

    当需要获取原生 DOM 事件对象（如鼠标位置、按键信息）时，可通过 $event 传递。

    1. 无参数时获取事件对象

    <button @click="showEvent">点击查看事件对象</button>

<script>
methods: {
  showEvent(event) { // 方法第一个参数自动接收事件对象
    console.log(event.target); // 触发事件的DOM元素（button）
    console.log(event.type); // 事件类型（"click"）
  }
}
</script>

    2. 带参数时获取事件对象

    需显式传递 $event 作为参数：

    <button @click="handleWithParams('参数', $event)">带参数+事件对象</button>

<script>
methods: {
  handleWithParams(msg, event) {
    console.log(msg); // "参数"
    console.log(event.clientX); // 鼠标点击的X坐标
  }
}
</script>
  • 事件修饰符

    Vue 提供了事件修饰符，用于简化常见的事件处理逻辑（如阻止冒泡、阻止默认行为），修饰符通过 . 连接在事件名后。

    修饰符作用等价原生操作
    .stop阻止事件冒泡（不触发父元素同名事件）event.stopPropagation()
    .prevent阻止默认行为（如表单提交、链接跳转）event.preventDefault()
    .self仅当事件在当前元素本身触发时才执行-
    .once事件仅触发一次-
    .capture事件捕获模式（从外层到内层触发）-
    .passive优化滚动性能（不阻止默认滚动）多用于 touchmove 等事件

    用法：

    说明：修饰符可链式使用，如 @click.stop.prevent（先阻止冒泡，再阻止默认行为）。

    <div id="app">
  <!-- 1. .stop：阻止冒泡 -->
  <div @click="parentClick" style="padding: 20px; border: 1px solid #000;">
    父元素
    <button @click.stop="childClick">子按钮（.stop阻止冒泡）</button>
  </div>

  <!-- 2. .prevent：阻止默认行为 -->
  <a href="https://vuejs.org" @click.prevent="handleLinkClick">
    点击不跳转（.prevent）
  </a>

  <!-- 3. .self：仅自身触发 -->
  <div @click.self="onlySelf" style="padding: 20px; border: 1px solid #000;">
    点击内部文字不触发，仅点击边框区域触发（.self）
  </div>

  <!-- 4. .once：仅触发一次 -->
  <button @click.once="onceClick">点击一次后失效（.once）</button>
</div>

<script>
new Vue({
  el: '#app',
  methods: {
    parentClick() { console.log('父元素被点击'); },
    childClick() { console.log('子按钮被点击'); },
    handleLinkClick() { console.log('链接被点击，但不跳转'); },
    onlySelf() { console.log('仅自身触发'); },
    onceClick() { console.log('只触发一次'); }
  }
})
</script>
  • 按键修饰符

    用于监听键盘事件（如 keydown、keyup）时，过滤特定按键触发的事件。

    1. 内置按键别名

    Vue 提供了常用按键的别名，无需记忆 keyCode：

    别名对应按键
    .enter回车键
    .tabTab 键（需配合 keydown）
    .delete删除键（Delete/Backspace）
    .esc退出键
    .space空格键
    .up/.down/.left/.right方向键

    用法：

    <div id="app">
  <!-- 按Enter键触发 -->
  <input 
    type="text" 
    @keyup.enter="handleEnter" 
    placeholder="按Enter提交"
  >

  <!-- 按Esc键触发 -->
  <input 
    type="text" 
    @keyup.esc="clearInput" 
    placeholder="按Esc清空" 
    v-model="inputVal"
  >
</div>

<script>
new Vue({
  el: '#app',
  data() {
    return { inputVal: '' };
  },
  methods: {
    handleEnter() {
      console.log('输入内容：', this.inputVal);
    },
    clearInput() {
      this.inputVal = '';
    }
  }
})
</script>

    自定义按键修饰符（仅Vue2）

    Vue2 中可通过 config.keyCodes 自定义按键别名（Vue3 已移除，需用按键名）：

    // Vue2 自定义 F1 键修饰符
Vue.config.keyCodes.f1 = 112;

// 使用
<input @keyup.f1="handleF1">
  • 鼠标修饰符
    修饰符作用
    .left仅左键点击触发
    .right仅右键点击触发
    .middle仅中键（滚轮）点击触发
    <button @click.right="rightClick">右键点击触发</button>

<script>
methods: {
  rightClick() {
    console.log('右键点击了按钮');
  }
}
</script>
  • Vue2 与 Vue3 事件处理的差异
    1. 按键修饰符：Vue3 移除了 keyCode 支持，仅保留按键别名（如 .enter），且新增 .exact 修饰符（精确匹配按键组合）。例：@click.ctrl.exact 仅在按住 Ctrl 键且无其他键时触发。
    1. v-on 绑定多个事件：Vue3 支持对象语法绑定多个事件：
      <button v-on="{ click: handleClick, mouseenter: handleEnter }">
  多事件绑定
</button>


    1. 组件事件：

    Vue3 中组件自定义事件需通过 emits 选项声明（更规范），而 Vue2 无需声明。

  • 核心原理

    Vue 的事件处理器本质是对原生 DOM 事件的封装：

    1. Vue 会在编译模板时，将 @事件名 转换为原生事件监听（如 addEventListener）。
    1. 事件修饰符由 Vue 内部处理（如 .stop 对应调用 event.stopPropagation()），避免在业务代码中直接操作 DOM 事件对象，符合 Vue 的 “声明式” 理念。
    1. 事件处理函数中的 this（Vue2 选项式 API）自动绑定为组件实例，方便访问 data 和 methods。
