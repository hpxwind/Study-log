API

  • XMLHttpRequest

    XMLHttpRequest（简称 XHR）是浏览器提供的一个API 对象，用于在不刷新页面的情况下与服务器交换数据，是实现 “异步请求”（AJAX，Asynchronous JavaScript and XML）的核心技术。

    使用XMLHttpRequest发起请求通常分为 4 步：创建对象→配置请求→发送请求→处理响应:

    // 1. 创建XMLHttpRequest对象
const xhr = new XMLHttpRequest();

// 2. 配置请求（方法、URL、是否异步）
// 语法：open(method, url, async)
// method: 请求方法（GET/POST/PUT/DELETE等）
// url: 请求地址
// async: 是否异步（true为异步，false为同步，推荐true）
xhr.open('GET', 'https://api.example.com/data', true);

// 3. 发送请求（GET请求无请求体，传null）
xhr.send(null);

// 4. 监听请求状态变化，处理响应
xhr.onreadystatechange = function() {
  // readyState === 4：请求已完成
  // status === 200：服务器成功响应
  if (xhr.readyState === 4 && xhr.status === 200) {
    // 响应数据在xhr.responseText（文本）或xhr.responseXML（XML）中
    const data = JSON.parse(xhr.responseText); // 解析JSON数据
    console.log('请求成功：', data);
  } else if (xhr.readyState === 4) {
    // 其他状态码（如404、500）表示请求失败
    console.error('请求失败，状态码：', xhr.status);
  }
};
    • 核心属性
      • readyState：请求状态码，取值说明：
        • 0：未初始化（未调用open()）。
        • 1：已打开（open()已调用，未调用send()）。
        • 2：已发送（send()已调用，未收到响应）。
        • 3：正在接收（已收到部分响应）。
        • 4：完成（响应全部接收完毕）。
      • status：服务器响应的 HTTP 状态码（如200成功、404未找到、500服务器错误）。
      • statusText：服务器返回的状态文本（如"OK"对应200）。
      • responseText：服务器返回的文本数据（适用于 JSON、HTML 等）。
      • responseXML：服务器返回的 XML 文档（已解析为Document对象，较少使用）。
    • 常用方法
      • open(method, url, async)：配置请求参数。
        • method：请求方法（GET/POST等）。
        • url：请求的服务器地址（相对路径或绝对路径）。
        • async：是否异步（默认true，建议保持异步避免阻塞页面）。
      • send(body)：发送请求。
        • body：请求体数据（GET请求传null，POST请求传表单数据、JSON 等）。
      • setRequestHeader(header, value)：设置请求头（需在open()之后、send()之前调用）。
        • 示例：xhr.setRequestHeader('Content-Type', 'application/json')（声明发送 JSON 数据）。
      • abort()：取消当前请求（如用户关闭页面或超时处理）。
    • 事件监听

      除了onreadystatechange，还有更直观的事件可监听：

      • onload：请求完成且成功（readyState === 4且status正常）时触发。
      • onerror：网络错误（如断网）时触发。
      • onabort：请求被abort()取消时触发。
      • onprogress：接收响应过程中触发（可用于显示进度条）。
    xhr.onload = function() {
  if (xhr.status === 200) {
    console.log('成功：', xhr.responseText);
  } else {
    console.error('失败：', xhr.status);
  }
};

xhr.onerror = function() {
  console.error('网络错误！');
};
    • post发送示例
      const xhr = new XMLHttpRequest();
xhr.open('POST', 'https://api.example.com/submit', true);

// 设置请求头：告诉服务器发送的是JSON数据
xhr.setRequestHeader('Content-Type', 'application/json');

// 监听响应
xhr.onload = function() {
  if (xhr.status === 200) {
    console.log('提交成功：', xhr.responseText);
  }
};

// 发送JSON数据（需转为字符串）
const data = JSON.stringify({ name: 'Alice', age: 25 });
xhr.send(data);
  • fetch

    fetch 是 JavaScript 中用于发起网络请求的现代 API，主要作用是从服务器获取资源（如数据、文件等）或向服务器发送数据，支持异步操作，是替代传统 XMLHttpRequest 的更简洁、更强大的方案。

    基本特点：

    1. 基于promise：fetch 方法返回一个 Promise 对象，可通过 then() 处理成功响应，catch() 捕获错误，也支持 async/await 语法，避免回调地狱。
    1. 默认不携带Cookie：需手动配置 credentials 参数才能携带 Cookie（如跨域请求时）。
    1. 仅网络错误会reject：HTTP 错误状态（如 404、500）不会触发 Promise 的 reject，需手动判断响应状态。
    1. 支持多种数据类型：可处理 JSON、文本、表单数据、二进制数据等。
    • 基本语法
      fetch(url, options)
  .then(response => {
    // 处理响应（如解析为JSON、文本等）
    return response.json(); // 解析为JSON
  })
  .then(data => {
    // 处理解析后的数据
    console.log(data);
  })
  .catch(error => {
    // 捕获网络错误
    console.error('请求失败:', error);
  });
      url：必选参数，请求的资源地址（字符串）。
options：可选参数，配置请求的对象，常用属性：
method：请求方法（GET、POST、PUT、DELETE 等，默认 GET）。
headers：请求头（如 {'Content-Type': 'application/json'}）。
body：请求体（发送的数据，适用于 POST、PUT 等，不能用于 GET）。
credentials：是否携带 Cookie（'include' 表示携带，跨域时需设置）。
mode：请求模式（'cors' 允许跨域，'no-cors' 限制跨域，'same-origin' 仅同源）。
    • get请求（获取数据）
      // 获取JSON数据
fetch('https://api.example.com/users')
  .then(response => {
    // 检查HTTP状态码（200-299表示成功）
    if (!response.ok) {
      throw new Error(`HTTP错误，状态码: ${response.status}`);
    }
    return response.json(); // 解析为JSON对象
  })
  .then(users => {
    console.log('用户列表:', users);
  })
  .catch(error => {
    console.error('请求失败:', error);
  });
    • post请求（发送数据）
      // 发送JSON数据
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json', // 声明发送的是JSON
  },
  body: JSON.stringify({ // 将对象转为JSON字符串
    name: 'Alice',
    age: 25
  }),
  credentials: 'include' // 携带Cookie（跨域时需要）
})
.then(response => {
  if (!response.ok) throw new Error('提交失败');
  return response.json();
})
.then(result => {
  console.log('提交成功:', result);
})
.catch(error => {
  console.error('错误:', error);
});
    • 使用async/await（更简洁）
      async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) throw new Error(`状态码: ${response.status}`);
    const data = await response.json(); // 等待解析完成
    console.log('数据:', data);
  } catch (error) {
    console.error('请求失败:', error);
  }
}

fetchData();
    • 处理不同类型的响应/响应对象
      fetch 的响应对象（response）提供多种方法解析数据：

response.json()：解析为 JSON 对象（最常用）。
response.text()：解析为文本字符串。
response.blob()：解析为二进制大对象（如图片、文件）。
response.formData()：解析为 FormData 表单数据。
      fetch('https://example.com/text.txt')
  .then(response => response.text())
  .then(text => console.log('文本内容:', text));
    • 注意
      1. HTTP 错误不触发 catch：即使服务器返回 404 或 500，fetch 的 Promise 仍会进入 resolve 状态，需通过 response.ok（true 表示 200-299）手动判断。
      1. 跨域请求需服务器支持：跨域时，服务器需设置 CORS 头（如 Access-Control-Allow-Origin），否则请求会失败。
      1. 默认不携带 Cookie：跨域请求需添加 credentials: 'include' 才能携带 Cookie，同源请求可省略。
      1. 无法取消请求（原生）：原生 fetch 本身不支持取消，需结合 AbortController 实现：
      const controller = new AbortController();
const signal = controller.signal;

fetch('https://api.example.com/data', { signal })
  .then(response => response.json())
  .catch(error => {
    if (error.name === 'AbortError') console.log('请求已取消');
  });

// 1秒后取消请求
setTimeout(() => controller.abort(), 1000);
