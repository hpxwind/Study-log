路由懒加载

Vue 路由懒加载（也叫路由组件懒加载）是一种性能优化手段，核心是将路由对应的组件打包成独立的代码块（chunk），仅在访问该路由时才加载对应的代码块，而非页面初始化时一次性加载所有组件代码。

这能大幅减小首屏加载的包体积，提升页面加载速度，尤其适用于大型应用。

：访问资源时候访问到什么就向后端请求什么，而不是一进入就加载所有

其实就是把传统导入改成箭头函数，在需要的地方再导入

component: () => import("@/views/Order")
  • 核心原理
    • 默认行为：不使用懒加载时，所有路由组件会被打包进同一个 app.js（或 main.js）文件，首屏加载需下载完整的包，体积大、加载慢。
    • 懒加载行为：通过动态导入（import() 语法）将每个路由组件拆分为独立的 chunk 文件（如 0.js、1.js），只有访问该路由时，浏览器才异步加载对应的 chunk，加载完成后渲染组件。
  • 基础实现
    // router/index.js
import Vue from 'vue'
import Router from 'vue-router'
Vue.use(Router)

// 非懒加载（对比）：静态导入，打包进主文件
// import Home from '@/components/Home'
// import About from '@/components/About'

// 懒加载：动态导入，拆分独立chunk
const router = new Router({
  routes: [
    {
      path: '/',
      name: 'Home',
      // 方式1：基础懒加载
      component: () => import('@/components/Home')
    },
    {
      path: '/about',
      name: 'About',
      // 方式2：带命名chunk（方便打包后识别）
      component: () => import(/* webpackChunkName: "about" */ '@/components/About')
    }
  ]
})

export default router
  • 批量懒加载（分组打包）

    多路由属于同一组件，可将其打包到用一个chunk中，可以减少请求数

    // 分组打包：user 相关组件打包到同一个 chunk（命名为 "user"）
const UserList = () => import(/* webpackChunkName: "user" */ '@/components/user/List')
const UserDetail = () => import(/* webpackChunkName: "user" */ '@/components/user/Detail')

const routes = [
  { path: '/user/list', component: UserList },
  { path: '/user/detail/:id', component: UserDetail }
]xz
  • 细节注意
    1. 注释 /* webpackChunkName: "xxx" */ 的作用
      这是 Webpack 的魔法注释（Vite 也兼容），用于给拆分后的 chunk 文件命名，默认 chunk 是数字（如 0.js），命名后变为 about.js/user.js，方便打包后调试和识别。

      相同 webpackChunkName 的组件会被打包到同一个 chunk 文件中（分组打包的核心）。

    1. 加载状态处理（可选）

      懒加载时，组件加载需要时间（尤其网络差时），可添加加载中 / 加载失败的提示：

      // 封装带加载状态的懒加载组件
const lazyLoad = (componentPath) => {
  const component = () => ({
    // 加载中组件
    loading: import('@/components/Loading.vue'),
    // 目标组件
    default: import(componentPath),
    // 加载失败组件
    error: import('@/components/Error.vue')
  })
  return component
}

// 使用
const routes = [
  {
    path: '/about',
    component: lazyLoad('@/components/About')
  }
]
    1. 预加载（可选）

      如果希望某些懒加载组件在空闲时提前加载（提升后续访问速度），可使用 Webpack 的 /* webpackPrefetch: true */ 注释

      // 访问首页时，空闲期预加载 about 组件
const About = () => import(/* webpackChunkName: "about", webpackPrefetch: true */ '@/components/About')

      webpackPrefetch：浏览器空闲时加载，不阻塞首屏；

      webpackPreload：与当前页面一起加载（优先级更高，适合大概率立即访问的组件）。
