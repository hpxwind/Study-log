字面量类型

字面量类型（Literal Type） 是对基础类型（字符串、数字、布尔值）的 “精确值约束”—— 它不再泛指整个基础类型（如 string），而是限定为唯一的、具体的值（如 "red"、100、true）。

  • 字符串字面量类型

    限定值为唯一的字符串常量，常用于枚举式的字符串约束（如按钮类型、状态值）。

    // 定义字符串字面量类型：仅允许值为 "primary"
type PrimaryBtn = "primary";

// 合法赋值
const btnType: PrimaryBtn = "primary";

// 错误赋值（值必须严格匹配）
// const btnType2: PrimaryBtn = "secondary";

// 结合联合类型：限定可选值范围（最常用）
type ButtonType = "primary" | "secondary" | "danger" | "success";
const btn1: ButtonType = "danger"; // 合法
const btn2: ButtonType = "warning"; // 报错：不在可选范围内
  • 数字字面量类型

    限定值为唯一的数字常量，常用于固定数值约束（如状态码、端口号）。

    // 定义数字字面量类型：仅允许值为 200
type SuccessCode = 200;

// 合法赋值
const code: SuccessCode = 200;

// 错误赋值
// const code2: SuccessCode = 404;

// 结合联合类型：限定状态码范围
type HttpCode = 200 | 400 | 404 | 500;
const resCode: HttpCode = 404; // 合法
const resCode2: HttpCode = 502; // 报错：不在可选范围内
  • 布尔字面量类型

    限定值为 true 或 false（布尔类型仅两个字面量值），单独使用意义不大，常结合联合类型 / 接口使用。

    // 布尔字面量类型
type TrueType = true;
type FalseType = false;

const isOk: TrueType = true; // 合法
// const isOk2: TrueType = false; // 报错

// 结合接口：约束属性为固定布尔值
interface Config {
  debug: false; // 仅允许 debug 为 false
}
const config: Config = { debug: false }; // 合法
// const config2: Config = { debug: true }; // 报错
  • 字面量类型与接口 / 对象结合

    用于约束对象属性的固定值范围，是业务场景中最常用的方式。

    // 接口中使用字面量类型
interface User {
  name: string;
  role: "admin" | "user" | "guest"; // 角色仅允许这三个值
  status: 0 | 1; // 状态：0（禁用）、1（启用）
}

// 合法对象
const user1: User = {
  name: "张三",
  role: "admin",
  status: 1,
};

// 错误对象（role 不在范围内）
// const user2: User = {
//   name: "李四",
//   role: "super_admin",
//   status: 0,
// };
  • 字面量类型作为函数参数 / 返回值

    约束函数的入参 / 返回值为固定值，避免非法值传入。

    // 函数参数：仅允许传入 "add" | "subtract"
function calculate(type: "add" | "subtract", a: number, b: number): number {
  if (type === "add") return a + b;
  if (type === "subtract") return a - b;
  // 联合类型覆盖所有情况，TS 不会提示“未返回值”
  return 0;
}

// 合法调用
calculate("add", 1, 2); // 3

// 错误调用（参数 type 非法）
// calculate("multiply", 1, 2);

// 函数返回值：固定返回字面量值
function getStatus(): "success" | "failed" {
  return Math.random() > 0.5 ? "success" : "failed";
}
const status = getStatus(); // 类型为 "success" | "failed"
  • 字面量类型与类型别名 / 泛型结合

    通过类型别名复用字面量组合，结合泛型实现动态约束。

    // 复用字面量联合类型
type Direction = "left" | "right" | "up" | "down";

// 泛型 + 字面量：约束参数为指定字面量
function setPosition<T extends Direction>(dir: T): { direction: T } {
  return { direction: dir };
}

// 调用时，TS 推导 dir 的类型为具体字面量
const pos = setPosition("left"); // pos.direction 类型为 "left"
  • 常量断言（as const）：推导字面量类型

    TS 默认会将 const 声明的基础类型值推断为宽泛类型（如 const str = "a" 推断为 string），通过 as const 可强制 TS 推导为字面量类型。

    // 默认推断：str 类型为 string
const str = "primary";
// 无法赋值给字面量类型变量
// const btnType: "primary" = str;

// 常量断言：强制推导为字面量类型 "primary"
const str2 = "primary" as const;
const btnType: "primary" = str2; // 合法

// 对象/数组的常量断言：所有属性/元素均为字面量类型
const config = {
  theme: "dark",
  size: 16,
} as const;
// config.theme 类型为 "dark"（而非 string），config.size 类型为 16（而非 number）

const arr = [1, 2, 3] as const;
// arr 类型为 readonly [1, 2, 3]（只读元组 + 数字字面量）

字面量类型与枚举（enum）的对比

特性字面量联合类型枚举（enum）
运行时存在❌ 仅编译期✅ 编译为 JS 对象
类型精准性✅ 直接约束具体值❌ 枚举成员是 “枚举类型”，可隐式转换为 number
灵活性✅ 可自由组合联合类型❌ 枚举成员固定，无法动态扩展
代码量✅ 无需额外定义❌ 需单独声明枚举
