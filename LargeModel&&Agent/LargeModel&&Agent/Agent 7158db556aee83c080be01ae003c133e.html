Agent

AI Agent是一个能够自主感知环境、进行思考规划，并采取行动来完成复杂目标的智能实体。核心是让大预言模型（LLM）根据任务需求，选择并调用工具，完成单靠模型自身无法解决复杂问题。

  • 以电商商品回答为例
  • Agent与普通Chain区别

基础流程（tool call）

流式输出

  • code
    # 修正你之前的代码并实现流式
from langchain.agents import create_agent
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage
import os

@tool
def get_weather(city: str) -> str:
    return f"{city} 天气晴朗"

agent = create_agent(
    model=ChatTongyi(model="qwen3-max", api_key=os.getenv("API_KEY")),
    tools=[get_weather],
    system_prompt="你是一个助手"
)

# 流式调用 Agent
for chunk in agent.stream(
    {"messages": [HumanMessage(content="乌鲁木齐天气如何？")]}
):
    # chunk 是一个字典，可能包含多个消息流的更新
    if "messages" in chunk:
        for msg in chunk["messages"]:
            if hasattr(msg, "content") and msg.content:
                print(msg.content, end="", flush=True)
Agent 的 stream 会返回多个事件（工具调用、思考过程、最终回答）。你需要提取最终的消息内容实时输出

ReAct

Agent ReAct(Reasoning+Acting)是大模型智能体的核心思考与行动与行动框架，是让Agent像人一样【思考问题 → 制定策略 → 执行行动 → 验证结果】

Tool Call 是底层能力（模型输出结构化工具调用指令）；
ReAct 是上层循环框架（用 Thought + Action + Observation 闭环，把 ToolCall 包装成多轮思考流程）。
ReAct 底层本质依赖 ToolCall 能力，但 ToolCall 可以单独裸用。

Reflection反思

Reflection 直译反思、自省、自我校验，是大模型智能体（Agent）里核心的自我纠错迭代机制。Agent 执行完动作（调用工具、输出回答）拿到结果后，停下来自我复盘：刚才的操作对不对、哪里出错、接下来怎么改，把复盘结论带入下一轮推理。

ReAct 是循环ToolCall
Reflection 是具有思考上一轮答案对错并总结给下一轮的ReAct
  • 解决的普通ReAct痛点
    • 参数错误不会自我修正：
      案例：订单填 300，系统标准 299，原生 ReAct 只会看到 “金额不匹配”，不一定知道要改成 299 重调用；Reflection 强制拆解：参数数值错误、正确值 299、下一步重调订单。
    • 遗忘多任务进度：
      你场景有「订单 + 日程」两个任务，原生模型容易做完一个就结束、重复调用同一个工具；Reflection 强制盘点：已完成 XX、未完成 XX，保证双任务闭环。
    • 格式、年份、命名类低级错误：
      如日程写成 2024、自创不存在工具名、大括号乱输出；自省环节会标记格式违规、年份错误，给出标准格式。
    • 大模型惰性、推理断裂：
      真实 API 模型容易空白思考、半途断逻辑；Reflection 是强制结构化输出，约束模型必须逐条分析，降低放飞概率。
    • 复杂长链路记忆衰减：
      多轮对话上下文变长，模型容易遗忘前面报错信息；Reflection 把关键结论提炼总结，固化到历史消息里，下一轮必可见。
  • 反思的两种触发时机

    方式一：工具调用后反思（Post-Action Reflection）

    触发时机：每次 Action → Observation 之后

    反思内容：

    • 工具选对了吗？参数填错没？
    • 返回的 Observation 能不能回答问题？
    • 已完成 / 待完成的任务清单

    作用：纠偏当前轮的错误，指导下一轮 Action

    当前代码已实现 — 就是 react_agent.py 中的 _build_reflect_prompt()：

    Observation → Reflection → 存入历史 → 下一轮 Thought 能看到反思

    方式二：最终答案前反思（Pre-Answer Reflection）

    触发时机：LLM 输出 FinalAnswer 之前（或之后做一次校验）

    反思内容：

    • 计算对不对？有没有幻觉？
    • 所有子任务都完成了吗？
    • 是否答非所问？参数信息是否齐全？

    作用：防止 LLM "过早收尾"——明明还有任务没做完就给 FinalAnswer

Middleware中间件

中间件的作用是对智能体的每一步工作进行控制和自定义的执行。

作用场景：

  • 日志记录，分析，调试
  • 转换提示词，工具选择
  • 重试，备用，提前终止等逻辑控制
  • 安全防护，个人身份检测等
  • 有无中间件对比

LangChain中内置了一些基础的中间件，中间件通过Hooks钩子来实现拦截，自定义中间件可以简单的使用装饰器来定义

节点式钩子：

  • before_agent: agent执行之前拦截
  • after_agent: agent执行后拦截
  • before_model: 模型执行前拦截
  • after_model: 模型执行后拦截

针对工具和模型的包装式钩子：

  • wrap_model_call: 每个模型调用时候拦截
  • wrap_tool_call: 每个工具调用时候拦截
  • 简单案例

    节点式钩子

    class NodeHooksMiddleware:
    """节点式钩子：在 Agent 执行的关键阶段前后触发"""

    async def before_agent(self, state):
        """Agent 开始执行前调用（可修改初始状态）"""
        print(f"[before_agent] 用户输入: {state.messages[-1].content}")
        # 可修改 state，例如添加系统提示
        state.messages.insert(0, SystemMessage("你是一个乐于助人的助手"))
        return state

    async def after_agent(self, state):
        """Agent 执行结束后调用（可修改最终响应）"""
        print(f"[after_agent] 最终答案: {state.output}")
        # 可对输出做后处理，例如添加签名
        state.output += "\n\n—— 由 AI 助手生成"
        return state

    async def before_model(self, state):
        """每次调用大模型前触发（可修改发给模型的提示词）"""
        print("[before_model] 即将调用 LLM...")
        # 可动态调整温度、max_tokens 等参数
        return state

    async def after_model(self, state):
        """每次调用大模型后触发（可修改模型响应）"""
        print(f"[after_model] 模型原始回复: {state.model_response}")
        # 可对模型回复做过滤或缓存
        return state

# 挂载到 Agent（伪代码）
agent = create_agent(
    model=...,
    tools=[...],
    middleware=[NodeHooksMiddleware()],
)

    包装式钩子

    from functools import wraps

def wrap_model_call(func):
    """装饰器：拦截每次 model.invoke() 调用"""
    @wraps(func)
    def wrapper(messages, **kwargs):
        print(f"[wrap_model_call] 原始提示词: {messages}")
        # 可修改提示词或添加 RAG 上下文
        modified_messages = [
            {"role": "system", "content": "请用繁体中文回答"},
            *messages
        ]
        response = func(modified_messages, **kwargs)
        print(f"[wrap_model_call] 模型返回: {response}")
        # 可缓存或修改响应
        return response
    return wrapper

def wrap_tool_call(func):
    """装饰器：拦截每次 tool 调用"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"[wrap_tool_call] 调用工具: {func.__name__}, 参数: {args}, {kwargs}")
        # 安全检查、参数校验、限流等
        if func.__name__ == "delete_file":
            raise PermissionError("禁止删除文件")
        result = func(*args, **kwargs)
        print(f"[wrap_tool_call] 工具返回: {result}")
        # 后处理，如格式化输出、记录审计日志
        return result
    return wrapper

# 使用时手动包装（或在中间件里自动应用）
get_weight = wrap_tool_call(get_weight)
get_height = wrap_tool_call(get_height)

# 模型包装示例（针对 ChatTongyi）
original_model = ChatTongyi(model="qwen3-max")
wrapped_model = wrap_model_call(original_model.invoke)
agent = create_agent(model=wrapped_model, tools=[get_weight, get_height])
  • 两种钩子的区别
    • 节点式钩子：关注 业务流程（agent 开始/结束、model 调用前后），通常可以访问完整的 state 对象，适合做状态修改、日志、权限验证。
    • 包装式钩子：关注 具体函数调用（model 的 invoke、tool 的执行），粒度更细，适合做参数校验、重试、缓存、性能监控。
