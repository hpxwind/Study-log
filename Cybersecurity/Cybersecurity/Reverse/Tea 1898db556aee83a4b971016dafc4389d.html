Tea

Tea系列包含原始TEA,XTEA（Extended TEA）,XXTEA（Conrrected Block TEA）

TEA系列均为对称分组密码,

核心是通过多轮迭代对固定长度的明文进行加密

解密为加密的逆过程

  • 原始TEA（1994）

    分组大小：64 位（8 字节，分为两个 32 位字 v0、v1）；

    密钥大小：128 位（16 字节，分为四个 32 位字 k0、k1、k2、k3）；

    轮数：推荐 32 轮（每轮包含两次加密操作，共 64 步）；

    核心轮函数：基于移位、加法和异或的混合运算，使用固定常量 delta = 0x9E3779B9（黄金比例相关的 32 位常量）。

    加密流程：

    void tea_encrypt(uint32_t* v, uint32_t* k) {
    uint32_t v0 = v[0], v1 = v[1], sum = 0;
    uint32_t delta = 0x9E3779B9; // 固定常量
    for (int i = 0; i < 32; i++) { // 32轮迭代
        sum += delta; // 每轮更新sum
        // 对v0和v1进行混合运算（移位、异或、加法）
        v0 += ((v1 << 4) + k[0]) ^ (v1 + sum) ^ ((v1 >> 5) + k[1]);
        v1 += ((v0 << 4) + k[2]) ^ (v0 + sum) ^ ((v0 >> 5) + k[3]);
    }
    v[0] = v0; v[1] = v1;
}

    解密过程：与加密对称，轮数相同，sum 初始化为 delta*32，每轮 sum -= delta，反向执行轮函数。

  • XTEA（1995）

    原始TEA存在密钥表攻击漏洞，XTEA对轮函数进行改造，

    分组 / 密钥大小：同 TEA（64 位分组，128 位密钥）；

    轮数：推荐 32 轮（每轮操作更对称）；

    核心改进：轮函数中移位和密钥的使用更均衡，避免原始 TEA 的弱密钥问题。

    加密流程：

    void xtea_encrypt(uint32_t* v, uint32_t* k) {
    uint32_t v0 = v[0], v1 = v[1], sum = 0;
    uint32_t delta = 0x9E3779B9;
    for (int i = 0; i < 32; i++) { // 32轮
        v0 += (((v1 << 4) ^ (v1 >> 5)) + v1) ^ (sum + k[sum >> 11 & 3]);
        sum += delta; // sum更新
        v1 += (((v0 << 4) ^ (v0 >> 5)) + v0) ^ (sum + k[sum & 3]);
    }
    v[0] = v0; v[1] = v1;
}

    关键差异：XTEA 中密钥 k 的索引由 sum 的移位 / 掩码决定（k[sum >> 11 & 3] 和 k[sum & 3]），轮函数对称性更强。

  • XXTEA（1998）

    XTEA 是对 TEA 的进一步改进，支持可变长度分组（不再局限于 64 位），更适合加密任意长度数据

    分组大小：可变（至少 32 位，推荐多个 32 位字，如 128 位、256 位等）；

    密钥大小：128 位（4 个 32 位字）；

    轮数：推荐 6+5*n（n 为分组字数，通常 32 轮）；

    核心改进：引入 “反馈机制”，每个字的加密依赖其他字，支持变长数据。

    加密流程：

    void xxtea_encrypt(uint32_t* v, int n, uint32_t* k) {
    uint32_t z = v[n-1], y = v[0], sum = 0;
    uint32_t delta = 0x9E3779B9;
    int q = 6 + 52/n; // 轮数计算
    while (q-- > 0) {
        sum += delta;
        uint32_t e = (sum >> 2) & 3;
        for (int p = 0; p < n-1; p++) {
            y = v[p+1];
            z = v[p] += ( (z >> 5 ^ y << 2) + (y >> 3 ^ z << 4) ^ (sum ^ y) + (k[p&3 ^ e] ^ z) );
        }
        y = v[0];
        z = v[n-1] += ( (z >> 5 ^ y << 2) + (y >> 3 ^ z << 4) ^ (sum ^ y) + (k[n-1&3 ^ e] ^ z) );
    }
}

    关键特征：分组长度 n（32 位字的数量）参与轮数计算，轮函数中每个字依赖相邻字，且密钥索引与 p&3 ^ e 相关。

  • 逆向中识别TEA的核心特征

    常量特征

    • delta = 0x9E3779B9：TEA/XTEA/XXTEA 均使用该 32 位常量（十六进制），反汇编中会以立即数形式出现（是识别的核心标志）；
    • 解密时可能出现 delta * 轮数（如 32 轮 TEA 解密，sum 初始值为 0x9E3779B9 * 32 = 0xC6EF3720）。

    2. 代码结构特征

    • 轮循环：存在固定轮数的循环（通常 32 轮），循环内包含 sum += delta（加密）或 sum -= delta（解密）；
    • 移位与异或：轮函数中频繁出现 <<4、>>5（TEA）或 <<4、>>5、<<2、>>3（XXTEA）等移位操作，且与异或（XOR）、加法混合使用；
    • 密钥索引：
      • TEA：直接使用 k[0]~k[3]；
      • XTEA：密钥索引由 sum 移位 / 掩码得到（如 sum >> 11 & 3、sum & 3）；
      • XXTEA：密钥索引与分组索引 p 和 e（sum >> 2 & 3）相关（如 k[p&3 ^ e]）；
    • 分组处理：
      • TEA/XTEA：固定处理 2 个 32 位字（64 位）；
      • XXTEA：处理可变数量的 32 位字（n 个字，n≥1），循环中包含对 n 的判断。

    3. 数据特征

    • 加密 / 解密函数参数通常包含：明文 / 密文（32 位字数组）、密钥（4 个 32 位字，16 字节）、分组长度（仅 XXTEA，n）；
    • 运算基于 32 位无符号整数（C 中 uint32_t），溢出自动取模（符合分组密码的 “模运算” 特征）。
  • 逆向分析的关键

    1. 定位加密函数

    通过上述特征在反汇编代码中定位目标函数：

    • 搜索常量 0x9E3779B9（十六进制），该常量是 TEA 系列的 “指纹”；
    • 查找包含 32 轮左右循环、sum += delta 操作、频繁移位（<<4、>>5）和异或的函数；
    • 分析函数参数：若输入为 8 字节数据（2 个 32 位字）+16 字节密钥，可能是 TEA/XTEA；若输入为可变长度数组 + 16 字节密钥 + 长度参数，可能是 XXTEA。

    2. 确定算法版本

    根据轮函数细节区分 TEA/XTEA/XXTEA：

    • TEA：轮函数中 v0 += ((v1 <<4) + k0) ^ (v1 + sum) ^ ((v1 >>5) + k1)，直接使用 k0~k3；
    • XTEA：轮函数中 v0 += (((v1 <<4) ^ (v1 >>5)) + v1) ^ (sum + k[sum>>11 &3])，密钥索引依赖 sum；
    • XXTEA：存在分组长度 n，轮函数中每个字依赖相邻字（如 v[p] += ... z ...，z 是前一个字）。

    3. 提取密钥

    TEA 系列密钥均为 16 字节（4 个 32 位字），提取方式：

    • 硬编码密钥：若密钥直接存储在程序数据段（如 0x12,0x34,0x56,0x78,...），可通过反汇编直接读取（注意字节序，通常为小端）；
    • 动态密钥：若密钥是动态生成（如从文件、网络读取），可通过调试工具（如 x64dbg）在加密函数调用前拦截密钥内存（跟踪密钥参数指针）；
    • 验证密钥：用提取的密钥对已知明文进行加密，对比程序输出的密文（一致则密钥正确）。

    4. 还原解密逻辑

    TEA 系列解密是加密的逆过程，逆向中可直接推导：

    • TEA/XTEA：解密轮数与加密相同，sum 初始值为 delta * 轮数，每轮 sum -= delta，轮函数操作反向；
    • XXTEA：解密逻辑与加密对称，轮数相同，sum 初始值为 delta * 总轮数，每轮 sum -= delta，轮函数反向计算。

    #include<stdio.h>

int main()
{
	unsigned int enc[6] = {0xC11EE75A, 0xA4AD0973, 0xF61C9018, 0x32E37BCD, 0x2DCC1F26, 0x344380CC};
    unsigned int key[4] = {0x10203, 0x4050607, 0x8090A0B, 0x0C0D0E0F};
	int i, j;
	long sum = 0, delta = 0x61C88647;
    // 解码
	for(i=0;i < 6;i+=2){
		sum =  0 - (32 * delta);
		for(j = 0; j < 32; j++) {
			enc[i+1] -= (((enc[i] >> 5) ^ (16 * enc[i])) + enc[i]) ^ (key[((sum >> 11) & 3)] + sum);
			sum += delta;
			enc[i] -= ((((enc[i+1] >> 5) ^ (16 * enc[i+1])) + enc[i+1]) ^ key[sum & 3] + sum);
		}
	}
    // 打印
	for (i = 0; i < 6; i++)
	{
		for (j = 0; j<=3; j++)
		{
			printf("%c", (enc[i] >> (j * 8)) & 0xFF);
		}
	}

	return 0;
}
