映射类型

基于已有类型创建新类型的核心工具，它通过遍历已有类型的属性（keyof 操作符），对每个属性进行 “修饰”（如可选、只读、类型转换），最终生成结构关联但特性不同的新类型。其核心价值是复用类型逻辑、减少重复定义、提升类型安全性。

映射类型的本质是：迭代一个类型的所有属性，对每个属性执行相同的类型转换规则，生成新类型。

  • 基本语法
    // 泛型 T 是源类型，K 是 T 的属性名（通过 keyof T 遍历）
type 映射类型名<T> = {
  [K in keyof T]: 转换规则; // 对 T 的每个属性 K 应用转换规则
};
    • keyof T：获取源类型 T 的所有属性名联合类型（如 T={a:number,b:string}，则 keyof T = "a" | "b"）。
    • K in keyof T：遍历联合类型 keyof T，K 依次代表每个属性名。
    • 右侧 转换规则：定义属性 K 的新类型（可保留原类型、修改类型、添加修饰符等）。

    示例：保留原类型的映射

    // 定义源类型
interface User {
  name: string;
  age: number;
  email: string;
}

// 映射类型：保留 User 的所有属性类型，仅生成新类型
type CopyUser<T> = {
  [K in keyof T]: T[K]; // T[K] 表示取原属性 K 的类型（索引访问类型）
};

// 使用：CopyUser<User> 与 User 完全一致
type UserCopy = CopyUser<User>; 
// UserCopy = { name: string; age: number; email: string }

  • 属性修饰符的映射

    映射类型可通过 +（添加修饰符）或 -（移除修饰符），修改属性的「可选性」和「只读性」（默认隐含 +）。

    1.可选修饰符（?）

    • +?：添加可选性（默认可省略 +，直接写 ?）。
    • ?：移除可选性（强制属性必选）。
    // 1. 所有属性变为可选（内置类型 Partial 的实现）
type MyPartial<T> = {
  [K in keyof T]?: T[K]; // 等价于 [K in keyof T]+?: T[K]
};

type PartialUser = MyPartial<User>;
// PartialUser = { name?: string; age?: number; email?: string }

// 2. 所有属性变为必选（内置类型 Required 的实现）
type MyRequired<T> = {
  [K in keyof T]-?: T[K]; // 移除可选修饰符
};

interface PartialUser2 {
  name?: string;
  age?: number;
}
type RequiredUser = MyRequired<PartialUser2>;
// RequiredUser = { name: string; age: number }（必选）

    2.只读修饰符（readonly）

    • +readonly：添加只读性（默认可省略 +）。
    • readonly：移除只读性。
    // 1. 所有属性变为只读（内置类型 Readonly 的实现）
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K]; // 等价于 +readonly [K in keyof T]: T[K]
};

type ReadonlyUser = MyReadonly<User>;
const user: ReadonlyUser = { name: "张三", age: 20, email: "a@qq.com" };
user.age = 21; // 报错：只读属性不能修改

// 2. 移除只读性
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

type MutableUser = Mutable<ReadonlyUser>;
const user2: MutableUser = { name: "李四", age: 22, email: "b@qq.com" };
user2.age = 23; // 正常：可修改
  • 内置映射类型
    内置类型作用实现核心逻辑
    Partial<T>所有属性变为可选[K in keyof T]?: T[K]
    Required<T>所有属性变为必选[K in keyof T]-?: T[K]
    Readonly<T>所有属性变为只读readonly [K in keyof T]: T[K]
    Pick<T, K extends keyof T>挑选 T 中指定的 K 属性（保留原类型）[P in K]: T[P]
    Omit<T, K extends keyof T>排除 T 中指定的 K 属性（保留其他）Pick<T, Exclude<keyof T, K>>
    Record<K extends keyof any, T>定义键为 K、值为 T 的对象类型[P in K]: T
    Exclude<T, U>从 T 中排除可赋值给 U 的类型（属性名）（辅助类型，常配合映射使用）
    Extract<T, U>从 T 中提取可赋值给 U 的类型（属性名）（辅助类型，常配合映射使用）
    // 1. Pick：挑选 User 的 name 和 age 属性
type PickUser = Pick<User, "name" | "age">;
// PickUser = { name: string; age: number }

// 2. Omit：排除 User 的 email 属性
type OmitUser = Omit<User, "email">;
// OmitUser = { name: string; age: number }（与 PickUser 等价）

// 3. Record：定义键为 id/name，值为 string 的对象
type InfoRecord = Record<"id" | "name", string>;
const info: InfoRecord = { id: "1001", name: "王五" };

// 4. Exclude：从 keyof User 中排除 "age"
type ExcludeAge = Exclude<keyof User, "age">; // "name" | "email"
  • 转换属性类型（类型映射）

    对每个属性的类型进行统一转换（如全部转为 string、null 可选等）。

    // 1. 所有属性类型转为 string
type Stringify<T> = {
  [K in keyof T]: string;
};

type StringUser = Stringify<User>;
// StringUser = { name: string; age: string; email: string }（age 从 number 转 string）

// 2. 所有属性类型转为「原类型 | null」（可空类型）
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

type NullableUser = Nullable<User>;
const user3: NullableUser = { name: "赵六", age: null, email: "c@qq.com" }; // 正常
  • 重命名属性（TS4.1+ 支持 as 从句）

    通过 as 对属性名进行重命名（结合模板字面量类型更强大）。

    // 1. 属性名转为大写（结合模板字面量类型 Uppercase）
type UppercaseKeys<T> = {
  [K in keyof T as Uppercase<string & K>]: T[K];
};

type UppercaseUser = UppercaseKeys<User>;
// UppercaseUser = { NAME: string; AGE: number; EMAIL: string }

// 2. 给属性名添加前缀 "prefix_"
type AddPrefix<T> = {
  [K in keyof T as `prefix_${string & K}`]: T[K];
};

type PrefixUser = AddPrefix<User>;
// PrefixUser = { prefix_name: string; prefix_age: number; prefix_email: string }
  • 过滤属性（结合条件类型）

    只保留满足条件的属性（如仅保留 number 类型的属性）。

    // 保留 T 中类型为 number 的属性
type PickNumberProps<T> = {
  [K in keyof T as T[K] extends number ? K : never]: T[K];
};

type NumberUser = PickNumberProps<User>;
// NumberUser = { age: number }（仅保留 number 类型的 age 属性）

// 解释：
// - T[K] extends number ? K : never：若属性 K 的类型是 number，保留 K；否则转为 never（never 会被自动过滤）

  • 递归映射（处理嵌套类型）

    默认映射仅处理浅层属性，如需处理嵌套对象，可结合递归逻辑。

    // 递归将所有属性转为可选（深层 Partial）
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};

interface UserWithAddress extends User {
  address: {
    province: string;
    city: string;
  };
}

type DeepPartialUser = DeepPartial<UserWithAddress>;
const user4: DeepPartialUser = {
  name: "孙七",
  address: { city: "北京" } // address 可选，province 可选（深层生效）
};
  • DTO类型转换（请求/响应数据处理）

    后端返回的原始数据（User）需转换为前端展示用的「可编辑类型」（所有属性可选）或「只读展示类型」。

    // 编辑用户时：所有属性可选（允许部分修改）
type UserEditDTO = Partial<User>;

// 展示用户时：所有属性只读（防止误修改）
type UserViewDTO = Readonly<User>;

  • 配置类型优化

    定义基础配置后，通过映射生成「必填配置」「只读配置」等变体。

    // 基础配置（部分可选）
interface BaseConfig {
  apiUrl: string;
  timeout?: number;
  debug?: boolean;
}

// 生产环境配置：所有属性必选（移除可选性）
type ProdConfig = Required<BaseConfig>;

// 开发环境配置：只读（防止代码中修改配置）
type DevConfig = Readonly<BaseConfig>;
  • 工具类型封装

    封装通用工具类型，复用类型逻辑（如 DeepPartial、Nullable 等）。

    // 封装：所有属性可空且可选
type PartialNullable<T> = Partial<Nullable<T>>;

type PartialNullableUser = PartialNullable<User>;
const user5: PartialNullableUser = { age: null }; // 仅传部分可空属性，正常
